# Dealix V9 — Real Productization & Multi-Tenant SaaS Layer

الهدف من V9 هو تحويل Dealix من نظام تشغيل داخلي وحزم ملفات إلى منتج SaaS/Managed SaaS قابل للبيع، القياس، التوسع، والفوترة.

## المبادئ
1. كل عميل داخل Workspace/Tenant مستقل.
2. كل حركة مهمة لها Audit Log.
3. كل استخدام AI محسوب ومحدود حسب الخطة.
4. كل خطة لها حدود واضحة: users, leads, agents, runs, reports.
5. لا يوجد outbound تلقائي بدون opt-in ومراجعة بشرية.
6. البداية تكون Managed SaaS: أنت تدير النظام للعميل ثم تفتح Self-Serve تدريجيًا.

## المسار الصحيح
- Phase 1: Internal Admin Console + Tenant Registry.
- Phase 2: Client Workspace محدود للعميل.
- Phase 3: Billing + Usage Metering.
- Phase 4: Agent Apps لكل قطاع.
- Phase 5: Marketplace/Partner Console.

## North Star
MRR موثق + retention + usage حقيقي + proof لكل claim.
