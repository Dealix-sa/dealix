# Diagnostic Script — سكربت التشخيص

**Duration / المدة:** 20–30 دقيقة. **التشخيص ليس مكالمة بيع — هو mini-product.**

> المخرج يُسجّل في `data/revenue/diagnostics.jsonl` وملخص في
> `reports/revenue/diagnostic_brief_template.md`. التسجيل عبر `sales/DIAGNOSTIC_SCORECARD.md`.

---

## 0. Frame / التأطير (1 دقيقة)
"هدفنا اليوم نفهم وين تتسرب الفرص والإيراد عندك، ونطلع بأهم 3 معوّقات وخطوة تالية واضحة.
ما فيه أي إرسال أو التزام — كل قرار خارجي يبقى بموافقتك."

## 1. Context / السياق (5 دقائق)
- إيش طبيعة خدماتكم B2B؟ ومن العميل المثالي؟
- كم حجم الفرص/المتابعات الشهرية تقريبًا؟
- مين صاحب القرار في تحسين هذا؟

## 2. Pain discovery / اكتشاف الألم (8 دقائق)
- وين تحسون الفرص تضيع؟ (مصدر / متابعة / تسليم / إثبات)
- كيف تثبتون الأثر للعميل بعد التسليم؟
- إيش اللي جربتوه وما اشتغل؟

## 3. Data & readiness / الجاهزية (5 دقائق)
- عندكم بيانات أولية (فرص، عملاء، متابعات) في أي شكل؟
- تقبلون نموذج "ذكاء يقترح، إنسان يوافق"؟
- ميزانية Sprint تأسيسي متاحة؟

## 4. Close / الإغلاق (3 دقائق)
- لخّص أهم 3 معوّقات بصوت عالٍ.
- صنّف النتيجة عبر الـ Scorecard.
- اقترح الخطوة التالية بحسب النتيجة (انظر النتائج أدناه).

---

## Outcomes / النتائج
1. **Not fit** — وضّح السبب بلطف، لا عرض.
2. **Nurture** — متابعة لاحقة، أرسل نموذج Proof Pack.
3. **Command Sprint offer** — فقط إذا score ≥ 70.
4. **Partner path** — إحالة لشريك إذا خارج النطاق.

> لا وعود إيراد مضمون · لا بيانات عميل لتدريب نماذج · لا حالة نجاح علنية بدون إذن.
