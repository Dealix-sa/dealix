## مبدأ السلّم / The ladder principle

نبدأ دائماً من أدنى رُتبة تناسب الإشارة، ونصعد بالدليل. الرُّتبة الأعلى ليست هدفاً — هي نتيجة لإثبات قابل للقياس.
*We always start at the lowest rung the signal supports, and climb on evidence. A higher rung is never the goal — it is the result of measurable proof.*

## Rung 0 — التشخيص المجاني / Free Diagnostic — 0 ريال

**الإشارة التي تبرّره / Signal that justifies it**
- أي تواصل أول. لا يحتاج تأهيلاً عالياً. / Any first contact. Requires no high qualification.
- قرار `qualify` = DIAGNOSTIC_ONLY أو REFRAME، أو ACCEPT في أول خطوة. / `qualify` decision = DIAGNOSTIC_ONLY or REFRAME, or ACCEPT as the first step.

**ما المتضمَّن / What's included**
- مراجعة محكومة لوضع الإيراد الحالي، تُسلَّم ثنائية اللغة خلال 24 ساعة. / A governed review of the current revenue picture, delivered bilingual within 24 hours.
- موجز (brief) يحدد فرصة واحدة قابلة للقياس + توصية بالرُّتبة التالية. / A brief naming one measurable opportunity + a recommendation for the next rung.
- لا أتمتة، لا إرسال خارجي، لا بيانات تُشارَك مع أي طرف. / No automation, no external send, no data shared with anyone.

**مسار الترقية / Upgrade path → Rung 1**
- إذا أظهر التشخيص فرصة محددة + مالك قرار + بيانات جاهزة → اعرض سبرنت 499 ريال. / If the diagnostic surfaces a specific opportunity + decision-owner + ready data → offer the 499 SAR Sprint.

## Rung 1 — سبرنت ذكاء الإيراد / Revenue Intelligence Sprint — 499 ريال

**الإشارة التي تبرّره / Signal that justifies it**
- الألم واضح (`pain_clear`) + مالك حاضر (`owner_present`) + بيانات جاهزة (`data_available`). / Pain clear + owner present + data ready.
- قرار `qualify` = ACCEPT (نقاط ≥ 85). / `qualify` decision = ACCEPT (score ≥ 85).

**ما المتضمَّن / What's included**
- سبرنت 7 أيام محكوم على فرصة إيراد واحدة. / A 7-day governed sprint on one revenue opportunity.
- مخرجات ضمن النطاق: حسابات مُرتَّبة بالأولوية، مسوّدات ثنائية اللغة، سجل قرارات حوكمة، حزمة إثبات (Proof Pack)، وأصل رأسمالي واحد قابل لإعادة الاستخدام. / Bounded outputs: prioritized accounts, bilingual drafts, a governance decisions log, a Proof Pack, and one reusable capital asset.
- وعد المقياس: هدف درجة الإثبات ≥ 80 وأصل رأسمالي ≥ 1 — **هدف تقديري، لا نتيجة مضمونة**. / Proof metric promise: target proof score ≥ 80 and capital asset ≥ 1 — **an estimated target, not a guaranteed outcome**.

**شروط الدفع / Payment terms**
- 50% عند القبول، 50% عند تسليم حزمة الإثبات. / 50% on acceptance, 50% on Proof Pack delivery.

**مسار الترقية / Upgrade path**
- → **Rung 2** إذا ظهر تصدير CSV/CRM يحتاج معالجة PII منظَّمة. / if a CSV/CRM export needs structured PII handling.
- → **Rung 3** إذا تحقق بعد التسليم: درجة إثبات ≥ 80 + تبنٍّ ≥ 70 + مالك سير عمل، عبر `adoption_os.retainer_readiness.evaluate(...)`. / if, after delivery, proof ≥ 80 + adoption ≥ 70 + a workflow owner, via `adoption_os.retainer_readiness.evaluate(...)`.

## Rung 2 — حزمة البيانات إلى إيراد / Data-to-Revenue Pack — 1,500 ريال

**الإشارة التي تبرّره / Signal that justifies it**
- كل إشارات الرُّتبة 1، **زائد** تصدير CSV / CRM فعلي يحتاج معالجة بيانات شخصية (PII). / All Rung 1 signals, **plus** a real CSV / CRM export needing PII handling.
- العميل يقبل الحوكمة و PDPL صراحةً. / Customer explicitly accepts governance and PDPL.

**ما المتضمَّن / What's included**
- كل ما في الرُّتبة 1، زائد طبقة معالجة بيانات محكومة: تنظيف، تصنيف، ومعالجة PII وفق نظام حماية البيانات. / Everything in Rung 1, plus a governed data layer: cleaning, classification, and PDPL-compliant PII handling.
- سجل تدقيق للبيانات يوضّح ما لُمس وكيف. / A data audit log showing what was touched and how.

**شروط الدفع / Payment terms**
- 50% عند القبول، 50% عند تسليم حزمة الإثبات. / 50% on acceptance, 50% on Proof Pack delivery.

**مسار الترقية / Upgrade path → Rung 3**
- نفس بوابة الرُّتبة 3: درجة إثبات ≥ 80 + تبنٍّ ≥ 70 + مالك سير عمل. / Same Rung 3 gate: proof ≥ 80 + adoption ≥ 70 + workflow owner.

## Rung 3 — إدارة عمليات الإيراد / Managed Revenue Ops — 2,999–4,999/شهر

**الإشارة التي تبرّره / Signal that justifies it**
- **بوابة صارمة:** `adoption_os.retainer_readiness.evaluate(...).eligible == True`، أي: درجة إثبات ≥ 80 **و** تبنٍّ ≥ 70 **و** وجود مالك لسير العمل. / **Hard gate:** `...eligible == True`, i.e. proof ≥ 80 **and** adoption ≥ 70 **and** a workflow owner exists.
- لا تُعرَض الرُّتبة 3 قبل تسليم حزمة إثبات حقيقية واحدة على الأقل. / Rung 3 is never pitched before at least one real Proof Pack has been delivered.

**ما المتضمَّن / What's included**
- إدارة شهرية محكومة لعمليات الإيراد المتكررة، مع لوحة تبنٍّ ومراجعات دورية. / Monthly governed management of recurring revenue ops, with an adoption dashboard and regular reviews.
- النطاق والسعر يُحدَّدان في مكالمة مجدولة، لا في محادثة. / Scope and price set in a scheduled call, not in chat.

**مسار الترقية / Upgrade path → Rung 4 / Enterprise**
- → **Rung 4** إذا اتسع النطاق إلى بناء مخصّص يتجاوز الإدارة. / if scope grows into custom build beyond management.
- → **Enterprise** إذا دخل بنك / شركة كبرى / كيان منظَّم. / if a bank / large corporate / regulated entity enters.

- إعداد محكوم مخصّص ضمن نطاق مكتوب وواضح، مع رسم شهري للتشغيل (1K/شهر). / A governed custom setup within a written, explicit scope, with a monthly run fee (1K/month).

**مسار الترقية / Upgrade path → Enterprise**
- → **Enterprise** عند دخول متطلبات حوكمة على مستوى مؤسسي / تنظيمي. / if enterprise-/regulator-level governance requirements appear.

## قواعد عبور الرُّتب / Rung-crossing rules

1. **ابدأ من الأدنى.** لا تعرض Rung 3 لعميل لم يَر حزمة إثبات. / **Start low.** Never pitch Rung 3 to a customer who has not seen a Proof Pack.
2. **اصعد بالدليل، لا بالرغبة.** البوابة الكمية (درجة/تبنٍّ) تسبق العرض. / **Climb on evidence, not appetite.** The quantitative gate (score/adoption) precedes the offer.
3. **النطاق محدود دائماً.** كل رُتبة تأتي مع استثناءاتها (اللاءات الـ 11). / **Scope is always bounded.** Each rung ships with its exclusions (the 11 non-negotiables).
4. **التقدير ليس وعداً.** كل رقم نتيجة هو هدف تقديري حتى يُتحقَّق ويُسجَّل. / **An estimate is not a promise.** Every outcome number is an estimated target until verified and logged.
