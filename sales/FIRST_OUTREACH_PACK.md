# First Outreach Pack — حزمة التواصل الأولى

**Purpose / الغرض:** إطارات رسائل أولية **للمراجعة والتخصيص اليدوي**. ليست للإرسال الآلي.

> كل رسالة تمر عبر `reports/revenue/outreach_approval_queue.md` وتحتاج موافقة المؤسس.
> لا تخصيص مفبرك · CTA واحد · لا ادعاءات مضمونة · لا scraping · لا واتساب بارد.

---

## Frame 1 — Warm intro / تعارف دافئ
> يُستخدم فقط عند وجود `warm_intro_reason`.

```
مرحبًا [الاسم]،
[سبب التواصل الحقيقي — مرجع/معرفة مشتركة].
لاحظت [ملاحظة محددة من evidence_url] — أظن فيه فرصة لترتيب [الألم المفترض].
نعمل مع شركات مثلكم على تحويل البيانات والمتابعات إلى إيراد، بموافقة بشرية في كل خطوة.
هل يناسبك 20 دقيقة لتشخيص سريع هذا الأسبوع؟
```
CTA: حجز تشخيص 20–30 دقيقة.

## Frame 2 — Proof-gap observation / فجوة إثبات
```
[الاسم]، رأيت [evidence: صفحة خدماتكم / حالة منشورة].
سؤال واحد: كيف تثبتون أثر عملكم للعميل بعد التسليم؟
هذا بالضبط ما نساعد فيه عبر Command Sprint من 7 أيام.
أقدر أرسل لك نموذج Proof Pack (بدون بيانات عميل) إذا يهمك.
```
CTA: إرسال نموذج Proof Pack.

## Frame 3 — Follow-up complexity / تعقيد المتابعة
```
[الاسم]، الوكالات اللي تتعامل مع [القطاع] غالبًا تفقد فرص في المتابعة.
عندنا طريقة تحوّل المتابعات والفرص إلى Revenue Map واضح خلال أسبوع — كل إرسال خارجي بموافقتك.
يناسبك مكالمة قصيرة؟
```
CTA: مكالمة تشخيص قصيرة.

## Personalization checklist / تخصيص حقيقي
- [ ] سطر واحد على الأقل مبني على `evidence_url` حقيقي للشركة.
- [ ] لا أرقام أو نتائج مخترعة.
- [ ] لا وعود إيراد مضمون.
- [ ] CTA واحد واضح.
