# Service Quality Gate

## قبل بدء Pilot
- Scope مكتوب.
- expected deliverables واضحة.
- client owner معروف.
- baseline موجود.
- invoice أو approval موجود.

## أثناء Pilot
- كل مخرج له proof item.
- كل claim له evidence.
- كل رسالة خارجية مراجعة.
- أي agent output يحتاج QA قبل الاستخدام الخارجي.

## قبل تسليم التقرير
- لا بيانات حساسة في proof report.
- لا وعود ROI مضمونة.
- next action واضح.
- retainer option منطقي.

## فشل البوابة
إذا فشل أي بند، لا ترسل التقرير ولا تعد العميل بنتيجة حتى تعالج الخلل.
