# مخطط بذرة الحسابات (ICP Seed Schema)

> هذا المخطط يصف الأعمدة التي تستهلكها سكربتات `scripts/launch_package/`.
> النسخة العاملة الافتراضية في `sample_data/icp_seed_accounts_saudi.csv` (بيانات أمثلة فقط).

## الأعمدة المطلوبة (9 أعمدة — يجب أن تتطابق مع `Prospect`)

| العمود | الوصف |
| --- | --- |
| `company` | اسم الشركة |
| `sector` | القطاع (مثال: Marketing Agency, Training & Consulting, B2B SaaS) |
| `city` | المدينة (الرياض، جدة...) |
| `website` | الموقع الإلكتروني (إن وُجد) |
| `public_contact` | جهة تواصل عامة أو سياق علاقة موثّق (لا بيانات شخصية غير لازمة) |
| `likely_pain` | فرضية الألم/فرصة النمو |
| `best_offer` | العرض الأنسب من سلّم العروض |
| `priority` | `High` / `Medium` / `Low` |
| `notes` | ملاحظات ومصدر التواصل |

## قواعد البيانات (متوافقة مع الدستور)

- لا scraping مخالف لشروط المنصات؛ استخدم مصادر عامة أو علاقة/إذن.
- لا تضف بيانات شخصية غير لازمة (data minimization).
- وثّق مصدر كل جهة تواصل في `notes`.
- **بيانات العملاء الحقيقية لا تُرفع إلى Git**: ضع ملفك الحقيقي تحت
  `data/prospects/` (مُستثناة في `.gitignore`) ومرّره عبر `--input`.

> ملاحظة: المخطط الأصلي في الحزمة الخارجية اقترح أعمدة إضافية
> (`contact_role`, `pain_hypothesis`, `offer_fit` ...). تمت مواءمته هنا مع
> الأعمدة التسعة التي تشغّلها السكربتات فعليًا؛ يمكنك إضافة أعمدة اختيارية
> مثل `phone` / `email` / `contact_role` وستظل السكربتات تعمل.
