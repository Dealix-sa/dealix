# Skill: assemble-proof-pack

## When to use
End of a Command Sprint (or any paid delivery) — before invoicing the remainder / proposing a retainer.

## Steps
1. Open the customer folder (`docs/04_delivery/CUSTOMER_FOLDER_TEMPLATE.md` structure).
2. Populate the Proof Pack from real evidence captured during delivery — never invent metrics.
3. Use `docs/04_delivery/PROOF_PACK_TEMPLATE.md` (cross-links `docs/phase-e/08_PROOF_PACK_TEMPLATE.md`).
4. Run the `audit-positioning` skill: no guaranteed-revenue language, no unapproved names, estimates framed.
5. Route to founder approval before anything leaves.
6. Only after delivery + approval, check retainer eligibility for the next rung.

## Rules
Evidence-based only. No fabricated proof. No public customer name without written approval. Founder approves before send.
