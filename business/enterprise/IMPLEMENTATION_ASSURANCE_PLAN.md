# Implementation Assurance Plan

## Phase 1 — Diagnostic Sprint (7 days)

1. Day 1: Kickoff workshop. Map customer's existing CRM, WhatsApp, review, billing, and reporting flows. Output: in-scope/out-of-scope matrix.
2. Day 2-3: Workflow review. Founder + customer commercial owner walk live process. Output: friction log.
3. Day 4-5: Draft the chosen OS module (Revenue OS / Review OS / Command Center / etc.).
4. Day 6: Customer review session. Adjust scope, pricing, deliverables.
5. Day 7: Signed SOW + first proof report.

## Phase 2 — First 30 Days

1. Module wired into customer's existing tools (read-only by default; write actions gated by approval).
2. First weekly proof report delivered.
3. First retention review.

## Phase 3 — Months 2-3

1. Expansion review: which adjacent OS module is next?
2. Customer success signals reviewed (`business/proof/CLIENT_SUCCESS_SIGNALS.md`).
3. Optional: case study draft (only with customer consent and real numbers).

## Acceptance criteria

- Each deliverable carries its own acceptance criteria (`business/contracts/ACCEPTANCE_CRITERIA_TEMPLATE.md`).
- Customer signs off in writing before the next deliverable starts.

## Rollback plan

- If a deliverable fails acceptance, Dealix rolls back the change, re-scopes, and re-delivers at no additional cost within the same phase.

## Exit plan

- 30-day notice on a managed retainer.
- Data return: CSV + JSON exports inside 14 days of notice.
- No data retention beyond the SOW termination + 30 days unless legally required.
