# Client Delivery OS

## Method

Map → Design → Build → Operate → Scale.

## Purpose

Turn each client request into a managed delivery workspace with intake, diagnosis, scope, blueprint, acceptance criteria, proof pack, and next-30-days plan.

## Required outputs

- Intake form.
- Current state map.
- Pain map.
- Scope card.
- Solution blueprint.
- Test plan.
- Proof pack.
- Next iteration plan.

## Acceptance criteria

No project starts without business outcome, acceptance criteria, risk list, and human approval gates.
