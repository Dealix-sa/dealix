# Offer Matching Rules

## Rule Engine

1. If sector = Healthcare AND pain = reviews/no-shows → Review & Reputation OS
2. If sector = B2B Services AND pain = follow-up/sales cycle → Revenue OS
3. If sector = Logistics/Retail AND pain = operations/tracking → Operations & Delivery OS
4. If sector = Any AND founder wants 360° view → Command Center OS
5. If budget < 20k SAR AND pain = unclear → Diagnostic Sprint
6. If budget > 100k SAR AND pain = multiple → Custom Enterprise OS
7. If post-pilot AND client wants continuity → Managed OS Retainer

## Governance
- Never match an offer without a confirmed pain signal.
- Always propose Diagnostic Sprint before a full OS if fit is unclear.
- Document the match reason in the CRM note.
