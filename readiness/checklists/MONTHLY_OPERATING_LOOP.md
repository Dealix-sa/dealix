# Monthly Operating Loop Checklist

> Last Sunday of every month. 90 minutes. Pairs with the regular Weekly Review.

## Pre-Work (30 min)

- [ ] Complete the regular Weekly Review for this Sunday first
- [ ] Pull financial reconciliation (bank vs `cash_collected.csv`)
- [ ] Pull win/loss list for the month
- [ ] Pull experiment results for the month
- [ ] Pull cohort retention data for active Managed Ops clients

## Monthly Strategy Update (45 min)

Open `docs/learning/MONTHLY_STRATEGY_UPDATE.md` template and complete:

- [ ] North-Star Status (cash, MRR, incidents, 90-day milestone progress)
- [ ] What We Learned (6 one-sentence learnings, one per learning source)
- [ ] What's Working (3 items)
- [ ] What's Not (2 items)
- [ ] Decisions For Next Month (BUILD / FIX / KILL / DEFER + optional ICP / pricing / sector / playbook)
- [ ] One Strategic Question To Answer Next Month
- [ ] Reading

## Board Memo (15 min)

Use `docs/founder/BOARD_MEMO_TEMPLATE.md`. Even with no board, write it.

- [ ] TL;DR (3 sentences)
- [ ] Key Metrics with deltas
- [ ] What Worked / Didn't Work
- [ ] Decisions Made
- [ ] Top 3 Risks
- [ ] Next Month's One Goal
- [ ] What I Need

## Storage + Distribution

- [ ] Save Monthly Strategy Update to `weekly_reviews/monthly-YYYY-MM.md` (private)
- [ ] Save Board Memo to `weekly_reviews/board-memo-YYYY-MM.md` (private)
- [ ] Public-safe excerpt of learnings (if any) → next LinkedIn post draft
- [ ] Append month's decisions to `DEALIX_EXECUTION_LEDGER.md`

## Stage Status Check (5 min)

- [ ] Update `DEALIX_STAGE_STATUS.md` if any super-system maturity changed
- [ ] If Stage Transition Rules met → trigger transition planning

## Hard Rules

- Cannot skip — write the abbreviated version if time is short
- Bad months get reported with the same precision as good ones
- "What I Need From The Board" cannot be empty; ask for help specifically
- Pricing / ICP / playbook changes get logged in the right place, not just in the memo
