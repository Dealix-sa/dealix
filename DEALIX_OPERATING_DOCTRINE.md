# Dealix Operating Doctrine

Dealix is a Saudi-native Revenue Company Operating System.

The company runs on five loops:

## 1. Revenue Loop

```
Lead → Reply → Call → Payment → Delivery → Proof → Retainer
```

## 2. Delivery Loop

```
Intake → Research → Score → Draft → QA → Handoff → Feedback
```

## 3. Trust Loop

```
Classify → Guard → Approve → Log → Review → Improve
```

## 4. Learning Loop

```
Experiment → Measure → Learn → Decide → Update
```

## 5. CEO Loop

```
Brief → Decide → Approve → Focus → Review
```

## The Operating Sentence

**AI prepares. Workflows execute. Trust governs. Founder approves critical moves. Metrics improve the system.**

## The Twelve Systems

01 Founder OS · 02 Strategy OS · 03 Revenue OS · 04 Acquisition OS ·
05 Sales OS · 06 Delivery OS · 07 Trust OS · 08 Finance OS ·
09 Client Success OS · 10 Product OS · 11 Content OS · 12 Learning OS.

Each system has a policy, a playbook, templates, metrics, an owner, an
automation level, an approval level, and a verification script.

## The Four Layers

- **CEO Layer** — the founder's daily and weekly system.
- **Company Systems Layer** — the twelve systems above.
- **Execution Layer** — workflows, agents, trust gates, scripts, CI.
- **Evidence Layer** — ledgers, decision log, weekly reviews, proof packs.

## Hard Rules

1. No claim public without evidence on file.
2. No A2 / A3 action without a DECISION_LOG entry.
3. No new system before the previous one has a verification script.
4. No feature without a revenue, delivery, trust, or founder-leverage reason.
5. No new revenue stream before the previous one has paying evidence.

## Done Looks Like

The company is operating when:

1. Daily Command Brief runs every working day.
2. Pipeline tracker is updated weekly.
3. Every lead has a stage and a next action.
4. Every proposal has an approval entry.
5. Every client delivery has a QA pass.
6. Every public claim has an evidence file.
7. Every week has a CEO Review.
8. Every month has a Strategy Update.
9. GitHub blocks merges on CI failures.
10. Client data lives only in the private ops repo.
11. Every feature has a revenue, delivery, or trust reason.
12. Every agent has boundaries, permissions, and an evaluation.

When all twelve are true, Dealix is no longer founder-dependent — it is
a system that the founder drives.
