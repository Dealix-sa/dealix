# reports/launch

Output directory for launch-pass reports produced by the Company OS commands
(`/dealix-audit`, `/dealix-verify`) and the launch agents.

Each report should capture: files changed + why · what's sellable now vs future ·
gate results (PASS/FAIL with exact failures) · blockers · next actions.

Reports are working artifacts — they must never contain unsafe claims, customer names without
written approval, or secrets.
