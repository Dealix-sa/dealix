# reports/revenue — Revenue Board Outputs

## What lives here

This directory holds the generated outputs of the founder's revenue board and the Weekly CEO Review. These are rendered artifacts and committed review pages, not source data.

- `daily_command.md` (when generated by `founder_daily_command.py`) — the day's revenue board: paid events, proof packs delivered, pipeline by stage, offer-ladder position, open approvals, blockers.
- Weekly review pages — the one-page decision output produced by the Weekly CEO Review (decisions with owners and dates, the priority account, ladder movement, bilingual coverage note).
- The revenue asset index (`dealix_revenue_asset_index.*`) — an index of revenue-relevant assets.

## How it is generated

The source of truth is append-only event data under `data/revenue/*.jsonl` (`payments.jsonl`, `revenue_events.jsonl`). Each line is one immutable event: payment received, proof pack delivered, paid sprint started, upsell opened.

`founder_daily_command.py` reads those events and renders the daily board. Rendered files are read-only — never hand-edit a generated board. To correct a number, fix or append the source event and regenerate.

## Rules

- No figure here is presented as a guaranteed return; values follow the value-tier language (estimated / observed / verified / client_confirmed).
- No customer PII in any committed page; use anonymized labels.
- No external action is marked done without a logged approval.
