## 1. الجوهر · Brand Essence

### الوعد · Core Promise
- **عربي:** صفقات ذكية. نمو حقيقي.
- **English:** Intelligent Deals. Real Growth.

### الفكرة الواحدة · Single Idea
ذكاء اصطناعي مبني للسوق السعودي، يحوّل البيانات إلى صفقات مغلقة ونمو مُثبَت.

AI built for the Saudi market that turns data into closed deals and measurable growth.

### الشخصية · Personality

| البُعد · Dimension | الصفة · Trait | الضد · Opposite |
|---|---|---|
| Voice  | واثق بدون غرور · Confident, not arrogant | متردّد ❌ |
| Tone   | محترف ودافئ · Professional, warm        | بارد ❌ |
| Approach | عملي ومباشر · Practical, direct       | مبهم ❌ |
| Values | موثوق وشفّاف · Trusted, transparent     | مخفي ❌ |

## 3. الصوت والنبرة · Voice & Tone

### القواعد · Rules
- **اللغة الأساسية:** عربي (RTL). الإنجليزية ثانوية. راجع
  [`../DESIGN.md` §1 — Arabic-first Principles](../DESIGN.md) للقواعد الكاملة.
- **النبرة:** تنفيذية سعودية — هادئة، دقيقة، بدون مبالغة، بدون كلمات شامية.
- **الأرقام:** هندية (٠١٢٣) داخل النصوص العربية، لاتينية (0123) داخل الجداول وKPIs.
- **علامات الترقيم:** فاصلة عربية `،` وعلامة استفهام `؟` داخل النصوص العربية.

### الكلمات المحظورة · Forbidden Words
لا تستخدم. القائمة موروثة من v1 ومُلزمة:
- ❌ "نضمن" / "guaranteed" — overclaim
- ❌ "ثوري" / "revolutionary" — overclaim
- ❌ "الأفضل" / "best" — لا يمكن إثباته
- ❌ "Disruptive" — كليشيه
- ❌ "AI-powered" لحالها — استخدم "AI يفهم السياق السعودي"
- ❌ أي مصطلح شامي (شلون، شي، منيح، هيك)

المرجع الرسمي للكلمات المحظورة:
[`docs/sales-kit/dealix_brand_guidelines.md` § الكلمات المحظورة](../../../docs/sales-kit/dealix_brand_guidelines.md)
و [`../DESIGN.md` § Forbidden copy](../DESIGN.md).

### Receipts قبل الادعاءات · Receipts before Claims
كل رقم في أي مادة Dealix لازم يكون مرتبط بـ source / proof / ledger entry. لا أرقام بدون دليل.

## 4. الألوان · Color Palette

### الألوان الأساسية · Core Tokens

| Token   | Hex       | RGB              | الدور · Role |
|---------|-----------|------------------|--------------|
| `navy`  | `#0B1220` | 11, 18, 32       | الخلفية الأساسية، الـ hero، الـ headers |
| `teal`  | `#00D1A1` | 0, 209, 161      | CTAs، النمو، الـ tagline، success |
| `silver`| `#B2BBC6` | 178, 187, 198    | نصوص ثانوية على الداكن، الفواصل |
| `slate` | `#0F1726` | 15, 23, 38       | الكروت والـ panels على الـ navy |
| `white` | `#FFFFFF` | 255, 255, 255    | النصوص الرئيسية على الداكن |

### قاعدة 60-30-10
- **60% Navy** — الخلفية والأسطح.
- **30% Silver/White** — النصوص والـ borders.
- **10% Teal** — CTAs، التركيز، الإشارات.

### الألوان الموسّعة · Extended
- `teal_light` `#34E3B5` — hover للـ CTA.
- `teal_dark`  `#00A37D` — active/pressed.
- `navy_alt`   `#070C16` — أعمق navy للـ footer.

### الألوان الدلالية · Semantic
| Token | Hex | الاستخدام |
|---|---|---|
| `success` | `#00D1A1` | live / approved (مرادف لـ teal) |
| `warn`    | `#E0A744` | pilot / partial |
| `danger`  | `#D45656` | blocked / error |
| `info`    | `#3B82F6` | informational notice |

### التباين · Contrast
- `white` على `navy` → AA Large + AAA Normal ✓
- `teal` على `navy` → AA Large ✓ — استخدمه للأزرار والعلامات، **لا** للنص الطويل.
- `silver` على `navy` → AA Large فقط — للنص الثانوي القصير.
- `navy` على `teal` → AAA ✓ — مثالي لنص الـ CTA.

> القاعدة: المصدر الوحيد للقِيَم هو [`tokens.json`](./tokens.json). لا تكتب hex خام في الكود إلا عند توليد المتغيرات.

## 5. الخطوط · Typography

### العائلات · Families

| Use | Family | Fallback |
|---|---|---|
| Display (Latin)   | **Sora**            | Space Grotesk, system-ui |
| Body (Latin)      | **Space Grotesk**   | Plus Jakarta Sans, system-ui |
| Arabic (كل شيء)   | **IBM Plex Sans Arabic** | Noto Sans Arabic, system-ui |
| Mono              | **JetBrains Mono**  | ui-monospace |

> Arabic لا يتغيّر بين v1 و v2 — IBM Plex Sans Arabic يظلّ الخط الرسمي.

### السلّم · Scale
يتبع `typography.scale` في [`tokens.json`](./tokens.json):

| Token | Size | Use |
|---|---|---|
| `display` | 64px | hero headline |
| `5xl`     | 48px | hero subhead |
| `4xl`     | 36px | H1 |
| `3xl`     | 30px | H2 |
| `2xl`     | 24px | section titles |
| `xl`      | 20px | card titles |
| `lg`      | 18px | lead paragraph |
| `base`    | 16px | body |
| `sm`      | 14px | helper, tables |
| `xs`      | 12px | captions, badges |

### الأوزان · Weights
- 400 regular — body
- 500 medium — emphasis
- 600 semibold — section titles
- 700 bold — display / wordmark

## 6. الشعار · Logo System

### الـ Lockups المعتمدة · Approved Lockups

| Lockup | File | الاستخدام |
|---|---|---|
| **Primary**    | [`assets/logos/primary.svg`](./assets/logos/primary.svg)       | غلاف الـ decks، splash، posters |
| **Icon only**  | [`assets/logos/icon.svg`](./assets/logos/icon.svg)             | avatars، app icons، favicons (>24px) |
| **Horizontal** | [`assets/logos/horizontal.svg`](./assets/logos/horizontal.svg) | navbars، email signatures، PDF headers |
| **Monochrome** | [`assets/logos/monochrome.svg`](./assets/logos/monochrome.svg) | طباعة single-ink، خلفيات صور |
| **Favicon**    | [`assets/logos/favicon-32.svg`](./assets/logos/favicon-32.svg) | browser tab |

### القواعد · Rules
- **Clear space:** المسافة الخالية حول الشعار = ارتفاع cap-height للحرف D.
- **Min sizes:**
  - Icon: 24px.
  - Horizontal: 120px wide.
- **خلفيات:** navy `#0B1220`, slate `#0F1726`, white `#FFFFFF` فقط. على الصور — استخدم scrim معتم 60%+.

### ممنوع · Don'ts
- ❌ لا تطبَع stretch / squeeze.
- ❌ لا تغيّر ألوان الشعار.
- ❌ لا تضيف ظلال أو effects.
- ❌ لا تدوّر الشعار.
- ❌ لا تضعه على خلفية مزدحمة بدون scrim.
- ❌ لا تستبدل الـ tagline.
- ❌ لا تفصل الـ icon عن الـ wordmark وتضيف عناصر بينهما.

### تحذير · Caveat
ملفات الـ SVG الحالية هي **approximations** عاملة. الـ designer master سيحلّ محلها لاحقًا (راجع
[`assets/logos/README.md`](./assets/logos/README.md)).

## 7. التطبيقات · Applications

### Deck / Pitch
- الغلاف: navy فل-بليد، primary logo في المنتصف، tagline أسفل.
- النصوص: white headlines، silver body.
- Highlights: teal فقط (KPI numbers، CTA boxes).
- المرجع: تعديل [`design-skills/investor-style-deck`](../../../design-skills/) لاستخدام tokens v2.

### Proposal PDF
- header: navy bar + horizontal logo (white).
- body: white background، navy نصوص (للنسخ المطبوعة).
- CTA / signature box: teal fill.
- المرجع: [`design-skills/proposal-page`](../../../design-skills/).

### Email Signature

```
الاسم · Name
Role · Dealix
[icon.svg 32×32]  Intelligent Deals. Real Growth.
dealix.sa  ·  +966 ...
```

### Business Card
- الوجه الأمامي: navy، icon على اليمين (RTL)، الاسم/الدور بـ silver.
- الوجه الخلفي: teal fill، الـ tagline بـ navy.

### Social Avatar
- استخدم `icon.svg` على navy مربع 1024×1024.
- LinkedIn banner: horizontal lockup + tagline على navy gradient.

### Favicon
- استخدم `favicon-32.svg` كمصدر. ولّد PNG 16/32/192/512 من نفس الـ viewBox.

### GitHub
- repo READMEs: استخدم horizontal lockup أعلى الصفحة (height 64px).
- org profile (`.github` repo): متابعة منفصلة — راجع [Roadmap](#9-خارطة-الطريق--roadmap).

### الحالة الحالية · Current State

| [`frontend/tailwind.config.ts`](../../../frontend/tailwind.config.ts) | ⚠️ يستخدم gold/emerald القديم — انتظار codegen task |
| [`landing/styles.css`](../../../landing/styles.css) | ⚠️ يستخدم cyan — انتظار rebinding task |

### قواعد الترحيل · Migration Rules
1. أي عمل **جديد** (deck، proposal، صفحة، component) يلتزم بـ v2.
2. الأعمال **القائمة** تبقى v1 حتى يصدر طلب ترحيل صريح.
3. **لا ترميم جزئي** — صفحة كاملة v2 أو v1، لا خلط.
4. عند الترحيل: حدّث الـ asset، عبّيء كل tokens من [`tokens.json`](./tokens.json)، لا تكتب hex خام.

## 9. خارطة الطريق · Roadmap
