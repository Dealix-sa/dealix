# Dealix Stage-Gated Roadmap

## Purpose
Prevent Dealix from overbuilding before revenue proof.

## Owner
Sami / Founder CEO.

## Review Cadence
Weekly.

## Stage 0 — Safety Foundation
Goal: protect public/private boundary and claims.

Exit Criteria:
- Trust Command Center exists.
- Approval Matrix exists.
- Autonomy Policy exists.
- Public/private boundary check passes.
- No banned claims in public repo.

Allowed:
- docs
- checks
- templates
- manual execution

Not Allowed:
- autonomous external sending
- sensitive data export
- guaranteed revenue claims

## Stage 1 — Revenue Sprint Execution
Goal: prove first paid Revenue Sprint.

Exit Criteria:
- 25 leads.
- 25 DMs sent.
- 3 samples prepared.
- 1 proposal sent.
- payment / PO / written approval pursued.

Allowed:
- founder-led outbound
- manual samples
- manual proposal
- daily/weekly CLI
- dashboard local only

Not Allowed:
- SaaS buildout
- complex agents
- enterprise platform
- white-label system

## Stage 2 — Delivery Proof
Goal: deliver first paid or approved sprint.

Exit Criteria:
- client intake completed.
- delivery report created.
- QA checklist passed.
- handoff completed.
- feedback requested.

Allowed:
- delivery templates
- QA improvements
- sample automation
- report generation drafts

Not Allowed:
- full automation without QA
- public case study without approval

## Stage 3 — Retainer Conversion
Goal: convert delivery proof into recurring revenue.

Exit Criteria:
- client health score.
- renewal / retainer proposal.
- one retainer attempt.
- feedback captured.

Allowed:
- weekly client report template
- retention playbook
- client success dashboard

## Stage 4 — Productization
Goal: automate repeated workflows only after evidence.

Exit Criteria:
- workflow repeated 3 times manually.
- template exists.
- QA pass rate acceptable.
- customer requested repeatability.

Allowed:
- semi-automation
- internal dashboard
- repeatable report generation
- productization candidates

## Stage 5 — Scale
Goal: delegate or automate proven workflows.

Exit Criteria:
- 3 paid sprints.
- 2 case studies or approved proof points.
- 1 retainer.
- repeated workflow evidence.

Allowed:
- VA/SDR support
- delivery analyst support
- partner system
- SaaS candidate planning

## Stage 6 — SaaS / Enterprise Candidate
Goal: build software only around proven repeated workflow.

Exit Criteria:
- 10 repeated workflow executions.
- clear customer demand.
- clear workflow ROI.
- trust controls tested.
- support burden understood.

Allowed:
- dashboard product
- API layer
- billing system
- customer portal
- multi-tenant architecture

## Rules
- Revenue proof before product expansion.
- Trust control before automation.
- Delivery proof before scale.
- Repetition before SaaS.
- Evidence before stage advancement.

## Metrics
- current stage.
- exit criteria completed.
- blocked criteria.
- next stage readiness.

## Evidence
- private pipeline.
- payment or PO.
- delivery report.
- feedback.
- weekly review.
- GitHub checks.
