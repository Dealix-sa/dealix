# Client Data Handling SOP (Arabic)

إجراءات تشغيلية موحّدة للتعامل مع بيانات العميل

## الغرض

تعريف كيفية جمع بيانات العميل وتخزينها والوصول إليها ومشاركتها
وحذفها عبر دورة حياة العميل الكاملة، مع الالتزام بنظام حماية
البيانات الشخصية السعودي (PDPL).

## دورة حياة البيانات

### 1. الجمع (Collection)

- **المصدر الموثّق فقط**: بيانات العميل تُجمع من مصادر Tier1
  موثّقة (راجع `revenue_os/source_registry.py`).
- **الحد الأدنى**: تُجمع فقط الحقول الضرورية للغرض المعلن.
- **إشعار الموافقة**: عند نقطة الجمع، يُعرض إشعار واضح يبيّن:
  الغرض، الأساس القانوني، فترة الاحتفاظ، حقوق صاحب البيانات.
- **تسجيل الموافقة**: الموافقة تُسجّل في جدول الموافقات مع:
  التوقيت، القناة، النص، نسخة السياسة، المعرّف.

### 2. التخزين (Storage)

- **التشفير**: البيانات مشفّرة في السكون (at rest) وأثناء النقل (TLS).
- **العزل**: بيانات كل مستأجر (tenant) معزولة عبر `tenant_id`.
- **ضوابط الوصول**: وصول على مبدأ أقل امتياز عبر
  `CLIENT_ACCESS_CONTROL_POLICY_AR.md`.
- **سجلات الوصول**: كل وصول لبيانات شخصية يُسجَّل في audit log.

### 3. الاستخدام (Use)

- **الغرض المُعلَن فقط**: لا تُستخدم البيانات لغرض غير المُعلَن
  دون موافقة جديدة.
- **لا تخصيب غير مصرّح**: التخصيب (enrichment) يخضع لسياسة
  المصادر.
- **لا مشاركة خارجية**: لا تُشارك البيانات مع طرف ثالث إلا
  بعقد معالجة بيانات (DPA) وضوابط مناسبة.

### 4. الاستبقاء (Retention)

- جدول الاستبقاء موثّق في `PDPL_RETENTION_POLICY.md`.
- البيانات تُحذف آمناً بعد انتهاء فترة الاستبقاء.
- استثناءات الاستبقاء القانوني موثّقة ومبرّرة.

### 5. الحذف (Deletion)

- طلب حذف من العميل يُعالَج خلال 30 يوماً عبر `DSAR_SOP_AR.md`.
- الحذف آمن ويشمل: البيانات الأساسية، النسخ الاحتياطية بحسب
  الجدول، السجلات المرتبطة، بيانات التتبّع.
- يُسجَّل في سجل عمليات الحذف.

## التصنيف

| فئة البيانات | مستوى الحساسية | ضوابط الوصول |
|---|---|---|
| معلومات الاتصال (إيميل، هاتف) | متوسط | موظفو المبيعات المصرّح لهم |
| بيانات مالية | عالي | فريق المالية فقط |
| بيانات الهوية | عالي | موظفو الامتثال فقط |
| سجلات الاستخدام | منخفض | فريق المنتج |
| المحتوى المولّد بالذكاء الاصطناعي | متوسط | صاحب المسودة + المراجِع |

## القنوات

### إيميل

- يتطلّب آلية إلغاء اشتراك (unsubscribe) في كل رسالة.
- يتطلّب فحص قائمة الإيقاف (suppression) قبل الإرسال.
- راجع `EMAIL_OUTREACH_SAFETY_POLICY.md`.

### واتساب

- يتطلّب موافقة موثّقة (opt-in) قبل أي إرسال.
- يتطلّب قالب معتمد من Meta.
- راجع `WHATSAPP_OUTREACH_SAFETY_POLICY.md`.

### رسائل نصية (SMS)

- يتطلّب موافقة موثّقة وآلية STOP/opt-out.
- راجع `SMS_OUTREACH_SAFETY_POLICY.md`.

## المصادقة

- **المسؤول**: مسؤول حماية البيانات (DPO) + فريق المنتج
- **المراجعة**: ربع سنوي أو عند تغيير جوهري في مسار البيانات
