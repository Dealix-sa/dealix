# PDPL Readiness Checklist (Arabic)

قائمة جاهزية نظام حماية البيانات الشخصية السعودي (PDPL)

## الغرض

تضمن هذه القائمة أن منصة Dealix ملتزمة بنظام حماية البيانات
الشخصية السعودي (PDPL) الصادر بالمرسوم الملكي رقم (م/19) وتنفيذه
العملي عبر جميع مراحل معالجة بيانات العميل.

## المبادئ الأساسية

1. **قانونية المعالجة** — كل معالجة بيانات شخصية يجب أن تتم على أساس
   قانوني واضح (موافقة، عقد، التزام قانوني، مصلحة مشروعة).
2. **الحد من الغرض** — البيانات تُجمع لغرض محدد ولا تُستخدم لغرض آخر
   دون موافقة جديدة.
3. **تقليل البيانات** — لا تُجمع إلا البيانات الضرورية للغرض المعلن.
4. **الدقة** — يجب الحفاظ على دقة البيانات وتحديثها.
5. **حد الاحتفاظ** — البيانات تُحفظ فقط للمدة الضرورية.
6. **الأمن** — حماية البيانات من الوصول غير المصرّح والتسريب.
7. **الشفافية** — إبلاغ صاحب البيانات بحقوقه وطرق المعالجة.

## قائمة الجاهزية التشغيلية

### 1. سياسات الإفصاح

- [ ] سياسة خصوصية منشورة باللغة العربية متاحة للعملاء
- [ ] إشعار جمع البيانات واضح عند نقطة التجميع
- [ ] توضيح الأساس القانوني لكل نوع معالجة
- [ ] بيان فترة الاحتفاظ بكل فئة بيانات

### 2. الموافقة (Consent)

- [ ] آلية تسجيل موافقة صريحة لكل قناة (إيميل، واتساب، رسائل نصية)
- [ ] سجل الموافقة يحتوي: التوقيت، القناة، النص المُوافَق عليه، نسخة
- [ ] الموافقة قابلة للسحب في أي وقت عبر آلية واضحة
- [ ] سحب الموافقة يُفعّل قائمة الإيقاف (suppression) فوراً

### 3. حقوق صاحب البيانات (DSAR)

- [ ] آلية استقبال طلبات الوصول/التصحيح/الحذف
- [ ] الرد على الطلب خلال 30 يوماً كحد أقصى
- [ ] توثيق كل طلب وسجل الردود
- [ ] آلية تصعيد للطلبات المعقدة أو المتنازع عليها

### 4. حماية البيانات والتقنية

- [ ] التشفير أثناء النقل (TLS) وفي السكون (at rest)
- [ ] ضوابط الوصول على مبدأ أقل امتياز
- [ ] سجلات تدقيق (audit logs) لكل وصول لبيانات شخصية
- [ ] اختبار اختراق دوري ومراجعة أمنية

### 5. خرق البيانات (Breach)

- [ ] خطة استجابة لخرق البيانات موثّقة ومُختبَرة
- [ ] إبلاغ الهيئة السعودية للبيانات خلال 72 ساعة من الاكتشاف
- [ ] إبلاغ المتأثرين إذا كان الخطر مرتفعاً
- [ ] سجل حوادث الخرق مع التحليل التصحيحي

### 6. نقل البيانات عبر الحدود

- [ ] توثيق وجهات نقل البيانات وموفري الخدمة
- [ ] التأكد من مستوى حماية كافٍ في وجهة النقل
- [ ] موافقة صريحة لنقل البيانات خارج المملكة عند الحاجة

### 7. الاحتفاظ والحذف

- [ ] جدول احتفاظ موثّق لكل فئة بيانات
- [ ] آلية حذف آمنة بعد انتهاء فترة الاحتفاظ
- [ ] سجل عمليات الحذف
- [ ] استثناءات الاحتفاظ القانوني موثّقة

## المصادقة

- **المسؤول**: مسؤول حماية البيانات (DPO)
- **المراجعة**: كل ربع سنوي أو عند تغيير جوهري في المعالجة
- **الدليل**: راجع `PDPL_RETENTION_POLICY.md` و `PDPL_BREACH_RUNBOOK.md`
