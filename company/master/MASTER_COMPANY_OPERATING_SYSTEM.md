# Dealix Master Company Operating System

## Purpose
This is the executive operating layer for Dealix.

It turns Dealix into a daily-running company system:
- production checks
- frontend checks
- lead research
- target scoring
- daily offer generation
- outreach drafts
- approval queue
- CRM update
- proposal stubs
- client delivery workflow
- finance/collection rules
- trust posture review
- CEO daily report

## Operating principle
The system prepares everything. The founder approves external actions.

## What can run automatically
- Generate targets
- Generate weakness analysis
- Generate drafts
- Generate proposal stubs
- Update internal CRM queues
- Create daily CEO report
- Create weekly board report
- Run production/frontend checks
- Upload GitHub artifacts

## What requires approval
- Sending WhatsApp messages
- Sending cold email
- Quoting final price
- Issuing invoices
- Charging clients
- Signing contracts
- Publishing public content
- Exporting customer data
- Changing production secrets

## Daily operating standard
Minimum every day:
- Production health: PASS
- Frontend verify: PASS or known issue
- 50 targets researched
- 20 draft-ready prospects
- 20 reviewed outbound messages
- 5 follow-ups
- 2 discovery calls targeted
- 1 proposal or diagnostic offer
- CRM updated
- CEO report generated

## Weekly operating standard
Every week:
- Review best segment
- Review reply rate
- Review proposals sent
- Review cash collected
- Review client delivery
- Review trust/security
- Decide next offer focus
