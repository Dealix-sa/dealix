# Dealix Definition of Done

## A System Is Done For Now When
- it has an operating document.
- it has templates if needed.
- it has a verifier.
- it is included in master blueprint.
- it has private ops evidence or clear reason why not yet.
- it does not violate trust or data boundary.
- it supports cash, proof, retention, trust, learning, or founder leverage.

## A Revenue Sprint Is Done When
- payment / PO / written approval exists.
- intake is complete.
- delivery report is complete.
- QA passed.
- handoff complete.
- feedback requested.
- retainer path evaluated.
- learning recorded.

## A Week Is Done When
- pipeline updated.
- revenue actions logged.
- trust review complete.
- weekly learning complete.
- one system update made.
- next week focus selected.

## A Month Is Done When
- board pack complete.
- finance review complete.
- pricing review complete.
- productization review complete.
- capital allocation reviewed.
- strategy decision made.

## A Product Feature Is Done When
- workflow repeated.
- data model stable.
- risk classified.
- QA exists.
- verifier exists.
- rollback/disable path exists.
- it saves time or improves revenue/delivery/trust.
