# User Guide

## Overview
This guide explains how the client team should use the delivered Dealix operating system day to day.

## Daily Workflow
1. open the operating page
2. review queues, risks, and decisions
3. approve or reject pending drafts
4. update key records after conversations or meetings
5. review today’s next actions

## Weekly Routine
1. review pipeline and follow-up health
2. review decision and risk registers
3. review proof or scorecard outputs
4. adjust process based on actual bottlenecks

## Safety Reminders
- keep outbound approval-gated
- do not enable live send casually
- avoid unreviewed AI claims
- review trust-sensitive content before sending
