# Multi-Threading System

Enterprise deals close on multiple threads. Single-threaded deals die when
one person leaves, gets reorganized, or loses authority. This system makes
multi-threading a default, not a hope.

## Thread definition

A *thread* is an active conversation cadence with one named stakeholder.
The conversation has a purpose, a frequency, and a last-touch date.

## Minimum threads per deal

| Account size | Minimum threads | Required roles |
|---|---|---|
| < 100k SAR ARR | 2 | Economic, Champion |
| 100k–500k SAR | 3 | Economic, Champion, Security/IT |
| ≥ 500k SAR | 4+ | Economic, Champion, Security/IT, Procurement/Legal |

Falling below the minimum for two weeks triggers a `WARN` in the daily
brief.

## Thread health checklist

For every thread, weekly:

- [ ] Last touch within 14 days
- [ ] Next step has a date
- [ ] Stakeholder has not changed role
- [ ] Touchpoint cadence respects stated preferences

## Recording

In [STAKEHOLDER_MAP](STAKEHOLDER_MAP.md) under each role's "touchpoint
cadence". Aggregate health rolls up into the daily brief for any account
in the [`docs/strategy/STRATEGIC_ACCOUNT_LIST.md`](../strategy/STRATEGIC_ACCOUNT_LIST.md).
