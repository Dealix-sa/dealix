## العربية

### الفلسفة
لا «اطلاق كبير». البداية بوكيل واحد، أداة واحدة، وKPI واحد. توسعة فقط بعد ظهور أوّل Evidence Pack.

### اليوم 0 — التهيئة (قبل البداية)
- توقيع NDA + DPA.
- تعيين راعي البرنامج، مالك الحوكمة، مالك البيانات، DPO.
- ورشة DPIA-light.
- قرار سيادة البيانات (داخل المملكة افتراضيًا).

### الأيام 1-30 — التأسيس
- إنشاء المستأجِر.
- نشر AI Use Policy (مُكيَّفة).
- تسجيل أوّل وكيل واحد (T0 أو T1) في Agent Registry.
- بناء Tool Permission Matrix لهذا الوكيل.
- إصدار أوّل 3 Evidence Packs.
- اجتماع مجلس الحوكمة الأول.

**معيار المرور للمرحلة التالية**: 3 Evidence Packs مكتملة، 0 إرسال خارجي بدون موافقة، 0 PII في السجلات.

### الأيام 31-60 — التوسع المحدود
- إضافة وكيل ثانٍ (يسمح به T0/T1 فقط).
- ربط CRM والمصادر المُصرَّح بها.
- مراجعة MCP لخادم واحد (إن وُجد).
- بدء تقرير القيمة الشهري.
- تمرين Incident Response الأول.

**معيار المرور**: 10 Evidence Packs، تقرير قيمة موقّع، تمرين ناجح.

### الأيام 61-90 — التشغيل المستقر
- إدخال أوّل وكيل T2 (بموافقة مزدوجة).
- مصادقة على Agent Governance Council كإيقاع شهري.
- نشر سياسة سحب الوكيل ودورة حياتها.
- مراجعة فصلية أولى.

**مخرَج المرحلة**: تشغيل مستقرّ، تقرير قيمة جاهز للمجلس، خارطة Q+1.

### المخرجات المُسلَّمة
- Agent Registry حيّ.
- Tool Permission Matrix حيّة.
- AI Use Policy مُعتمَدة.
- أوّل 10 Evidence Packs مُتاحة للتدقيق.
- تقرير القيمة الأول.
- خطّة Q+1.

### ما لا يحدث في 90 يومًا
- لا «تحوّل ذكاء اصطناعي» شامل.
- لا 50 وكيلًا.
- لا أتمتة خارجية بدون موافقات.
- لا أرقام إيراد مُتحقَّق دون Evidence Packs مكتملة.

## English

### Philosophy
No "big bang". Start with one agent, one tool, one KPI. Expand only after the first Evidence Pack appears.

### Day 0 — Setup (before start)
- Signed NDA + DPA.
- Assign executive sponsor, governance lead, data owner, DPO.
- DPIA-light workshop.
- Data residency decision (Kingdom by default).

### Days 1-30 — Foundation
- Provision the tenant.
- Publish a tailored AI Use Policy.
- Register the first single agent (T0 or T1) in the Agent Registry.
- Build the Tool Permission Matrix for that agent.
- Produce the first 3 Evidence Packs.
- Hold the first Governance Council meeting.

**Gate to next phase**: 3 complete Evidence Packs, 0 external sends without approval, 0 PII in logs.

### Days 31-60 — Limited Expansion
- Add a second agent (T0/T1 only).
- Wire CRM and authorized sources.
- One MCP server review (if applicable).
- Begin the monthly Value Report.
- First Incident Response tabletop.

**Gate**: 10 Evidence Packs, signed Value Report, successful tabletop.

### Days 61-90 — Stable Operation
- Introduce the first T2 agent (with dual approval).
- Ratify the AI Agent Governance Council as a monthly cadence.
- Publish the agent retirement policy and lifecycle.
- Conduct the first quarterly review.

**Phase exit**: stable operation, board-ready Value Report, Q+1 roadmap.

### Delivered Outputs
- Live Agent Registry.
- Live Tool Permission Matrix.
- Adopted AI Use Policy.
- First 10 Evidence Packs available for audit.
- First Value Report.
- Q+1 plan.

### What Does Not Happen in 90 Days
- No company-wide "AI transformation".
- No 50-agent estate.
- No external automation without approvals.
- No verified revenue figures without complete Evidence Packs.
