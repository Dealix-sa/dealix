# Enterprise Sales Motion

How Dealix sells into accounts with procurement, legal, security review, and
multiple stakeholders. Builds on the existing positioning material in
[`ENTERPRISE_POSITIONING`](ENTERPRISE_POSITIONING.md) and the long-form plan
in [`ROAD_TO_ENTERPRISE`](ROAD_TO_ENTERPRISE.md).

## The stages

```
1. Discovery       → STAKEHOLDER_MAP filled
2. Validation      → 1-2 working pilots, signed by champion
3. Procurement     → PROCUREMENT_READINESS checklist passed
4. Security review → SECURITY_REVIEW_PACKET sent
5. Proposal        → ENTERPRISE_PROPOSAL_FLOW
6. Negotiation     → DEAL_DESK_SYSTEM
7. Signed          → kickoff
```

Each stage has an entry condition and an exit condition; deals that try to
jump stages get caught in the [`docs/revenue/PIPELINE_REVIEW_RHYTHM.md`](../revenue/PIPELINE_REVIEW_RHYTHM.md)
weekly review.

## Cadence

- Per-deal: weekly status row in [`docs/ops/CEO_TOP50_TRACKER.csv`](../ops/CEO_TOP50_TRACKER.csv)
- Per-account: monthly review against the [STAKEHOLDER_MAP](STAKEHOLDER_MAP.md)

## Outputs

- Stage transitions logged to evidence tracker
- Decisions logged to [`docs/founder/DECISION_LOG_SYSTEM.md`](../founder/DECISION_LOG_SYSTEM.md)

## Non-negotiables

Enterprise deals still flow through the approval-center gates and deal-desk
process. No commitments to terms, refunds, or guarantees outside the
documented offers. See [`docs/founder/DO_NOT_SAY.md`](../founder/DO_NOT_SAY.md).
