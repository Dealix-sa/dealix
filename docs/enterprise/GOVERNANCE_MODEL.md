## العربية

### 1. المبدأ
الحوكمة عند Dealix ليست لجنة مراجعة، بل **سطح تشغيل** يومي: Agent Registry، Tool Permission Matrix، Evidence Packs، وAI Run Ledger.

### 2. الأدوار المؤسسية
- **راعي البرنامج (Executive Sponsor)** — يتحمّل المسؤولية النهائية.
- **مالك الحوكمة (Governance Lead)** — يدير Agent Registry والمصفوفة شهريًا.
- **مالك البيانات (Data Owner)** — يُصادق على نطاق البيانات لكل وكيل.
- **مالك المخاطر (Risk Owner)** — يصنّف وكلاء T2 وT3.
- **DPO** — يوقّع على ما يمس بيانات شخصية.
- **مالك الوكيل (Agent Owner)** — اسم بشري لكل وكيل.

### 3. مجلس حوكمة الوكلاء (AI Agent Governance Council)
- اجتماع شهري.
- مراجعة انحراف Agent Registry.
- مراجعة تغييرات Tool Permission Matrix.
- مراجعة عيّنة Evidence Packs.
- تصديق على وكلاء جدد بمستوى T2 فأعلى.

### 5. الإيقاعات
| الإيقاع | المخرَج |
|---|---|
| يومي | تقرير AI Run Ledger للأحداث الحرجة |
| أسبوعي | مراجعة الموافقات المُعلَّقة |
| شهري | تقرير انحراف Agent Registry + عيّنة Evidence Packs |
| فصلي | تمرين Incident Response Tabletop + مراجعة سياسة |
| سنوي | مراجعة قُدمى الوكلاء والاستغناء عمّا لم يُستخدم |

### 6. عتبات التصعيد
- T0: تنفيذ تلقائي، تدقيق دوري.
- T1: تنفيذ مع توقيع المالك.
- T2: تنفيذ مع موافقة المالك + مالك المخاطر.
- T3: تنفيذ مع موافقتين مستقلّتين + توقيع المؤسس.

### 7. مقاييس الحوكمة
- نسبة الوكلاء بمالك بشر مُسمّى = **100%**.
- متوسط زمن مراجعة تغيير في المصفوفة.
- عدد الحوادث المُحتواة قبل الإرسال الخارجي.
- نسبة Evidence Packs المُتاحة للتدقيق.

## English

### 1. Principle
Governance at Dealix is not a review committee; it is a daily **operating surface**: Agent Registry, Tool Permission Matrix, Evidence Packs, and AI Run Ledger.

### 2. Enterprise Roles
- **Executive Sponsor** — ultimate accountability.
- **Governance Lead** — runs monthly Agent Registry and matrix reviews.
- **Data Owner** — certifies the data scope for each agent.
- **Risk Owner** — classifies T2 and T3 agents.
- **DPO** — signs off on anything touching personal data.
- **Agent Owner** — named human per agent.

### 3. AI Agent Governance Council
- Monthly meeting.
- Reviews Agent Registry drift.
- Reviews Tool Permission Matrix changes.
- Reviews an Evidence Pack sample.
- Certifies new T2-and-above agents.

### 4. Policies
- [AI Use Policy](../trust/AI_USE_POLICY_SAMPLE.md)

- [Incident Response](../trust/INCIDENT_RESPONSE_SAMPLE.md)

### 5. Cadences
| Cadence | Output |
|---|---|
| Daily | AI Run Ledger digest for critical events |
| Weekly | Pending approvals review |
| Monthly | Agent Registry drift report + Evidence Pack sample |
| Quarterly | Incident Response tabletop + policy review |
| Annual | Agent vintage review and retirement |

### 6. Escalation Thresholds
- T0: auto-executed, periodic audit.
- T1: executed with owner sign-off.
- T2: executed with owner + risk-owner approval.
- T3: executed with two independent approvals + founder sign-off.

### 7. Governance Metrics
- Share of agents with a named human owner = **100%**.
- Mean time to review a matrix change.
- Number of incidents contained before external send.
- Share of Evidence Packs available for audit.
