## 3. الحل المقترح — Dealix Executive Command Center

### ما يفعله Dealix لـ [اسم الشركة]

**المرحلة الأولى — قراءة الـ pipeline:**
- Dealix يقرأ pipeline العميل الحالي من المصدر الذي يتيحه العميل
  (CRM export، جدول، أو سجل فرص)
- لا scraping، لا قوائم مشتراة — فقط بيانات العميل نفسه

**المرحلة الثانية — ترتيب الفرص:**
- ترتيب الفرص حسب الأولوية مع تبرير واضح لكل ترتيب
- تصنيف استشاري للفرصة (أولوية عالية / متوسطة / منخفضة)
- التصنيف **استشاري وغير مُلزم** — القرار النهائي للعميل

**المرحلة الثالثة — صياغة المسوّدات:**
- مسوّدات تواصل ومتابعة بالعربي الخليجي لكل فرصة ذات أولوية
- **مسوّدة فقط** — يراجع العميل كل مسوّدة ويوافق ويرسلها بنفسه
- لا إرسال آلي، لا حجز اجتماعات آلي

**المرحلة الرابعة — الإثبات:**
- قبل كل اجتماع: ملخّص فرصة جاهز (1 صفحة) للمراجعة
- Proof Pack موثق يوضّح المشكلة والمدخلات والعمل والقيمة القابلة للقياس،
  مع تمييز ما هو مُتحقَّق عمّا هو تقديري
