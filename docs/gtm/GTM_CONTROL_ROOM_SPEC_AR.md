# GTM Control Room — مواصفة غرفة التحكم التجارية — Spec for a future ops page

> **هذه مواصفة (spec) لا واجهة قائمة.** واجهة `/[locale]/ops/gtm-control` موجة لاحقة؛ صفحات `ops` الحالية كلها **stubs** (founder, war-room, marketing, sales, partners, approvals, evidence, targeting, support) ولا توجد صفحة `gtm-control` بعد. هذه الوثيقة **رابطة**: تحدد ما ستعرضه الصفحة وأي مخرجات قائمة ستقرؤها، دون تكرار منطق الوحدات. **This is a spec, not a live UI.**

---

## 1) الغرض — Purpose (AR)

غرفة تحكم واحدة تعطي المؤسس **أمر اليوم** في شاشة واحدة: حالة إنتاج 250 مسودة، أعلى 50 في طابور الموافقة، خطة الإرسال المعتمَدة، الردود الإيجابية، أفضل قطاع، صحة الدومين، تنبيهات الخصوصية، قيمة خط الأنابيب، وتوصية الغد. الصفحة **تعرض ولا تُرسل**: كل إرسال أو تسعير أو التزام يبقى قرار المؤسس عبر [`os/06_APPROVAL_GATES.yml`](../../os/06_APPROVAL_GATES.yml).

## 1) Purpose (EN)

One control room that gives the founder the **daily order** on a single screen: 250-draft production status, the top 50 in the approval queue, the approved sending plan, positive replies, the best sector, domain health, privacy warnings, pipeline value, and tomorrow's recommendation. The page **displays, it does not send**: every send, price, or commitment stays a founder decision via [`os/06_APPROVAL_GATES.yml`](../../os/06_APPROVAL_GATES.yml).

## 2) ما تقرؤه الصفحة — Data sources (AR)

الصفحة لا تحسب من جديد؛ تُصيِّر مخرجات قائمة:

- **الأمر اليومي:** [`reports/gtm/GTM_DAILY_COMMAND.md`](../../reports/gtm/GTM_DAILY_COMMAND.md) المولَّد من [`scripts/gtm_daily_command.py`](../../scripts/gtm_daily_command.py).
- **بوابة الجودة:** `summarize_gate_results()` في [`auto_client_acquisition/gtm_os/draft_quality_gate.py`](../../auto_client_acquisition/gtm_os/draft_quality_gate.py) (`approval_ready_ids`, `blocked_ids`, `top_failure_reasons`).
- **خطة الإرسال/التدرّج:** `plan_sending_batches()` و`ramp_stage_for()` في [`auto_client_acquisition/gtm_os/sending_ramp.py`](../../auto_client_acquisition/gtm_os/sending_ramp.py).
- **العملاء والإشارات:** [`auto_client_acquisition/gtm_os/records.py`](../../auto_client_acquisition/gtm_os/records.py) + [`prospects/PROSPECT_OS_AR.md`](prospects/PROSPECT_OS_AR.md).

## 2) Data Sources (EN)

The page does not recompute; it renders existing outputs:

- **Daily command:** [`reports/gtm/GTM_DAILY_COMMAND.md`](../../reports/gtm/GTM_DAILY_COMMAND.md) generated by [`scripts/gtm_daily_command.py`](../../scripts/gtm_daily_command.py).
- **Quality gate:** `summarize_gate_results()` in [`gtm_os/draft_quality_gate.py`](../../auto_client_acquisition/gtm_os/draft_quality_gate.py).
- **Sending / ramp:** `plan_sending_batches()` and `ramp_stage_for()` in [`gtm_os/sending_ramp.py`](../../auto_client_acquisition/gtm_os/sending_ramp.py).
- **Prospects / signals:** [`gtm_os/records.py`](../../auto_client_acquisition/gtm_os/records.py) + [`prospects/PROSPECT_OS_AR.md`](prospects/PROSPECT_OS_AR.md).

## 4) البطاقات العلوية — Top cards (AR/EN)

| البطاقة — Card | ماذا تُظهر — What it shows | المصدر — Source |
|---|---|---|
| **250 Draft Production Status** | المحمَّل / الجاهز / المحجوب اليوم | `summarize_gate_results()` |
| **Top 50 Approval Queue** | أعلى 50 مسودة جاهزة للموافقة | `approval_ready_ids[:50]` |
| **Approved Sending Batch** | الدفعات المعتمَدة + السقف الفعّال | `plan_sending_batches()` |
| **Positive Replies** | عدد الردود `positive` وتوجيهها | `route_reply` |
| **Best Sector Today** | أفضل قطاع أداءً (تقديري) | تجميع الدفعات/الردود |
| **Domain Health** | صحة الدومين ومرحلة التدرّج | `ramp_stage_for()` / `domain_health` |
| **Privacy Warnings** | تنبيهات PII / كبح / opt-in | فحوص PII + قائمة الكبح |
| **Pipeline Value** | قيمة خط الأنابيب **التقديرية** | تجميع داخلي |
| **Tomorrow Recommendation** | ضاعِف الأفضل، أوقف أسوأ 20% | قسم «توصية الغد» في الأمر اليومي |

البطاقات تعكس أقسام [`reports/gtm/GTM_DAILY_COMMAND.md`](../../reports/gtm/GTM_DAILY_COMMAND.md) واحدًا لواحد قدر الإمكان، فيبقى مصدر الحقيقة هو السكربت لا الواجهة.

The cards mirror the sections of [`reports/gtm/GTM_DAILY_COMMAND.md`](../../reports/gtm/GTM_DAILY_COMMAND.md) one-to-one where possible, so the script — not the UI — stays the source of truth. "Pipeline Value" is explicitly an **estimate**.

## 5) أزرار الإجراء — قرار بشري فقط — Action buttons are human-only (AR)

أزرار «اعتماد / رفض / إعادة صياغة / اختصار / اجعله رسميًا / غيّر العرض / انقله للتنشئة / لا تتواصل» تطابق قرارات المؤسس في [`FOUNDER_APPROVAL_QUEUE_AR.md`](FOUNDER_APPROVAL_QUEUE_AR.md) وتُمرَّر إلى [`approval_center/`](../../auto_client_acquisition/approval_center/). لا زر «إرسال للجميع»، لا أتمتة إرسال، لا واتساب بارد. الإرسال الفعلي لواتساب يبقى خلف `WHATSAPP_ALLOW_LIVE_SEND` (افتراضي false).

## 5) Action Buttons Are Human-Only (EN)

"Approve / Reject / Rewrite / Shorten / Make formal / Change offer / Move to nurture / Do not contact" match the founder decisions in [`FOUNDER_APPROVAL_QUEUE_AR.md`](FOUNDER_APPROVAL_QUEUE_AR.md) and pass to [`approval_center/`](../../auto_client_acquisition/approval_center/). No "send to all" button, no send automation, no cold WhatsApp. Live WhatsApp send stays behind `WHATSAPP_ALLOW_LIVE_SEND` (default false).

## 6) حالة التنفيذ — Build status (AR)

- **اليوم:** صفحات `ops` الحالية stubs؛ لا صفحة `gtm-control`. الأمر اليومي يُولَّد بالفعل في [`reports/gtm/GTM_DAILY_COMMAND.md`](../../reports/gtm/GTM_DAILY_COMMAND.md).
- **الموجة اللاحقة:** صفحة `/[locale]/ops/gtm-control` تُصيِّر هذا التقرير + مخرجات البوابة/التدرّج. أزرار القرار تُمرَّر للموافقة فقط.
- **خارج النطاق:** أي إرسال مباشر، أي قناة جديدة، أي سعر مُخترَع.

## 6) Build Status (EN)

- **Today:** existing `ops` pages are stubs; there is no `gtm-control` page. The daily command is already generated at [`reports/gtm/GTM_DAILY_COMMAND.md`](../../reports/gtm/GTM_DAILY_COMMAND.md).
- **Follow-up wave:** a `/[locale]/ops/gtm-control` page renders this report plus gate/ramp outputs. Decision buttons route to approval only.
- **Out of scope:** any direct send, any new channel, any invented price.
