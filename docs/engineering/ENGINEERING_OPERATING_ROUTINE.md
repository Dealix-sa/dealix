# Engineering Operating Routine

## Daily
Only fix engineering if it blocks revenue, delivery, trust, or CEO execution.

## Weekly
- review failed checks
- review automation candidates
- update one verifier if needed
- keep CI green

## Monthly
- run productization review
- review repeated workflows
- review SaaS readiness gate
- review engineering complexity

## Rules
- Do not build new platform layer before evidence.
- Do not add agent capabilities before release gate.
- Do not add external automation before trust approval.
- Keep the system boring, traceable, and useful.
