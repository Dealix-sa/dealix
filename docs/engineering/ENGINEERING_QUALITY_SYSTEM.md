# Engineering Quality System

## Purpose
Keep Dealix engineering reliable, simple, and aligned with business evidence.

## Current Quality Gates
- compileall passes
- verifiers pass
- data boundary passes
- no autonomous external actions
- trust terms scan passes
- public/private boundary respected

## Required Checks
- implementation audit
- data architecture
- revenue ops
- delivery client success
- execution assurance
- strategic CEO checks
- security checks

## Engineering Principles
- business evidence before product scope
- small scripts before platforms
- readable code before clever code
- tests and verifiers before expansion
- safety checks before agents
- local/private data boundary always

## DORA Later
Track only when there is real software delivery:
- deployment frequency
- lead time for changes
- change failure rate
- failed deployment recovery time

## Rule
Do not optimize engineering metrics before engineering is the bottleneck.
