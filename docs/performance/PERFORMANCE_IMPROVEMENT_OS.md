# Performance Improvement OS

A weekly loop that turns ledger data into one prioritised gap and one
proposed experiment.

## 1. The loop

```
Monday  → KPI snapshot (performance_analyst).
Tuesday → Gap prioritisation (founder review).
Wed     → Experiment proposed (performance_analyst).
Thu     → Experiment approved or rejected (founder).
Fri     → Experiment launched in-system (no external send change).
Next Mon→ Re-measure.
```

## 5. Banned patterns

- ❌ Adding KPIs without removing or de-prioritising others.
- ❌ Running > 3 concurrent experiments without explicit founder OK.
- ❌ Claiming a win without a re-measurement on fresh data.
