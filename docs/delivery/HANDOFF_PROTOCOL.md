# Handoff Protocol

## Purpose
Make sure the client knows exactly what to do after receiving the Revenue Sprint report.

## Handoff Must Include
- what was delivered
- how to use the lead table
- first action to take
- outreach cadence
- what to track
- when to review results
- optional next Dealix support

## Handoff Rule
No delivery is complete until the client has a clear next action.

## Recommended Cadence
- Day 1: send first outreach.
- Day 3: follow up.
- Day 7: send value follow-up.
- Day 14: close, recycle, or escalate.

## Metrics
- client understood next action
- feedback received
- retainer conversation opened

## Anti-Patterns
- "Here is the report, good luck."
- Handoff without a calendar suggestion.
- Handoff without a feedback request window.
- Pushing a retainer in the same message as the deliverable.

## Trust Note
The handoff is the moment a one-time delivery becomes a relationship.
Do not optimize for upsell; optimize for the client's next correct action.
