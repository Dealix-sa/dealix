# Delivery Kickoff Protocol

## Purpose
Ensure every delivery starts with clear scope, inputs, timeline, and trust boundaries.

## Trigger
Use after:
- payment confirmation
- purchase order
- written approval

## Required Before Kickoff
- approved proposal
- start condition confirmed
- client workspace created
- intake template ready
- QA checklist ready
- delivery target date set

## Kickoff Questions
1. What is the client selling?
2. Who is their best customer?
3. Which sectors matter most?
4. Which sectors should be excluded?
5. What makes a bad lead?
6. What would make this sprint valuable?
7. Who approves final delivery?
8. Are there any data or confidentiality constraints?

## Output
- completed intake
- confirmed scope
- client success definition
- delivery timeline
- risk notes

## Rule
If scope is unclear, delivery does not start.

## Anti-Patterns
- Starting research without an intake.
- Promising specific revenue numbers in the kickoff call.
- Skipping the bad-fit definition.
- Skipping the approval contact.

## Trust Note
Kickoff is a trust act, not a sales act. Set realistic expectations: Dealix
delivers opportunity intelligence, not guaranteed sales.
