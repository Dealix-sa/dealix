# Scope Engine

Scope is the main **profit and protection** tool. If it is not written, it is **not binding**.

## Every scope must include

```markdown
# Scope

## Service
Which Dealix service / SKU?

## Client Goal
What outcome does the client want? (one metric preferred)

## Included
What we will do.

## Excluded
What we will not do.

## Inputs Required
What the client must provide (data, access, approvers).

## Timeline
Milestones and dates.

## Review Points
When the client reviews (format: demo / call / async).

## Acceptance Criteria
What objectively counts as done.

## Change Request Rule
Anything outside this scope becomes a **paid change**, **new sprint**, or **explicit retainer item** — see [`CHANGE_REQUEST_SYSTEM.md`](CHANGE_REQUEST_SYSTEM.md).
```

## Rule

Any client demand **not** reflected in signed/written scope = **not committed**. Route through change request.

**Templates:** `clients/_TEMPLATE/00_scope.md`, [`../contracts/`](../contracts/) where applicable.
