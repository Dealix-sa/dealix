# Delivery Rescue Pack

## Purpose
Recover any delivery at risk of delay, poor quality, or scope mismatch.

## Trigger
Use when:
- QA fails
- delivery is late
- scope unclear
- evidence weak
- client expectation mismatch

## Actions
1. Re-read proposal scope.
2. Re-run client intake.
3. Identify missing evidence.
4. Run QA checklist.
5. Remove weak leads.
6. Rewrite executive memo.
7. Get approval before handoff.

## Metrics
- QA fixes
- rework count
- delivery delay
- client feedback

## Evidence
- QA checklist
- delivery report
- approval log
