# Revenue Sprint QA Checklist

## Purpose
Block low-quality, unsafe, or unevidenced deliveries before they leave Dealix.

## Owner
Sami / Delivery owner.

## When To Run
Day 6 of every Revenue Sprint, before handoff.

## Pre-handoff Checks

### Scope
- [ ] Delivered scope matches the signed proposal.
- [ ] Any scope drift is documented and accepted.

### Evidence
- [ ] Every A-priority account has an evidence link.
- [ ] No claim in the memo is unsupported.
- [ ] Source list is recorded in the delivery folder.

### Trust
- [ ] No banned claim language anywhere in the pack (see NO_OVERCLAIM_POLICY).
- [ ] All public-language items reframed to evidence/process language where needed.
- [ ] No PII of unrelated third parties included.

### Quality
- [ ] Memo is one page. Findings are concise. Action plan has dates.
- [ ] Outreach pack has AR and EN versions where required.
- [ ] All file names follow the standard.

### Governance
- [ ] Founder approval recorded.
- [ ] QA outcome logged.

### Privacy
- [ ] Client name does not appear in public repo commits made during the sprint.
- [ ] Any sample shared publicly uses synthetic data.
