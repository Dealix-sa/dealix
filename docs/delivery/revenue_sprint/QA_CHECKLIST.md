# Revenue Sprint QA Checklist

Before a wrap-up brief leaves the building, the Delivery Lead signs this checklist. Every item is binary: yes or no. A "no" blocks the sprint from closing.

## Scope

- [ ] The named bottleneck in the kick-off note is the same one addressed in the artefact.
- [ ] No out-of-scope work was silently added.
- [ ] If a Change Request was raised mid-sprint, it is logged in the AI Run Ledger.

## Evidence

- [ ] AI Run Ledger has at least one entry tagged to this sprint ID.
- [ ] The Proof Pack snippet has been generated and attached.
- [ ] The wrap-up brief references at least one verifiable artefact path.

## No overclaim

- [ ] No "guaranteed revenue" language anywhere in deliverables.
- [ ] No "100% compliance" language anywhere in deliverables.
- [ ] No "fully autonomous" language.
- [ ] No "no human needed" language.
- [ ] No "guaranteed sales" language.
- [ ] Every revenue figure carries an estimated / observed / verified tag.

## Approvals

- [ ] All outbound messages were A1-approved by a human.
- [ ] No A3 actions were attempted (regulatory, legal, third-party data export).
- [ ] If a public claim is being made (case study), A2 sign-off is on file.
