# Revenue Sprint — Delivery QA Checklist (Public)

This is the public version of the QA checklist used to gate every
Revenue Sprint delivery. The private working copy lives in
`dealix-ops-private/offers/revenue_sprint/qa_checklist.md`.

## Lead Quality
- [ ] Every lead matches the agreed ICP.
- [ ] Every A-priority lead has evidence.
- [ ] Duplicates removed.
- [ ] Bad-fit leads removed.
- [ ] Sector relevance is clear for each lead.

## Outreach Quality
- [ ] Messages are specific to the account.
- [ ] Messages are short.
- [ ] CTA is clear.
- [ ] No fake urgency.
- [ ] No guaranteed results.

## Trust Quality
- [ ] No sensitive client data exposed in any deliverable.
- [ ] No unsupported public claim.
- [ ] No compliance claim.
- [ ] No client name used publicly without written approval.
- [ ] Approval requirement checked for any external publication.

## Delivery Quality
- [ ] Executive summary included.
- [ ] Lead table included.
- [ ] Outreach pack included.
- [ ] 14-day action plan included.
- [ ] Next step included.

## Final Decision
PASS / FIX / BLOCKED

## Logging
- Each sprint records the result of this checklist in the private
  delivery log under `dealix-ops-private/offers/revenue_sprint/`.
- A sprint cannot be marked delivered without a PASS result.
