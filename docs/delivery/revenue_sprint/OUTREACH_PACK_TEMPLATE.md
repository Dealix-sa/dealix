# Outreach Pack — Template

> Per-account, per-priority, bilingual.

## Inputs
- Account row (from Lead Table)
- ICP definition
- Tone preference
- Sector pack (if any)

## Outputs

### DM (LinkedIn) — Arabic

```
السلام عليكم [الاسم الأول]،

لاحظنا [إشارة محددة] في [اسم الشركة]. هذا يعني عادةً [استنتاج محدد قابل للقياس].

أعدّينا نموذج لـ 3 حسابات شبيهة بحساباتكم في [القطاع]. مدته دقيقتين قراءة.

تحب نشاركه؟

— سامي
```

### DM (LinkedIn) — English

```
Hi [first name],

Saw [specific signal] at [company]. That usually means [specific
inference].

We put together a 3-account sample for [their sector] — 2-minute read.

Want it?

— Sami
```

### Email — Arabic

```
الموضوع: [إشارة محددة] في [الشركة]

[الاسم الأول]،

[فقرة واحدة: ماذا لاحظنا، ولماذا يهم].

[فقرة واحدة: ما عندنا لكم — نموذج 3 حسابات / Sprint كامل].

[سؤال واحد فقط].

— سامي
[رابط]
```

### Email — English

```
Subject: [specific signal] at [company]

[First name],

[One paragraph: what we noticed and why it matters].

[One paragraph: what we have — 3-account sample / full Sprint].

[Exactly one question].

— Sami
[link]
```

## Quality Standards

- Every claim about the prospect has an evidence reference (from the
  lead row).
- No overclaim language (`NO_OVERCLAIM_POLICY.md`).
- No "AI-powered" / "guaranteed" / "best".
- Bilingual versions say the same thing; not a literal translation.
- Subject line is specific, not generic.

## Per-priority differentiation

| Priority | Personalisation depth |
|----------|----------------------|
| A | Full personalisation; founder-signed; tailored hook |
| B | Templated personalisation; one specific hook |
| C | Light personalisation; nurture only |
| Disqualified | No outreach |

## Approval

- A: founder T2 per account before send.
- B: founder T2 batch approves a list.
- C: nurture; no immediate send.
