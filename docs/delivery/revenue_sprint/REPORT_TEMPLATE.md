# Report Template — Revenue Sprint

The single document the client receives at the end of the sprint.

## Sections
1. **Executive summary** — five bullets the client's CEO can read in 60 seconds.
2. **Sector frame** — what slice of the market this sprint covered.
3. **Method** — sources used, scoring approach, exclusions.
4. **Top leads (A tier)** — short rationale per lead.
5. **Lead table** — full A/B/C table.
6. **Suggested message angles** — three angles tied to specific leads.
7. **Recommended cadence** — calibrated to the client's team capacity.
8. **What we did not do** — explicit out-of-scope.
9. **Next step** — one concrete recommended next engagement (often a Managed Pilot or Revenue Desk).

## Formatting
- One PDF, branded, with page numbers.
- Arabic and English versions if requested.
- Embedded sources for any external claim.

## Approval
- Internal QA against `QA_CHECKLIST.md`.
- Founder sign-off for the first three sprints, then random sampling.

## Rule
A report that does not end with one concrete next step is half-delivered.
