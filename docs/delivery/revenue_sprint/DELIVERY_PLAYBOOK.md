# Revenue Sprint Delivery Playbook

## Purpose
Operate a 7-day Revenue Sprint end-to-end with a predictable, evidence-backed result.

## Owner
Sami / Delivery owner.

## Sprint Outline (7 days)

### Day 1 — Intake & Scope
- Confirm proposal scope and ICP with buyer.
- Lock down: sector, geography, deal-size band, evidence sources.
- Record approval and scope in the delivery folder.

### Day 2 — Source Passport
- List the data sources Dealix will use.
- Confirm each source is allowed (PDPL-safe, public, license-compatible).
- Log refused sources with reason.

### Day 3 — Account Scoring
- Build the candidate account list.
- Score using evidence (sector signal, recency, fit).
- Reject A-priority candidates with no evidence.

### Day 4 — Outreach Pack Draft
- Draft per-segment outreach copy in AR + EN.
- Apply NO_OVERCLAIM_POLICY and SAFE_LANGUAGE_LIBRARY.

### Day 5 — Executive Memo Draft
- One page. Three sections: situation, what we found, recommended next 14 days.
- Each finding cites its evidence.

### Day 6 — Governance Review
- Run QA_CHECKLIST.md.
- Founder approves anything client-visible.
- Resolve all flagged issues before handoff.

### Day 7 — Handoff
- Send the approved pack using HANDOFF_TEMPLATE.md.
- Schedule the 14-day follow-up checkpoint.
- Capture client feedback in feedback log.

## Definition of Done
- All artefacts produced.
- QA checklist passed.
- Founder approval recorded.
- Client received handoff and acknowledged.
- Feedback captured.
- Retainer eligibility evaluated.
