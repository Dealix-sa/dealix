# Revenue Sprint — Delivery Playbook

The end-to-end playbook for one Revenue Sprint engagement.

## Pipeline
```
Intake → Research → Scoring → Outreach Pack → Report → QA → Handoff → Feedback → Case Study
```

## Phase 1 — Intake (Day 0)
- Receive payment confirmation.
- Run `CLIENT_INTAKE.md` checklist.
- Confirm scope and out-of-scope in writing.
- Schedule the kickoff call within 48 hours.

## Phase 2 — Research (Day 1–3)
- Run `RESEARCH_PROCESS.md`.
- Source candidate leads, sector data, competitive view.
- Document every source.

## Phase 3 — Scoring (Day 3–4)
- Apply `SCORING_RULES.md`.
- Drop anything that fails ICP or Claim Guard.

## Phase 4 — Outreach Pack (Day 4–6)
- Use `OUTREACH_PACK_TEMPLATE.md`.
- Build messages that pass `MESSAGE_QUALITY_STANDARD.md`.
- Internal QA before client review.

## Phase 5 — Report (Day 6–8)
- Use `REPORT_TEMPLATE.md`.
- Include lead table, sector view, recommended next actions.

## Phase 6 — QA (Day 8)
- Run `QA_CHECKLIST.md` end-to-end.
- Founder sign-off required for first three sprints, then sampled.

## Phase 7 — Handoff (Day 9–10)
- Use `HANDOFF_TEMPLATE.md`.
- 30-minute handoff call.
- Provide the report, the lead table, and the recommended cadence.

## Phase 8 — Feedback (Day 10–14)
- Send the feedback form.
- Log responses in `docs/learning/MESSAGE_PERFORMANCE.md` and `docs/learning/WIN_LOSS_REVIEW.md`.

## Phase 9 — Case Study (Day 14+)
- If results are public-shareable, run `CASE_STUDY_CAPTURE.md`.
- Add to `docs/content/PROOF_LIBRARY.md`.

## Rule
A sprint without a feedback step and a case-study attempt is a sprint that did not compound. Both are mandatory.
