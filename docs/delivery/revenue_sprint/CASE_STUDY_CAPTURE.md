## Consent For Public Use

Capture explicit consent (date + name):
- [ ] Sanitized case study draft OK (with pre-publication review)
- [ ] Logo on website OK
- [ ] LinkedIn mention OK
- [ ] Quote with name OK
- [ ] Quote with pseudonym only
- [ ] No public mention

## Path To Public Case Study

If consent for sanitized case study:
1. Draft in `content/drafts/case-study-{pseudonym}.md`
2. Run claim_guard.py
3. Founder review
4. Send to client for pre-publication review (give 5 business days)
5. Address comments
6. Trust signoff
7. Publish (public repo `content/case_studies/` if shareable, else private only)
8. Distribute on LinkedIn per Content OS policy

If no consent: keep as internal learning only. Aggregate (n ≥ 3) before any composite case study.

## Storage + Privacy

- Notes file in private repo
- Verbatim quotes redacted of identifiers before any public use
- Numbers anonymized to ranges (e.g., "10–20 replies") in public versions
- Client data never moves to public repo without explicit consent

## What Capture Refuses

- Putting words in client's mouth
- Inflating numbers
- Skipping consent
- Treating "yes verbally" as written consent for publication
- Using a case study before client sees the final version
