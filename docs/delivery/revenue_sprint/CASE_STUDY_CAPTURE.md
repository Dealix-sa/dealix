# Case Study Capture

Every sprint is an opportunity for a public proof.

## Template

```
## Client Type
[Sector, size, location — no names unless approved]

## Before Dealix
[The problem in plain language]

## What Dealix Delivered
[Concrete deliverables — lead table, report, outreach pack]

## Result
[Quantitative or qualitative outcome — replies booked, meetings, deals]

## Client Quote
[Verbatim, only if approved]

## Public Version Approved?
Yes / No
```

## Capture process
1. Send the feedback form within 24 hours of handoff.
2. Two weeks later, request a short result update.
3. Draft the case study from the form and the update.
4. Send to the client for approval before publication.
5. Add the approved version to `docs/content/PROOF_LIBRARY.md`.

## Rule
No case study is published without written client approval. Anonymous case studies are allowed and useful.
