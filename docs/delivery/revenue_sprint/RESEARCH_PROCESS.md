# Research Process — Revenue Sprint

## Inputs
- Client intake (sector, geography, "good lead" definition).
- Target sector references from `docs/sector-reports/`.

## Steps
1. **Sector frame** — define the slice (e.g., Riyadh-based mid-market accounting firms with 10–50 staff).
2. **Source set** — collect 80–150 candidate companies from allowed sources only (see `docs/acquisition/LEAD_SOURCING_SYSTEM.md`).
3. **Enrichment** — for each candidate, record: name, sector, size estimate, public contact, public signals.
4. **De-duplication** — remove duplicates and entries already in the client's exclusion list.
5. **Source log** — record the source URL or origin for every candidate.

## Outputs
- A raw candidate table (CSV or sheet) inside the client folder.
- A short sector frame note (one page) for the report.

## Time budget
- 1.5 working days for a standard sprint.
- If the sector is new, allow an extra day and log it as an exception.

## Rule
A candidate without a recorded source does not enter the table.
