# Proof → Upsell Playbook — من الإثبات إلى التوسعة

How a delivered, proven engagement becomes the next, larger commitment — without
overclaiming. Proof first, claim second (always).

## The ladder (reminder)

| Rung | Offer | Price (SAR) |
|---|---|---|
| 0 | Free AI Ops Diagnostic | 0 |
| 1 | 7-Day Revenue Intelligence Sprint | 499 |
| 2 | Data-to-Revenue Pack | 1,500 |
| 3 | Managed Revenue Ops | 2,999–4,999 / mo |
| 4 | Custom AI Service Setup | 5,000–25,000 + 1,000 / mo |
| Enterprise | AI Governance Review | 25,000–50,000 |

## The mapping rule

The upsell is chosen by **what the Proof Pack actually showed**, not by what we
want to sell:

| What the Proof Pack proved | Recommended next rung |
|---|---|
| Clean data + scored accounts, founder wants more lists | Rung 2 — Data-to-Revenue Pack |
| Sprint output is valuable but needs ongoing operation | Rung 3 — Managed Revenue Ops |
| Repeatable workflow worth automating end-to-end | Rung 4 — Custom AI Service Setup |
| Governance / audit is the real pain | Enterprise — AI Governance Review |

## The four checks before recommending an upsell

1. **Proof exists** — a Proof Pack with score ≥ 70 is on file.
2. **Capital Asset registered** — something reusable was produced.
3. **Claim is sourced** — every number cites a `source_ref` (non-negotiable #4 & #7).
4. **Tier honesty** — `estimated` value is labeled estimated, never "verified".

## Output

The recommendation is written to the customer's `11_upsell_recommendation.md` and
queued (not sent) for founder approval per
[No External Action Without Approval](../governance/NO_EXTERNAL_ACTION_WITHOUT_APPROVAL.md).
