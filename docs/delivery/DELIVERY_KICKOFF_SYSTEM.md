# Delivery Kickoff System
## Purpose
Ensure paid or approved work starts with clear scope, inputs, and trust boundaries.
## Owner
Sami / Delivery owner.
## Kickoff Inputs
- signed/approved proposal.
- start condition.
- client intake.
- target sectors.
- exclusions.
- success definition.
- approval contact.
- data constraints.
## Kickoff Rules
- No kickoff without start condition.
- No delivery without intake.
- No scope expansion without approval.
- No public proof discussion before delivery success.
## Kickoff Output
- intake completed.
- delivery folder created.
- QA checklist prepared.
- delivery timeline confirmed.
## Evidence
- clients/client_name_private/intake.md
- proposal.md
- qa_checklist.md
