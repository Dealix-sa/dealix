# Delivery Operating Routine

## When Payment / PO Is Confirmed
1. Create client folder.
2. Copy client template files.
3. Fill client_os.md.
4. Complete intake.
5. Confirm scope and delivery date.
6. Prepare QA checklist.

## During Delivery
1. Research leads.
2. Score leads.
3. Prepare outreach pack.
4. Write executive summary.
5. Run QA score.
6. Fix anything below quality bar.

## Handoff
1. Send delivery report.
2. Explain how to use it.
3. Give 14-day action plan.
4. Ask for feedback.
5. Track retention signal.

## After Delivery
1. Update health score.
2. Send retainer ask if value is clear.
3. Request proof approval if appropriate.
4. Update weekly learning.
5. Update productization candidates if workflow repeated.

## Daily Discipline
- Touch the active client folder every day.
- Update client_os.md status whenever it changes.
- Log every governance event the moment it happens.

## Weekly Discipline
- Review every active client_success_dashboard row.
- Review qa_score_log.csv for systemic gaps.
- Review retention_tracker.csv for retainer triggers.
- Review productization_candidates.md for repeated patterns.

## Stop Conditions
- Stop delivery if scope, payment, or trust constraints are unclear.
- Stop public proof if approval is missing or weak.
- Stop retainer pitch if value is not yet visible.
