# Dealix Command Sprint — Offer

> **Status:** CANONICAL · **Owner:** Founder · **Last reviewed:** 2026-06-05
>
> The first paid product. Replaces the old "Revenue Proof Sprint" framing.

---

## 1. Summary

| Field | Value |
|---|---|
| Name | **Dealix Command Sprint** |
| Duration | 7 days |
| Founder price | 499–1,500 SAR |
| Goal | Prove the first operating loop inside the customer's company |
| Format | Founder-led service (not self-serve SaaS) |
| Approval | Every external/customer-facing draft requires human approval |

## 2. What it covers

| System | What it answers |
|---|---|
| Revenue OS | Where do opportunities and follow-up break? |
| Proof OS | What evidence exists? What evidence is missing? |
| Command OS | What should the manager decide this week? |
| Governance OS | What needs human approval? |
| Client OS Lite | What do we know about the customer / opportunities? |
| Delivery OS Lite | What are the next execution steps? |

## 3. The deliverable — Dealix Command Pack

```
Dealix Command Pack
├─ Revenue Map
├─ Next Action Board
├─ Follow-up Drafts        (drafts only — never auto-sent)
├─ Offer Upgrade
├─ Objection Intelligence
├─ Proof Register
├─ Approval Register
└─ Executive Command Brief
```

Each artifact is built from the customer's real intake data and labeled with
`source`, `confidence`, and `approval_required`. See the delivery spec:
[`../04_delivery/COMMAND_SPRINT_DELIVERY_OS.md`](../04_delivery/COMMAND_SPRINT_DELIVERY_OS.md).

## 4. The 7-day flow

| Day | Activity |
|---|---|
| 1 | Intake + consent + scope; collect minimum data (see Data Policy). |
| 2 | Revenue Map + Client Brain Lite. |
| 3 | Follow-up drafts + Offer Upgrade. |
| 4 | Objection Intelligence + Proof Register. |
| 5 | Approval Register + Governance review. |
| 6 | Executive Command Brief + Next 30 days. |
| 7 | Review call with customer + Proof Pack handoff + upsell conversation. |

## 5. Minimum data we collect (and nothing more)

company name · industry · contact role · current sales/delivery flow ·
top 5 opportunities · current offer · follow-up examples · pain points ·
approval contact · consent.

**Forbidden at intake:** passwords, full CRM dumps, unnecessary sensitive data,
customer API keys, large personal files, scraped data from platforms that do not
allow it.

## 6. What the Sprint is NOT

- It is not a promise of new revenue within 7 days.
- It does not send anything to the customer's contacts automatically.
- It does not make pricing, legal, or financial commitments on the customer's behalf.

## 7. Pricing authority

Price changes (including discounts) follow
[`PRICE_AUTHORITY.md`](PRICE_AUTHORITY.md). No discount without reason, term,
counterpart, approval, and a logged record.
