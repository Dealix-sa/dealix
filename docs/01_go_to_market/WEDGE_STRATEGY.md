# Dealix — Wedge Strategy

> **Status:** CANONICAL · **Owner:** Founder · **Last reviewed:** 2026-06-05

---

## 1. The strategic decision

Do **not** choose between "huge platform" and "sellable now." Choose both, in
this order:

| Level | Message |
|---|---|
| Vision | Dealix AI Business Operating System |
| Market entry | Command Sprint |
| First product | Command + Revenue + Proof |
| Expansion | Client + Delivery + Support |
| Enterprise | Finance + Governance + Data |
| Ecosystem | Partner + Academy + Venture |

**Rule:** Widen the vision, narrow the execution, sell the first wedge, then
expand from inside the customer.

## 2. Why a wedge

A 14-module platform is impossible to sell in one conversation. The wedge gives a
prospect one clear, fast, low-risk experience that proves value, then we expand
account-by-account using proof we generated *for that customer*.

## 3. The wedge = Command Sprint

7 days · 499–1,500 SAR · proves the first operating loop:

```
Revenue OS   → where do opportunities & follow-up break?
Proof OS     → what evidence exists, what is missing?
Command OS   → what should the manager decide this week?
Governance   → what needs human approval?
Client Lite  → what do we know about the customer/opportunities?
Delivery Lite→ what are the next execution steps?
```

Output: the **Command Pack** (see `COMMAND_SPRINT_OFFER.md`).

## 4. Land-and-expand path

1. **Land:** Command Sprint → Command Pack → "these outputs are useful."
2. **Convert:** Managed Business OS (Starter Command, 2,999 SAR/mo).
3. **Expand inside customer:** add Client → Delivery → Support.
4. **Deepen:** Finance + Governance + Data (enterprise).
5. **Multiply:** Partner channels + Academy + Venture.

## 5. What we do NOT do at wedge stage

- No SaaS self-serve build before the first 3 paying customers.
- No 10-sector spray. One ICP (Saudi B2B services).
- No public launch before 3 paid Sprints + 1 approved case study.
- No auto-send, no scraping, no "certified" claims.

## 6. Sequencing guardrail

Each expansion step requires the previous step's gates to pass (see
`docs/00_platform_truth/MODULE_STATUS_MAP.md`). We never sell a module above its
declared status.
