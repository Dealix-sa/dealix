## القسم العربي (أساسي)

### لماذا الإيرادات + الإثبات + القيادة هو المدخل؟

- **الإيرادات** هي الألم الذي يدفع المدير للشراء اليوم.
- **الإثبات** يحوّل الوعد إلى أدلة، فيبني الثقة في القرار الأول.
- **القيادة** تعطي المدير قراراً واضحاً هذا الأسبوع، لا لوحة أخرى.

هذه الثلاثة معاً تشكّل أصغر نسخة مفيدة من المنصة: قيمة فورية بمخاطر منخفضة.

### القاعدة الذهبية: وسّع الهوية، ضيّق التنفيذ

نُعرّف Dealix كنظام تشغيل أعمال (هوية واسعة)، لكننا ننفّذ أولاً المدخل الثلاثي فقط (تنفيذ ضيّق). نبيع المدخل، ونبني المنصة، ولا نخلط بين الدخول إلى السوق وهوية الشركة.

### تسلسل التوسّع في خمس مراحل

| المرحلة | المحتوى |
|---|---|
| 0 — إعادة التموضع | تثبيت هوية "نظام تشغيل أعمال" في كل المستندات. |
| 1 — الثلاثي الجوهري | الإيرادات + الإثبات + القيادة (+ الحوكمة + البيانات). |
| 2 — العميل/التسليم/الدعم | Client + Delivery + Support. |
| 3 — تعميق المالية والحوكمة | Finance + تعميق Governance. |
| 4 — الشركاء والأكاديمية | Partner + Academy. |
| 5 — المشاريع | Venture. |

### سكربت اكتشاف اللقاء الأول

كيف تعمل الشركة اليوم؟
- كيف تدخل الفرص إلى الشركة؟
- كيف تتحوّل الفرصة إلى عرض؟
- كيف يُتابَع العرض حتى الإغلاق؟
- كيف يُنفَّذ ما تم بيعه (التسليم)؟
- كيف تُجمَع الأدلة على القيمة المُقدَّمة؟
- ما القرار الذي يحتاجه المدير هذا الأسبوع؟

ثم نختم: **نبدأ بأصغر نسخة مفيدة: الإيرادات + الإثبات + القيادة.**

## English Section

### Why Revenue + Proof + Command is the wedge

- **Revenue** is the pain that makes a manager buy today.
- **Proof** turns a promise into evidence, building trust in the first decision.
- **Command** gives the manager a clear decision this week, not another dashboard.

Together these three form the smallest useful version of the platform: immediate value at low risk.

### The golden rule: widen the identity, narrow the execution

We define Dealix as a Business Operating System (wide identity), but we execute only the core triad first (narrow execution). Sell the wedge, build the platform, and never confuse market entry with company identity.

### Five-phase expansion sequence

| Phase | Scope |
|---|---|
| 0 — Reposition | Establish "Business OS" identity across all docs. |
| 1 — Core triad | Revenue + Proof + Command (+ Governance + Data). |
| 2 — Client/Delivery/Support | Client + Delivery + Support. |
| 3 — Finance/Governance deepening | Finance + deeper Governance. |
| 4 — Partner/Academy | Partner + Academy. |
| 5 — Venture | Venture. |

### First-meeting discovery script

How does the company work today?
- How do opportunities enter the company?
- How does an opportunity become an offer?
- How is the offer followed up to close?
- How is what was sold delivered?
- How is evidence of delivered value collected?
- What decision does the manager need this week?

Then close: **We start with the smallest useful version: Revenue + Proof + Command.**
