## Heuristic limits

- A router that imports a `tenant_id`-bearing model AND issues a query (select/execute/scalars/first/all/one) AND never mentions `tenant_id` in its source is flagged as **needs review**.
- A router that imports the model but never queries (e.g. uses the class for type hints only) is marked **imports-only** — almost always safe.
- Pydantic models, dataclasses, and JSON dicts named tenant_id but not in `db/models.py` are NOT scoped — this scan looks only at the SQLAlchemy classes there.
- This is a smoke check, not a security review. A `needs review` flag means a human must confirm the query is correctly scoped (via session.execute(stmt.where(Model.tenant_id == ...)) or via a tenant-aware base session).
