# LinkedIn — مقدّمات دافئة فقط — Warm Intros Only (NOT automation)

LinkedIn في Dealix = **بحث يدوي + رسائل يدوية + مقدّمات دافئة عبر معارف مشتركين فقط**. صراحةً: **ليست خدمة أتمتة، ولا استخراج بيانات، ولا إرسال جماعي**. LinkedIn مصنّف 🔴 في [SAUDI_DATA_SOURCE_CATALOG.md](../../ops/SAUDI_DATA_SOURCE_CATALOG.md) — لا توريد منه إطلاقاً.

> أداة المسودة: [design-skills/dealix-linkedin-warm-intro-draft](../../../design-skills/dealix-linkedin-warm-intro-draft/SKILL.md) — تنتج مسودة AR/EN وتفرض `approval_required_before_send` و`no_guaranteed_claims` و`no_external_http`.

## القاعدة

- لا اتصال آلي، لا InMail جماعي، لا أدوات استخراج.
- المقدّمة الدافئة تمرّ **عبر معرفة مشتركة حقيقية** توافق على التعريف.
- كل مسودة تُراجَع وتُرسَل **يدوياً** بعد موافقة.

## قالب طلب المقدّمة (إلى المعرفة المشتركة)

```
مرحباً {اسم المعرفة}،
ألاحظ أنك على تواصل مع {اسم العميل} في {الشركة}.
نعمل في Dealix على معالجة {الفجوة} لشركات {القطاع} بمخرجات ملموسة.
هل يناسبك تعريفي بهم إن رأيت ذلك مفيداً؟ بلا أي التزام منك.
```

## قالب الرسالة الدافئة (بعد موافقة الطرفين)

```
شكراً {اسم المعرفة} على التعريف.
مرحباً {اسم العميل} — باختصار: لاحظنا {الفجوة العامة}، ونساعد شركات مثلكم على تنظيمها.
إن رغبت، أشارك ملخّص صفحة واحدة. لا وعود بأرقام، فقط فجوة وخطوات.
```

## ممنوع صراحةً

- أي أداة أتمتة اتصال/رسائل على LinkedIn.
- استخراج قوائم أو بيانات من المنصّة.
- التعريف بلا موافقة المعرفة المشتركة.
