## مبادئ PDPL المطبَّقة

1. **أساس معالجة مشروع:** بيانات أعمال عامة منشورة من الكيان نفسه، أو موافقة صريحة. لا أرقام شخصية تحت «المصلحة المشروعة».
2. **تقليل البيانات:** نجمع ما يلزم للتواصل التجاري فقط.
3. **الشفافية:** هوية المُرسِل + سبب التواصل في كل رسالة.
4. **حق الانسحاب:** فوري في كل رسالة + تنفيذ خلال أقصر وقت.
5. **عدم تخزين PII حسّاس** في السجلّات؛ تُجزَّأ المعرّفات.
6. **لا بيانات حسّاسة** (صحية/مالية/دينية) بلا أساس قانوني وترخيص.
7. **سجل المعالجة:** مصدر كل بيان + سجل الموافقة + تاريخ الانسحاب.

## قواعد واتساب (إلزامية)

| القاعدة | التفصيل |
|---------|----------|
| الحساب | أعمال موثّق من Meta فقط |
| الرسالة الأولى | قالب معتمد + opt-in خلال 24 ساعة للرسائل الحرّة |
| ساعات الهدوء | 09:00–21:00 بتوقيت السعودية فقط |
| الحدّ اليومي | وفق حدود Meta للأعمال الجديدة (ramp-up) |
| الهوية | تعريف «Dealix» في كل رسالة |
| الانسحاب | تعليمات إيقاف في كل رسالة + تنفيذ فوري |
| المنع | لا واتساب بارد، لا أتمتة جماعية، لا إخفاء هوية |
