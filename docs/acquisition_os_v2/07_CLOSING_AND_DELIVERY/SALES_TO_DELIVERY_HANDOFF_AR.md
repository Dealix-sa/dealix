إجراء تسليم نظيف بعد الإغلاق، يمنع ضياع السياق ويبدأ التشغيل بثقة. يبني على: [P1_DELIVERY_SOP_AR.md](../../delivery_os/P1_DELIVERY_SOP_AR.md).

## شرط بدء التسليم (DoD للإغلاق)

- [ ] دفع مستلَم أو التزام مكتوب.
- [ ] نطاق متّفق عليه (أي رتبة من [OFFERS_AND_PRICING_AR.md](../04_SALES_PACKAGES/OFFERS_AND_PRICING_AR.md)).
- [ ] سجل موافقة العميل + قنوات التواصل المسموحة.
- [ ] لا التزامات شفهية خارج النطاق المكتوب.

## بطاقة التسليم (تُملأ من البيع)

| الحقل | القيمة |
|-------|--------|
| العميل / القطاع | |
| الرتبة المتّفق عليها | |
| الفجوة الأساسية المُثبتة | |
| المخرجات الموعودة (مسودات/تقرير/تشغيل) | |
| القنوات المسموحة + سجل الموافقة | |
| المخرجات **خارج** النطاق (صريحة) | |
| تاريخ تسليم Proof المستهدف (≤ 7 أيام للتشخيص) | |

## قواعد

- لا يُسلَّم وعد بيع لم يُكتَب في النطاق.
- كل البيانات المستلمة تُصنَّف `source_type=owned` بسجل موافقة.
- التسليم القياسي = مسودات؛ لا إرسال خارجي بلا موافقة لكل دفعة.
- أي PII يُعامَل وفق PDPL (القسم 08).

## أول 48 ساعة بعد التسليم

1. اجتماع انطلاق قصير مع العميل.
2. استلام بياناته (إن وُجدت) + تصنيف المصدر.
3. تأكيد المخرجات والتواريخ كتابياً.
4. بدء خطة النجاح: [ONBOARDING_SUCCESS_PLAN_AR.md](ONBOARDING_SUCCESS_PLAN_AR.md).
