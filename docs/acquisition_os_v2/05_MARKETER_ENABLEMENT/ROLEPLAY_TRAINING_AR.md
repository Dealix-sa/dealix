# تدريب لعب الأدوار — Roleplay Training

سيناريوهات تدريب عملية للمسوّق الجديد قبل التواصل الحقيقي. كل سيناريو يُدرَّب حتى يلتزم بالحدود غير القابلة للنقض.

## آلية التدريب

1. متدرّب = مسوّق، آخر = عميل.
2. 5–7 دقائق لكل جولة.
3. مراجعة بـ [QUALITY_RUBRIC_AR.md](QUALITY_RUBRIC_AR.md) بعد كل جولة.
4. لا انتقال للتواصل الحقيقي قبل اجتياز 3 سيناريوهات.

## السيناريوهات

### س1 — افتتاح قائم على فجوة
العميل متحفّظ. الهدف: تعريف + فجوة + إذن + انسحاب خلال 45 ثانية. **يفشل** إن وعد بأرقام أو نسي الانسحاب.

### س2 — «من أين رقمي؟»
الهدف: شرح المصدر العام بثقة + عرض الانسحاب. **يفشل** إن ادّعى مصدراً غير حقيقي.

### س3 — «كم ستزيدون مبيعاتي؟»
الهدف: «فرص مُثبتة بأدلة» بلا ضمان. **يفشل** إن قال «نضمن».

### س4 — طلب انسحاب صريح
الهدف: تنفيذ فوري + تأكيد واحد + توقّف تام. **يفشل** إن حاول الإقناع بعد الانسحاب.

### س5 — اعتراض PDPL
الهدف: طمأنة بالتزام PDPL + التسليم كمسودات. **يفشل** إن طلب بيانات حسّاسة بلا مبرر.

## معايير الاجتياز

- التزام بالحدود الـ11 في كل جولة.
- لغة تنفيذية واقعية بلا مبالغة.
- إثبات الفجوة من مصدر عام.
