# دليل المسوّق الميداني — Marketer Field Manual

الدليل الكامل لأي مسوّق ينفّذ ضمن Acquisition OS v2. اقرأه قبل أول تواصل. يبني على دليل بيع المؤسس: [FOUNDER_SALES_PLAYBOOK_AR.md](../../commercial/FOUNDER_SALES_PLAYBOOK_AR.md) وسكربت الاكتشاف: [DEALIX_DISCOVERY_SCRIPT_AR.md](../../commercial/sales/DEALIX_DISCOVERY_SCRIPT_AR.md).

> القاعدة الأولى: **لا رسالة تخرج بلا فجوة عامة مُثبتة + موافقة + مخرج انسحاب.** مخالفة هذه القاعدة = إيقاف فوري.

## 1) عقليّة التشغيل

- نبيع **معالجة فجوة محدّدة بمخرجات ملموسة**، لا «AI عام» ولا وعوداً.
- لغة تنفيذية واقعية. ممنوع: «نحوّل أعمالك»، «AI خارق».
- كل ادعاء له دليل أو يُحذف. كل إحصائية خارجية باسم مصدرها.

## 2) المسار اليومي للمسوّق

1. استلم شريحة A من السكورينج ([LEAD_SCORING_MODEL_V2_AR.md](../02_DAILY_LEAD_ENGINE/LEAD_SCORING_MODEL_V2_AR.md)).
2. تحقّق من قائمة منع التواصل لكل حساب.
3. اكتب رسالة قائمة على الفجوة (القسم 03) — لا قالب عام.
4. أرسل ضمن ساعات الهدوء + opt-in فقط.
5. تابع وفق كادنس 14 يوماً.
6. سجّل كل لمسة في [pipeline_template.csv](../10_DASHBOARDS_CSV/pipeline_template.csv).
7. أغلق اليوم بـ [DAILY_REPORT_TEMPLATE_AR.md](../02_DAILY_LEAD_ENGINE/DAILY_REPORT_TEMPLATE_AR.md).

| الفجوة | العرض المناسب |
|--------|----------------|
| واتساب غير منظَّم | تنظيم استقبال + Sprint |
| متابعة ضعيفة | كادنس + Managed Ops |
| عروض غير مقنعة | إعادة صياغة + one-pager |
| CRM غير مستخدَم | Data Pack + سكورينج |
| تقارير لا تتحول لقرار | تقرير تنفيذي مربوط بقرار |

## 5) خطوط حمراء للمسوّق

- لا scraping، لا قوائم مشتراة، لا أرقام شخصية.
- لا ضمان مبيعات (استخدم «فرص مُثبتة بأدلة»).
- لا تجاهل لطلب انسحاب.
- لا إرسال خارجي بلا موافقة بشرية على المسودة.
