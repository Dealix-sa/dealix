# Data Freshness Policy
## Purpose
Prevent CEO decisions from relying on stale operating data.
## Owner
Sami / Founder CEO.
## Review Cadence
Daily.
## Freshness Rules
| Data | Freshness Requirement |
|---|---|
| pipeline_tracker.csv | daily during sprint |
| revenue_action_log.csv | daily |
| stage_exit_checklist.csv | after every evidence update |
| ceo_business_score.md | daily |
| control_tower_brief.md | daily |
| weekly_intelligence_review.md | weekly |
| finance review | monthly or after cash change |
| board pack | monthly |
## Rules
- No CEO decision from stale pipeline.
- No stage advancement from stale evidence.
- No finance decision from outdated expenses/cash.
- No productization decision without latest weekly review.
## Evidence
- file modified dates
- generated reports
- audit logs
