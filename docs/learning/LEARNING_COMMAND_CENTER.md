# Learning Command Center

## Purpose
Turn operating results into better playbooks, messages, pricing, delivery, and trust controls.

## Owner
Sami / Learning owner.

## Review Cadence
Weekly.

## Inputs
- weekly metrics
- replies
- calls
- proposals
- payments
- delivery QA
- trust escalations
- objections
- founder bottlenecks

## Outputs
- one learning decision
- one updated playbook/checklist/template/policy
- one next-week experiment

## Learning Rules
- If issue appears once: log it.
- Twice: add checklist item.
- Three times: update playbook.
- Five times: template it.
- Ten times: automate or productize it.

## Weekly Required Decision
Build / Fix / Kill / Defer / Continue

## Metrics
- learning decisions
- playbook updates
- repeated issues reduced
- conversion improvement
- delivery rework reduction

## Evidence
- weekly review
- git commit
- updated file
- decision log
