# Sector Performance — أداء القطاعات

## Purpose

Which Saudi sectors convert for Dealix, at what value, with what proof base. The sector view is reviewed quarterly with one decision: trim, hold, or add. No sector stays in scope by inertia.

## Definitions

- **Active sector** — a sector with at least one named pipeline opportunity and one case-safe template in the last 180 days.
- **Pipeline value** — sum of estimated ACV across opportunities, marked estimated.
- **Conversion** — proposals signed divided by proposals sent in the sector, last 180 days.
- **Proof base** — count of case-safe summaries plus published sector reports for the sector.

## Metrics tracked per sector

| Metric | Source | Cadence |
|---|---|---|
| Pipeline value (estimated) | CRM | weekly |
| Proposals sent / signed | CRM | weekly |
| Average days to close | CRM | monthly |
| Case-safe summaries | [docs/07_proof_os/CASE_SAFE_SUMMARY.md](../07_proof_os/CASE_SAFE_SUMMARY.md) | quarterly |
| Sector reports published | [docs/sector-reports/](../sector-reports/) | quarterly |
| Win/loss root causes | [WIN_LOSS_REVIEW.md](./WIN_LOSS_REVIEW.md) | quarterly |

## Quarterly review

| Outcome | Trigger |
|---|---|
| Trim | conversion below threshold for two consecutive quarters, no proof base growth |
| Hold | conversion stable, proof base growing, capacity available |
| Add | repeated inbound from a new sector, two unprompted intros, fit with operating equation |

## Operations

1. Founder opens the sector view at the start of the last week of each quarter.
2. Each sector receives a one-line verdict and a written reason.
3. Trims are logged in [COMPANY_MEMORY.md](./COMPANY_MEMORY.md) under kills with the disqualifying evidence.
4. Adds require a 30-day discovery plan before any proposal goes out.

## Non-negotiables

- Sector reports use methodology and aggregated patterns only — no confidential client metrics. See [docs/14_trust_os/](../14_trust_os/) and [docs/sector-reports/](../sector-reports/).
- No sector is added because of a single warm conversation. Add requires repeated signal.
