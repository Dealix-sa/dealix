# Company Memory

The company needs memory, not just files. A smart company is not the
one that knows the most. It is the one that does not repeat the same
mistake twice.

Dealix remembers and improves through:

## Decision Memory
Stored in `founder/decision_log.md` (private founder repo).

## Customer Memory
Stored in `clients/*` inside the private repo.

## Market Memory
Stored in `strategy/market_notes.md`.

## Revenue Memory
Stored in `pipeline/win_loss_log.md` and `revenue/pricing_experiments.md`.

## Delivery Memory
Stored in `delivery/qa/` and `delivery/case_studies_private/`.

## Trust Memory
Stored in `trust/approval_log.csv` and `trust/data_incidents.md`.

## Learning Rule
Every repeated issue must become:
- a checklist item,
- a template update,
- a workflow update,
- or a product feature candidate.

## Memory Audit

Every quarter the founder asks one question per memory store:

1. Decision Memory: which decisions, in hindsight, were correct?
2. Customer Memory: which clients should we never accept again?
3. Market Memory: which sectors deserve more attention?
4. Revenue Memory: which pricing experiment changed the most cash?
5. Delivery Memory: which template saved the most hours?
6. Trust Memory: which incident pattern is recurring?
