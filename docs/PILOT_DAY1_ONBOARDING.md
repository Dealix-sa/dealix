# Pilot Day 1 Onboarding Checklist

**Founder:** This is your checklist for the first 3 hours with a new pilot customer.

**Duration:** 3 hours (founder + customer + AI system)

**Outcome:** Customer dashboard live with first data loaded; team trained

---

## Pre-Call Prep (15 min, Founder Only)

### Intake Prep

- [ ] Customer intake form filled (see section below)
- [ ] Company sector identified (Real Estate, Distribution, SaaS, etc.)
- [ ] Pain angle documented
- [ ] Decision maker contact confirmed
- [ ] Pilot contract signed + payment received (499 SAR via Moyasar)

### Technical Prep

- [ ] Dashboard template created (customer email + password generated)
- [ ] Data mapping plan (what data will customer load?)
- [ ] Integration plan (direct data import, Zapier, manual CSV upload?)
- [ ] AI approvals system ready (for this customer's requests)

---

## Customer Onboarding Call (30-45 min)

**Format:** Video call (Zoom or WhatsApp)

**Participants:**
- Founder (you)
- Customer (decision maker + 1-2 team members)
- Screen share enabled

### 1. Welcome & Housekeeping (3 min)

```
السلام عليكم ورحمة الله وبركاته 👋

شكراً إنكم اخترتوا ديليكس
أنا سامي، الـ founder
سأكون معاكم كل يوم لـ 14 يوم الجاي
أي أسئلة، WhatsApp مباشرة — ما فيه وقت انتظار

محاور اليوم:
1. شوفة سريعة للنظام (5 دقايق)
2. بيانات عملاءك — كيف نحملها (10 دقايق)
3. أول خطوة بنفسك (15 دقايق)
4. أسئلة منكم (10 دقايق)

تمام؟
```

### 2. System Demo (5 min)

Show:
- [ ] Dashboard landing page (prospect list view)
- [ ] Single prospect card (name, phone, last contact, next follow-up)
- [ ] "Send Follow-up" button (shows what auto-message founder will approve)
- [ ] Approval system (how follow-ups flow through founder before sending)

**Say:**
```
كل عميل/prospect في النظام له:
- آخر اتصال (تاريخ وملاحظات)
- آخر deal/عرض
- موعد متابعة التالي

كل يوم:
- النظام يقترح متابعات
- أنا بأراجع وأوافق
- تطير WhatsApp تلقائي

الفريق ما بيحتاج يتذكر — النظام بيتذكر
```

### 3. Data Mapping & Import (10-15 min)

**Option A: Direct CRM Export (fastest)**
- [ ] Customer exports prospect list from their CRM as CSV
- [ ] Upload to Dealix dashboard
- [ ] System auto-maps fields: Name, Phone, Company, Last Contact, etc.
- [ ] Founder reviews mapping; confirms

**Option B: Manual Spreadsheet (if no CRM)**
- [ ] Founder provides Google Sheets template
- [ ] Customer fills in: Name, Phone, Company, Last Contact Date, Notes
- [ ] Founder copies data into Dealix
- [ ] Takes 30 min for 50-100 prospects

**Option C: Zapier Integration (future)**
- [ ] Customer connects their CRM directly
- [ ] Automatic daily sync
- [ ] (Not available yet; mention for future)

**Action Steps:**
1. [ ] Founder asks: "What's your main source of prospects right now?"
2. [ ] Customer answers: CRM, spreadsheet, or manual list
3. [ ] Founder walks through upload process (2-3 min)
4. [ ] Customer loads first 10-20 prospects as test
5. [ ] System validates data; shows preview
6. [ ] Founder confirms; loads all data

**Red flag:** If data is incomplete (no phone numbers), this won't work. Founder should push back: "We need at least phone for follow-ups to work."

### 4. First Action (10-15 min)

**Have customer send their first follow-up while you watch:**

1. Customer picks 1 prospect from the list
2. Founder drafts a natural follow-up message in Khaliji Arabic:
   ```
   السلام عليكم [name] 👋
   
   متابعة على الطلب اللي تكلمنا عليه امس
   عندك وقت اليوم نختم الموضوع؟
   ```
3. Customer approves the message (this is how approval queue works)
4. Founder sends via WhatsApp on customer's behalf
5. Customer sees it land in their WhatsApp
6. Celebration: "That's it. Every day, dozens of these."

- [ ] Customer team created + passwords emailed

- [ ] Approval workflow configured (all follow-ups → founder)
- [ ] Notification settings: founder gets daily digest

### Data Validation

- [ ] Check: 100% of prospects have phone numbers
- [ ] Check: Date fields are in standard format (YYYY-MM-DD)
- [ ] Check: Company names are consistent (no "Al-Hamad Real Estate" and "hamad real estate")
- [ ] Resolve: Any missing critical fields

### Daily Automation Setup

- [ ] 9:00 AM: System analyzes prospects due for follow-up
- [ ] 10:00 AM: Founder approves draft messages (via `/decisions.html`)
- [ ] 11:00 AM: Messages auto-send on customer's behalf

## Day 2-14 Routine

After Day 1 setup, each day is simple:

**Founder morning (15 min):**
1. Log into `/decisions.html`
2. Review 3-5 proposed follow-ups for this customer
3. Approve or edit each one
4. Send approved messages
5. Log in Slack/WhatsApp: "Day X check-in: X prospects, X follow-ups sent, on track"

**Customer responsibility:**
- Check WhatsApp for outbound messages their team sent
- Respond to any inbound replies
- Weekly sync with founder (Friday 2 PM, 30 min)

**Founder tracking:**
- Log proof events: # prospects added, # deals advanced, # follow-ups working
- Track sentiment: Is customer engaged? Do they see value?
- Adjust: If something isn't working, pivot the strategy

## Common Day 1 Issues & Fixes

### Issue: "Customer has no CRM; all data is in notebooks/memory"

**Fix:**
- Founder takes 15 min notes from customer (name, company, last contact, notes)
- Enter into Google Sheets template
- Upload to Dealix
- Less clean data = less automation, but still works

### Issue: "Phone numbers are inconsistent (some Saudi, some international, some only 9 digits)"

**Fix:**
- Founder spends 10 min normalizing format: all `+966XXXXXXXXX`
- Use this regex: `^(\+966|00966|0)(.*)$` → replace with `+966$2`
- Dealix WhatsApp will fail on bad numbers; catch early

### Issue: "Data has lots of duplicates (same company listed 3 times)"

**Fix:**
- Founder de-dupes manually (takes 15 min for 100 entries)
- Or: Customer cleans their own data with founder's template
- Move on; don't spend more than 30 min on this

### Issue: "Customer says 'This will take us 2 hours to prepare data' "

**Fix:**
- You don't have 2 hours; sprint is 14 days
- Founder decision: Either customer preps today (before Day 2), or start with manual 20 prospects
- Say: "Start small, add more data on Day 3-4 once you see it working"

## Remember

**Day 1 is about:**
- Building confidence (it works!)
- Getting data in (so automation can work)
- Creating the first "win" (first follow-up sent)

**Day 1 is NOT about:**
- Perfect data (80/20 is fine)
- Advanced features (keep it simple)
- Upselling (focus on the pilot)

Get them to Day 7 with 1 deal advanced, and renewal is 80% likely.
