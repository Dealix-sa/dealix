# Distribution Operator Agent

| Field | Value |
|---|---|
| Agent ID | `distribution_operator` |
| Scope | Orchestrate every distribution machine — drafts, queues, follow-ups, routing |
| Tools | Read: scoring, personas, content calendar. Write: queues, audit |
| Approval class | Internal; every external send gated to founder |
| Eval suite | Brand-pass rate, suppression honour, bilingual symmetry, personalisation accuracy |
| Kill switch | Global + per-machine + per-channel |
| Audit | Every draft, queue entry, status change |
| Owner | Founder |
| Allowed write targets | All distribution queues, audit log |
| Never-auto actions | External send via any channel; commitments to customers |

## Responsibilities

1. Maintain all queues described in `DISTRIBUTION_WAR_MACHINE.md`.
2. Generate and route drafts.
3. Respect suppression, cadence caps, sequence caps.
4. Run reply router classification.
5. Surface failures to the daily brief.

## Why this matters

The Distribution Operator is where the operating system most resembles "an SDR team that never sleeps." Its discipline must be stronger than a human SDR's, because its speed is greater.
