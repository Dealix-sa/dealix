# Command Sprint — Delivery OS

> **Status:** CANONICAL · **Owner:** Founder · **Last reviewed:** 2026-06-05
>
> How to actually deliver a 7-day Command Sprint, repeatably.

---

## 1. Customer folder structure

```
customers/
  customer_name/
    00_intake.md
    01_revenue_map.md
    02_client_brain.md
    03_followup_pack.md
    04_offer_upgrade.md
    05_proof_register.md
    06_approval_register.md
    07_executive_command_brief.md
    08_next_30_days.md
```

## 2. Every artifact carries the same fields

```
source
analysis
assumptions
confidence
recommendation
approval_required
next_action
```

This makes every output honest, traceable, and safe to hand to the customer.

## 3. The 7-day flow

| Day | Output file(s) |
|---|---|
| 1 | `00_intake.md` — consent, scope, minimum data |
| 2 | `01_revenue_map.md`, `02_client_brain.md` |
| 3 | `03_followup_pack.md`, `04_offer_upgrade.md` |
| 4 | `05_proof_register.md` (+ objection intelligence) |
| 5 | `06_approval_register.md` + governance review |
| 6 | `07_executive_command_brief.md`, `08_next_30_days.md` |
| 7 | Review call + Proof Pack handoff + upsell conversation |

## 4. Doctrine checks before handoff

- [ ] No artifact contains auto-sendable actions presented as "done."
- [ ] Every customer-facing draft is marked `approval_required: true`.
- [ ] No claim without evidence; no proof published without approval.
- [ ] Only minimum data collected; consent captured in `00_intake.md`.
- [ ] Every recommendation has a `confidence` value.

## 5. Handoff

Assemble the artifacts into the **Command Pack** (see `PROOF_PACK_TEMPLATE.md` and
`EXECUTIVE_BRIEF_TEMPLATE.md`), walk the customer through it on the day-7 call, run
the Customer Success questions, and — if the sale rule is met — offer Managed
Business OS.

## 6. Customer Success questions (day 7)

- What was clearest to you?
- Which decision became easier?
- Which output will you actually use?
- What, if it continued weekly, would give you value?

Aim for proof level L2/L3. Do not chase L5 early.
