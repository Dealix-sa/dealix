# Executive Command Brief — Template

> **Status:** CANONICAL · **Owner:** Founder · **Last reviewed:** 2026-06-05
>
> The single most important artifact for the company owner. One page. Decisions,
> not dashboards.

---

## Template

```
Dealix — Executive Command Brief
Customer: <name>            Week of: <date>
Prepared by: <founder>      Approval contact: <name>

1. WHAT IS HAPPENING
   - <3–5 bullets, source-tagged>

2. WHAT BROKE / RISKS
   - <top risks from the risk register, with confidence>

3. DECISIONS NEEDED THIS WEEK
   | Decision | Owner | By when | Evidence | Approval class |
   |---|---|---|---|---|
   |  |  |  |  |  |

4. NEXT ACTIONS (Next Action Board top 5)
   - [ ] <action> — owner — due

5. WHAT NEEDS HUMAN APPROVAL
   - <from the Approval Register>

6. EVIDENCE / PROOF THIS WEEK
   - <proof level + link>
```

## Rules

- Maximum one page. If it doesn't fit, it isn't a command brief.
- Every claim is source-tagged and carries confidence.
- Every customer-facing action is a draft requiring approval.
- No invented metrics. If a number isn't measured, say so.
