# COMMAND SPRINT — DELIVERY PLAYBOOK

> How the **Dealix Command Sprint** (Rung 1, 499 SAR) is delivered: governed, time-boxed,
> approval-first, proof-ending. Pairs with `docs/01_go_to_market/COMMERCIAL_LADDER.md` and the
> `dealix-delivery` agent. **Zero external auto-send at any step.**

## Deliverables (the 8)

| # | Deliverable | What it is | Ends in |
|---|---|---|---|
| 1 | Market Intelligence Lite | governed read of the customer's market signals (founder inputs, no scraping) | Why-Now notes |
| 2 | Revenue Map | where revenue/ops leak + prioritized opportunities | scored map |
| 3 | Proof Register | the evidence trail for every claim made | Claims Register rows |
| 4 | Executive Command Brief | one-screen "what's happening / what's next" | brief doc |
| 5 | Approval Register | every external action queued for human approval | approval queue |
| 6 | Next Action Board | the prioritized next-actions | action board |
| 7 | Delivery Lite | a small, real operating output shipped (draft pack) | drafts (approved before send) |
| 8 | Upsell Recommendation | the honest next rung, only if proof supports it | recommendation |

## Delivery rhythm (records to ledgers)

Source Passport → DQ score → Account scoring → Draft pack → Governance review →
Proof Pack assembly → Capital asset registration → Retainer eligibility check.

Each step writes to the appropriate ledger under `docs/ledgers/`. See `.claude/agents/dealix-delivery.md`.

## Governance gates (non-negotiable)

- No external send. Every customer-facing action is a **draft queued for founder approval**.
- No proof without evidence (`docs/03_governance/PROOF_PACK_POLICY.md`).
- No upsell without proof. No guaranteed-revenue language.
- No customer name/logo/quote published without written approval.
- Every delivered capability carries a status label (`docs/00_platform_truth/MODULE_STATUS_MAP.md`).

## Day shape (indicative, 7-day)

- **Day 1–2:** Source Passport + Market Intelligence Lite + DQ score.
- **Day 3–4:** Revenue Map + Account scoring + Draft pack (drafts only).
- **Day 5:** Governance review + Approval Register.
- **Day 6:** Proof Pack assembly + Executive Command Brief + Next Action Board.
- **Day 7:** Founder review → Upsell Recommendation → retainer eligibility check.

## Definition of done

A signed-off **Proof Pack** + a populated **Claims Register** + a clear next-rung recommendation —
with every external action still pending the customer's approval, never auto-sent.
