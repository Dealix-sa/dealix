# Customer Folder Template

> كل عميل يدفع → مجلد منظم، كل ملف له front-matter حوكمة.

Generated by `scripts/targeting_delivery_handoff.py` at `customers/{slug}/`.

---

## Layout

```
customers/{slug}/
  00_intake.md                 # company, sector, offer, weakness→OS, paid date
  01_company_intelligence.md   # consolidated profile
  02_diagnostic_summary.md     # findings from the diagnostic call
  03_command_sprint_scope.md   # deliverables for the offer
  04_revenue_map.md            # opportunities + leakage points
  05_proof_register.md         # claims ↔ evidence (proof levels)
  06_next_action_board.md      # who / what / when
  07_executive_command_brief.md# the one-page exec view
  08_delivery_log.md           # each delivery step + artifact
  09_proof_pack.md             # proof levels L1→L5
  10_upsell_recommendation.md  # upsell / renewal decision
```

---

## Front-matter (every file)

```markdown
# {Title} — {Company}

- **source:** …
- **evidence:** …
- **assumption:** …
- **confidence:** low | medium | high
- **recommendation:** …
- **approval_required:** founder
- **next_action:** …
- **owner:** …
- **due_date:** YYYY-MM-DD
```

---

## Rules

- The folder is created **only** for a paid company.
- `slug` is filesystem-safe (lowercased, ASCII-folded, dashes).
- Files are scaffolds: populated during delivery, never overclaimed.

Tested by `tests/test_targeting_delivery_handoff.py`.
