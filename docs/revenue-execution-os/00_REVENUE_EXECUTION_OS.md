# Revenue Execution OS — نظام تنفيذ الإيرادات

## الغرض (AR)
نظام تنفيذ الإيرادات يحوّل كل الأنظمة السابقة إلى آلة يومية لتوليد الدخل. يأخذ الفرص الخام ويحوّلها عبر مسار واضح: بحث، ثم مسودات يجهّزها الذكاء الاصطناعي، ثم مراجعة المؤسس، ثم رسائل يدوية آمنة، ثم مكالمات، ثم تشخيصات مدفوعة، ثم تجارب (Pilots)، ثم عقود شهرية (Retainers)، مع إثبات التسليم والتقارير والمقاييس وحلقات القرار. المبدأ الحاكم: **الذكاء الاصطناعي يجهّز، والمؤسس يوافق، والإجراء يدوي فقط، ولا إرسال خارجي تلقائي.**

## Purpose (EN)
The Revenue Execution OS converts every prior system into a daily money-generating machine. It takes raw opportunities and moves them through a disciplined loop until they become realized revenue, while keeping every external action manual and founder-approved.

## حلقة التنفيذ / The Execution Loop
```
opportunities → drafts → founder review → safe manual messages → calls
→ paid diagnostics → pilots → retainers → delivery proof → reports
→ metrics → decision loops
```

- **opportunities / الفرص:** فرص خام من الأنظمة السابقة.
- **drafts / المسودات:** يجهّزها الذكاء الاصطناعي للمراجعة فقط.
- **founder review / مراجعة المؤسس:** بوابة بشرية إلزامية.
- **safe manual messages / رسائل يدوية آمنة:** يرسلها المؤسس بنفسه.
- **calls → diagnostics → pilots → retainers:** سلّم القيمة المتصاعد.
- **delivery proof → reports → metrics:** الإثبات والقياس.
- **decision loops / حلقات القرار:** كرّر، أوقف، أو وسّع.

## قواعد السلامة / Safety Posture
- لا إرسال تلقائي للبريد أو واتساب أو لينكدإن.
- لا كشط بيانات، ولا تعبئة نماذج تلقائية، ولا إطلاق إعلانات مدفوعة.
- لا جذب وهمي، ولا عائد مضمون، ولا ادعاءات غير مثبتة.
- لا أسرار أو مفاتيح API في المستودع.
- كل إجراء خارجي يدوي ومُعتمد من المؤسس.
