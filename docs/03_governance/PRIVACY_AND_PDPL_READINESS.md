# Privacy & PDPL Readiness

> **Status:** CANONICAL · **Owner:** Founder · **Last reviewed:** 2026-06-05
>
> Dealix is **PDPL-aware**, not "PDPL certified." This document tracks readiness,
> not certification.

---

## 1. The four disclosures we always make

A recent study of 100 Saudi e-commerce sites found only 31% disclosed all four of
these. Dealix discloses **all four**:

1. **Retention period** — see `DATA_RETENTION.md`.
2. **Right to request deletion** — supported and logged.
3. **Right to request a copy** — supported (A4 action).
4. **Complaints mechanism** — a named contact channel for data concerns.

## 2. PDPL posture

| Element | Posture |
|---|---|
| Consent | Captured at intake; lawful basis recorded. |
| Lawful basis | Consent / legitimate interest, documented per data category. |
| Retention | Defined defaults; deletable on request. |
| Suppression | Honored; logged. |
| Breach | Incident response process (see below). |
| Transfer | Sub-processors disclosed; cross-border posture reviewed before enterprise. |

## 3. What we never claim

"PDPL certified." We say **PDPL-aware** until an actual certification is held.
