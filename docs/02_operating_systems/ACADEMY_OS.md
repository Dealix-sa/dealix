# 14 — Academy OS — نظام الأكاديمية

**الحالة — Status: `DOCS_ONLY` (المنهج موجود كوثائق؛ التشغيل كقناة تبنّي `FUTURE`)**

> الأكاديمية ليست إضافة جانبية. هي قناة تبنّي وتوسّع: تدريب → تبنّي → استخدام → إثبات → تجديد.

## الغرض — Purpose

Academy OS هو طبقة التدريب والتبنّي: كيف يستخدم العميل ما سلّمناه استخدامًا آمنًا ومسؤولًا حتى يصبح جزءًا من إيقاعه التشغيلي. الأداة التي لا تُستخدَم لا تُجدَّد — لذلك التدريب ليس خدمة لاحقة بل **محرّك تبنّي وتوسّع** مرتبط مباشرة بالتجديد.

## القيمة — Value

- يرفع التبنّي الفعلي للأدوات بعد التسليم — استخدام لا رفّ.
- يحوّل التدريب إلى قناة توسّع (expansion) لا مجرّد دعم.
- يربط الاستخدام بالإثبات والتجديد عبر **حلقة تبنّي الأكاديمية**.

## سياق خارجي — External Context

تُظهر أبحاث الذكاء التوليدي (GenAI) في السعودية **اهتمامًا قويًا بتدريب مُنظَّم** يغطّي المهارات الأساسية، والتطبيقات القطاعية، والخصوصية، والأخلاقيات، والاستخدام المسؤول. *(سياق بحثي خارجي للتأطير، لا ادعاء عن دِيليكس.)* الفرصة: دِيليكس يُلبّي هذا الطلب بمنهج بنية واضحة يربط المهارة بالاستخدام الآمن.

## القدرات — Capabilities

- **تدريب العميل (Client training)** على ما سُلّم.
- **محو الأمية بالذكاء الاصطناعي (AI literacy)** للفِرق.
- **الاستخدام الآمن والمسؤول (Safe / responsible use)** والخصوصية والأخلاقيات.
- **إدخال الفريق (Team onboarding)** والإيقاع التشغيلي (operating rhythm).
- **الأدلة (Playbooks)** القابلة لإعادة الاستخدام.

## المُدخلات والمُخرجات — Inputs / Outputs

| المُدخل | المُخرج |
|---|---|
| تسليم مكتمل + قوالب من Knowledge OS | منهج تدريب جاهز |
| جلسة تدريب فريق | تبنّي مُقاس + إيقاع تشغيلي |
| استخدام فعلي | إشارة إثبات تُغذّي التجديد |

## البوابات والقواعد — Gates / Rules

- **لا نشر اسم عميل بلا موافقة** في مادة التدريب أو الأمثلة.
- **لا إثبات مزيّف** — الأمثلة حقيقية أو مُعلَّمة «نموذجية / case-safe».
- **تغطية الخصوصية والاستخدام المسؤول إلزامية** في كل منهج.
- **لا وعد بنتائج كحقيقة** — التبنّي يُقاس، لا يُضمَن.
