# Command OS

> **Status:** `BETA` · **Plane:** Decision · **Owner:** Founder · **Last reviewed:** 2026-06-05
>
> Command OS is the executive brain of Dealix. Company owners don't want more
> dashboards — they want a clear decision.

---

## Purpose

Turn everything happening across the company into a small set of clear decisions
each day and each week.

## Functions

- CEO daily brief
- Weekly Command Pack
- Risk register
- Decision log
- Priority stack
- Blockers
- Next actions
- Approvals queue

## The output (what the owner sees)

```
What is happening?
What broke?
What is the decision?
Who is responsible?
When is it executed?
What is the evidence?
```

## Inside the Command Sprint

Command OS produces the **Executive Command Brief** and the **Next Action Board**
in the Command Pack. It pulls from Revenue OS (pipeline), Proof OS (evidence),
and Governance OS (what needs approval).

## Inputs

Revenue Map, Proof Register, Approval Register, delivery blockers, customer
intake.

## Outputs

Executive Command Brief, Next Action Board, weekly decision log, risk register.

## Approval & doctrine

Command OS **recommends** decisions; it never executes external actions. Anything
customer-facing it suggests is a draft requiring human approval (see Governance
OS). Every recommendation carries `confidence` and `evidence`.
