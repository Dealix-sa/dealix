# 01 — Command OS

**Status: BETA** · العقل التنفيذي — The Executive Brain

> العميل لا يشتري لوحة تحكّم. العميل يشتري **وضوح القرار** — صفحة واحدة يقرأها صاحب القرار في 20 دقيقة ويتصرّف بناءً عليها.

## الغرض — Purpose

Command OS هو **العقل التنفيذي** لنظام تشغيل Dealix. مهمّته تحويل ضجيج الأنظمة السبعة (سوق، إيراد، أدلة، تسليم، عميل، دعم) إلى **حزمة قرار واحدة**: ماذا يحدث، ما الأهم، ما الخطر، وما الخطوة التالية. كل الأنظمة الأخرى تُغذّي Command OS؛ Command OS لا يجمع بيانات نيابة عنها — بل يُرتّبها للقرار.

## القيمة للعميل — Value to Customer

- **Decision clarity, not a dashboard.** المُخرَج الأساسي هو الـ **Executive Command Brief**: صفحة قرار، لا تقرير طويل.
- **إيقاع تشغيلي ثابت** يومي/أسبوعي يمنع المفاجآت الصامتة.
- **سجل قرار** قابل للتدقيق: مَن قرّر، متى، وعلى أي دليل.

## القدرات الأساسية — Core Capabilities

- **CEO Daily Brief** — موجز يومي للمدير التنفيذي.
- **Weekly Command Pack** — حزمة أسبوعية تجمع الأولويات والمخاطر والإنجاز.
- **Decision Log** — سجل القرارات المؤرّخ والمربوط بدليله.
- **Risk Register** — سجل المخاطر مع المالك ومستوى الخطورة.
- **Priority Stack** — ترتيب الأولويات بالأثر.
- **Blockers Board** — العوائق المفتوحة ومالك حلّها.
- **Next Actions** — 3 إجراءات تالية بمالك ومُهلة لكل واحد.
- **Approvals** — البوّابة التنفيذية: لا إجراء خارجي بلا موافقة مُسجَّلة.

## بنية الـ Executive Command Brief — Brief Structure

صفحة واحدة، بترتيب ثابت:

1. **Situation — الوضع:** سطران عن حالة الحساب/المشروع اليوم.
2. **What changed — ما تغيّر:** أهم 3 إشارات منذ آخر موجز، كل إشارة بدليلها.
3. **Priority stack — الأولويات:** أعلى 3 بنود بالأثر.
4. **Risks — المخاطر:** أعلى خطر واحد + مالكه.
5. **Decision needed — قرار مطلوب:** سؤال واحد واضح لصاحب القرار (نعم/لا أو خيار من خيارات).
6. **Next action — الخطوة التالية:** إجراء واحد، مالك، مُهلة.

## المُدخلات والمُخرجات — Inputs / Outputs

| المُدخلات — Inputs | المُخرجات — Outputs |
|---|---|
| إشارات من Market Intelligence OS | Executive Command Brief (يومي) |
| حالة خط الإيراد من Revenue OS | Weekly Command Pack |
| Proof Register من Proof OS | Decision Log + Risk Register |
| حالة التسليم من Delivery OS | Approval Register |
| تنبيهات من Support OS | Next Action Board |

## البوّابات والقواعد — Gates / Rules

- **No external customer-facing action without founder approval.** كل إجراء خارجي يمرّ عبر Approval Register.
- **No guaranteed revenue claims.** الموجز يعرض فرصًا مُثبتة بأدلة، لا أرقام مبيعات مضمونة.
- **Every decision logged** — لا قرار تنفيذي بلا قيد في Decision Log.
- **No silent delay** — تأخير التسليم يُسجَّل في Risk Register، لا يُخفى.

## الإيقاع التشغيلي — Operating Rhythm

- **Daily:** قراءة الـ CEO Daily Brief، تحديث Blockers، تأكيد/رفض القرار المطلوب.
- **Weekly:** بناء Weekly Command Pack، مراجعة Risk Register، إعادة ترتيب Priority Stack.
