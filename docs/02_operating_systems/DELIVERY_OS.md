# Delivery OS

> **Status:** `INTERNAL` (delivered as "Delivery Lite" in the Sprint) · **Plane:** Execution · **Owner:** Founder · **Last reviewed:** 2026-06-05

---

## Purpose

The delivery system. Turns Dealix from "a sales system" into "an operating system
for a services company."

## Functions

- Project plan
- SLA
- Blockers
- Change requests
- Owner
- Delivery evidence
- Completion pack

## Value

Services companies live or die on delivery. Delivery OS makes commitments,
owners, blockers, and completion evidence explicit — which also feeds Proof OS.

## Inside the Command Sprint

Delivered as **Delivery Lite** (`08_next_30_days.md` next-steps board): the next
execution steps with owners and a 30-day view.

## Inputs

Active projects, SLAs, blockers, change requests.

## Outputs

Delivery board, blocker list, completion packs, delivery evidence.

## Approval & doctrine

Delivery evidence feeds Proof OS but is published only with customer approval.
Status changes are logged.
