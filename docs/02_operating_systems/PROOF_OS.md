# 04 — Proof OS

**Status: BETA** · سلاح Dealix الحقيقي — Dealix's Real Weapon

> القاعدة الأم: **لا claim بدون evidence.** كل ادعاء يُربَط بمصدر/دليل، وإلا لا يُسلَّم.

## الغرض — Purpose

Proof OS هو الطبقة التي تجعل كل ما تقوله Dealix **قابلًا للتدقيق**. يحتفظ بسجل الأدلة، يجمع Proof Pack لكل عميل مدفوع، يتحقّق من الادعاءات، يوثّق قبل/بعد، ويدير اعتماد الحالات الدراسية. الدليل هو المنتج؛ Proof OS يحرسه.

## القيمة للعميل — Value to Customer

- كل رقم في التسليم مربوط بمصدره — لا أرقام بلا مرجع.
- **Proof Pack** قابلة للقراءة من صاحب القرار، تثبت ما حدث فعلًا.
- مصداقية تتراكم: الدليل اليوم يصبح حالة دراسية موافَقًا عليها غدًا.

## القدرات الأساسية — Core Capabilities

- **Evidence ledger** — سجل الأدلة المؤرّخ غير القابل للتلاعب.
- **Proof Pack** — حزمة الإثبات للعميل المدفوع.
- **Claims validation** — التحقق من كل ادعاء مقابل مصدر.
- **Before/After** — توثيق الحالة قبل وبعد بدليل.
- **Case-study approval** — مسار اعتماد العميل قبل أي نشر.
- **Proof-to-Market flywheel** — تحويل الدليل المعتمد إلى مادّة سوق.

## مستويات الإثبات — Proof Levels (L1–L5)

| المستوى | التعريف — Definition |
|---|---|
| **L1** | Dealix أنتجت مُخرَجًا (output created). |
| **L2** | العميل راجع المُخرَج (customer reviewed). |
| **L3** | العميل اعتمد المُخرَج (customer approved). |
| **L4** | العميل استخدم المُخرَج فعلًا (customer used). |
| **L5** | أثر رقمي قابل للقياس (measurable digital impact). |

**الهدف:** أول 30 يومًا → اللحاق بـ **L2/L3**. بعد 90 يومًا → ملاحقة **L4/L5**. لا نقفز إلى L5 بادعاء؛ نتدرّج بالدليل.

## ماذا تحتوي الـ Proof Pack — What a Proof Pack Contains

- **Intake & scope** — ما طُلب وما سُلّم.
- **Source Passport** — مصدر كل بيانات.
- **Evidence ledger** — قيود الأدلة المؤرّخة.
- **Claims table** — كل ادعاء + مصدره + مستوى الإثبات (L1–L5).
- **Before/After** — الحالة قبل وبعد.
- **Governance & approval log** — مَن وافق ومتى.
- **Limitations** — ما لم نثبته صراحةً.
- **Methodology** — كيف وصلنا للنتيجة.
- **Proof score** — درجة اكتمال الإثبات.

## المُدخلات والمُخرجات — Inputs / Outputs

| المُدخلات — Inputs | المُخرجات — Outputs |
|---|---|
| مُخرجات Delivery OS | Proof Pack مكتملة |
| ادعاءات Revenue OS و Market Intelligence | Claims validation result |
| موافقة العميل | Case-study معتمدة (اختياري) |

## البوّابات والقواعد — Gates / Rules

- **No fake proof** — لا دليل مُلفَّق، لا قبل/بعد مُجمَّل.
- **لا claim بدون evidence** — أي ادعاء بلا مصدر لا يدخل الحزمة.
- **No client name publishing without approval** — لا نشر بلا اعتماد L3 صريح.
- **No guaranteed revenue claims** — الأثر يُوصَف كـ estimated/observed، لا verified إلا بدليل قياس.
