# Role Architecture

## Purpose
Describe the named roles at Dealix today and in the near future.

## Current roles
- **Founder CEO** (Sami): owns strategy, sales, delivery, finance, trust.
- **Engineer sub-agent**: code, tests, migrations under founder direction.
- **Sales sub-agent**: qualifies leads, drafts proposals, never sends.
- **Delivery sub-agent**: per-customer sprint execution, no external messages.
- **Content sub-agent**: bilingual docs, SOPs, proposal templates.
- **PM sub-agent**: coordinates the other agents around milestones.

## Near-future roles (post-Sprint 9 + first paying customer)
- **Lead generation contractor (10 hr/week)**: hand-curates lead lists per ICP.
- **Bilingual content contractor (5 hr/week)**: drafts Arabic versions of approved English content.

## Mid-future roles (post-3 paying customers)
- **Delivery operator**: handles weekly retainer cadence under founder direction.
- **Finance contractor**: monthly book close.

## Far-future roles (post-SaaS gate)
- **Engineering lead**.
- **Customer success lead**.

## Each role gets
- A role doc with scope, decisions they can make, decisions escalated.
- Onboarding checklist.
- Access scope per `ACCESS_CONTROL_SYSTEM.md`.

## Forbidden roles for now
- A separate "growth marketer" before the operating system is producing pipeline manually.
- A separate "head of product" before the SaaS gate.

## Anti-patterns
- Title inflation.
- Hiring against vague needs.
- Founders abdicating accountability when delegating.
