# Role Map

Who owns what, even when the founder is doing several roles.

| Role | Owns | First-hire trigger |
|---|---|---|
| Founder / CEO | Strategy, key sales, A2 / A3 approvals, capital allocation | N/A |
| SDR / VA | Lead sourcing, first outreach, follow-up cadence | 100+ DMs/week is sustained |
| Delivery Analyst | Research, reports, lead tables | 3+ active engagements per week |
| Ops Manager | Workflows, QA, vendor coordination | 5+ retainers |
| Customer Success | Retention, renewals, upsell motions | 10+ retainers |
| Engineer | Product and automation | Product roadmap committed for the quarter |

## How the founder uses this map
- Every week, ask: which role am I doing the most this week?
- If the answer is the same role four weeks in a row, that role is overdue to be filled.
- Update `HIRING_TRIGGERS.md` when the threshold is crossed.

## Rule
Hiring is a decision logged in `DECISION_LOG.md`, not a vibe.
