# Delegation Ladder

## Purpose
Move tasks down a graded ladder so the founder eventually owns only the irreducible work.

## Rungs
1. **Founder solo**: only the founder does it.
2. **Founder + AI assist**: AI drafts, founder reviews and ships.
3. **Founder + contractor draft**: contractor drafts, founder approves and ships.
4. **Contractor solo**: contractor owns it; founder reviews monthly.
5. **Productized**: customers experience it as part of an offer; founder oversees system, not work.

## Rules
- A task moves up one rung at a time.
- A task moves up only after 30 days of clean execution at the current rung.
- A task can move back down at any time if quality drops.

## Per-task documentation requirement
Before a task moves above rung 2, the founder writes:
- **What good looks like** (sample artifact).
- **What bad looks like** (anti-examples).
- **The decision rules** the operator should follow.
- **Where to record evidence**.

## Logging
- `dealix-ops-private/people/delegation_log.csv` per move.
- Notes in the task's home doc/folder describing the rung change.

## Anti-patterns
- Moving a task two rungs in one step.
- "Trial week" delegations without recording the outcome.
- Delegating to AI without explicit boundary.
