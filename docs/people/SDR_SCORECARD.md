# SDR Scorecard

> 90-day evaluation. Measurable.

## Period
- Start date: yyyy-mm-dd
- End date: yyyy-mm-dd (90 days later)
- Reviewer: founder

## Core Metrics

| Metric | Target (90d) | Actual |
|--------|--------------|--------|
| Researched accounts added to pipeline | 300 | |
| Personalised DMs sent | 600 | |
| Replies received | 90 | |
| Calls booked (founder) | 20 | |
| Founder hours saved per week (vs pre-hire) | ≥ 15 | |
| Errors caught in audit (overclaim, off-ICP) | 0 | |
| Trust incidents involving SDR work | 0 | |

## Quality Checks (audit sample weekly)

- Personalisation depth (named signal cited per DM)
- ICP fit (accounts on-target ≥ 90%)
- Tone (matches founder voice + SAFE_LANGUAGE_LIBRARY)
- Bilingual handling (Arabic + English mix where appropriate)

## Outcomes Map

| 90-day result | Decision |
|---------------|----------|
| Metrics ≥ 90% targets + 0 trust incidents | Extend 6 months |
| Metrics 70–89% + 0 trust incidents | Extend 90 days with named focus |
| Metrics < 70% or any trust incident | End amicably |

## Documentation

- `dealix-ops-private/people/scorecards/sdr/<name>/<period>.md`
- Includes: audit samples, conversion data, founder observations.

## Anti-Patterns

- Scorecard built around "effort" not output.
- Forgiving missed targets because the SDR is "trying hard".
- Extending past 90 days without a written decision.
