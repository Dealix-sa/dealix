# Contractor Onboarding System

## Purpose
Bring a contractor up to safe productivity within 1 week.

## Pre-onboarding (Day 0)
- Written contract signed.
- NDA signed.
- Role doc shared.
- Access scope decided and logged in `people/access_log.csv`.

## Day 1
- Walk through the operating system map.
- Hand off the role doc and the relevant OS docs.
- Set up restricted access (minimum scope).
- Agree on weekly check-in.

## Days 2–3
- Contractor reads the OS docs.
- Contractor produces one micro-deliverable to confirm understanding.

## Days 4–7
- Contractor takes ownership of the first real task.
- Founder reviews and gives written feedback.

## Week 2 and beyond
- Weekly check-in (30 min).
- Async updates in agreed channel.
- Monthly written review.

## Access principles
- Read-only by default.
- Write access only to the contractor's task folder.
- Production access only after 30 days clean.

## Communication
- All work conversations in one agreed channel.
- All commits attributed to the contractor.
- All deliverables stored in the canonical folder.

## Exit
- 30 days' notice (either side) unless contractually otherwise.
- Access revoked within 1 business day of last day.
- Exit checklist signed in `people/access_log.csv`.

## Anti-patterns
- Giving "admin" access to save setup time.
- Letting contractors work in personal accounts.
- Skipping the documented role doc because "we'll figure it out".
