# Contractor Onboarding System

## Purpose
Onboard contractors safely without exposing unnecessary data or lowering quality.

## Before Contractor Starts
- define role
- define output
- define quality bar
- define data access
- define approval requirements
- define communication channel
- define payment terms

## Access Rules
- least privilege only
- no full private ops access unless necessary
- no secrets
- no payment data
- no client confidential data unless approved
- no GitHub write access unless trusted and needed

## Onboarding Checklist
- role scorecard shared
- SOP shared
- sample output shown
- QA checklist shared
- trust rules explained
- data boundary explained
- first task is low-risk

## First Week Review
- output quality
- rework needed
- communication quality
- trust compliance
- speed
- next decision: continue / fix / stop

## Rule
Never delegate trust-sensitive work before trust is proven.
