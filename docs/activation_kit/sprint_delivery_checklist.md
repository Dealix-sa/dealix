# Sprint Delivery Checklist — قائمة تسليم السبرنت

> The 7-day Revenue Intelligence Sprint as a day-by-day checklist. Each day has tasks, a gate that must pass before the next day, and an artifact. Calendar starts the first business day after acceptance + first-payment clearance.

---

## القسم العربي

### قبل اليوم 1 — بوابة البدء

- [ ] تم قبول العرض (رقم المشروع مُسجّل).
- [ ] تم تحصيل 50% الأولى عبر رابط ميسر، وصدرت فاتورة ZATCA.
- [ ] تم تحديد مالك سير عمل مُسمّى من جانب العميل.
- [ ] تم الاتفاق على سير عمل واحد لهذا السبرنت.

**البوابة:** لا يبدأ التقويم قبل قبول العرض + تحصيل الدفعة الأولى.

### اليوم 1 — الانطلاق + Source Passport

- [ ] جلسة انطلاق مع العميل.
- [ ] توقيع Source Passport (ملكية البيانات، الاستخدام المسموح، الاحتفاظ).
- [ ] تأكيد سير العمل الواحد والمالك المُسمّى.

**البوابة:** لا يُستورد أي بيان قبل توقيع Source Passport.
**الأثر:** Source Passport موقّع ومحفوظ.

### اليوم 2 — استيراد البيانات + درجة DQ الأساسية

- [ ] استيراد ملف الحسابات (CSV / تصدير CRM).
- [ ] حساب درجة جودة البيانات الأساسية عبر ست أبعاد.
- [ ] تشغيل كشف التكرار وتوثيق قواعد الدمج.

**البوابة:** درجة DQ محسوبة وموثّقة قبل التقييم.
**الأثر:** درجة DQ أساسية + تقرير إزالة التكرار.

### اليوم 3 — تقييم الحسابات + ترتيب أفضل 10

- [ ] تقييم الحسابات بمعيار قابل للتفسير (الملاءمة، قوة الإشارة، مخاطر الحوكمة).
- [ ] إخراج قائمة أفضل 10 حسابات مرتّبة.

**البوابة:** كل حساب في القائمة له سبب ترتيب مُوثّق بمصدر.
**الأثر:** قائمة 10 حسابات مرتّبة.

### اليوم 4 — توليد المسودات ثنائية اللغة + مراجعة الحوكمة

- [ ] توليد مسودات عربية وإنجليزية، كلها بعلامة `draft_only`.
- [ ] مراجعة الحوكمة (مصفوفة 7 قرارات) وتوثيق القرارات.
- [ ] حجب أو تنقيح أي مسودة غير آمنة وتسجيلها.

**البوابة:** لا مسودة تُمرَّر دون مراجعة حوكمة موثّقة. لا إرسال خارجي.
**الأثر:** حزمة مسودات ثنائية اللغة + سجل قرارات الحوكمة + ملخص الحجب.

### اليوم 5 — تجميع حزمة الإثبات

- [ ] تعبئة أقسام حزمة الإثبات الـ 14 من `proof_pack_template.md`.
- [ ] حساب نقاط الإثبات، الهدف ≥ 80.

**البوابة:** نقاط الإثبات ≥ 80 وكل الأقسام الـ 14 غير فارغة.
**الأثر:** حزمة إثبات من 14 قسماً بنقاط محسوبة.

### اليوم 6 — التسليم + تقييم جاهزية الاحتفاظ

- [ ] جلسة تسليم ثنائية اللغة (60 دقيقة) لاستعراض حزمة الإثبات.
- [ ] تقييم جاهزية الاحتفاظ: proof_score ≥ 80؟ adoption_score ≥ 70؟ مالك سير عمل باقٍ؟

**البوابة:** العميل استلم حزمة الإثبات واطّلع عليها.
**الأثر:** جلسة تسليم مُنجزة + تقييم جاهزية احتفاظ موثّق.

### اليوم 7 — تسجيل الأصل الرأسمالي + الملخص الآمن

- [ ] تسجيل أصل رأسمالي واحد على الأقل (قاعدة / قالب / رؤية قابلة لإعادة الاستخدام).
- [ ] إعداد ملخص حالة آمن (بلا PII، مُعلّم "قالب افتراضي / آمن للحالة" إن لم يُذكر عميل).
- [ ] تحصيل 50% المتبقية عبر رابط ميسر، وإصدار فاتورة ZATCA.
- [ ] إذا تحقّقت معايير الاحتفاظ → اعرض إدارة عمليات الإيرادات (2,999 / 4,999 ريال شهرياً).

**البوابة:** لا يُغلق المشروع دون حزمة إثبات من 14 قسماً ودون أصل رأسمالي مُسجّل.
**الأثر:** أصل رأسمالي مُسجّل + ملخص حالة آمن + قرار مسار الاحتفاظ.

## English Section

### Before Day 1 — start gate

- [ ] Proposal accepted (engagement ID logged).
- [ ] First 50% collected via Moyasar link, ZATCA invoice issued.
- [ ] Named workflow owner identified on the customer side.
- [ ] One workflow agreed for this Sprint.

**Gate:** the calendar does not start before acceptance + first-payment clearance.

### Day 1 — Kickoff + Source Passport

- [ ] Kickoff session with the customer.
- [ ] Source Passport signed (data ownership, allowed use, retention).
- [ ] Single workflow and named owner confirmed.

**Gate:** no data is imported before the Source Passport is signed.
**Artifact:** signed Source Passport on file.

### Day 2 — Data import + DQ baseline score

- [ ] Import the account file (CSV / CRM export).
- [ ] Compute the Data Quality baseline score across six dimensions.
- [ ] Run duplicate detection and document merge rules.

**Gate:** DQ score computed and documented before scoring.
**Artifact:** DQ baseline score + deduplication report.

### Day 3 — Account scoring + top-10 ranking

- [ ] Score accounts on an explainable rubric (fit, signal strength, governance risk).
- [ ] Produce the ranked top-10 account list.

**Gate:** every account on the list has a source-documented ranking reason.
**Artifact:** 10 ranked accounts.

### Day 4 — Bilingual draft generation + governance review

- [ ] Generate Arabic and English drafts, all marked `draft_only`.
- [ ] Governance review (7-decision matrix), decisions logged.
- [ ] Block or redact any unsafe draft and log it.

**Gate:** no draft passes without a logged governance review. No external sending.
**Artifact:** bilingual draft pack + governance decisions log + redaction/block summary.

### Day 5 — Proof Pack assembly

- [ ] Fill the 14 Proof Pack sections from `proof_pack_template.md`.
- [ ] Compute the proof score, target ≥ 80.

**Gate:** proof score ≥ 80 and all 14 sections non-empty.
**Artifact:** 14-section Proof Pack with a computed score.

### Day 6 — Handoff + retainer-readiness assessment

- [ ] Bilingual handoff session (60 minutes) walking through the Proof Pack.
- [ ] Retainer-readiness assessment: proof_score ≥ 80? adoption_score ≥ 70? workflow owner still named?

**Gate:** the customer has received and reviewed the Proof Pack.
**Artifact:** completed handoff session + logged retainer-readiness assessment.

### Day 7 — Capital asset registration + case-safe summary

- [ ] Register at least one Capital Asset (a reusable rule, template, or insight).
- [ ] Prepare a case-safe summary (no PII, labelled "Hypothetical / case-safe template" if no customer is named).
- [ ] Collect the remaining 50% via Moyasar link, issue ZATCA invoice.
- [ ] If retainer criteria are met → offer Managed Revenue Ops (2,999 / 4,999 SAR/mo).

**Gate:** no project closure without a 14-section Proof Pack and without a registered Capital Asset.
**Artifact:** registered Capital Asset + case-safe summary + retainer-path decision.
