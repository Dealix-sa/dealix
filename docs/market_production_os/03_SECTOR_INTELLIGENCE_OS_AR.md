## 1. وكالات التسويق — Marketing Agencies

- **الألم:** leads كثيرة لا تُغلق؛ متابعات متفرقة على WhatsApp.
- **صاحب القرار:** المؤسس / مدير النمو.
- **محفّز الشراء:** نهاية ربع بلا إغلاق كافٍ؛ توظيف مدير مبيعات.
- **العرض الأول:** الدرجة 1 (Sprint 499).
- **سير العمل الأول:** تشخيص تسرّب pipeline + Top 20 متابعة.
- **زاوية البريد البارد:** «leads عندكم لا تُغلق — نُظهر أين تضيع خلال 7 أيام.»
- **واتساب بعد الرد:** ملخص الفجوة + اقتراح جلسة 20 دقيقة.
- **زاوية العرض:** خريطة زوايا العرض + ذكاء اعتراضات.
- **حزمة الإثبات:** Proof Pack تحسّن ترتيب الأولويات (L3).
- **الاعتراضات:** «عندنا CRM» → نرتّب التشغيل لا الأداة.
- **لا تستهدف:** وكالة تطلب قوائم/أتمتة باردة.

## 2. شركات التدريب — Training Companies

- **الألم:** استفسارات لا تتحول إلى حجوزات.
- **صاحب القرار:** مدير البرامج / المالك.
- **محفّز الشراء:** موسم تسجيل / إطلاق دفعة جديدة.
- **العرض الأول:** الدرجة 1 ← 3.
- **سير العمل الأول:** طابور متابعة الاستفسارات + scorecard.
- **زاوية البريد البارد:** «استفسارات التدريب تتسرّب قبل الحجز — نرتّب المتابعة.»
- **واتساب بعد الرد:** تذكير حجز + رابط برنامج.
- **زاوية العرض:** خطة تشغيل 30 يوماً للتسجيل.
- **حزمة الإثبات:** Proof Pack نسبة تحويل استفسار←حجز (تقديري، L3).
- **الاعتراضات:** «الموسم متقلب» → نقيس مقابل baseline موسمي.
- **لا تستهدف:** تدريب أفراد B2C صرف بلا pipeline.

## 3. العيادات — Clinics

- **الألم:** حجوزات ومتابعات متناثرة؛ بيانات مرضى حساسة.
- **صاحب القرار:** مدير العيادة / المالك.
- **محفّز الشراء:** فرع جديد / حملة موسمية.
- **العرض الأول:** الدرجة 0 ← 1 (بحذر PDPL).
- **سير العمل الأول:** ترتيب متابعات الحجوزات (بلا بيانات صحية حساسة في المسودات).
- **زاوية البريد البارد:** «متابعات الحجوزات تضيع — نرتّبها بأمان وبموافقتكم.»
- **واتساب بعد الرد:** تذكير موعد فقط، بموافقة، بلا تفاصيل طبية.
- **زاوية العرض:** تشغيل متابعة مع حدود PDPL صارمة.
- **حزمة الإثبات:** Proof Pack انضباط المتابعة (L3) — بلا أي PII.
- **الاعتراضات:** «بيانات حساسة» → DPIA + حد أدنى + لا بيانات صحية في prompts/logs.
- **لا تستهدف:** أي طلب يمسّ بيانات صحية في رسائل خارجية.

## 4. فرق العقار — Real Estate Teams

- **الألم:** leads كثيرة، استجابة بطيئة، أولويات غير واضحة.
- **صاحب القرار:** مدير المبيعات / مالك المكتب.
- **محفّز الشراء:** إطلاق مشروع / مخزون جديد.
- **العرض الأول:** الدرجة 1.
- **سير العمل الأول:** scoring + Top 20 متابعة سريعة.
- **زاوية البريد البارد:** «سرعة الرد على leads العقار تصنع الصفقة — نرتّب الأولويات.»
- **واتساب بعد الرد:** بطاقة وحدة + موعد معاينة.
- **زاوية العرض:** قائمة أولويات يومية + scorecard.
- **حزمة الإثبات:** Proof Pack زمن الاستجابة (تقديري، L3).
- **الاعتراضات:** «السوق بطيء» → نركّز على الفرص العالية.
- **لا تستهدف:** وسطاء يطلبون قوائم مشتراة.

## 5. وكالات التوظيف — Recruitment Agencies

- **الألم:** مرشحون وعملاء كثر، متابعة مزدوجة فوضوية.
- **صاحب القرار:** مدير التوظيف / الشريك.
- **محفّز الشراء:** عقد توظيف كبير / توسّع عميل.
- **العرض الأول:** الدرجة 1 ← 2.
- **سير العمل الأول:** ترتيب pipeline العملاء (لا المرشحين كـ PII خارجي).
- **زاوية البريد البارد:** «pipeline عملاء التوظيف يحتاج ترتيباً — نُظهر أين تتعطل الصفقات.»
- **واتساب بعد الرد:** تحديث حالة شاغر، بموافقة.
- **زاوية العرض:** تشغيل أسبوعي لجانب العملاء.
- **حزمة الإثبات:** Proof Pack انضباط متابعة العملاء (L3).
- **الاعتراضات:** «بيانات مرشحين» → لا PII مرشحين في المسودات/السجلات.
- **لا تستهدف:** من يطلب scraping سير ذاتية أو قوائم.

## 6. الخدمات المهنية — Professional Services

- **الألم:** pipeline استشاري غير منظم؛ عروض تتأخر.
- **صاحب القرار:** الشريك / مدير التطوير.
- **محفّز الشراء:** نمو الطلب / فقد عرض كبير.
- **العرض الأول:** الدرجة 1 ← 3.
- **سير العمل الأول:** تشخيص pipeline + طابور عروض.
- **زاوية البريد البارد:** «عروضكم المهنية تتأخر — نرتّب المتابعة والأولوية.»
- **واتساب بعد الرد:** ملخص نطاق + موعد.
- **زاوية العرض:** تشغيل شهري مع scorecard.
- **حزمة الإثبات:** Proof Pack سرعة العرض (تقديري، L3).
- **الاعتراضات:** «خدماتنا معقدة» → نرتّب العملية لا المحتوى الفني.
- **لا تستهدف:** فرد مستقل بلا pipeline.

## 7. مزوّدو التعليم — Education Providers

- **الألم:** استفسارات تسجيل كثيفة موسمياً، متابعة ضعيفة.
- **صاحب القرار:** مدير القبول / المالك.
- **محفّز الشراء:** فتح تسجيل فصل / برنامج جديد.
- **العرض الأول:** الدرجة 1.
- **سير العمل الأول:** طابور متابعة القبول + scorecard.
- **زاوية البريد البارد:** «استفسارات التسجيل تذروة ثم تضيع — نرتّب المتابعة.»
- **واتساب بعد الرد:** تذكير موعد تسجيل، بموافقة.
- **زاوية العرض:** خطة 30 يوماً لموسم القبول.
- **حزمة الإثبات:** Proof Pack تحويل استفسار←تسجيل (تقديري، L3).
- **الاعتراضات:** «الموسمية» → baseline موسمي واضح.
- **لا تستهدف:** بيانات طلاب قُصّر في رسائل خارجية.

## 8. مجموعات المطاعم — Restaurant Groups

- **الألم:** علاقات موردين/فعاليات/B2B catering غير منظمة.
- **صاحب القرار:** مدير العمليات / المالك.
- **محفّز الشراء:** فرع جديد / عقد catering مؤسسي.
- **العرض الأول:** الدرجة 0 ← 1 (الجانب B2B فقط).
- **سير العمل الأول:** ترتيب متابعات عقود الجملة/الفعاليات.
- **زاوية البريد البارد:** «عقود الـ catering المؤسسي تحتاج متابعة منظمة.»
- **واتساب بعد الرد:** عرض باقة فعالية، بموافقة.
- **زاوية العرض:** تشغيل متابعة B2B.
- **حزمة الإثبات:** Proof Pack انضباط عقود الجملة (L3).
- **الاعتراضات:** «نحن B2C» → نخدم جانب B2B فقط.
- **لا تستهدف:** متابعة عملاء أفراد (خارج النطاق).

## 9. اللوجستيات — Logistics

- **الألم:** عروض أسعار وعقود شحن متفرقة، متابعة بطيئة.
- **صاحب القرار:** مدير المبيعات / مدير العمليات.
- **محفّز الشراء:** عميل شحن كبير / توسّع مسار.
- **العرض الأول:** الدرجة 1 ← 2.
- **سير العمل الأول:** scoring حسابات + طابور عروض الشحن.
- **زاوية البريد البارد:** «عروض الشحن تتأخر فتُفقد — نرتّب الأولوية والمتابعة.»
- **واتساب بعد الرد:** ملخص عرض سعر، بموافقة.
- **زاوية العرض:** تشغيل أسبوعي + scorecard.
- **حزمة الإثبات:** Proof Pack زمن دورة العرض (تقديري، L3).
- **الاعتراضات:** «أسعار متقلبة» → نركّز على سرعة المتابعة لا التسعير.
- **لا تستهدف:** وسيط يطلب قوائم شاحنين مشتراة.

## 10. شركات SaaS/الخدمات المحلية — Local SaaS / Service Firms

- **الألم:** تجارب لا تتحول لاشتراكات؛ onboarding غير منظم.
- **صاحب القرار:** المؤسس / مدير النمو.
- **محفّز الشراء:** إطلاق ميزة / جولة نمو.
- **العرض الأول:** الدرجة 1 ← 3.
- **سير العمل الأول:** تشخيص قمع التجربة←الاشتراك + Top 20 متابعة.
- **زاوية البريد البارد:** «تجاربكم تتسرّب قبل الاشتراك — نرتّب المتابعة بدليل.»
- **واتساب بعد الرد:** تذكير قيمة + موعد عرض، بموافقة.
- **زاوية العرض:** تشغيل شهري مع scorecard اشتراكات.
- **حزمة الإثبات:** Proof Pack تحويل trial←paid (تقديري، L3).
- **الاعتراضات:** «عندنا فريق نمو» → نكمّل التشغيل لا نستبدله.
- **لا تستهدف:** منتج بلا أي قاعدة مستخدمين / بلا بيانات.

## EN — Sector Intelligence OS Summary

This document turns 10 Saudi B2B sectors into usable operating angles: marketing agencies, training companies, clinics, real estate teams, recruitment agencies, professional services, education providers, restaurant groups, logistics, and local SaaS/service firms. For each sector it defines pain, decision maker, buying trigger, first offer (drawn from the five-rung ladder), first workflow, cold-email angle, WhatsApp-after-reply angle, proposal angle, proof-pack angle, objections, and do-not-target signals. The governing rule is that the sector selects the first offer, never the reverse, and every cold-email angle maps to exactly one catalog offer with all outreach starting as a draft under founder approval. Forbidden items — scraping, LinkedIn automation, cold WhatsApp, bulk/cold outreach, purchased lists — appear only as refusals; any manual public-company lookup is `linkedin_company_search (manual, founder-approved per call)`. Sensitive data (health records, minors' data) is never placed in external messages, prompts, or logs. Proof-pack figures are estimated (L3) until verified, with zero PII. A machine-readable seed lives at `auto_client_acquisition/market_production_os/seeds/sectors.yaml` (built separately); this is its human-readable counterpart. Cross-links: master spine, product catalog, brand OS, PDPL guard.
