# Offer Ladder — Dealix Ascension Path — سُلّم العروض

**Dealix — نظام تشغيل الإيرادات للشركات السعودية / Saudi B2B Revenue Operating System.**

السُّلّم يصف **كيف يتدرّج العميل بين العروض المعتمدة فقط** — لا يخترع البائع درجة جديدة ولا سعراً جديداً. مصدر الأسعار: [./DEALIX_REVOPS_PACKAGES_AR.md](./DEALIX_REVOPS_PACKAGES_AR.md). تعريف كل عرض: [./PRODUCT_CATALOG_AR.md](./PRODUCT_CATALOG_AR.md). كل الأسعار **نطاقات حواجز معتمدة من المؤسس — لا تُفعَّل تلقائياً.**

The ladder describes **how a customer ascends through the approved offers only** — a seller never invents a rung or a price. Prices: [./DEALIX_REVOPS_PACKAGES_AR.md](./DEALIX_REVOPS_PACKAGES_AR.md). Offer definitions: [./PRODUCT_CATALOG_AR.md](./PRODUCT_CATALOG_AR.md). All prices are **founder-approved guardrail ranges — never auto-charged.**

## الدرجات / The rungs

| # | الدرجة / Rung | السعر / Price (SAR) | غرضها / Its purpose |
|---|---|---|---|
| 0 | Free AI Ops Diagnostic | 0 | تأهيل + كشف الفجوة الأولى / Qualify + surface the first gap |
| 1 | Revenue Diagnostic | 3,500 → ~4,500 | إثبات وجود التسرّب بتقرير / Prove the leakage with a report |
| 2 | **Lead Intelligence Sprint** | 9,500 → ~18,000 | الدخول الافتراضي: بيانات + Top 50 + مسودات / Default entry |
| 3 | Pilot Conversion Sprint | 22,000 → ~45,000 | تشغيل حي 30 يوماً بأدلة / Live 30-day operation |
| 4 | Monthly RevOps OS | 15,000 / 25,000 شهرياً | استمرارية بإيقاع شهري / Monthly continuity |
| 5 | Enterprise AI Revenue OS | من 85,000 + 35,000–120,000/شهر | عميل كبير جاهز فقط / Large, ready customer only |

**نقطة الدخول الافتراضية / Default entry point:** Lead Intelligence Sprint (الدرجة 2). نبدأ منها ما لم يطلب العميل تشخيصاً أولاً أو لم تتوفر بياناته بعد.

## متى نرفع العميل درجة / When to move a customer up

كل صعود يحتاج **إشارة جاهزية + دليل من الدرجة السابقة**، لا مجرد حماس.

Every ascent needs a **readiness signal + proof from the prior rung**, not enthusiasm alone.

| الانتقال / Move | الشرط — متى نرفع / Condition to ascend |
|---|---|
| Free → Diagnostic | فجوة واضحة لكن العميل غير متأكد من حجمها ويملك عيّنة بيانات. |
| Free → Sprint | بيانات جاهزة (CSV/CRM export) + ألم متابعة واضح + جهة قرار → نتخطّى التشخيص. |
| Diagnostic → Sprint | التقرير أظهر فرصاً قابلة للتنفيذ والعميل وافق على معالجة البيانات (DPA). |
| Sprint → Pilot | Top 50 أنتجت اهتماماً حقيقياً والعميل يريد تشغيلاً حياً مع تتبع أسبوعي. |
| Pilot → Monthly RevOps | Proof pack الختامي أظهر إيقاعاً مستداماً والعميل يريد استمرارية. |
| Monthly → Enterprise | حجم كبير + حاجة لتكاملات/حوكمة + جاهزية تنفيذ داخلية + فريق مالك. |

**لا تقفز درجتين بلا دليل / Do not skip two rungs without proof.** عادة لا نبيع Enterprise قبل 3–5 عملاء Sprint ناجحين.

## متى لا نرفع — إشارات التوقف / When NOT to move up

- لا توجد أدلة من الدرجة الحالية (لا Proof Pack، لا نتيجة موثّقة).
- العميل يطلب ضماناً رقمياً للنتيجة — نوقف الصعود ونعيد ضبط التوقعات.
- لا جهة قرار واضحة أو لا قدرة دفع للدرجة الأعلى.
- طلب قناة محظورة (scraping، إرسال بارد آلي) — نرفض ونصحّح، لا نصعد.
- لم يُغلق التزام الدرجة الحالية مالياً وتشغيلياً بعد.

## القاعدة الحاكمة — موافقة المؤسس / Governing rule — founder approval

**أي سعر، أي خصم، أي التزام تعاقدي، وأي قفزة درجة = تحتاج موافقة المؤسس قبل تثبيتها مع العميل.** البائع/الوكيل يقترح ويجهّز المسودة؛ المؤسس يعتمد. لا سحب تلقائي للمبالغ. تفصيل الحدود: [./PRICING_GUARDRAILS_AR.md](./PRICING_GUARDRAILS_AR.md).

**Any price, any discount, any contractual commitment, and any rung jump = require founder approval before it is fixed with the customer.** The seller/agent proposes and drafts; the founder approves. No auto-charge. Bands detail: [./PRICING_GUARDRAILS_AR.md](./PRICING_GUARDRAILS_AR.md).

## إيقاع الصعود / Ascension rhythm

1. أنهِ تسليم الدرجة الحالية بـ Proof Pack موثّق.
2. اعرض الدرجة التالية بنتيجة محددة، لا بميزات عامة.
3. جهّز المسودة (نطاق + سعر ضمن النطاق) → ارفعها للمؤسس.
4. بعد موافقة المؤسس، ثبّت الالتزام واطلب الموافقة على معالجة البيانات حيث ينطبق.
5. سجّل الانتقال في سجل الفرص.

## القواعد غير القابلة للمساومة / Non-negotiables

- لا عرض خارج الكتالوج المعتمد. لا scraping، لا إرسال بارد آلي، لا أتمتة LinkedIn.
- لا ضمان نتائج — **فرص مُثبتة بأدلة / evidenced opportunities**.
- لا إجراء خارجي بلا موافقة. لا مشروع بلا Proof Pack.
