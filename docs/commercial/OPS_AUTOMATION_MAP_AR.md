## 1. المبدأ — نُؤتمت المكتب الخلفي، ونُبقي البيع بشرياً

تحت **التجميد التجاري النشط** (`../ops/COMMERCIAL_FREEZE.md`)، كلمة "أتمتة كاملة" لها قراءة واحدة صحيحة فقط: **نُؤتمت المكتب الخلفي — التقارير، اللقطات، المراقبة، الملخّصات، السجلّات — ونُبقي البيع قراراً بشرياً بموافقة مسبقة.** البنية الآلية موجودة فعلاً في المستودع؛ هذه الوثيقة توثّقها كخريطة مرجعية واحدة حتى يعرف المؤسّس بدقّة ما الذي يعمل تلقائياً، وما الذي يبقى يدوياً بحُكم العقيدة.

> **القاعدة الذهبية:** كل ما يقرأ الحالة ويُلخّصها ويراقبها — يُؤتمت. كل ما يُرسل رسالة خارجية أو يُقرّر بيعاً أو يلتقي عميلاً — يبقى بشرياً بموافقة صريحة. الآلة تُجهّز الدليل؛ المؤسّس يتّخذ القرار.

The principle: automate everything that reads, summarizes, and monitors system state. Keep human everything that sends an external message, decides a sale, or meets a customer. Machines prepare the evidence; the founder makes the decision.

## 2. المهام المؤتمتة — Automated jobs

كل صف أدناه مؤكَّد موجوداً في المستودع، مع جدوله الزمني الفعلي المقروء من `.github/workflows/`. كل المخرجات إمّا artifact (للتدقيق) أو بريد للمؤسّس فقط — **لا إرسال خارجي**.

### يومياً — Daily

| المهمة | ماذا تفعل | الجدول / المُشغّل | المخرَج | المالك |
|--------|-----------|-------------------|---------|--------|
| `daily_digest.yml` → `dealix_morning_digest.py` | يجمع ملخّص حلقة النمو: daily-brief، مرشّحو تفعيل الخدمة، partner-radar، أولويات geo_aio_radar، حالة المحيط | `cron: 0 4 * * *` — 7ص توقيت السعودية | بريد للمؤسّس فقط (Resend) | المؤسّس |
| `daily_snapshot.yml` → `dealix_snapshot.py` | يكتب لقطة تدقيق صامتة لحالة النظام | `cron: 0 5 * * *` — 8ص توقيت السعودية | artifact بصيغة JSON، احتفاظ 90 يوماً | المؤسّس |
| `daily-revenue-machine.yml` | يولّد مسوّدات (drafts) ومتابعات وتقرير يومي في وضع `draft_only` فقط | `cron: 0 4 * * *` — 7ص توقيت السعودية | artifacts للمسوّدات، احتفاظ 14 يوماً — **مسوّدات لا تُرسَل** | المؤسّس |

### أسبوعياً — Weekly

## 3. يدوي بحُكم العقيدة — Manual by doctrine

ما يلي **ليس مؤتمتاً ولن يُؤتمت** — لا بسبب نقص بنية، بل بحُكم العقيدة. المرجع: القرار الخامس في `COMMERCIAL_CONTROL_TOWER_AR.md` وعدم التفاوض عليه تحت `../ops/COMMERCIAL_FREEZE.md`.

| النشاط | الحالة | لماذا يبقى يدوياً |
|--------|--------|-------------------|
| إرسال رسالة لعميل محتمل | يدوي بموافقة | Dealix لا يرسل رسائل خارجية نيابة عن العميل دون موافقته الصريحة. |
| Outreach جماعي / cold WhatsApp | غير معروض إطلاقاً | ليس خدمة معروضة — القرار الخامس صريح. |
| أتمتة LinkedIn / scraping | غير معروض إطلاقاً | ليس خدمة معروضة — القرار الخامس صريح. |
| مكالمات العرض التوضيحي (Demo) | يدوي بالكامل | العرض 12 دقيقة بشرية تثبت نقطة واحدة، لا سكربت آلي. |
| الموافقة على المسوّدات والاقتراحات | يدوي بالكامل | الآلة تُنتج `draft_only`؛ المؤسّس يراجع ويعتمد قبل أي إرسال. |
| إرسال التقرير التنفيذي الأسبوعي | يدوي بحُكم المادة 4 | `dealix_weekly_executive_pack.py` لا يُرسل بريداً آلياً — المؤسّس ينسخه يدوياً. |
| اعتماد التجديد الشهري | يدوي بالكامل | `monthly_cadence_runner.py` يجهّز التقرير؛ المؤسّس يراجع ويوافق. |
| تسجيل النقد والإيراد | يدوي بدليل | كل رقم تقديري إلى أن يُتحقَّق منه — لا ادّعاء إيراد آلي. |

القاعدة الموجزة: **المسوّدة آلية، الإرسال بشري.** أي توسعة آلية تمسّ أحد البنود أعلاه مرفوضة افتراضياً.

## 5. ثغرات P0/P1 معروفة — للمؤسّس (لا تُبنى الآن)

هذه أفعال **إعداد** يقوم بها المؤسّس، وليست عملاً برمجياً. مُدرجة هنا للشفافية فقط.

| الثغرة | المستوى | الفعل المطلوب من المؤسّس |
|--------|---------|--------------------------|
| `RESEND_API_KEY` غير مُهيّأ | P1 | بدونه يتخطّى `daily_digest` الإرسال بصمت — يُضاف عبر Settings → Secrets and variables → Actions |
| `DEALIX_API_BASE` / `DEALIX_API_KEY` غير مُهيّأة | P1 | بدونها يتخطّى `daily-revenue-machine` تشغيله — تُضاف كأسرار للمستودع |
| Sentry DSN غير مُهيّأ | P1 | لا تتبّع أخطاء مركزي — يُهيّأ DSN في إعدادات الإنتاج |
| UptimeRobot غير مُهيّأ | P1 | حالياً `scheduled_healthcheck.yml` (كل 15 دقيقة) يقوم مقامه كبديل — تهيئة UptimeRobot تعطي مراقبة مستقلّة عن GitHub |
| `WATCHDOG_SLACK_URL` غير مُهيّأ | P1 | بدونه لا تصل تنبيهات انحراف Slack — يُضاف كسرّ للمستودع |
| `STAGING_REDIS_URL` / `PRODUCTION_REDIS_URL` غير مُهيّأة | P1 | بدونها يتخطّى `dlq_check` فحص عمق DLQ — تُضاف كأسرار لكل بيئة |

Known config gaps handed to the founder. None of these is a build task; each is a one-time secret or service configuration the founder owns.
