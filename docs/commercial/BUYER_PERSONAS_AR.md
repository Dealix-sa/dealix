# Buyer Personas — Dealix Decision-Makers — شخصيات المشترين

**Dealix — نظام تشغيل الإيرادات للشركات السعودية / Saudi B2B Revenue Operating System.**

خمس شخصيات نخاطبها داخل الحساب الواحد. لكل شخصية: ما يهمّها، ألمها، اعتراضها، وزاوية الرسالة. هذه شخصيات نمطية (أنماط أدوار)، **ليست عملاء حقيقيين**. العروض: [./PRODUCT_CATALOG_AR.md](./PRODUCT_CATALOG_AR.md). الاعتراضات الكاملة: [./OBJECTION_BANK_AR.md](./OBJECTION_BANK_AR.md). التأهيل: [./ICP_MATRIX_AR.md](./ICP_MATRIX_AR.md).

Five personas we address inside one account. For each: what they care about, their pain, their objection, and the message angle. These are role archetypes, **not real customers**. Offers: [./PRODUCT_CATALOG_AR.md](./PRODUCT_CATALOG_AR.md). Full objections: [./OBJECTION_BANK_AR.md](./OBJECTION_BANK_AR.md).

## 1) المؤسس / المدير العام — Founder / GM

- **الدور / Role:** المالك النهائي للقرار والميزانية. يرى الصورة الكاملة للإيراد.
- **ما يهمّه / Cares about:** نمو إيراد يمكن إثباته، انضباط تشغيلي، تقليل الاعتماد على أفراد، سمعة الشركة تحت PDPL.
- **الألم / Pain:** «أنفق على التسويق وأرى فرصاً تضيع، ولا أملك دليلاً واضحاً على أين يتسرّب الإيراد.»
- **الاعتراض / Objection:** «هل أرى عائداً يستحق الالتزام؟ ولماذا الآن؟»
- **زاوية الرسالة / Message angle:** ابدأ بـ Revenue Diagnostic أو Lead Intelligence Sprint بنتيجة محددة وProof Pack قبل أي التزام شهري — **فرص مُثبتة بأدلة**، لا وعود. اعرض المسار التصاعدي في [./OFFER_LADDER_AR.md](./OFFER_LADDER_AR.md).

## 2) قائد المبيعات — Sales Lead

- **الدور / Role:** مسؤول الـ pipeline والأرقام الربعية. يقود الفريق التنفيذي.
- **ما يهمّه / Cares about:** فرص مؤهلة جاهزة، ترتيب أولويات الاتصال، رسائل تُحرّك الردود، تقارير تُظهر التقدّم.
- **الألم / Pain:** «الفريق يلاحق عشوائياً؛ لا نعرف من نتصل أولاً ولا بأي رسالة، والمتابعة تضيع بعد أول تواصل.»
- **الاعتراض / Objection:** «عندنا CRM بالفعل، ما الجديد؟»
- **زاوية الرسالة / Message angle:** CRM يخزّن الصفقة؛ Lead Intelligence Sprint يرتّب **من نلاحق ولماذا وبأي دليل** — Top 50 + خطة 10 إجراءات + حتى 20 مسودة جاهزة للمراجعة.

## 3) قائد التسويق — Marketing Lead

- **الدور / Role:** مسؤول توليد الطلب وجودة الفرص المسلَّمة للمبيعات.
- **ما يهمّه / Cares about:** تحويل أعلى للفرص الموجودة، محاذاة تسويق-مبيعات، إثبات أثر الإنفاق التسويقي.
- **الألم / Pain:** «أُسلّم فرصاً وتذوب في المبيعات بلا متابعة؛ يصعب إثبات أن التسويق ينتج إيراداً.»
- **الاعتراض / Objection:** «هل هذه أتمتة إرسال جماعي؟ لن أخاطر بسمعة الدومين.»
- **زاوية الرسالة / Message angle:** لا إرسال بارد آلي إطلاقاً — **مسودات بموافقة بشرية فقط**، وكثافة في الإنتاج مع حذر في الإرسال (راجع [../market_production_os/README.md](../market_production_os/README.md)). النتيجة: استعادة المتابعة الضائعة لفرص لديك أصلاً.

## 4) مدير العمليات / CRM — Ops / CRM Manager

- **الدور / Role:** حارس البيانات والعمليات والأدوات الداخلية.
- **ما يهمّه / Cares about:** نظافة البيانات، dedupe، حوكمة، تكامل سلس، امتثال PDPL، سجل تدقيق.
- **الألم / Pain:** «البيانات مكرّرة ومبعثرة ومصادرها غير موثّقة؛ كل تقرير يحتاج تنظيفاً يدوياً.»
- **الاعتراض / Objection:** «أين تُخزَّن البيانات؟ وهل تلتزمون PDPL وحدود المصادر؟»
- **زاوية الرسالة / Message angle:** dedupe + التحقق من المصدر + قسم امتثال داخل التسليم، DPA عند التعاقد، وحدود مصادر صريحة (لا scraping). كل ذلك ضمن Lead Intelligence Sprint وما بعده.

## 5) المدير المالي / المشتريات — Finance / Procurement (شخصية ثانوية)

- **الدور / Role:** يحكم الالتزام المالي وشروط التعاقد.
- **ما يهمّه / Cares about:** وضوح السعر، لا مفاجآت، لا سحب تلقائي، قيمة مقابل الالتزام.
- **الألم / Pain:** «أخشى اشتراكاً يتجدد صامتاً بلا إثبات قيمة.»
- **الاعتراض / Objection:** «السعر عالٍ / نلتزم شهرياً بلا ضمان؟»
- **زاوية الرسالة / Message angle:** كل الأسعار **نطاقات حواجز معتمدة من المؤسس — لا سحب تلقائي**، والاشتراك الشهري يُراجَع بالأدلة كل دورة. ابدأ صغيراً (Diagnostic/Sprint) قبل الالتزام الأكبر. راجع [./PRICING_GUARDRAILS_AR.md](./PRICING_GUARDRAILS_AR.md).

## قواعد المخاطبة / Addressing rules

- لكل شخصية **زاوية واحدة واضحة**، لا رسالة عامة تخاطب الجميع.
- استخدم **فرص مُثبتة بأدلة / evidenced opportunities** بدل أي وعد رقمي.
- لا تَعِد بإرسال نيابة عن العميل — **مسودات بموافقة** دائماً.

## القواعد غير القابلة للمساومة / Non-negotiables

- لا scraping، لا إرسال بارد آلي، لا أتمتة LinkedIn.
- لا ضمان نتائج. لا PII في السجلّات. لا إجراء خارجي بلا موافقة.
