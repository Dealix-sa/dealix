# Dealix — تنفيذ دخول السوق للمؤسس

## القرار المؤسس

Dealix لا ينتقل من «الكود موجود» إلى «جاهز للتوسع» مباشرة. مسار السوق الرسمي:

1. **Evidence Required** — بناء واختبار داخلي فقط.
2. **Private Pilot Ready** — تجربة دافئة خاصة، بقيادة المؤسس، وكل إجراء خارجي بموافقة.
3. **Limited Launch Ready** — إطلاق محدود بعد صحة الإنتاج وثلاث تجارب مدفوعة وثلاث حزم إثبات.
4. **Scale Ready** — توسع بعد تكرار التسليم والاحتفاظ وقياس الهامش.

وجود ملف أو اختبار لا يثبت نتيجة عميل. الـProof Pack لا يصبح دراسة حالة عامة إلا
إذا كانت النتيجة منسوبة، والعميل وافق على النشر.

## عرض الدخول الوحيد

- الـCTA العام قبل اعتماد السعر هو **محادثة تأهيل**؛ لا شراء فوري ولا Checkout.
- العرض الأساسي لأول 90 يومًا هو **Revenue Command Pilot — 30 يومًا**:
  ICP واحد، سير عمل إيرادي واحد، baseline من بيانات الطرف الأول، هدف تشغيلي
  قابل للقياس، قراءة تنفيذية أسبوعية، وProof Pack نهائي مع قرار
  stop/redesign/expand.
- السعر العام غير معتمد حتى تكتمل خمس محادثات مؤهلة، ومراجعة الهامش المحمّل
  بوقت المؤسس، والضريبة، والفوترة الإلكترونية، ثم يسجّل قرار المؤسس في
  `dealix/config/first_launch_offer_gate.yaml`.
- عرض 7 أيام/499 ر.س. تراثي داخلي للتوافق فقط؛ لا يُنشر ولا يُقتبس ولا يُربط
  بالدفع.
- كل عرض أو خصم أو اقتباس خارجي يحتاج موافقة منفصلة. الـRetainer الشهري
  quote-only ولا يُعرض إلا بعد Proof Pack منسوب وقبول حالة التوسع.

## ICP البداية

ابدأ بشركة B2B سعودية لديها:

- Pipeline أو قائمة عملاء تملكها الشركة وتسمح بمعالجتها.
- مشكلة واحدة قابلة للقياس في المتابعة أو ترتيب الفرص أو إعداد العروض.
- صاحب قرار معروف وقادر على اعتماد Pilot صغير.
- استعداد لتوفير baseline ومراجعة المخرجات وتوقيع Proof Pack.

لا تبدأ بحساب مجهول، ولا بيانات مشتراة أو scraped، ولا WhatsApp بارد.

## بوابات المراحل

### تجربة خاصة

- عرض واحد وسعر واحد معتمدان.
- Dry run كامل من intake إلى Proof Pack.
- Approval Queue تعمل دون إرسال خارجي.
- مراجعة PDPL لنموذج البيانات.
- خمسة حسابات دافئة مع مصدر وموافقة قناة.

### إطلاق محدود

- `/health` موثق وCI المطلوب أخضر وأسرار الإنتاج مدوّرة.
- مسار الدفع والعقد والاسترداد معتمد.
- 3 Pilots مدفوعة مكتملة.
- 3 Proof Packs موقعة.
- دراسة حالة واحدة بموافقة نشر.
- 3 عينات هامش Pilot، وهامش إجمالي 55% أو أكثر بعد احتساب وقت المؤسس.
- سعة تسليم 3 Pilots شهريًا دون خفض الجودة.
- عقد Pilot وDPA ومسار الحوادث والاسترداد معتمدة ومختبرة.

### توسع

- 5 Pilots مدفوعة على الأقل.
- 3 عملاء Retainer نشطين.
- 5 عينات تسليم ونسبة التزام بالوقت 90% أو أكثر.
- وسيط الوصول لأول قيمة 7 أيام أو أقل.
- 3 قرارات تجديد ونسبة تجديد 70% أو أكثر.
- 5 عينات CAC شاملة وقت المؤسس وفترة استرداد 3 أشهر أو أقل.

هذه حدود قرار أولية قابلة للمراجعة، وليست ادعاءً بأن Dealix حققها.

## التنفيذ

```bash
python scripts/commercial/run_founder_market_entry.py
python scripts/commercial/verify_market_entry_readiness.py
```

المشغل يكتب JSON وMarkdown وجداول CSV متوافقة مع Sheets/Airtable تشمل:
Strategy Backlog وAction Queue وApproval Queue وOpportunity Graph وProof Ledger
وSelf Improvement وContacts Radar. جداول العملاء تبقى فارغة حتى إدخال بيانات
حقيقية مسموحة؛ لا يولّد النظام أسماء أو إثباتات وهمية.

## ما يحتاج موافقة منفصلة

- الإرسال أو النشر الخارجي.
- تشغيل الدفع الحقيقي.
- تغيير السعر أو الخصم.
- دمج `main` أو ترقية إنتاج.
- نشر اسم العميل أو نتائج دراسة الحالة.
