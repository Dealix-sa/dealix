# مواصفات المحركات الاثني عشر — The 12 Engines Spec

> **قاعدة ذهبية:** كل محرك يُصمَّم الآن، ويُبنى عند فتح بوابته. وكل محرك — بلا استثناء — يتوقّف عند **بوابة موافقة بشرية**. الأتمتة تُجهّز القرار؛ الإنسان يتخذه.
> **Golden rule:** Every engine is designed now, built at its gate, and stops at a human-approval gate. Automation prepares the decision; the human makes it.

## النطاق / Scope

- **هذا الملف:** المواصفة التفصيلية لكل محرك من المحركات الاثني عشر (E1–E12): الغرض، ما يُؤتمت مقابل ما يبقى بشرياً، بوابة الفتح، الوكيل المالك، المدخلات والمخرجات، والمخاطر والحوكمة.
- **ليس هذا الملف:** الاستراتيجية الحاكمة أو الفهرس الرئيسي ([`LAUNCH_MASTER_PLAN.md`](LAUNCH_MASTER_PLAN.md))، ولا معايير البوابات التفصيلية ([`GATE_CRITERIA.md`](GATE_CRITERIA.md))، ولا قائمة الأسعار (المصدر القانوني: [`../OFFER_LADDER_AND_PRICING.md`](../OFFER_LADDER_AND_PRICING.md)).
- **الجمهور:** المؤسس والوكلاء التشغيليون (dealix-pm / dealix-sales / dealix-delivery / dealix-content / dealix-engineer).

## كيف تُقرأ هذه المواصفة — How to read this spec

- **بوابة الفتح (unlock gate):** متى يُسمح ببناء المحرك — `الآن / G0 / G1 / G2 / G3`. المعايير الكاملة لكل بوابة في [`GATE_CRITERIA.md`](GATE_CRITERIA.md).
- **الحالة:** المحركات `E2 / E3 / E7 / E11` متوافقة مع Commercial Freeze ونشطة الآن ([`../ops/COMMERCIAL_FREEZE.md`](../ops/COMMERCIAL_FREEZE.md)). البقية تُصمَّم الآن وتُبنى عند بوابتها — البناء عند الطلب تحكمه [`../sales-kit/CONDITIONAL_BUILD_TRIGGERS.md`](../sales-kit/CONDITIONAL_BUILD_TRIGGERS.md).
- **الحوكمة:** كل محرك يُسمّي صراحةً أي بند من **اللوائح الإحدى عشرة غير القابلة للمساومة** يحميه.

### اللوائح الإحدى عشرة غير القابلة للمساومة — The 11 non-negotiables

1. لا scraping. 2. لا cold WhatsApp/LinkedIn automation. 3. لا fake proof. 4. لا وعود نتيجة أو ROI مضمونة. 5. لا PII في السجلات. 6. لا ادعاء بلا مصدر. 7. لا مخرج AI للعميل بلا QA بشري. 8. لا live send. 9. لا live charge. 10. موافقة بشرية لكل إجراء خارجي. 11. لا تقدّم مرحلة بلا دليل مُتحقَّق.

## E1 — Revenue Activation

| الحقل | التفاصيل |
|-------|----------|
| **الغرض** | تفعيل قناة قبض المال: ربط Moyasar، توليد payment links، إصدار invoices مرتبطة بالـ SOW. بلا هذا المحرك لا يوجد عرض مدفوع. |
| **بوابة الفتح** | **G0** — يُصمَّم الآن، يُبنى عند فتح G0. |
| **الوكيل المالك** | dealix-engineer |
| **ما يُؤتمت (داخلي)** | توليد رابط الدفع لكل عرض، تجهيز مسودة الفاتورة المرتبطة بالـ SOW، تأكيد استلام الدفع من Moyasar، تسجيل الحدث في capital ledger. |
| **ما يبقى بشرياً (مُبوَّب)** | موافقة المؤسس على إرسال رابط الدفع، مراجعة الفاتورة قبل الإرسال، تأكيد الاعتراف بالإيراد. لا live charge — لا خصم تلقائي بدون قبول مكتوب. |
| **المدخلات** | عرض مقبول، SOW موقّع، بيانات العميل التجارية (لا PII حساسة في السجل). |
| **المخرجات** | رابط دفع جاهز، فاتورة، حدث `invoice_sent` / `payment_received` في السجل. |
| **المخاطر والحوكمة** | يحمي البند 9 (لا live charge) والبند 10 (موافقة لكل إجراء خارجي). المرجع المالي: [`FINANCIAL_MODEL.md`](FINANCIAL_MODEL.md). |
