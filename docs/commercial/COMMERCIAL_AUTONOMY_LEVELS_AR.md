# مستويات الاستقلالية التجارية (Commercial Autonomy Levels)

يعمل النظام بأربعة مستويات استقلالية. المستوى الافتراضي هو **A0** الأكثر أماناً.

## A0 — مسودة فقط (DRAFT_ONLY) — الافتراضي

- يولّد توصيات ورسائل وعروضاً وخيارات حجز.
- **لا** إرسال خارجي ولا كتابة تقويم إطلاقاً.
- كل بطاقة تحمل `owner_decision=pending` و`next_action`.

## A1 — مشغّل مُساعَد (ASSISTED_OPERATOR)

- يجهّز النظام الحمولات (payloads) الدقيقة.
- يوافق المستخدم على كل إجراء في غرفة القيادة.
- المستخدم يرسل يدوياً أو يضع الحالة `approved`.

## A2 — مباشر مُتحكَّم به (CONTROLLED_LIVE)

- يستطيع النظام إرسال رسائل منخفضة المخاطر معتمدة وإنشاء أحداث تقويم.
- **فقط** بعد اجتياز كل بوابات الأمان.
- كل إجراء يُسجَّل (audit) وتُفرض حدود المعدّل (rate limits).

## A3 — مُقيَّد (RESTRICTED) — يتطلب موافقة المؤسس/العميل دائماً

لا يُسمح به مطلقاً بدون موافقة صريحة:

- التسعير النهائي.
- الخصومات.
- الشروط القانونية.
- قبول العقد.
- التزامات الدفع.
- قرارات الاسترداد.
- التواصل البارد عبر واتساب (Cold WhatsApp).
- الإرسال الآلي عبر LinkedIn.
- البريد البارد عالي الحجم.

## كيف يُفرض المستوى تقنياً؟

أعلام البيئة (Environment flags) تتحكم بالحد الأقصى المسموح، وكلها افتراضياً
آمنة:

```
EXTERNAL_SEND_ENABLED=false
EMAIL_SEND_ENABLED=false
WHATSAPP_SEND_ENABLED=false
WHATSAPP_ALLOW_LIVE_SEND=false
SMS_SEND_ENABLED=false
CALENDAR_WRITE_ENABLED=false
PROPOSAL_FINALIZATION_ENABLED=false
OUTBOUND_MODE=draft_only
```

تتحقق الدوال في `app/commercial/safety.py` من هذه الأعلام + الموافقات +
حالة العميل قبل السماح بأي إجراء حي. أي إخفاق في أي شرط ⇒ إنشاء مسودة + سبب،
دون إرسال.
