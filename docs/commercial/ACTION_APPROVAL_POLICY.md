# Dealix Action Approval Policy

Dealix runs approval-first. The system may prepare internal artifacts, but external or high-impact actions require founder approval.

## Allowed automatically

- Read local/repo context.
- Score companies and opportunities.
- Draft messages and content.
- Generate reports.
- Write internal action queues.
- Write approval queues.
- Write proof logs.
- Prepare internal fix plans.

## Requires explicit approval

- Sending email.
- Sending WhatsApp.
- Posting to public social channels.
- Merging pull requests.
- Changing production settings.
- Charging, invoicing, or activating payment links.
- Sharing client-facing proof packs.
- Using a client name publicly.

## Blocked by default

- Cold WhatsApp campaigns.
- Mass LinkedIn automation.
- Scraping that violates terms.
- Guaranteed revenue claims.
- Government access claims.
- Fake proof or fabricated case studies.
- Hardcoded secrets.
- Public local-model endpoints without authentication and network controls.

## Safety principle

AI drafts and ranks. Deterministic workflows create internal artifacts. The founder approves external commitments.
