# مكتب الحجز والتقويم (Booking & Calendar Desk)

## الغرض

توليد خيارات اجتماع جاهزة للعميل دون لمس التقويم.

## ما يُنتجه المكتب لكل بطاقة

- **3 خيارات حجز** مع مواعيد مقترحة (بتوقيت `Asia/Riyadh`).
- **أجندة** مناسبة لنوع الحركة (مبيعات / شراكة / عرض).
- **قائمة حضور** مقترحة.
- **ملاحظات تحضير**: مراجعة درجة ICP، إحضار أمثلة صادقة فقط، تجهيز **نطاق
  سعري** وليس سعراً نهائياً.

## قاعدة الكتابة في التقويم

`calendar_write_enabled = false` افتراضياً. لا يكتب النظام حدثاً إلا إذا:

```
CALENDAR_WRITE_ENABLED=true
OUTBOUND_MODE=controlled_live
owner_decision=book
العميل غير مستبعَد (لم يُلغِ الاشتراك)
```

تتحقق `safety.can_write_calendar(...)` من هذه الشروط. أي إخفاق ⇒ يبقى الحجز
`proposed` فقط مع سبب.

## الحالات

`proposed → confirmed → booked → declined`

الانتقال إلى `booked` يستلزم قرار `book` صريحاً من المالك + اجتياز البوابة.
