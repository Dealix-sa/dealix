# كتيّب المسوّق — افهم كل شيء واشرحه لأي عميل — Marketer Playbook — Understand Everything & Explain It to Any Client

> هدف هذا الملف: أي مسوّق أو شريك يقرأه مرة، فيقدر يشرح Dealix، يؤهّل العميل، يعرض الحزمة الصحيحة، ويسلّم للمؤسس للإغلاق — بدون كسر الحوكمة.
>
> Goal: any marketer or partner reads this once and can explain Dealix, qualify the client, pitch the right package, and hand off to the founder to close — without breaking governance.

---

## 1) اشرح Dealix في ٤ طبقات / Explain Dealix in 4 layers

ابنِ الشرح من الأعلى للأسفل. لا تبدأ بالمخرجات؛ ابدأ بالفئة.
Build the explanation top-down. Don't start with deliverables; start with the category.

| الطبقة / Layer | الجملة / The line |
|---|---|
| **١. الفئة / Category** | «عمليات إيراد محوكمة لشركات B2B — مو أداة AI، مو شات بوت.» / "Governed revenue operations for B2B — not an AI tool, not a chatbot." |
| **٢. المحرك اليومي / Daily engine** | «يقرأ أرقامك كل يوم، يطلّع الفرص المتجاهَلة، يكتب الرسالة، تعتمدها أنت.» / "Reads your numbers daily, surfaces ignored opportunities, writes the message, you approve." |
| **٣. السُّلّم / The ladder** | «تبدأ مجاني، ثم سبرنت ٤٩٩، ثم تشغيل شهري — بعد ما تثبت القيمة.» / "Start free, then a 499 sprint, then monthly ops — after value is proven." |
| **٤. الإثبات / Proof** | «كل عمل مدفوع يطلّع Proof Pack: أدلة من بياناتك، مو وعود.» / "Every paid engagement produces a Proof Pack: evidence from your data, not promises." |

## 2) لمن نبيع / من نتجنّب / Who we sell to / who we avoid

**العميل المثالي / Ideal client:**
- وكالة تسويق عندها leads ولا تغلقها / a marketing agency with leads it doesn't close
- شركة تدريب عندها استفسارات لا تتحوّل لحجوزات / a training company with inquiries not converting
- شركة خدمات B2B عندها pipeline غير منظّم / a B2B services firm with a messy pipeline
- عيادة/متجر/شركة تشغيل عندها متابعات وفواتير متفرقة / a clinic/retailer/ops company with scattered follow-ups & invoices
- مؤسس يعيش في واتساب وإكسل / a founder living in WhatsApp + Excel

**نتجنّب (أو نعيد التأطير) / Avoid (or reframe):**
- من يطلب «أرسل ٥٠٠٠ رسالة باردة» / anyone asking to "blast 5,000 cold messages"
- من يطلب قوائم مشتراة أو scraping / anyone asking for bought lists or scraping
- من يريد «ضمان مبيعات» مكتوباً / anyone wanting a written "sales guarantee"

> هؤلاء ليسوا رفضاً نهائياً — أعد تأطيرهم للطريقة الآمنة (راجع §6 والاعتراضات). إذا أصرّوا على المخالفة، اعتذر بوضوح.
> These aren't a hard no — reframe them to the safe method (see §6 and objections). If they insist on the violation, decline cleanly.

## 3) التأهيل — ٨ أسئلة قبل أي عرض / Qualification — 8 questions before any pitch

اسأل (أو استنتج) هذه الثمانية. الإجابات تحدد القرار.
Ask (or infer) these eight. The answers drive the decision.

| # | السؤال / Question | نبحث عن / Looking for |
|---|---|---|
| 1 | هل الألم واضح؟ / Pain clear? | فرص تضيع فعلاً / real lost opportunities |
| 2 | هل صاحب القرار حاضر؟ / Owner present? | من يقدر يقول نعم / who can say yes |
| 3 | هل البيانات متاحة؟ / Data available? | leads/عروض/فواتير ولو فوضى / leads/quotes/invoices, even messy |
| 4 | هل يقبل الحوكمة؟ / Accepts governance? | يقبل الموافقة قبل الإرسال / accepts approval-before-send |
| 5 | هل عنده ميزانية؟ / Has budget? | ولو ٤٩٩ للبداية / even 499 to start |
| 6 | هل يريد طرقاً آمنة؟ / Wants safe methods? | **لا** يطلب scraping/بارد/ضمان / **not** asking for scraping/cold/guarantee |
| 7 | هل مسار الإثبات واضح؟ / Proof path visible? | نقدر نطلّع Proof Pack / we can produce a Proof Pack |
| 8 | هل مسار الريتينر واضح؟ / Retainer path visible? | احتمال تشغيل شهري لاحقاً / likely monthly ops later |

**القرار / The decision:**

| الحالة / Case | القرار / Decision |
|---|---|
| أغلب النعم + يريد طرقاً آمنة / mostly yes + wants safe methods | **ACCEPT** — اعرض السُّلّم من التشخيص / pitch the ladder from the diagnostic |
| ألم واضح لكن بلا ميزانية/قرار الآن / pain clear but no budget/decision yet | **DIAGNOSTIC_ONLY** — ابدأ بالتشخيص المجاني / start with the free diagnostic |
| يطلب مخالفة (بارد/scraping/ضمان) / asks for a violation | **REFRAME** — حوّله للطريقة الآمنة / convert to the safe method |
| يصرّ على المخالفة / insists on the violation | **REJECT** بأدب + بديل آمن / decline politely + safe alternative |
| خارج نطاقنا تماماً / fully out of scope | **REFER_OUT** — أحِله لجهة مناسبة / refer elsewhere |

| إشارة العميل / Client signal | الحزمة / Package |
|---|---|
| أول تواصل، يبي يشوف قيمة / first contact, wants to see value | [التشخيص المجاني / Free Diagnostic](./PACKAGE_1_FREE_DIAGNOSTIC.md) — 0 |
| ألم واضح + صاحب قرار + بيانات جاهزة / pain + owner + data ready | [سبرنت ٤٩٩ / 499 Sprint](./PACKAGE_2_REVENUE_PROOF_SPRINT.md) |
| بيانات كثيرة وفوضى، يحتاج تنظيف/دمج / lots of messy data, needs cleaning/dedup | [حزمة البيانات / Data Pack](./PACKAGE_3_DATA_TO_REVENUE_PACK.md) — 1,500 |
| شاف Proof Pack + يبي تشغيل مستمر / saw a Proof Pack + wants ongoing | [النمو الشهري / Growth Ops](./PACKAGE_4_GROWTH_OPS_MONTHLY.md) — 2,999/شهر |
| ضغط تذاكر دعم كبير / heavy support-ticket load | [دعم Support OS / Support OS](./PACKAGE_5_SUPPORT_OS_ADDON.md) — 1,500/شهر (إضافة) |
| صاحب قرار يبي لوحة/غرفة قيادة / decision-maker wants a dashboard/command center | [غرفة القيادة / Command Center](./PACKAGE_6_EXECUTIVE_COMMAND_CENTER.md) — 7,500/شهر |

**قاعدة السُّلّم:** لا تعرض حزمة أعلى قبل إثبات الحزمة تحتها. لا ريتينر قبل Proof Pack.
**Ladder rule:** Don't pitch a higher rung before proving the one below. No retainer before a Proof Pack.

## 5) سكربت الاكتشاف (≤ ٣٠ دقيقة، ٥ أقسام) / Discovery script (≤30 min, 5 sections)

1. **الوضع الحالي (٥ د) / Current state (5m):** «كيف تدخل الـ leads عندك اليوم، ووين تروح بعدها؟» / "How do leads come in today, and where do they go after?"
2. **الألم بالأرقام (٧ د) / Pain in numbers (7m):** «كم lead/استفسار بالشهر؟ كم يتحوّل؟ وين تحس إنها تضيع؟» / "How many leads/inquiries a month? How many convert? Where do they feel lost?"
3. **الحوكمة (٥ د) / Governance (5m):** «نشتغل بطريقة آمنة: لا بارد، لا scraping، كل رسالة بموافقتك. مريح؟» / "We work the safe way: no cold, no scraping, every message with your approval. Comfortable?"
4. **العرض المناسب (٨ د) / The fit (8m):** اربط إشارته بحزمة من §4، واشرح المخرجات والسعر / map their signal to a package from §4, explain deliverables + price.
5. **الخطوة التالية (٥ د) / Next step (5m):** «نبدأ بتشخيص مجاني ٢٤ ساعة. نعبّي النموذج الحين؟» / "Let's start with a free 24-hour diagnostic. Shall we fill the form now?"

> سجّل ملاحظاتك بعد المكالمة وسلّمها للمؤسس. **أنت لا ترسل عروضاً أو فواتير بنفسك** — المؤسس يعتمد ويرسل.
> Log your notes after the call and hand them to the founder. **You don't send proposals or invoices yourself** — the founder approves and sends.

## 6) العرض الحي (Demo) — ٤ لقطات / The live demo — 4 shots

1. **مخرج التشخيص / Diagnostic output:** أرِه شكل «٣ فرص + سبب + مسودة واحدة» / show "3 opportunities + reason + 1 draft."
2. **عيّنة Proof Pack / Proof Pack sample:** أرِه كيف تُوثَّق الفرص بالأدلة / show how opportunities are evidenced.
3. **صندوق الموافقات / Approval inbox:** أرِه «اعتمد/عدّل/تجاهل» — هو يتحكم / show approve/edit/skip — they're in control.
4. **السُّلّم والسعر / Ladder & price:** أرِه جدول الحزم السبع وحدّد رُتبته / show the 7-package table and mark their rung.

اقفل بـ: «الخطوة المجانية الأولى بلا مخاطرة. نبدأ؟» / Close with: "The first free step is risk-free. Shall we start?"

## 8) ممنوعات المسوّق — لا تتجاوزها أبداً / Marketer red lines — never cross

| لا تَعِد بـ / Never promise | قل بدلاً منها / Say instead |
|---|---|
| «نرسل آلاف الرسائل تلقائياً» / "we auto-send thousands" | «نجهّز مسودات، وأنت تعتمد» / "we prepare drafts, you approve" |
| «نجيب لك قوائم/أرقام جاهزة» (scraping) / "we get you ready lists" | «نشتغل على بياناتك أنت + مصادر موافَق عليها» / "we work on your data + approved sources" |
| «نضمن لك X صفقة» / "we guarantee X deals" | «نطلّع فرصاً مُثبتة بأدلة» / "we surface evidenced opportunities" |
| ذكر شهادات/أرقام لا نقدر نوثّقها / unverifiable certs/numbers | تكلّم عن المبدأ: PDPL-aligned، بياناتك تحت سيطرتك / speak to principle |
| إرسال عرض/فاتورة بنفسك / sending a proposal/invoice yourself | سلّم للمؤسس ليعتمد ويرسل / hand to the founder to approve & send |

السبب: هذه ليست مجرد قواعد تسويق — هي **محظورات محوكمة** تحمي الشركة والعميل قانونياً وسمعةً.
Why: these aren't just marketing rules — they are **governed non-negotiables** that protect the company and the client legally and reputationally.

## 9) التسليم للمؤسس / Hand-off to the founder

بعد مكالمة مؤهَّلة، سلّم: (١) ملخص الألم بالأرقام، (٢) الحزمة الموصى بها + السبب، (٣) قرار التأهيل (ACCEPT/…)، (٤) أي اعتراض مفتوح. المؤسس يعتمد العرض، يرسل الفاتورة، ويبدأ التسليم.

After a qualified call, hand over: (1) pain summary in numbers, (2) recommended package + why, (3) qualification decision (ACCEPT/…), (4) any open objection. The founder approves the proposal, sends the invoice, and starts delivery.
