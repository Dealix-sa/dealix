# Proof Pack Delivery Standard

A proof pack is an **honest** record of a real engagement. Never fabricate.

## Contents
- Problem diagnosed
- Target list
- Messages drafted
- Approvals
- Manual sends
- Replies
- Calls
- Invoice status
- Payment status
- Delivered work
- Next actions
- Screenshots / evidence references (if available)
- Before/after summary (real data only)
- Risk notes

## Rules
- Requires real `payment_received` and `work_delivered` events.
- If any evidence is missing, write **MISSING** — do not invent it.
- No fake results, no fake clients, no guaranteed claims.
