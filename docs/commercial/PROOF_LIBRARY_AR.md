# Proof Library — Evidence Levels L0–L5 — مكتبة الأدلة

**Dealix — نظام تشغيل الإيرادات للشركات السعودية / Saudi B2B Revenue Operating System.**

هذه المكتبة تحدّد **ما الدليل الموجود اليوم وما يجب كسبه**، وكيف يربط كل عرض بمستوى أدلة. **القاعدة الحاكمة: صفر عملاء مدفوعين اليوم هو صدق، لا ضعف.** نعوّض بأدلة قابلة للتحقق، لا بادعاءات. العروض: [./PRODUCT_CATALOG_AR.md](./PRODUCT_CATALOG_AR.md). الأسعار: [./DEALIX_REVOPS_PACKAGES_AR.md](./DEALIX_REVOPS_PACKAGES_AR.md).

This library defines **what proof exists today and what must be earned**, and how each offer maps to an evidence level. **Governing rule: zero paid customers today is honesty, not weakness.** We compensate with verifiable proof, never with claims.

## مستويات الأدلة / Evidence levels

| المستوى / Level | الوصف / Description | أمثلة / Examples | الحالة اليوم / Status today |
|---|---|---|---|
| **L0** | منهجية + مبادئ | البنود غير القابلة للمساومة، الحوكمة، approval-first | **موجود / exists** |
| **L1** | قوالب + أصول جاهزة | قوالب النطاق، قوائم التسليم، CSV تجريبي | **موجود / exists** |
| **L2** | تقرير/مخرَج عيّنة | [./samples/LEAD_INTELLIGENCE_SPRINT_SAMPLE_REPORT_AR.md](./samples/LEAD_INTELLIGENCE_SPRINT_SAMPLE_REPORT_AR.md) | **موجود / exists** |
| **L3** | Proof Pack تسليم لمشروع فعلي | سجل تسليم موثّق + ledger أسبوعي لعميل | **يُكتسَب / to be earned** |
| **L4** | نتيجة تشغيل تراكمية | أداء عبر دورات شهرية موثّقة | **يُكتسَب / to be earned** |
| **L5** | مرجع عميل قابل للنسب + دراسة حالة | شهادة/مرجع موثّق باسم (بإذن) | **يُكتسَب / to be earned** |

**خط الصدق / Honesty line:** اليوم نملك **L0–L2** فقط. L3–L5 **تُكتسَب بالتسليم**، ولا تُدّعى قبل وقوعها. أي دراسة حالة بلا عميل مُسمّى تُوسَم صراحة «نموذج افتراضي آمن / hypothetical, case-safe template».

## ما الدليل الموجود اليوم / Proof that exists today

- **منهجية محكومة (L0):** 11 بنداً غير قابلاً للمساومة، approval-first، هوية وكيل، Proof Pack لكل مشروع.
- **قوالب وأصول (L1):** قالب نطاق Sprint، قائمة تسليم، CSV تجريبي ببيانات وهمية، عرض صفحة واحدة [./OFFER_LEAD_INTELLIGENCE_SPRINT_AR.md](./OFFER_LEAD_INTELLIGENCE_SPRINT_AR.md).
- **مخرَج عيّنة (L2):** تقرير Sprint عيّنة يُظهر الشكل والجودة المتوقعة.

## ما يجب كسبه / Proof that must be earned

- **L3** — أول Proof Pack تسليم لعميل مدفوع (Sprint/Pilot) مع ledger موثّق.
- **L4** — أداء تراكمي عبر دورات Monthly RevOps.
- **L5** — مرجع عميل قابل للنسب ودراسة حالة باسم (بإذن صريح، بلا PII).

**قاعدة الترقية / Promotion rule:** لا يُرفع ادعاء من مستوى لأعلى إلا بوجود الدليل الفعلي. لا نقفز من L2 إلى لغة L5 في أي مادة بيع.

## عرض الـ Founding Partner / Founding partner framing

بما أننا في L2، نعرض على أوائل العملاء وضع **founding partner**: سعر ضمن النطاق المعتمد + قرب من المؤسس + أولوية تسليم، مقابل **إذن النسب لاحقاً** لبناء L3–L5. صراحة كاملة عن المرحلة — لا ادعاء عملاء غير موجودين. تُستخدم في الرد على اعتراض الثقة في [./OBJECTION_BANK_AR.md](./OBJECTION_BANK_AR.md).

## القواعد غير القابلة للمساومة / Non-negotiables

- لا ادعاء بلا مصدر. لا ادعاء عميل/مرجع غير موجود. لا PII في الأدلة.
- لا ضمان نتائج — **فرص مُثبتة بأدلة / evidenced opportunities**.
- لا مشروع بلا Proof Pack ولا أصل رأسمالي.
