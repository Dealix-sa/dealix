# نموذج تشغيل غرفة القيادة (Command Room Operating Model)

## الغرض

تجميع كل مخرجات التشغيل في لقطة واحدة (snapshot) + تقرير بشري، تعطي الإدارة
رؤية يومية وقائمة قرارات واضحة.

## المخرجات

- `reports/commercial/growth_os/latest.json` — اللقطة الكاملة + حزمة الإثبات +
  أحداث خط الأنابيب.
- `reports/commercial/growth_os/latest.md` — تقرير بشري.

## محتوى اللقطة

| القسم | المحتوى |
|-------|---------|
| `summary` | إحصائيات + الوضع الأمني |
| `accounts` / `cards` / `replies` | الحسابات والبطاقات والردود |
| `negotiation_drafts` | مسودات التفاوض |
| `booking_options` | خيارات الحجز |
| `proposal_briefs` | ملخصات العروض (بموافقة) |
| `followup_tasks` | مهام المتابعة D1/D3/D7 |
| `delivery_handoffs` | تسليمات التنفيذ |
| `risks` | المخاطر (مصدر مفقود، غير قابل للتواصل، بطاقة عالية المخاطر) |
| `decision_queue` | القرارات المطلوبة |
| `next_actions` | أهم 10 إجراءات |

## دورة التشغيل اليومية

1. `make commercial-growth-os` لتوليد اللقطة.
2. مراجعة `decision_queue`: اعتماد/تعليق/استبعاد البطاقات.
3. معالجة التصعيدات عالية المخاطر (طلبات العقود، إلغاء الاشتراك).
4. اعتماد ملخصات العروض (نطاق سعري فقط).
5. تنفيذ أهم 10 إجراءات.
6. `make commercial-growth-verify` للتأكد من بقاء الوضع آمناً.

## ضمان الأمان

يبدأ التقرير دائماً بـ "الوضع الأمني" ويؤكد في نهايته أنه لم تُرسَل أي رسالة
خارجية ولم يُكتَب أي حدث تقويم ولم يُحدَّد أي سعر نهائي.
