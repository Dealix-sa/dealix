# Open Design Integration

This directory explains how Dealix adapts useful concepts from `nexu-io/open-design` into a Dealix-native Design Command Room OS.

## Files

- `OPEN_DESIGN_TO_DEALIX_OS.md` — strategy and architecture
- `ATTRIBUTION.md` — attribution and license boundary note
- `VALIDATION.md` — validation and merge posture
- `NEXT_STEPS.md` — recommended follow-up implementation phases

## Rule

Do not vendor Open Design source, media, Electron runtime, or dependencies into Dealix without a separate technical and license review.
