# Daily Command Center — Dealix Launch — مركز القيادة اليومي

The founder's daily loop during launch. Short, repeatable, and safe. Nothing in this loop sends externally without the founder's manual approval.

## The loop — الحلقة

Run this once each morning and once each evening during launch week, then once daily after.

### Morning — الصباح

1. **Site check.** All public routes load; `/status` healthy. — تحقق الموقع.
2. **Queue check.** Open the founder review queue; triage new intake. — تحقق قائمة المراجعة.
3. **Draft review.** Review AI drafts ranked by priority; approve, edit, or reject. — مراجعة المسودات.
4. **Manual send.** Send only what you approved, by hand. — إرسال يدوي.
5. **Calls.** Confirm today's discovery calls and diagnostics. — المكالمات.

### Evening — المساء

1. **Outcome log.** Record what you sent and what came back. — سجل النتائج.
2. **Opportunity update.** Mark opportunities as evidenced, not guaranteed. — تحديث الفرص.
3. **Incident scan.** Any send anomaly, broken route, or claim issue? — مسح الحوادث.
4. **Tomorrow.** Set the top three actions for tomorrow. — الغد.

## Daily safety checks — فحوص السلامة اليومية

- [ ] No external message left the system automatically.
- [ ] No bulk or cold outreach occurred.
- [ ] No sensitive data processed before an agreement.
- [ ] No new public claim violates the forbidden-claims list.
- [ ] Any value figure shown carries the estimated-value disclosure.

## Daily metrics (review-only) — مؤشرات يومية (للمراجعة فقط)

| Metric | Source | Note |
|---|---|---|
| Routes healthy | Site check | Red blocks the day |
| New intake items | Review queue | Triaged same day |
| Drafts reviewed | Draft review | Approved / edited / rejected |
| Manual sends | Founder log | All approved by hand |
| Calls held | Calendar | Discovery + diagnostics |
| Opportunities (evidenced) | Outcome log | Never labeled guaranteed |

Metrics are for the founder's review only. No metric triggers an automated action.

## Escalation — التصعيد

If any safety check fails, stop the loop and open `06_FAILURE_RESPONSE_PLAYBOOK.md`.
