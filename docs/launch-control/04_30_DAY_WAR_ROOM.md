# 30-Day War Room — غرفة عمليات الـ30 يومًا

> A concrete four-week plan for the founder. All outreach is manual and happens only after the founder approves a specific draft. Nothing in this plan sends external messages automatically.
>
> خطة ملموسة لأربعة أسابيع للمؤسس. كل تواصل يدوي ويحدث فقط بعد موافقة المؤسس على مسودة محددة. لا شيء في هذه الخطة يرسل رسائل خارجية تلقائيًا.

---

## EN — The plan

### Week 1 — Foundation and first revenue
- Polish the public website (copy, mobile layout, Arabic + English parity, CTA, metadata). Run `scripts/site_launch_static_check.py`.
- Load the first 100 real leads into the CRM (no email/phone stored where the schema forbids it).
- Generate drafts: `python scripts/commercial_generate_400_drafts.py`; run `scripts/commercial_safety_audit.py`.
- Founder reviews the **top 50** in `outputs/commercial_launch/latest/top_50_priority.md`.
- Publish 5 founder posts (drafted by Media OS, posted by hand).
- Sell the first paid diagnostics (499 SAR Sprint).

### Week 2 — Learn and run the loop daily
- Improve outreach copy using real reply patterns (anonymized, no PII in notes).
- Run the 400-draft factory **daily**; review top 50 each morning.
- Book and run discovery calls across the 5 verticals.
- Deliver the first diagnostics; capture findings as case-safe patterns.
- Send the first pilot proposals (generated as templates, sent manually).

### Week 3 — Proof and conversion
- Build case-style proof assets, labeled "Hypothetical / case-safe template" unless a real customer consents to be named.
- Execute pilot engagements; track delivery against the Delivery OS.
- Open retainer conversations (2,999–4,999 SAR/mo Managed Ops).
- Produce content from real learnings (no confidential metrics).

### Week 4 — Scale and report
- Double down on the verticals showing the strongest evidenced opportunities.
- Update the offer ladder if pricing or scope needs adjusting.
- Build a simple referral loop (manual ask after a satisfied delivery).
- Prepare a monthly board report from `daily_metrics.json` and CRM stages.

### Offer ladder reference
Free Diagnostic → 499 SAR Sprint → 1,500 SAR Data Pack → 2,999–4,999 SAR/mo Managed Ops → 5K–25K SAR Custom AI.

## AR — الخطة

### الأسبوع 1 — الأساس وأول إيراد
- صقل الموقع العام (النص، تخطيط الجوال، تكافؤ العربية والإنجليزية، CTA، البيانات الوصفية). شغّل `scripts/site_launch_static_check.py`.
- تحميل أول 100 عميل حقيقي في CRM (دون تخزين بريد/هاتف حيث يمنعه المخطط).
- توليد المسودات: `python scripts/commercial_generate_400_drafts.py`؛ شغّل `scripts/commercial_safety_audit.py`.
- يراجع المؤسس **أعلى 50** في `outputs/commercial_launch/latest/top_50_priority.md`.
- نشر 5 منشورات للمؤسس (يكتبها نظام الإعلام، ينشرها المؤسس يدويًا).
- بيع أول تشخيص مدفوع (سبرنت 499 ريال).

### الأسبوع 2 — التعلّم وتشغيل الحلقة يوميًا
- تحسين نص التواصل باستخدام أنماط الردود الحقيقية (مجهّلة، دون PII في الملاحظات).
- تشغيل مصنع الـ400 مسودة **يوميًا**؛ مراجعة أعلى 50 كل صباح.
- حجز وإجراء مكالمات استكشاف عبر القطاعات الخمسة.
- تسليم أول تشخيصات؛ التقاط النتائج كأنماط آمنة للحالة.
- إرسال أول عروض تجريبية (تُولَّد كقوالب، تُرسل يدويًا).

### الأسبوع 3 — الإثبات والتحويل
- بناء أصول إثبات على شكل دراسة حالة، موسومة "افتراضي / قالب آمن للحالة" ما لم يوافق عميل حقيقي على ذكر اسمه.
- تنفيذ الارتباطات التجريبية؛ تتبع التسليم مقابل نظام التسليم.
- فتح محادثات العقود الشهرية (إدارة العمليات 2,999–4,999 ريال/شهر).
- إنتاج محتوى من التعلّم الحقيقي (دون مقاييس سرية).

### الأسبوع 4 — التوسّع والتقرير
- التركيز المضاعف على القطاعات ذات الفرص المُثبتة بأدلة الأقوى.
- تحديث سلّم العروض إن احتاج التسعير أو النطاق تعديلًا.
- بناء حلقة إحالة بسيطة (طلب يدوي بعد تسليم مُرضٍ).
- إعداد تقرير مجلس شهري من `daily_metrics.json` ومراحل CRM.

### مرجع سلّم العروض
تشخيص مجاني ← سبرنت 499 ريال ← حزمة بيانات 1,500 ريال ← إدارة عمليات 2,999–4,999 ريال/شهر ← ذكاء اصطناعي مخصص 5–25 ألف ريال.
