# 02 — Go / No-Go Matrix

القرار واضح: ما المسموح إطلاقه الآن، وما الممنوع حتى تُستوفى الشروط.
A clear matrix of what is authorized now vs. what is blocked by doctrine.

## ✅ GO — authorized now
| Item | Why it is safe |
|---|---|
| Website public launch | Static, no sensitive-data forms, no exaggerated claims |
| Commercial docs / positioning | Internal artifacts, founder-owned |
| 400+ draft generation | Review-only; `send_allowed=false` for every draft |
| Founder review | Human-in-the-loop |
| Manual outreach **after approval** | Sent by a human, individually |
| Paid diagnostics | Value exchange, agreement-based |
| Discovery calls | Inbound/booked, consent-based |
| Proposal generation | Internal artifact |
| Content calendar | Plan only, no auto-post |
| Manual social posting | Human posts each item |

## ⛔ NO-GO — blocked until conditions are met
| Item | Condition to lift |
|---|---|
| Automated email sending | SPF/DKIM/DMARC + Postmaster + reputation monitoring + explicit founder decision |
| Cold messaging automation | Never (platform terms + doctrine) |
| Professional-network automation | Never (platform terms prohibit bots/automation) |
| Website form auto-submit | Never |
| Bulk sending | Never (manual, individual outreach only) |
| Paid ads live launch | Landing-page tracking + UTM + conversion event + privacy review + budget/audience/claim review + negative keywords + approval owner |
| Collecting sensitive data before agreement | Signed agreement + PDPL basis |
| Sending from GitHub Actions | Never (artifact-only workflows) |

## القاعدة الذهبية / Golden rule
> AI explores, analyzes, and recommends. Deterministic workflows execute artifacts.
> **Humans approve and perform every external commitment.**
