# Founder Execution Checklist — Dealix Launch — قائمة تنفيذ المؤسس

The founder's personal, end-to-end checklist for taking Dealix live. Work it top to bottom. Do not skip the safety gates.

## Before launch day — قبل يوم الإطلاق

- [ ] Read every public page in AR and EN (`docs/site-launch/03_COPY_DECK_AR_EN.md`).
- [ ] Complete the manual QA checklist (`docs/site-launch/04_MANUAL_QA_CHECKLIST.md`).
- [ ] Score readiness (`01_LAUNCH_SCORECARD.md`); all P0 Green.
- [ ] Confirm the GO list conditions are met (`02_GO_NO_GO_MATRIX.md`).
- [ ] Confirm every NO-GO item is off.
- [ ] Generate and read the evidence pack (`03_EVIDENCE_PACK.md`).
- [ ] Confirm the API is read-only (`docs/ops/API_COMMERCIAL_LAUNCH_QA.md`).
- [ ] Confirm the 400 drafts are review-only and none auto-sent.

## Safety gates — بوابات السلامة (must all pass)

- [ ] No automated email sending is enabled.
- [ ] No WhatsApp cold outreach is configured.
- [ ] No LinkedIn automation is configured.
- [ ] Website form creates a review item only; no auto-submit.
- [ ] No bulk sending path exists.
- [ ] No paid ads live without tracking and compliance.
- [ ] No sensitive data processed before an agreement.
- [ ] No external sending from GitHub Actions.

If any gate fails: **No-Go.** Open `06_FAILURE_RESPONSE_PLAYBOOK.md`.

## Launch day — يوم الإطلاق

- [ ] Publish the website.
- [ ] Verify all routes load in AR and EN.
- [ ] Confirm `/status`, `robots.txt`, `sitemap.xml` reachable.
- [ ] Post the planned media/social items manually.
- [ ] Open the founder review queue.
- [ ] Record the GO decision with date and owner.

## First week — الأسبوع الأول

- [ ] Run the daily command center loop (`05_DAILY_COMMAND_CENTER.md`).
- [ ] Triage all intake same day.
- [ ] Review and approve drafts; send only by hand.
- [ ] Hold discovery calls; deliver diagnostics per SOP.
- [ ] Send proposals manually after review.

## Ongoing discipline — انضباط مستمر

- [ ] Treat every opportunity as evidenced, never guaranteed.
- [ ] Keep the estimated-value disclosure on every customer-facing figure.
- [ ] Never describe scraping, cold WhatsApp, LinkedIn automation, or bulk outreach as a service.
- [ ] Never imply the system sends external messages on a customer's behalf.
- [ ] Re-score weekly; refresh evidence when the build changes.

## Arabic summary — ملخص عربي

قائمة المؤسس تأخذك من قبل الإطلاق إلى ما بعده: اقرأ كل صفحة بالعربية والإنجليزية، أكمل الفحص اليدوي، قيّم الجاهزية، تحقق من قائمتي الانطلاق والحظر، اقرأ حزمة الأدلة، وأكد أن واجهة البرمجة للقراءة فقط. بوابات السلامة الثماني يجب أن تنجح كلها وإلا فلا انطلاق. في الأسبوع الأول شغّل الحلقة اليومية وأرسل يدويًا فقط.
