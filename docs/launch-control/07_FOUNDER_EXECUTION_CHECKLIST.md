# 07 — Founder Execution Checklist

قائمة المؤسس اليومية والأسبوعية. بسيطة، قابلة للتنفيذ، ومحكومة بالدكترين.
The founder's daily and weekly checklist — simple, actionable, doctrine-bound.

## Daily
- [ ] 08:00 — Generate 400+ drafts.
- [ ] 08:15 — Safety audit PASS (else stop).
- [ ] 08:30 — Review top 50; approve/edit/reject.
- [ ] 10:00 — Send approved outreach **manually**.
- [ ] 13:00 — Discovery calls / follow-ups.
- [ ] 16:00 — Publish 1 post **manually**.
- [ ] 18:00 — Update CRM stages; honor suppression.
- [ ] 20:00 — Log metrics; set tomorrow's top 3.

## Weekly
- [ ] Run full control tower (`final_launch_control_verify.py` PASS).
- [ ] Refresh lead list with real, consent-based accounts.
- [ ] Publish 5 founder posts; 1 long-form.
- [ ] Deliver diagnostics; send pilot proposals.
- [ ] Update offer ladder from learnings.
- [ ] Review friction log; close one improvement.
- [ ] Confirm NO-GO items remain blocked (no auto-send, no ads-live, no sensitive data pre-agreement).

## Never (non-negotiable)
- [ ] Never automate email/messaging/professional-network outreach.
- [ ] Never auto-submit website forms or bulk-send.
- [ ] Never launch paid ads before the ads readiness gate.
- [ ] Never collect sensitive data before a signed agreement.
- [ ] Never send anything from GitHub Actions.
