# Content Production System
## Purpose
Create a repeatable system for turning Dealix learning into safe, useful content.
## Weekly Content Inputs
- weekly intelligence review
- win/loss review
- delivery feedback
- customer intelligence
- sector performance
- founder decisions
- productization candidates
## Content Output Types
- founder post
- short insight
- sector observation
- operating system note
- anonymized lesson
- case study
- proof-backed thread
- landing page section
- one-pager
## Weekly Content Cadence
Minimum:
- 2 founder posts
- 1 operating lesson
- 1 proof or anonymized insight if available
## Content Workflow
1. Choose evidence.
2. Select proof level.
3. Draft content.
4. Review claims.
5. Redact sensitive data.
6. Approve if needed.
7. Publish.
8. Log result.
## Rule
No evidence, no strong claim.
