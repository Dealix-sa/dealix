# Case Study System — نظام دراسات الحالة

## Purpose
A consistent pipeline that turns delivered engagements into case studies — anonymized by default, named only with written A2 consent. Case studies are evidence; they are not marketing fiction.

## Pipeline

| Stage | Owner | Output |
|---|---|---|
| 1. Trigger | Founder | Engagement closed; proof pack accepted; health score ≥80 |
| 2. Anonymized draft | Founder or analyst | First draft using case-safe labels; numbers from delivery log |
| 3. Internal review | Founder | Voice + claims check |
| 4. Consent ask (named version) | Founder | A2 written request to client per `docs/governance/APPROVAL_MATRIX.md` |
| 5. Client review (if named) | Client | Client edits; final approval |
| 6. Publish (anonymized or named) | Founder | LinkedIn + repo storage |
| 7. Archive | Founder | Stored in `docs/case-studies/` |

## Consent rules
- Named case studies require written A2 consent. Email or signed message acceptable.
- Consent scope is explicit: which channels (LinkedIn, repo, deck), which version (anonymized only / named), and time-bound (e.g., 12 months, renewable).
- Renewal of consent is requested at the time-bound expiry; published item is anonymized or pulled if consent lapses.
- Anonymized publication does not require consent but does require: no identifying detail combination that re-identifies the client.

## Structure (every case study)
- Headline: outcome in one line; estimated, not Verified.
- Client label: sector, size band, location band, role.
- Problem: one paragraph.
- Approach: numbered steps; named artefacts.
- Outcome: numbers from delivery log; estimated impact labeled.
- Lessons: 2-3 specific points.
- Disclosure: "Estimated value is not Verified value / القيمة التقديرية ليست قيمة مُتحقَّقة".
- Case label (when anonymized): "Hypothetical / case-safe template" or "Anonymized real engagement".

## Rules
- No fabricated outcomes. No "composite client" presented as one client.
- No PII: no names, faces, phone numbers, emails, national IDs, real company names absent consent.
- No claim of guaranteed sales. Use "evidenced opportunities".
- No claim that exceeds the proof pack's measured scope.

## Operations
- Each closed engagement enters the pipeline within 14 days of close.
- Quarterly: founder reviews backlog and publishes one batch.

## Evidence
- Each case study in repo links to its proof pack, consent record (if any), and publish links.

## Owner & cadence
- Owner: Founder.
- Cadence: per engagement close + quarterly batch publish.

## القسم العربي

**خط الإنتاج:** محفّز (إغلاق العقد + قبول حزمة إثبات + نقاط ≥80) → مسودة مجهولة الاسم → مراجعة داخلية → طلب موافقة A2 لنسخة مُسمّاة → مراجعة العميل → نشر → أرشفة.

**قواعد الموافقة:** الاسم يحتاج A2 مكتوبة. نطاق الموافقة صريح (قنوات، نسخة، مدة). تجديد الموافقة عند الانتهاء. المجهول لا يحتاج موافقة ولكن لا يجوز جمع تفاصيل تكشف هوية.

**الهيكل:** عنوان (نتيجة تقديرية)، تصنيف العميل (قطاع، حجم، موقع، دور)، المشكلة، الخطوات، النتيجة (أرقام مع تسمية التقديري)، 2-3 دروس، إفصاح، وسم "Hypothetical / case-safe template" أو "Anonymized real engagement".

**القواعد:** لا نتائج مفبركة، لا "عميل مركّب" يُقدَّم كعميل واحد، لا PII، لا ضمان مبيعات (استخدم "فرص مُثبتة بأدلة")، لا ادعاء يتجاوز حزمة الإثبات.

**المالك:** المؤسس.
