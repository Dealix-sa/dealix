## المبدأ — The principle

المحتوى لا يُخترع، بل يُستخرج. مصادره: عمل سبرنت مُسلَّم، إشارة توظيف مرصودة، اعتراض مبيعات متكرر، قرار حوكمة، حزمة إثبات. الدور هنا تحويل العمل إلى قطعة قابلة للنشر، لا اصطناع قصة. لا أرقام مُختلَقة، ولا عملاء وهميون، ولا وعود.

هذا المحرّك يُغذّي القنوات الموجودة، ولا يستبدلها. التقويم الشهري في [LINKEDIN_CONTENT_CALENDAR_MAY2026.md](../marketing/LINKEDIN_CONTENT_CALENDAR_MAY2026.md) وخطة الإيقاع في [LINKEDIN_CADENCE_PLAN.md](./LINKEDIN_CADENCE_PLAN.md) هما المخرج النهائي؛ هذا المستند هو خط الإنتاج الذي يملؤهما.

Content is not invented, it is extracted — from delivered sprint work, an observed job signal, a recurring objection, a governance decision, a Proof Pack. This engine feeds the existing channels; it does not replace them.

## العناصر الستة لكل قطعة — The six elements per piece

كل منشور، مهما قصُر، يحمل هذه السلسلة صراحةً أو ضمناً:

| العنصر / Element | السؤال الذي يجيب عنه / The question it answers |
|---|---|
| القطاع / Sector | لمن هذا؟ (تأمين، خدمات مهنية، SaaS، لوجستيات…) |
| الألم / Pain | ما المشكلة المتكرّرة؟ |
| الـworkflow / Workflow | كيف يُعالَج عملياً داخل Dealix؟ |
| الإثبات / Proof | ما الدليل؟ (نمط آمن، درجة جودة، سجل قرار) |
| العرض / Offer | أي عرض من السلّم يناسب؟ |
| الـCTA / CTA | ما الخطوة التالية المحدّدة؟ |

إن غاب أي عنصر، فالقطعة غير جاهزة. غياب «الإثبات» يحوّل المنشور إلى ادعاء؛ غياب «الخطوة التالية» يحوّله إلى تأمل بلا فائدة تجارية.

Every piece, however short, carries this chain. A missing Proof element turns a post into a claim; a missing next step turns it into a reflection with no commercial use.

## الإنتاج اليومي — Daily production

أربع قطع قصيرة يومياً، كل واحدة من زاوية مختلفة:

1. **رؤية المؤسس / Founder insight** — ملاحظة من واقع التشغيل اليوم. القطاع + الألم.
2. **منشور ألم قطاعي / Sector pain post** — مشكلة متكرّرة في قطاع محدّد، بلا اسم عميل. القطاع + الألم + العرض.
3. **منشور إثبات/تعلُّم / Proof or learning post** — درجة جودة، قرار حوكمة، أو خطأ صُحّح. الإثبات + الـworkflow.
4. **منشور قصير بأسلوب حالة / Short case-style post** — نمط آمن مُجهَّل، مُعلَّم «Hypothetical / case-safe template». الإثبات + العرض + الـCTA.

القطع اليومية مسودات. لا تُنشر كلها كل يوم؛ تُختار منها بحسب التقويم. لا نشر آلي — الاختيار والاعتماد بشريان.

Four short drafts daily, each from a different angle. Daily pieces are drafts; not all publish every day, and selection plus approval are human. No automated publishing.

## الإنتاج الأسبوعي — Weekly production

خمس قطع أطول أسبوعياً:

1. **منشور LinkedIn طويل / Long LinkedIn post** — موضوع واحد بعمق، ثنائي اللغة. يتبع [LINKEDIN_SYSTEM_AR.md](./LINKEDIN_SYSTEM_AR.md).
2. **سكربت كاروسيل / Carousel script** — 6–8 شرائح، كل شريحة جملة واحدة، تشرح workflow أو قراراً.
3. **مسودة دراسة حالة / Case study draft** — وفق [CASE_STUDY_SYSTEM_AR.md](./CASE_STUDY_SYSTEM_AR.md)، مُعلَّمة قالباً آمناً حتى توقيع العميل على إذن نشر.
4. **شرح منتج / Product explainer** — كيف يعمل عرض واحد (مثلاً Support Draft OS) خطوة بخطوة.
5. **درس مؤسس / Founder lesson** — قرار اتُّخذ، رفضٌ واعٍ، أو خطأ ومعالجته.

Five longer pieces weekly: a long LinkedIn post, a carousel script, a case study draft, a product explainer, and a founder lesson.

## حلقة التحويل — The conversion loop

تتبع حلقة [MARKETING_FACTORY.md](../marketing/MARKETING_FACTORY.md) وتمدّها بمصدرين جديدين:

اعتراض مبيعات → منشور LinkedIn → كاروسيل → قسم نشرة → سؤال شائع → بريد مبيعات → أصل للشريك.

**المصدر الإضافي الأول:** إشارة توظيف من [docs/signals/JOB_SIGNAL_PLAYBOOK_AR.md](../signals/JOB_SIGNAL_PLAYBOOK_AR.md). النمط — لا الشركة — يصبح منشور ألم قطاعي.

**المصدر الإضافي الثاني:** حزمة إثبات مُسلَّمة. تتحوّل عبر [PROOF_CONTENT_SYSTEM_AR.md](./PROOF_CONTENT_SYSTEM_AR.md) إلى منشور إثبات، مع التزام صارم بمستويات الأدلة وما هو قابل للنشر.

The loop extends MARKETING_FACTORY with two new inputs: a job-signal pattern (the pattern, never the company) and a delivered Proof Pack routed through the proof-content system.

## بوابات الجودة — Quality gates

قبل أي نشر، تمرّ المسودة على هذه البوابات:

- **لا توكن محظور / Forbidden tokens:** لا «نضمن»، لا «guaranteed»، لا «10x»، لا أسماء نماذج، لا إيموجي، لا ROI كحقيقة.
- **اكتمال العناصر الستة / Six-element completeness:** القطاع، الألم، الـworkflow، الإثبات، العرض، الـCTA.
- **سلامة الإثبات / Proof safety:** أي رقم نمطٌ آمن أو درجة محسوبة، لا ادعاء سوق.
- **خصوصية / Privacy:** لا اسم عميل، لا بيانات شخصية، لا مقياس سرّي.
- **اعتماد المؤسس / Founder sign-off:** لا قطعة تُنشر دون مراجعة المؤسس مقابل اللاءات.

Before publishing: no forbidden tokens, all six elements present, proof safety, privacy, and founder sign-off.

## القياس — Measurement

نقيس الردود التي تتحوّل إلى تشخيص مجاني، لا عدد المتابعين. كل تحوُّل يُسجَّل ويُربط بمصدره (إشارة، إثبات، اعتراض). KPI المرجعي يبقى كما في [MARKETING_FACTORY.md](../marketing/MARKETING_FACTORY.md): إيقاع نشر ثابت، نشرة أسبوعية، طلبات حزمة إثبات، اجتماعات مؤهَّلة من المحتوى.

الخطوة التالية الافتراضية لأي قطعة: ربطها بعرض من السلّم — التشخيص المجاني (0) ثم سبرنت ذكاء الإيرادات (499) — لا قفز إلى عقد شهري.

We measure replies that convert to a Free Diagnostic, not follower counts. Every conversion is logged and tied to its source. The default next step for any piece is a ladder offer — Free Diagnostic then 499 Sprint — no leap to a retainer.
