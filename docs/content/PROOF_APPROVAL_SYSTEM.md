# Proof Approval System

## Purpose
Create trust-safe proof assets from client delivery.

## Proof Levels
- P0 = internal only.
- P1 = anonymized public insight.
- P2 = approved named proof.
- P3 = full case study.

## Required Before Public Proof
- client approval
- claim evidence
- no sensitive data
- no inflated results
- no guaranteed revenue language

## Proof Assets
- anonymized insight
- approved quote
- case study
- sector report
- founder lesson

## Rules
- No client name without approval.
- No metrics without evidence.
- No private data.
- No public proof from weak delivery.

## Approval Flow
1. Draft proof internally at level P0.
2. Anonymize for P1 review.
3. Send wording for client approval (P2/P3) using
   clients/<client_name_private>/proof_approval.md.
4. Record decision (approved / rewrite / no).
5. Only publish once decision = approved.

## Anti-Patterns
- "Working with major Saudi brand" implying a client we cannot name.
- Quoting numbers ("3x more meetings") with no traceable source.
- Reusing P0 internal language in public posts.
