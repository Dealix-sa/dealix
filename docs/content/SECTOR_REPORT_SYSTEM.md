# Sector Report System

> One sector report per quarter. Evidence-backed. Aggregated. Trust-approved.

## Purpose
- Establish founder authority in Tier 1 sectors
- Generate inbound from named sectors
- Aggregate learnings into a public-safe artifact
- Build the moat (sector knowledge compounding)

## Cadence
- One report per quarter
- Topic = the Tier 1 sector with the most accumulated data

## Format
- 8–12 pages PDF
- Arabic + English versions
- Sections:
  1. Executive summary (1 page)
  2. Sector snapshot (size, segments, key dynamics, sources cited)
  3. Sales motion patterns observed (anonymized aggregate)
  4. Sample artifacts (sanitized, with SAMPLE labels)
  5. Common pitfalls + how to avoid
  6. Tooling landscape (without endorsements)
  7. References + methodology + limitations
  8. How Dealix engages with this sector (offer one-pager)

## Sourcing Rules
- All quantitative claims cited
- All client-specific data anonymized to ranges (n ≥ 3)
- All examples either synthetic with clear labeling OR publicly cited
- Limitations section is mandatory (small sample, not statistical)

## Production Workflow
1. Pick sector (most accumulated sprint data)
2. Outline → founder + advisor review
3. Draft → claim_guard pass + sanitization check
4. Internal review (founder)
5. External review (advisor — 1 week)
6. Final claim_guard pass
7. Publish (PDF + LinkedIn distribution)

## Approval
- A3 always
- Advisor required for first report in each sector
- Trust signoff required before publish

## Distribution
- Published as PDF on landing page
- Announced via LinkedIn (one post + one comment thread)
- Sent to active customers and key prospects as a soft touch
- Available for download in exchange for: nothing (no email gate this quarter — friction kills trust on first contact)

## Measurement
- Downloads (if instrumented)
- Inbound leads citing the report
- Comments + reshares
- Direct outreach citing the report

## Kill Switch
- If a report draws zero inbound or zero substantive engagement → don't do another in that sector for 2 quarters
- If a report draws negative feedback on accuracy → retract, fix, republish with errata

## Anti-Patterns
- Reports that just summarize publicly-available articles
- Reports with single-client data (privacy + sample size)
- Reports with conclusions that don't follow from the evidence
- Reports timed to news cycles (chasing virality)
- Reports gated by lead-capture forms (this quarter we don't gate)
- "Vendor benchmarks" (we're not Gartner)

## Limitations Section (mandatory wording)

Every report includes a Limitations section:
> This report is based on Dealix's working observations from a limited
> set of engagements (n=X) in the {sector} sector during {period}.
> It is not statistically representative of the sector at large.
> Numbers cited from third-party sources are linked in the References section.
> This report is for operator-level insight, not investment advice or
> regulatory guidance.

## Archival
- All reports in `content/sector_reports/{year}-Q{n}-{sector}.pdf` (public-safe after Trust signoff)
- Source data + working notes in private repo `content/case_study_notes/`
