# LinkedIn System — نظام LinkedIn

## Purpose
LinkedIn is Dealix's primary channel for reaching KSA decision-makers. Cadence is consistent. Post types are limited and disciplined. Approval is required before publish.

## Cadence
- One long-form founder note per week (Sunday 8pm KSA default).
- 2-3 lightweight posts per week (short pattern observations or quick takes).
- One case-safe summary per closed engagement (when consent exists).
- One sector report announcement per quarter.

## Post types

| Type | Length | Frequency | Voice |
|---|---|---|---|
| Founder note | 800-1500 chars | weekly | First-person; one lesson; evidence cited |
| Pattern post | 300-600 chars | 2-3/week | One observation; one data point; no calls to action besides "ask if useful" |
| Case-safe summary | 1000-1500 chars | per closure | Anonymized unless A2 consent; AR + EN parallel |
| Sector report announcement | 400-700 chars | quarterly | Methodology one-liner; link to the report |
| Repost with analysis | 300-500 chars | rare | Only when Dealix adds substantive analysis |

## Approval flow
1. Drafter (founder or analyst) writes the post.
2. Founder reads aloud, checks against [`FOUNDER_VOICE.md`](FOUNDER_VOICE.md) and the Content Rule.
3. Any claim is verified against [`PROOF_LIBRARY.md`](PROOF_LIBRARY.md).
4. Client name? Confirm A2 consent.
5. Schedule or publish.

## Rules
- No automation for posting on LinkedIn beyond a basic scheduler. No comment automation. No connection-request automation. No DM automation.
- No engagement pods.
- No "agree?" or "comment YES" engagement bait.
- No reposted screenshots of private chats.
- No PII in screenshots. Names, faces, phone numbers, emails redacted.

## Operations
- Backlog maintained in `docs/content/calendar/YYYY-MM.md`.
- Each scheduled post links to its proof row and consent record if applicable.

## Evidence
- Published post links archived per quarter in `docs/content/calendar/YYYY-Qn_published.md`.

## القسم العربي

**الإيقاع:** ملاحظة مؤسس مطوّلة أسبوعيًا (الأحد 8م افتراضيًا)، 2-3 منشورات خفيفة، case-safe عند إغلاق العقد بموافقة، إعلان تقرير قطاعي ربعي.

**القواعد:** لا أتمتة تتجاوز الجدولة. لا engagement pods. لا "هل توافق؟". لا لقطات محادثات خاصة. لا PII في الصور.
