# LinkedIn System

> The operating model for founder LinkedIn presence.
> One post per week. Considered comments. No volume-game.

## Posting Schedule

- **Day:** Thursday morning (Saudi time)
- **Frequency:** 1 main post per week
- **Format rotation:** text / carousel / single-image — alternate to learn what lands

## Post Structure

A working template (not mandatory, but a default that works):

```
Hook (1–2 lines, specific not generic)

Setup (3–5 lines, context)

The insight or evidence (5–8 lines, the meat)

What we learned / what we refused (2–4 lines)

CTA — optional, ≤ 1 line ("DM me / book Free Diagnostic")
```

Length: 200–600 words. Single emoji max (if any). Arabic primary; English version comments-thread.

## Topic Buckets (for the week)

Rotate through:
1. **Process** — show how the work happens (transparency)
2. **Trust** — show the approval matrix, claim_guard catches, etc.
3. **Sector learning** — what we learned from this sector this month
4. **Founder OS** — Daily Brief / Weekly Review meta-content
5. **Refusal** — what we said no to and why

## Comment Strategy

- 3 considered comments per week on Tier 1 ICP buyer posts
- Each comment must add a specific operator perspective, not "great post"
- Never pitch in comments
- Engage with replies to your own posts within 24 hr

## Pre-Publish Checklist

- [ ] One specific claim, not five generic ones
- [ ] All numbers cited or labeled (or removed)
- [ ] claim_guard.py pass
- [ ] No banned language
- [ ] Public-safety: no client data, no buyer name without consent
- [ ] Arabic version (or written in Arabic primary)
- [ ] Founder review

## Measurement

Per post, track in `learning/message_performance.csv` (private):
- post_id
- date
- topic_bucket
- format
- impressions (LinkedIn analytics)
- engagement rate
- inbound DMs received (and which → became leads)
- founder hours to produce

## Kill Switch

If content shows zero inbound after 6 weeks of consistent posting → reformat (video, carousel) and try 6 more weeks. If still zero → kill and re-evaluate channel mix.

## Anti-Patterns

- Posting before you have a real insight to share
- Posting because it's Thursday and you must
- Posting about "AI is changing everything"
- Posting to compete with peers' personal brand
- Engagement bait ("agree? 👇")
- Recycling old posts every quarter
- Long threads of self-replies

## Comment Etiquette

- Disagree civilly
- Don't argue with bots
- Don't engage when you'd be unhappy with the words being permanent
- Don't tag random people for visibility
- Don't comment outside ICP-relevant zones

## Sensitive Subjects

- Politics: never
- Religion: never (Saudi context: extra caution)
- Specific competitor names: never
- Industry conflicts / drama: never
- Internal team disputes: never

## Recovery From A Bad Post

If a post draws negative response or violates trust:
- Delete + edit + repost (with explanation if material)
- Log in `trust/data_incidents.md` if material
- Capture learning
- Update content guidance
