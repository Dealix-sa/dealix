# Proof Level Policy

## Purpose
Map every external claim to a proof level and what's required to use it.

## Levels (ascending)
1. **Opinion** — labeled as opinion; no proof required beyond the founder's view.
2. **Method** — describes our process generically; no client attribution.
3. **Anonymized case** — sector + role + outcome; no name.
4. **Named case** — client name with explicit written approval.

## Per-level requirements
| Level | Citation required | Approval needed |
|---|---|---|
| Opinion | No | None |
| Method | Yes (link to method doc) | None |
| Anonymized case | Yes (private artifact path) | Founder review |
| Named case | Yes + client written approval | Founder + Trust workflow |

## Using numbers
- A specific number requires a specific source.
- Round numbers ("a few" / "several") may appear at Opinion or Method level only when an exact count cannot be cited.

## Examples
- ✅ Method: "We score every lead 0–100 across five dimensions; the rubric is here."
- ✅ Anonymized: "A KSA insurance brokerage we worked with cut their qualification time from 3 days to 1."
- ✅ Named (with approval): "<Client Name> reduced qualification time from 3 days to 1 after a 7-day Dealix sprint."
- ❌ Forbidden: "Our customers see 3x improvement" with no source.

## Operating rule
If a claim cannot be elevated to the required level, downgrade or remove it.
