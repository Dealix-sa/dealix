# Proof to Demand Machine / ماكينة الإثبات إلى الطلب

## Purpose / الغرض

Turn delivery learning and approved proof into inbound demand.

تحويل التعلم من التسليم والإثبات المعتمد إلى طلب وارد.

## Proof Types / أنواع الإثبات

- anonymized insight / رؤية مجهولة الهوية
- approved quote / اقتباس معتمد
- case study / دراسة حالة
- sector report / تقرير قطاعي
- before/after process / عملية قبل/بعد
- founder lesson / درس المؤسس

## Content Outputs / مخرجات المحتوى

- LinkedIn post / منشور LinkedIn
- website proof block / كتلة إثبات على الموقع
- sector insight / رؤية قطاعية
- email nurture asset / أصل بريد إلكتروني للرعاية
- proposal proof slide / شريحة إثبات في العرض
- partner enablement note / مذكرة تمكين الشركاء

## Rules / القواعد

- no client name without approval / لا اسم عميل بدون موافقة
- no metric without evidence / لا مقياس بدون دليل
- no guaranteed results / لا نتائج مضمونة
- redact sensitive data / تنقية البيانات الحساسة

## Rule / القاعدة

Proof should create trust, not hype.

الإثبات يجب أن يخلق الثقة، لا الضجيج.
