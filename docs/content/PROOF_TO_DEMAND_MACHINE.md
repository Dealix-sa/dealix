# Proof to Demand Machine

## Purpose
Turn delivery learning and approved proof into inbound demand.

## Proof Types
- anonymized insight
- approved quote
- case study
- sector report
- before/after process
- founder lesson

## Content Outputs
- LinkedIn post
- website proof block
- sector insight
- email nurture asset
- proposal proof slide
- partner enablement note

## Rules
- no client name without approval
- no metric without evidence
- no guaranteed results
- redact sensitive data

## Rule
Proof should create trust, not hype.
