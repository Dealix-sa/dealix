# نموذج الأمان والخصوصية والثقة

## الأمان

- Secrets في env vars فقط.
- HTTPS/TLS لكل الاتصالات.
- PostgreSQL محمي بكلمات مرور قوية.
- Redis محمي بكلمة مرور في الإنتاج.
- النسخ الاحتياطي مشفر.

## الخصوصية

- نلتزم بـ PDPL.
- تقليل البيانات: نجمع فقط ما نحتاج.
- لا بيانات شخصية حساسة بدون موافقة.
- اتفاقية معالجة بيانات (DPA) مع كل عميل.

## الثقة

- مخرجات AI مُعلَّمة كمسودات.
- مراجعة بشرية قبل التسليم.
- Audit logs متاحة.
- لا وعود بنتائج مضمونة.

## الأدوار

- **Founder/CEO:** مسؤول النهائي عن الأمان والامتثال.
- **CTO:** تنفيذ الضوابط التقنية.
- **Compliance Lead:** مراجعة السياسات والسجلات.

## الحوادث

- أي تسريب يُبلّغ فورًا.
- خطوات الاستجابة في `docs/SECURITY_RUNBOOK.md`.
