# Dealix CEO Operating System

The CEO OS is the founder's weekly + daily operating contract: what to
look at, in what order, and what to do about each item.

## Daily brief (5 minutes)

Generated by `make ceo-daily-brief` (`scripts/generate_ceo_daily_brief.py`).
Output: `data/ceo_briefs/<YYYY-MM-DD>.md`.

Contents:
1. Yesterday's pipeline delta (leads in, leads qualified, leads stalled).
2. Today's approvals queue (count + top 3 high-risk items).
3. Today's revenue checkpoint (vs day-target, MTD cumulative).
4. Friction events last 24h (top 3 by severity).
5. One forced decision the founder owes today.

## Weekly review (45 minutes)

Generated by `make ceo-weekly-review` (`scripts/generate_ceo_weekly_review.py`).
Output: `data/ceo_briefs/weekly/<ISO-week>.md`.

Sections:
- North-star score delta.
- Pipeline health (added / qualified / closed-won / closed-lost).
- Capital allocation report (`make capital-allocation`).
- Doctrine compliance report (governance + proof pack + capital asset stats).
- Top 3 customer signals worth a personal call.
- Next-week 3-bet decisions.

## Cadence

| Day | Anchor |
|---|---|
| Sunday | Weekly review + plan |
| Mon–Thu | Daily brief + 2x deep-work blocks |
| Friday | Quiet day, deep customer work |
| Saturday | Optional reading + reflection |

## Doctrine coverage

The CEO OS is bound to the non-negotiables: any week showing a doctrine
violation (e.g., an external send without approval) blocks weekly review
PASS until the violation is logged and remediated.
