# نظام المؤسس — Dealix

> **كيف يُدير المؤسس الشركة يوميًا وأسبوعيًا وشهريًا.**
> الفرع: `phase/startup-architecture-brand-os`
> كل التواصل الخارجي مسودة فقط افتراضيًا (`OUTBOUND_MODE=draft_only`).
> لا قرارات خارجية ذاتية. الذكاء الاصطناعي يوصي، الإنسان يعتمد.

---

## 1. الهدف من نظام المؤسس

نظام المؤسس هو طبقة التشغيل التي تضمن أن المؤسس:
- يرى الحالة الكاملة للشركة كل صباح.
- يتخذ القرارات الحرجة في وقتها.
- يُفوّض ما لا يحتاج قراره المباشر.
- لا يُرسل أي شيء للخارج دون اعتماد صريح.
- يُسجّل كل قرار في دماغ الشركة ليتعلم منه النظام.

المؤسس في ديالكس هو المشغّل الأول. لا توجد طبقة إدارة متوسطة في المرحلة الحالية. النظام يعوّض غياب الفريق الكبير بالانضباط والإيقاع والأتمتة المحكومة.

## 3. الجدول الأسبوعي

### الأحد — مراجعة الأسبوع
- مراجعة كل المؤشرات الأسبوعية (تشخيصات، تجريبيات، اشتراكات، حزم إثبات).
- مراجعة الأنابيب الكاملة في `ledgers/deals_pipeline.csv`.
- تحديث أولويات الأسبوع القادم.

### الإثنين — بحث القطاعات
- عمق بحث قطاع واحد من القطاعات الثمانية المستهدفة.
- تحديث `data/outreach/saudi_icp_segments.json`.
- مراجعة إشارات المنافسين لذلك القطاع.

### الثلاثاء — التسليم والعملاء
- مراجعة حالة كل عميل نشط (Discovery/Build/Test/Report).
- التأكد من أن كل عميل لديه خط أساس (baseline) واضح.
- تحديد العملاء القريبين من تقرير النجاح.

### الأربعاء — العروض والأسعار
- مراجعة سُلّم العروض.
- تحديث محرك التسعير بناءً على إشارات السوق.
- مراجعة عروض قيد الإعداد.

### الخميس — الثقة والمراجعة
- مراجعة سجل الادعاءات الممنوعة (الهدف: صفر مخالفات).
- مراجعة حالة أعلام الإرسال لكل بيئة.
- مراجعة اكتمال سجل التدقيق.
- تقرير أسبوعي مختصر في دماغ الشركة.

### الجمعة — راحة (لا عمليات تجارية)
### السبت — تخطيط الأسبوع القادم (ساعة واحدة فقط)
- قراءة تقرير الأسبوع.
- تحديد 3 أولويات للأسبوع القادم.
- لا عمليات إرسال.

## 4. الجدول الشهري

### آخر يوم عمل في الشهر — المراجعة الشهرية
- مراجعة الإيراد المتكرر الشهري (MRR) بالريال.
- مراجعة عدد الاشتراكات النشطة.
- حساب معدل تحويل التجريبي → اشتراك.
- حساب التسرّب.
- حساب الهامش الإجمالي.
- حساب مدة التشغيل (Runway) بالأشهر.
- مراجعة مشغلات التوظيف: هل اشتعل أي مشغل؟

### اليوم الأول من الشهر — التخطيط
- تحديد هدف الإيراد الشهري (سيناريو + ثقة + فرضيات).
- تحديد عدد التشخيصات المستهدفة.
- تحديد عدد التجريبيات المستهدفة.
- تحديث حزمة العرض للقطاعات ذات الأولوية.
- مراجعة استراتيجية السوق السعودي B2B (انظر `SAUDI_B2B_MARKET_STRATEGY.md`).

### منتصف الشهر — التصحيح
- مقارنة التقدم الفعلي بالخطة.
- إن كان هناك انحراف > 20%، مراجعة الأسباب وتعديل الخطة.
- مراجعة الأنابيب المتوقّدة.

## 5. القرارات اليومية

القرارات التي يتخذها المؤسس يوميًا:

| القرار | التكرار | الأداة |
|---|---|---|
| اعتماد/رفض مسودة تواصل | عدة مرات | مكتب قرارات المؤسس |
| تحديث حالة فرصة في الأنابيب | عدة مرات | غرفة قيادة الإيرادات |
| تحديد أولوية عميل تسليم | يوميًا | نظام تسليم العملاء |
| اعتماد عرض تشخيص | عند الحاجة | نظام ذكاء العروض |
| تصنيف رد عميل | عند الردود | نظام متابعة الواتساب والبريد |
| تسجيل تعلّم | يوميًا | دماغ الشركة |

القرارات التي لا يتخذها المؤسس يوميًا (تُفوّض أو تؤجّل):
- تغيير استراتيجية التسعير (شهريًا فقط).
- تفعيل تواصل حي (يتطلب مراجعة قانونية + opt-in).
- توظيف (يتطلب مشغلًا حقيقيًا).

## 7. مصفوفة التفويض

في مرحلة المؤسس، التفويض محدود لأن الفريق صغير. لكن هناك قرارات يمكن تفويضها للنظام (الأتمتة المحكومة) وقرارات لا يمكن تفويضها إطلاقًا.

| النشاط | المؤسس | النظام (AI) | ملاحظة |
|---|---|---|---|
| توليد مسودة تواصل | — | ✓ | مسودة فقط، لا إرسال |
| اعتماد مسودة للإرسال | ✓ | — | قرار إنساني صريح |
| الإرسال الفعلي | ✓ (يدوي) | ✗ | يتطلب أعلامًا مفعّلة |
| تصنيف رد عميل | مراجعة | ✓ (اقتراح) | النظام يقترح، المؤسس يصدر |
| تحديث الأنابيب | ✓ | ✓ (مساعدة) | |
| توليد عرض سعر | — | ✓ (اقتراح) | عبر نظام ذكاء العروض |
| اعتماد عرض سعر | ✓ | — | |
| بناء نظام لعميل | مشاركة | ✓ (تنفيذ) | نظام تسليم العملاء |
| تقرير نجاح عميل | ✓ | ✓ (مسودة) | حزمة إثبات تنفيذية |
| ادعاء نتائج خارجي | ✓ | ✗ | يتطلب حزمة إثبات حقيقية |
| تفعيل دفع حي | ✓ | ✗ | `MOYASAR_LIVE_MODE=true` |
| تفعيل تواصل حي | ✓ | ✗ | أعلام + opt-in + قانوني |
| توظيف | ✓ | ✗ | يتطلب مشغلًا حقيقيًا |
| تسجيل تعلّم | ✓ | ✓ (مساعدة) | دماغ الشركة |

## 8. قواعد الانضباط

1. **لا إرسال قبل الاعتماد.** كل مسودة تمرّ بطابور الاعتماد. لا استثناء.
2. **لا ادعاءات بدون إثبات.** كل ادعاء خارجي يحتاج حزمة إثبات: قبل ← بعد ← فرق موثّق.
3. **لا توظيف بالتوقع.** التوظيف يحكمه مشغل حقيقي اشتعل فعليًا.
4. **لا توقع بدون فرضيات.** كل توقع يحمل مستوى ثقة + فرضيات صريحة.
5. **سجّل كل قرار.** كل قرار يُسجّل في دماغ الشركة ليتعلم منه النظام.
6. **الإغلاق اليومي إلزامي.** تقرير المدير اليومي ليس اختياريًا.
7. **الراحة الأسبوعية محمية.** الجمعة لا عمليات تجارية. السبت ساعة تخطيط فقط.
