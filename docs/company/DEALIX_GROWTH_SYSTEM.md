# Dealix Growth System — نظام النمو

Dealix لا تحتاج "حملات". تحتاج **growth system** قابل للقياس والتكرار.

## الغرض — Purpose

إنشاء نظام **قابل للتكرار** لاكتساب، تحويل، تسليم، والاحتفاظ بعملاء B2B في السعودية.

## المالك — Owner

Sami / Founder CEO.

## إيقاع المراجعة — Review Cadence

أسبوعي.

## الحلقة — Growth Loop

```
Lead → DM → Reply → Sample → Proposal → Payment → Delivery → Feedback → Retainer → Proof → More Leads
```

كل حلقة تنتج **proof**، وكل proof يولّد **leads** جدد.

## القنوات — Channels

1. Founder-led outbound (DMs, email, calls).
2. Samples.
3. Referrals.
4. Content مبني على proof.
5. Partners.
6. Retainers / expansion.

## أولوية القنوات الحالية — Current Channel Priority

1. **Founder-led outbound** — الأساس.
2. **Sample packs** — proof of skill.
3. **Proposal follow-up** — تحويل المهتمّين.
4. **Delivery proof** — أصول لاحقة.
5. **Client referral** — أرخص lead.

## القواعد — Rules

- **لا تُوسّع قناة** قبل وجود conversion.
- **لا تنشر proof** بدون موافقة.
- **لا تضف قنوات** قبل أن تتعلّم من الأولى.
- **كل experiment** له metric محدد.
- **لا scaling قبل manual proof of repeatability** — راجع [`GROWTH_ANTI_PATTERNS.md`](GROWTH_ANTI_PATTERNS.md).
