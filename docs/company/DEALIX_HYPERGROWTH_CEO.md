# Dealix Hypergrowth CEO Layer

This is the founder's lever-routing manual. Used when revenue is
growing > 30% MoM and decisions multiply faster than the founder can
serialize.

## Founder leverage focus areas

| Area | Lever | Default ownership |
|---|---|---|
| Sales | Approval queue throughput | dealix-sales + founder |
| Delivery | Proof Pack assembly | dealix-delivery |
| Product | Capital Asset registry | dealix-delivery + founder |
| Governance | Eval gate enforcement | dealix-pm |
| Capital | Allocation report cadence | founder |

## Weekly loop

1. **Monday** — Strategy scorecard refresh (`make strategy-scorecard`).
2. **Tuesday** — Capital allocation report review.
3. **Wednesday** — Customer call cadence (top 3 by signal).
4. **Thursday** — Founder deep work (no calls).
5. **Friday** — Weekly review + next-week 3 bets.
6. **Sunday** — Doctrine compliance audit.

## Automation budget per week

| Activity | Automation budget |
|---|---|
| Drafting outreach | <= 4h founder time (rest = dealix-sales) |
| Generating Proof Packs | <= 8h founder time per paid engagement |
| Capital Asset packaging | <= 2h founder time per asset |
| Approval queue triage | <= 1h founder time per day |

If any line exceeds budget for 2 weeks running, the dealix-pm raises a
`founder-leverage` decision card.

## Halt conditions

- Doctrine violation in audit log -> halt all outbound + new sales.
- Capital allocation runway buffer < 10% -> halt hiring + new tooling.
- Founder time > 60h/week -> halt new sprint sales.
