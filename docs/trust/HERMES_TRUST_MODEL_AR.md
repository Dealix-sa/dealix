# 🛡️ نموذج الثقة وحوكمة الموافقات — ديلكس هيرمس

نموذج متكامل لتفويض الصلاحيات وضمان أمان البيانات بما يطابق نظام حماية البيانات الشخصية السعودي (PDPL) وإطار إدارة مخاطر الذكاء الاصطناعي NIST AI RMF.

---

## 1. فلسفة الأمان (Zero-Trust AI Principle)
* **لا إجراء خارجي دون موافقة بشرية مسبقة:** لا يقوم النظام بإرسال أي رسالة مبيعات أو محتوى خارجي أو عروض أسعار تلقائياً دون تسجيل رمز موافقة معتمد (`APP-ID`) موقّع من المؤسس سامي عسيري.
* **فصل قنوات البيانات:** بيانات العملاء والتقارير الفنية الحساسة معزولة تماماً في خوادم محلية ولا تمرر لأي طرف خارجي أو نماذج تدريب عامة.

## 2. مستويات الحوكمة والصلاحيات
1. **L0 (غير آمن):** استخدام عام للذكاء الاصطناعي دون مراجعة أو قيود (محظور تماماً في ديلكس).
2. **L1 (مرحلة المراجعة البشرية):** النظام يقترح المسودة، والموظف يراجع ويعدل ويرسل يدوياً (الوضع الافتراضي الحالي لـ Dealix).
3. **L2 (مرحلة الموافقات المنظمة):** النظام يعتمد مخرجات محدودة الحساسية بعد التحقق التلقائي ومطابقة شروط الفحص.
4. **L3 (التشغيل الآلي المقيد):** إطلاق تلقائي لرسائل تصحيح العناوين والتدفقات اللوجستية العامة التي لا تحتوي على بيانات مالية أو PII حساسة.
5. **L4 (كامل الأمان والمراجعة):** مخرجات وحزم إثبات معتمدة وموثقة وموقعة برمجياً وجاهزة للعميل.

## 3. حماية البيانات وأنظمة الامتثال
* **توافق PDPL:** حصر معالجة البيانات الشخصية، مسح البيانات بعد نهاية المشروع، وتوقيع اتفاقيات حماية معالجة البيانات (DPA).
* **إدارة مخاطر NIST AI RMF:**
  - **Govern:** تأسيس ثقافة الحوكمة عبر دفاتر السجلات.
  - **Map:** تحديد المخاطر المرتبطة بالوكلاء وتدفقات المحادثات.
  - **Measure:** قياس مستويات الجودة ونسب الخطأ التشغيلي.
  - **Manage:** إدارة المخاوف بشكل مباشر عبر معالجة الاعتراضات ووقف الحملات المخالفة.
