# Messaging Hierarchy

**Owner:** Founder + Brand Lead
**Source of truth:** `docs/positioning/DEALIX_POSITIONING.md` + `docs/brand/DEALIX_BRAND_VOICE.md`

## Purpose

Every external surface — landing page, deck, proposal, outbound draft, social post — uses a consistent message stack. This doc defines the H1 / H2 / H3 hierarchy so every author writes from the same shelf.

## The hierarchy

| Level | Function | Length | Example |
|---|---|---|---|
| H1 | Category claim | 6-10 words | "Saudi B2B Revenue Operating System." |
| H2 | What it does | 12-18 words | "Intelligent deal flow, founder-approved growth, trust-gated AI execution." |
| H3 | Who it is for | 15-22 words | "For Saudi B2B founders running revenue themselves, with 5-50-person teams and 1-3 commercial operators." |
| Sub | Proof anchor | 12-20 words | "Every action passes a named approval class. Every sprint produces a Proof Pack." |
| CTA | Next step | 5-9 words | "Start with a Diagnostic." |

## Landing page (long-form surface)

```
H1: Saudi B2B Revenue Operating System.
H2: Intelligent deal flow, founder-approved growth, trust-gated AI execution.
H3: For Saudi B2B founders running revenue themselves.
Sub: Every action passes a named approval class. Every sprint produces a Proof Pack.
CTA: Start with a Diagnostic.
```

## Cold outreach draft (queued for founder approval)

```
Subject: One question about your deal flow at [Company]
Body (3-5 lines):
- Sentence 1 — Observed signal about the customer (sourced from public, sanctioned data).
- Sentence 2 — One specific question about their revenue operation.
- Sentence 3 — Dealix one-line ("Saudi B2B Revenue OS").
- Sentence 4 — Offer: a 30-minute Diagnostic call.
- Sentence 5 — Sign-off, founder name.
```

No engagement tricks, no postscripts about how busy they are, no manufactured urgency.

## LinkedIn post (founder voice)

```
Hook (1 line) — A principle or an observation.
Body (3-5 lines) — Evidence, dated, anonymized.
Bridge (1 line) — What Dealix is doing about it.
CTA (1 line) — Specific peer ask, not a generic sell.
Disclosure — Standard footer.
```

See `docs/brand/DEALIX_SOCIAL_MEDIA_KIT.md` for templates.

## Proposal cover and exec summary

```
H1: Sprint Proposal — [Sprint Type] — [Sector]
H2 (exec summary lead): "This proposal commits Dealix to deliver [named outputs] within [N] weeks for [SAR amount]."
```

The proposal exec summary leads with the commitment, not with company background.

## Sector report (public-facing)

```
H1: [Sector] — Saudi B2B Sector Scorecard.
H2: Methodology-led, aggregated patterns, no named accounts.
H3: Sprint engagement pattern for operators in this sector.
```

## Messaging "shelf" — banned at every level

- "AI-powered" anywhere in the hierarchy.
- "Transform your business" or any "transform/supercharge/unlock" verb.
- "Guaranteed" — replace with "evidenced" or "estimated."
- "We help businesses" — too generic, replace with the named customer profile.

See `docs/brand/DEALIX_BRAND_VOICE.md` for the full banned phrase list.

## Adapting the hierarchy to a sector

For a sector-specific landing (e.g., cybersecurity B2B):

```
H1: Saudi B2B Revenue OS — Cybersecurity Sector.
H2: Account scoring and gated outreach for Saudi cybersecurity vendors.
H3: For Saudi cybersecurity founders selling to mid-market and enterprise.
Sub: Sprint catalog mapped to cybersecurity buyer cycles.
CTA: Start with a Cybersecurity Sector Diagnostic.
```

The H1 substitutes the sector. The H2 substitutes the named buyer cycle. The rest holds.

## Adapting the hierarchy to a partner audience

```
H1: Build your sprint catalog on Dealix Revenue OS.
H2: Partner program for Saudi B2B integrators and consultancies.
H3: For partners with an existing book of Saudi B2B service customers.
Sub: Shared trust gates, shared Proof Pack standard, shared category language.
CTA: Apply to the Dealix Partner Program.
```

## Failure mode

- A new surface invents its own H1, fragmenting the category claim.
- Sub-line on a landing page contains a guarantee word.
- Arabic H1 reads softer than the English H1.

## Recovery path

1. Re-anchor to this doc.
2. Re-run the voice checklist in `DEALIX_BRAND_VOICE.md`.
3. Re-publish.

## Disclaimer

Messaging hierarchy is positional, not promissory. Estimated value is not Verified value / القيمة التقديرية ليست قيمة مُتحقَّقة.
