# Cold Email Sequences — تسلسلات البريد البارد (English)

**Cold email is Dealix's primary acquisition channel**, operated inside PDPL and CAN-SPAM limits: accurate sender identity, a working opt-out in every message, opt-outs honored immediately, no purchased lists, no misleading subjects, and no fake "Re:" or "Fwd:". Agents draft; the founder approves; nothing is sent externally without approval.

This file gives ready-to-adapt copy for five touch types: `first_touch`, `follow_up_1`, `follow_up_2`, `proposal_intro`, and `breakup`. Packages and reference prices live in [DEALIX_REVOPS_PACKAGES_AR.md](../commercial/DEALIX_REVOPS_PACKAGES_AR.md). For sector angles see the [sectors index](../sectors/README.md). For production logic see the [market production OS](../market_production_os/README.md). The Arabic parallel is [COLD_EMAIL_SEQUENCES_AR.md](COLD_EMAIL_SEQUENCES_AR.md).

## Sender-identity requirement (non-negotiable)

- A **real name** of a real person at Dealix — no aliases, no anonymous "sales team".
- A **real reply address** that receives and is read — no unmonitored inboxes.
- A signature with the company name, a clear contact method, and a physical address on request.
- No domain spoofing and no display name that hides the true sender.
- Every message carries one clear opt-out line (below).

---

## Personalization slot guide

Every `first_touch` builds on a real research note placed in the personalization slot. The note is one line proving you actually studied the company: a new branch, a sales-role hire, a sector expansion, or another sourced public signal. Levels:

| Level | Definition | first_touch allowed? |
|-------|------------|----------------------|
| **P0** | No note — generic name only | **No** — forbidden |
| **P1** | Sector/size signal + clear reason to reach out | Yes (minimum) |
| **P2** | A specific sourced event (hire, branch, launch) | Yes (preferred) |
| **P3** | Event + precise tie to an addressable operating pain | Yes (strongest) |

**Governing rule:** no `first_touch` below level **P1**. Any P0 draft is rejected before approval. The personalization slot is never left empty or filled with a generic phrase.

Standard opt-out line in every message:

> To opt out, reply STOP and we will remove you immediately.
