# دليل التواصل المبني على الأنظمة
# System-Based Outreach Playbook

---

## المبدأ الجوهري — Core Principle

**النظام أولاً، ليس AI عاماً.**

كل رسالة تواصل تُحدد نظاماً تشغيلياً محدداً يحل مشكلة محددة. لا رسائل عامة عن "قدرات الذكاء الاصطناعي". لا وعود بنتائج مضمونة.

**System first, not generic AI.**

Every outreach message names a specific operational system that solves a specific problem. No generic AI capabilities messages. No guaranteed outcomes.

---

## قيود الإرسال — Sending Constraints

قبل أي رسالة تواصل، يجب التحقق من:

1. **موافقة المؤسس مكتوبة:** لا تُرسَل أي رسالة بدون موافقة المؤسس الصريحة على المسودة
2. **Need Fit Score ≥ 7:** الحساب المستهدف مؤهل ومُحدَّد
3. **سياسة Do Not Contact:** التحقق من `docs/privacy/DO_NOT_CONTACT_AND_SUPPRESSION_POLICY_AR.md`
4. **Domain Health سليم:** SPF/DKIM/DMARC جاهز قبل أي إرسال
5. **المسودة في SYSTEM_EMAIL_DRAFTS_REVIEW:** المراجعة قبل الإرسال

---

## بنية الرسالة — Message Structure

كل رسالة تتبع هذا الهيكل:

```
Hook      — جملة تربط المشكلة بالواقع المُحدد للمستلم
Pain      — وصف دقيق للتكلفة التشغيلية للمشكلة
System    — اسم النظام التشغيلي المقترح (لا "AI" وحده)
Sprint    — العرض المحدد مع الوقت والسعر
CTA       — خطوة واحدة واضحة
```

## معايير جودة الرسالة — Message Quality Standards

كل رسالة قبل الموافقة عليها يجب أن:

- [ ] تُسمي نظاماً تشغيلياً محدداً (لا "AI" وحده)
- [ ] Need Fit Score للحساب المستهدف ≥ 7
- [ ] لا تحتوي على وعد بزيادة مبيعات أو ROI محدد
- [ ] لا تحتوي على "مضمون" أو "نضمن" أو "100%"
- [ ] تُحدد وقت تسليم واضح (7 أيام)
- [ ] تُحدد سعراً واضحاً
- [ ] تطلب خطوة واحدة فقط (15 دقيقة / فحص مجاني)
- [ ] حاصلة على موافقة المؤسس

---

## ما هو محظور في التواصل — Prohibited in Outreach

1. الإرسال التلقائي بدون موافقة المؤسس على كل دفعة
2. استخدام أسماء عملاء حقيقيين كمراجع بدون إذن موثق
3. الوعد بنتائج مبيعاتية أو مالية محددة
4. ادعاء أن Dealix يُرسل الرسائل نيابة عن العميل تلقائياً
5. جمع بيانات من مصادر غير مصرّح بها
