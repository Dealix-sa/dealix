# Inbound Engine OS

## Purpose
Create demand through founder content, proof, sector insights, and useful operating frameworks.

## Inbound Assets
- Founder posts.
- Sector mini-reports.
- Revenue Sprint examples.
- Anonymized lessons.
- Proof posts.
- Landing page.
- Lead magnet.
- Newsletter later.

## Weekly Outputs
- 3 founder posts.
- 1 sector insight.
- 1 useful checklist.
- 1 proof / learning post if approved.

## Lead Capture
Every content asset should point to one CTA:
- Ask for a sample.
- Request Revenue Sprint.
- DM Sami.
- Join waitlist later.

## Rules
- No fake traction.
- No unsupported claims.
- No client name without approval.
- Content should create trust, not hype.
