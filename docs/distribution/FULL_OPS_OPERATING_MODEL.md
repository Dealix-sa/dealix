# Dealix — Full-Ops Operating Model
<!-- PHASE 3 | Owner: Founder | Date: 2026-05-18 -->
<!-- Arabic primary — العربية أولاً -->

كيف يُدار Dealix كعملية تجارية شبه ذاتية التشغيل: المؤسس هو **طبقة القرار
والموافقة** — لا طبقة العمل اليدوي. كل ما يمكن أتمتته داخلياً مُؤتمت؛ كل فعل
خارجي أو غير قابل للتراجع يمرّ بموافقة بشرية. هذا المستند يربط المكائن السبع،
فريق الوكلاء، والإيقاع اليومي في نظام واحد قابل للتشغيل.

How Dealix runs as a near-autonomous commercial operation: the founder is the
**decision-and-approval layer**, not the manual-labor layer. Everything that can
be automated internally is automated; every external or irreversible action
passes a human approval gate.

> **حدّ الواقع — Reality boundary.** المنصة جاهزة ومُتحقَّق منها. المشروع تحت
> **تجميد تجاري** ([`../ops/COMMERCIAL_FREEZE.md`](../ops/COMMERCIAL_FREEZE.md)):
> "Full ops" هنا = **تشغيل المكائن المبنية بأقصى كثافة**، لا بناء مكائن جديدة.
> القيد ليس الكود — بل البيع وتفعيل الدفع. "Full ops" means *operating* the
> machines already built — not building new ones.

## 1. المبدأ التشغيلي — The operating principle

**أتمتة داخلية كاملة · موافقة بشرية على كل فعل خارجي.**
Full internal automation · human approval on every external action.

ثلاث حلقات لكل فعل في النظام — every action falls in one of three rings:

| الحلقة Ring | التعريف | من ينفّذ |
|---|---|---|
| 🟢 **آلي Auto** | داخلي، قابل للتراجع، لا يلمس عميلاً | النظام/الوكيل بلا انتظار |
| 🟡 **بموافقة Approval-gated** | خارجي أو غير قابل للتراجع | الوكيل يُجهّز مسودة → المؤسس يوافق |
| 🔴 **محظور Banned** | يخالف العقيدة | لا يحدث أبداً — يُرفض في الكود |

المحظورات الـ11 (`no_live_send`, `no_live_charge`, `no_cold_whatsapp`,
`no_scraping`, `no_fake_proof`, `no_unconsented_data`, `no_unverified_outcomes`,
`no_hidden_pricing`, `no_silent_failures`, `no_unbounded_agents`,
`no_unaudited_changes`) هي السياج. لا "full ops" يكسر هذا السياج.

## 2. هرم الوكلاء — The agent team

فريق تنفيذي من الوكلاء المتخصصين، المؤسس على القمة كسلطة القرار:

```
                    ┌─────────────────┐
                    │     المؤسس       │  ← القرار + الموافقة + Moyasar
                    │   Founder/CEO    │
                    └────────┬────────┘
                    ┌────────┴────────┐
                    │   dealix-pm     │  ← المنسّق: الحالة، التتابع، البوابات
                    │  (orchestrator) │
                    └────────┬────────┘
        ┌──────────────┬─────┴──────┬──────────────┐
   dealix-sales   dealix-content  dealix-delivery  dealix-engineer
   تأهيل/عروض/    محتوى ثنائي     سبرنت 7 أيام →   كود/اختبارات/
   تواصل دافئ     اللغة، SOPs     Proof Pack       هجرات (P0/P1 فقط أثناء التجميد)
```

| الوكيل | يفعل | متى تستدعيه | يكتب كوداً؟ |
|---|---|---|---|
| **dealix-pm** | يملك خطة الـ90 يوم، التتابع، البوابات، مراجعة سجل الاحتكاك | "ما الحالة؟ ما التالي؟ نفّذ الخطة" | لا |
| **dealix-sales** | تأهيل العملاء، توليد العروض، مسودات التواصل الدافئ، توصية العرض من السلّم | عند lead جديد أو فرصة | لا |
| **dealix-content** | وثائق AR+EN، SOPs، دراسات حالة، قوالب عروض، منشورات | عند الحاجة لمحتوى/نسخة | لا |
| **dealix-delivery** | سبرنت إثبات الإيرادات 7 أيام، تجميع Proof Pack، التسليم | عند توقيع pilot | لا |
| **dealix-engineer** | كود Python، FastAPI، اختبارات، هجرات — **أثناء التجميد: P0/P1 فقط** | إصلاح عاجل أو نطاق الدرجة 0–1 | نعم |

قاعدة حوكمة الوكلاء (`no_unbounded_agents`): كل وكيل له حدود واضحة، ولا وكيل
يرسل رسالة خارجية أو يشحن مبلغاً — فقط يُجهّز مسودة للموافقة.

## 4. مصفوفة الأتمتة — what runs without you / waits for you / forbidden

| 🟢 آلي — يعمل بلا انتظارك | 🟡 ينتظر موافقتك | 🔴 محظور تماماً |
|---|---|---|
| التقاط وتسجيل العملاء المحتملين | إرسال أي رسالة خارجية | scraping / استخراج بيانات |
| تسجيل النقاط وترتيب الأولوية | إرسال أي فاتورة | شحن مالي تلقائي |
| صياغة الرسائل والعروض (مسودات) | أي خصم أو استرداد | واتساب بارد |
| تذكيرات المتابعة، موجزات الاجتماعات | تقديم KYC في Moyasar | أتمتة LinkedIn / DM جماعي |
| تشغيل سبرنت 7 أيام وتجميع Proof Pack | إصدار Proof Pack النهائي | دليل مزيّف / إحصاءات ملفّقة |
| التسوية الليلية، إعادة تشغيل DLQ | اعتماد شريك / صرف عمولة | ادعاء عائد مضمون |
| اللوحة اليومية، الموجز الأسبوعي | نشر دراسة حالة باسم عميل | تغييرات بلا تدقيق |
| مُحقِّقات الجاهزية، سلسلة التدقيق | إصدار board memo | عرض/تسعير مخفي |

**القاعدة:** إن كان الفعل خارجياً أو غير قابل للتراجع → 🟡. إن كان داخلياً
وقابلاً للتراجع → 🟢. إن خالف العقيدة → 🔴.

## 5. الإيقاع التشغيلي — The operating cadence

### يومياً — Daily loop
المرجع الكامل: [`../ops/DAILY_COMMERCIAL_LOOP_AR.md`](../ops/DAILY_COMMERCIAL_LOOP_AR.md)

| الوقت | الفعل | الحلقة |
|---|---|---|
| صباحاً | افتح اللوحة اليومية + قائمة قرارات الموافقة | 🟢 تُولَّد آلياً |
| صباحاً | راجع ووافق على مسودات التواصل (≤10 لمسات) | 🟡 موافقتك |
| الظهر | متابعات اليوم (≤5)، محادثة شريك واحدة | 🟡 موافقتك |
| مساءً | حدّث غرف البيع، سجّل الأرقام في scorecard | 🟢 + 🟡 |

### أسبوعياً — Weekly
المرجع: [`../ops/POST_LAUNCH_SCORECARD.md`](../ops/POST_LAUNCH_SCORECARD.md) — Pilots نشطة + أحداث إثبات مُسلَّمة. أي صف أحمر = action item بمالك وتاريخ.

### شهرياً — Monthly
مراجعة بنمط مجلس الإدارة: الإيراد، الأنبوب، التسليم، الدعم، الشركاء، الحوكمة،
الرهان التالي.

### قواعد التشخيص — Diagnostic rules
- 7 أيام بلا ردود → غيّر الشريحة أو الرسالة.
- ردود بلا demos → غيّر CTA.
- demos بلا دفع → غيّر العرض أو السعر أو الدليل.

## 7. طابور قرارات المؤسس — The founder's decision queue

في نظام full-ops، عملك الوحيد هو هذه القرارات. لا شيء آخر يحتاجك:

1. ✅ الموافقة على مسودات التواصل الخارجي (يومياً).
2. ✅ الموافقة على إرسال الفواتير.
3. ✅ **تفعيل Moyasar KYC** — العائق #1، إجراء يخصّك وحدك
   ([`../ops/MOYASAR_KYC_CHECKLIST.md`](../ops/MOYASAR_KYC_CHECKLIST.md)).
4. ✅ الموافقة على الخصومات والاستردادات.
5. ✅ الموافقة على إصدار Proof Pack النهائي.
6. ✅ الموافقة على اعتماد الشركاء وصرف العمولات.
7. ✅ الموافقة على نشر دراسات الحالة وذكر أسماء العملاء.

كل ما عداه — التقاط، تسجيل، صياغة، تذكير، تقارير، تسوية — يجري بلا تدخّلك.

## 9. ما لا يعنيه "Full ops" الآن — Freeze boundary

- ❌ ليس بناء واجهات/راوترات/وحدات جديدة للدرجات 2–5.
- ❌ ليس إعادة تصميم الواجهة الأمامية أو لوحات جديدة.
- ❌ ليس إرسالاً خارجياً تلقائياً أو شحناً مالياً تلقائياً.
- ❌ ليس استبدال موافقة المؤسس على الأفعال الخارجية.
- ✅ هو تشغيل المكائن المبنية بأقصى كثافة، بأقصى أتمتة داخلية، وبموافقة بشرية
  منضبطة على كل فعل خارجي — حتى يُسلَّم أول Pilot مدفوع، ثم تُحكَم الخطوة التالية
  بـ[`../90_DAY_BUSINESS_EXECUTION_PLAN.md`](../90_DAY_BUSINESS_EXECUTION_PLAN.md).
