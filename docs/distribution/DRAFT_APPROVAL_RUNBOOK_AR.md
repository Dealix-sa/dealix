# Draft Approval Runbook — دليل اعتماد المسودات

> روتين المؤسس اليومي لتصريف المنتجات بأمان. الهدف: قرارات سريعة على مسودات
> جاهزة، دون أي إرسال آلي.

## 1. التشغيل الصباحي

```bash
make doctor            # فحص صحة الريبو (env + alembic + security smoke)
make distribution-day  # توليد المسودات + مراجعة الطابور
```

ثم افتح:

```
reports/distribution/DRAFT_GENERATION_REPORT.md
reports/distribution/DRAFT_QUEUE_REVIEW.md
```

## 2. مراجعة كل مسودة (قائمة تحقّق)

لكل مسودة في `pending_approval`، تأكّد:

- [ ] الشركة/القطاع مناسبان فعلًا (ملاءمة ICP).
- [ ] الرسالة دقيقة وتعكس ألمًا حقيقيًا.
- [ ] **لا** وعد بنتيجة مضمونة.
- [ ] **لا** بيانات شخصية (PII) في النص.
- [ ] القناة صحيحة (وكلها يدوية).
- [ ] مستوى الإثبات منطقي (المسودة الأولى = L1).

أي مسودة في `needs_edit` تحمل `policy_issues` — **لا تعتمدها** قبل إصلاح السبب.

## 3. القرار

| القرار | الإجراء |
| --- | --- |
| Approve | اعتمد النص، ثم انسخه يدويًا للقناة. |
| Edit | عدّل النص ثم أعد المراجعة. |
| Reject | استبعد المسودة مع سبب مختصر. |
| Postpone | أجّل (مثلًا توقيت غير مناسب). |

> v1 لا يضبط الحالة آليًا بعد القرار البشري؛ الاعتماد/النسخ يتمّ يدويًا. مسار
> الـ API/الواجهة لضبط الحالات مخطّط في P1.

## 4. الإرسال — يدوي فقط

| القناة | كيف |
| --- | --- |
| email | انسخ النص أو احفظه كمسودة في عميل البريد. |
| whatsapp_manual | انسخ والصق يدويًا فقط. **لا** أتمتة. |
| linkedin_manual | انسخ والصق يدويًا فقط. **لا** أتمتة. |
| proposal_pdf | صدّر PDF / مسودة. |
| phone_script | استخدم النص كنقاط مكالمة. |

## 5. ممنوع منعًا باتًا

- ❌ cold WhatsApp automation.
- ❌ LinkedIn automation.
- ❌ scraping أو قوائم مشتراة.
- ❌ ادعاء ROI مضمون.
- ❌ أي إرسال خارجي بدون موافقة.

هذه القواعد مفروضة في الكود عبر `policy_check_draft` و`enforce_doctrine_non_negotiables`
و`pii_flags_for_row`، ومقفولة في `tests/test_distribution_drafts.py`.

## 6. إيقاع مقترح ليوم واحد

```
اعتمد 10 مسودات
عدّل 3
ارفض 5
احجز مكالمتي استكشاف
ولّد عرضًا واحدًا
```

هذا هو التصريف الحقيقي: قابل للتنفيذ، آمن، وقابل للقياس.
