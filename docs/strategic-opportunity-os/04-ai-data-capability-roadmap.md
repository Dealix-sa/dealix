# AI and Data Capability Roadmap

## Purpose
Turn Dealix AI from feature output into a governed capability with quality, data advantage, and trust.

## Capability layers

| Layer | Goal | Artifact |
|---|---|---|
| Data inventory | Know what data exists and who owns it | Data owner map |
| Data quality | Make data reliable for decisions | Data quality checks |
| AI use case registry | Know every AI workflow | AI tool registry |
| Evaluation | Test output quality before release | AI evaluation checklist |
| Human review | Keep high-impact outputs owned | Review SOP |
| Observability | Trace model use, quality, and cost | Trace dashboard |
| Incident response | Handle AI errors with trust | AI incident playbook |
| Customer proof | Convert results into evidence | Proof ledger |

## 90-day AI/data plan

| Phase | Focus | Output |
|---|---|---|
| Days 1-30 | Inventory and governance | AI use case registry and data map |
| Days 31-60 | Evaluation and observability | Test sets and trace dashboard |
| Days 61-90 | Customer proof loop | Proof ledger and quality review |

## High-value AI workflows

| Workflow | Value | Control |
|---|---|---|
| Account research summary | Faster sales prep | Source review |
| Opportunity scoring explanation | Better prioritization | Explainability and owner |
| Outreach draft | Faster founder-led sales | Human approval |
| Customer health summary | Earlier risk detection | CS review |
| Executive weekly memo | Faster decisions | Founder approval |
| Proof pack assembly | Better conversion | Evidence mapping |

## Data advantage thesis

Dealix should compound data in four places:

1. Account fit and segment data.
2. Opportunity stage and objection data.
3. Customer workflow and proof data.
4. AI output quality and review data.

## Metrics

- AI workflows registered.
- AI outputs reviewed.
- Evaluation pass rate.
- Unsupported claim count.
- Time saved per workflow.
- Customer proof assets created.
- Data quality issue count.

## Guardrails

- No customer-facing AI output without review unless formally approved.
- No high-impact score without explanation.
- No unsupported public claim.
- No sensitive data sent to a tool without documented purpose.
- No AI automation without fallback.
