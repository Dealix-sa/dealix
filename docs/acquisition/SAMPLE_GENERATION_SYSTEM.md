# Sample Generation System

How Dealix produces the Signal Sample — the artifact that turns a reply into a call.

## What a Signal Sample is
A short, specific, useful artifact tailored to one prospect. Not a brochure. Not a generic deck.

## Sample types
- **Lead Sample** — 5–10 named, public-source leads inside the prospect's ICP, with reasoning.
- **Market Snapshot** — one-page view of the prospect's sector with three concrete signals.
- **Pain Hypothesis** — a short note naming one likely problem and how Dealix would diagnose it.

## Production rules
- Built from public data only unless the prospect provides data with consent.
- One page maximum. PDF or short doc.
- Branded as "sample" — never used as a delivered report.
- Passes the same Claim Guard as a paid deliverable.

## Time budget
- Lead Sample: 30 minutes.
- Market Snapshot: 60 minutes.
- Pain Hypothesis: 30 minutes.

If a sample takes longer than the budget, the ICP is wrong or the sourcing system needs work — log it in `EXPERIMENT_LOG.md`.

## Rule
A Signal Sample that is not specific to the prospect is not a Signal Sample. It is a brochure, and brochures do not earn calls.
