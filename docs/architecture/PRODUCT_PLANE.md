# Hermes Product Plane

The product plane defines the *Packages* Dealix sells and the
*Playbooks* used to deliver them. The proposal factory, growth
engine, and partner program all read from the same registry — there is
no duplicated source of truth.

## Packages (initial shelf)

| Package | Buyer | Price band (SAR) |
| --- | --- | --- |
| Revenue Hunter Pilot | B2B founders + sales leaders | 7,500–25,000 |
| AI Trust Kit | Companies running AI without governance | 5,000–25,000 |
| Agency White-Label Kit | Agencies | 15,000–75,000 |
| Founder OS Setup | Founders | 8,000–18,000 |
| Market Radar Report | GMs / CMOs | 12,000–40,000 |
| Executive PMO Lite | Executive teams | 20,000–60,000 |
| MCP Risk Review | MCP server operators | 10,000–35,000 |

## Playbooks

Each package binds to a Playbook in `dealix.hermes.delivery`. A
playbook lists steps, owners, quality gates, approval gates, and
outcome metrics. Without an explicit playbook, a package cannot be
sold.
