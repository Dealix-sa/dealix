# Dealix V4 — خطة الإنتاج والنمو الحقيقي

## الفكرة
V3 جعل Dealix ماكينة إيراد داخلية. V4 يجعلها قابلة للتشغيل الخارجي مع عملاء فعليين: نشر، تتبع، CRM contracts، قياس، شراكات، Vertical packs، QA، وضبط مخاطر.

## أولويات V4
1. **Production Website:** صفحات الموقع يجب أن تقود إلى Diagnostic أو Book Call.
2. **Database Contract:** كل lead/opportunity/interaction له schema واضح.
3. **Observability:** كل يوم تعرف: من دخل؟ من تأهل؟ من يحتاج متابعة؟ ماذا بيع؟ ماذا تعطل؟
4. **Verticalized Selling:** التدريب، العقار، الوكالات، العيادات، الخدمات المهنية.
5. **Partner Channel:** مستشارين ومسوّقين وشركات تنفيذ يمكنهم إدخال leads مقابل referral.
6. **Delivery Governance:** قبل استلام أي عميل، يوجد checklist واضح ومخاطر SLA.
7. **Founder Data Room:** كل شيء جاهز لشريك، مستثمر، موظف أول، أو عميل Enterprise.

## المخرجات المطلوبة بعد الدمج
- موقع منشور.
- lead API مربوط بقاعدة بيانات أو ledger مؤقت.
- pipeline report يومي.
- عروض جاهزة حسب القطاع.
- pack لكل قطاع.
- financial forecast.
- security smoke داخل CI.
- data room فيه narrative وmetrics وassets.

## قاعدة القرار
أي ملف لا يساعد على أحد هذه الأربعة لا يدخل الأولوية: lead، sale، delivery، retention.
