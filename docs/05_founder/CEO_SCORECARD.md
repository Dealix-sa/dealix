# CEO Scorecard — بطاقة أداء المؤسس الأسبوعية — The Founder's Weekly Scorecard

> رقم واحد لا يكذب أهم من عشر صفحات تطمين. هذه الوثيقة هي **بطاقة الأداء القابلة للتسجيل**: هدف، فعلي، حالة — تُملأ مرة أسبوعيًا في مجلس المؤسس.
>
> القاعدة العليا — **The Supreme Rule:** أي عمل لا يجلب فرصة، أو يحوّلها، أو يسلّم قيمة، أو يثبت نتيجة، أو يوسّع داخل عميل — يُؤجَّل. كل مؤشّر هنا يخدم واحدة من هذه الخمس.

ربط: [`FOUNDER_DAILY_COMMAND`](FOUNDER_DAILY_COMMAND.md) · [`WEEKLY_BOARD_REVIEW`](WEEKLY_BOARD_REVIEW.md) · [`LAUNCH_GO_NO_GO`](LAUNCH_GO_NO_GO.md) · [`COMMAND_SPRINT_OFFER`](../01_go_to_market/COMMAND_SPRINT_OFFER.md) · [`PROOF_PACK_STANDARD`](../07_proof_os/PROOF_PACK_STANDARD.md) · [`APPROVAL_POLICY`](../05_governance_os/APPROVAL_POLICY.md)

**الإيقاع:** يُملأ أسبوعيًا. الأهداف أدناه هي **أهداف أول 30 يومًا (first-30-day targets)** ما لم يُذكر غير ذلك.

**مفتاح الحالة — Status key:** ✅ على الهدف · 🟡 قريب (≥70%) · 🔴 تحت الهدف (<70%) · ⛔ خرق ثقة — توقّف فوري.

---

## 1. مؤشّرات النموّ — Growth KPIs

> هذه مؤشّرات **حركة الفرص**: من مرشّح خام إلى تحويل مُدار. لا قيمة لرقم أعلى في القمة إذا انقطع في الأسفل — اقرأ العمود من فوق لتحت كـ**قمع (funnel)**.

| # | المؤشّر — KPI | الهدف (30 يومًا) — Target | الفعلي — Actual | الحالة — Status |
|---|---|---|---|---|
| G1 | Raw candidates — مرشّحون خام | ≥ 800 | | |
| G2 | Qualified — مؤهَّلون (Targeting Scorecard) | ≥ 100 | | |
| G3 | Manual messages — رسائل يدوية معتمدة | 30 | | |
| G4 | Replies — ردود | 5 – 10 | | |
| G5 | Diagnostic calls — مكالمات تشخيص | 3 – 6 | | |
| G6 | Paid Sprints — Sprints مدفوعة | 3 | | |
| G7 | Proof Packs — حِزم أدلة مُسلَّمة | 3 | | |
| G8 | Managed conversion — تحويل إلى علاقة مُدارة | 1 | | |

**قراءة القمع:** إذا كان G3 على الهدف لكن G4 تحته → مشكلة **رسالة/استهداف**. إذا كان G5 على الهدف لكن G6 تحته → مشكلة **عرض/سعر**. إذا كان G6 على الهدف لكن G7 تحته → مشكلة **تسليم** (خطر مباشر على بوابة الإطلاق).

**حواجز:** G3 يدوي فقط، بموافقة A2 مُسجَّلة، لا دفعة واحدة، لا واتساب جماعي. رقم G3 مرتفع بإرسال تلقائي = ⛔ لا ✅.

---

## 2. مؤشّرات الثقة — Trust KPIs

> هذه ليست مؤشّرات «أعلى أفضل». هذه **خطوط حمراء**. الهدف صفر، والفعلي يجب أن يطابق الهدف. أي رقم غير صفر = ⛔ + بند إلزامي في [`WEEKLY_BOARD_REVIEW`](WEEKLY_BOARD_REVIEW.md) (سجلّ القرار).

| # | المؤشّر — KPI | الهدف — Target | الفعلي — Actual | الحالة — Status |
|---|---|---|---|---|
| T1 | Auto-send incidents — حوادث إرسال تلقائي | 0 | | |
| T2 | Claims without evidence — ادعاءات بلا دليل | 0 | | |
| T3 | Customer files complete — اكتمال ملفات العملاء | 100% | | |
| T4 | Proof Packs delivered — تسليم الأدلة المستحقّة | 100% | | |
| T5 | Data misuse — سوء استخدام بيانات (تدريب/مشاركة) | 0 | | |
| T6 | Unauthorized publish — نشر اسم/حالة بلا إذن | 0 | | |

**قاعدة لا تُكسَر:** T1–T6 تُراجَع قبل أي مؤشّر آخر. بطاقة فيها خرق ثقة لا تُعتبر «أسبوعًا جيدًا» مهما كان النموّ. النموّ على حساب الثقة = دَين لا أصل.

---

## 3. مؤشّرات المنتج — Product KPIs

> هذه مؤشّرات **اتجاه (trend)** لا قيمة مطلقة: هل التسليم يصبح أرخص وأسرع وأكثر قابلية للتكرار مع كل Sprint؟ سجّل قيمة هذا الأسبوع والاتجاه عن الأسبوع الماضي (↓ هابط، → ثابت، ↑ صاعد).

| # | المؤشّر — KPI | الاتجاه المطلوب — Target trend | فعلي هذا الأسبوع | اتجاه | الحالة |
|---|---|---|---|---|---|
| P1 | Delivery time per Sprint — زمن التسليم لكل Sprint | ↓ هابط (ضمن 7-Day SLA) | | | |
| P2 | Template reuse — إعادة استخدام القوالب | ↑ صاعد | | | |
| P3 | Blockers — العوائق التشغيلية | ↓ هابط | | | |
| P4 | Manual work per client — العمل اليدوي لكل عميل | ↓ هابط | | | |
| P5 | Proof quality — جودة الأدلة (كل ادعاء بمصدر) | ↑ صاعد | | | |

**قراءة الاتجاه:** P1 و P4 يجب أن يهبطا معًا — إن هبط الزمن وارتفع العمل اليدوي فالجودة مهدّدة. P2 الصاعد هو ما يجعل السعر يرتفع بأمان لاحقًا (انظر سلّم السعر في [`COMMAND_SPRINT_OFFER`](../01_go_to_market/COMMAND_SPRINT_OFFER.md)).

## خلاصة البطاقة — Scorecard Summary (صفحة واحدة)

```text
الأسبوع رقم: ___            التاريخ: ___
النموّ — Growth:   على الهدف ___/8   |   تحت الهدف: ___
الثقة — Trust:    خروقات: ___ (يجب 0)   ← أي رقم > 0 = توقّف
المنتج — Product: اتجاهات إيجابية ___/5
أكبر فجوة هذا الأسبوع: ___
السبب الجذري: ___
تحرّك واحد لإغلاقها الأسبوع القادم: ___
أثر البطاقة على بوابة الإطلاق (Go/No-Go): ___
```

**قاعدة الإغلاق:** لا تُحفظ البطاقة بدون **فجوة واحدة + سبب جذري + تحرّك واحد**. رقم بلا سبب وبلا تحرّك = ملاحظة، لا قرار.
