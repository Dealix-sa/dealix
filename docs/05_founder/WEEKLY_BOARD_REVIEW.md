# Weekly Board Review

> **Status:** CANONICAL · **Owner:** Founder · **Last reviewed:** 2026-06-05

---

## 1. Cadence

Every Friday. 60–90 minutes. The founder reviews the week as if reporting to a
board, then sets the next 3 decisions.

## 2. Scorecard (fill weekly)

| Area | KPI | This week | Target |
|---|---|---|---|
| Sales | calls, offers, closes | | |
| Delivery | packs delivered | | |
| Proof | L2/L3 proof events | | |
| Product | bugs / blockers cleared | | |
| Finance | cash / MRR | | |
| Trust | approvals / claims clean | | |
| Founder | next 3 decisions set | | |

## 3. Review questions

- Did we move the wedge (Sprints sold, packs delivered)?
- Did any doctrine rule get bent? (auto-send, overclaim, undisclosed status)
- What is the single biggest blocker to the next paid Sprint?
- What proof did we capture that we can (with approval) reuse?
- What are the next 3 decisions?

## 4. Milestone gates to check

- 30 days: 3 paid Sprints, 3 Command Packs, 1 monthly client.
- 60 days: 5–7 paid Sprints, 1–2 managed clients, 3k–10k SAR MRR, 1 case study.
- 90 days: Limited public launch gate (see `BUSINESS_METRICS.md`).

## 5. Output

A short written record: scorecard + the next 3 decisions + the biggest blocker.
Feeds the next week's Founder Daily Command.
