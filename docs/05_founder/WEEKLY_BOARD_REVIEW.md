# Weekly Board Review — مجلس المؤسس الأسبوعي — The Founder Self-Board

> المؤسس واحد، لكن القرار يُدار كأنه أمام مجلس. مرة واحدة كل أسبوع، يجلس المؤسس على الجانبين: يقدّم الأرقام، ويستجوبها. هذه الوثيقة هي **جدول أعمال المجلس** + سجلّ القرار + سجلّ المخاطر.

ربط: [`FOUNDER_DAILY_COMMAND`](FOUNDER_DAILY_COMMAND.md) · [`CEO_SCORECARD`](CEO_SCORECARD.md) · [`LAUNCH_GO_NO_GO`](LAUNCH_GO_NO_GO.md) · [`COMMAND_SPRINT_OFFER`](../01_go_to_market/COMMAND_SPRINT_OFFER.md) · [`PROOF_PACK_STANDARD`](../07_proof_os/PROOF_PACK_STANDARD.md)

**الإيقاع:** مرة أسبوعيًا (مثلاً صباح الأحد)، 60 دقيقة، مغلق. المخرَج: 3 قرارات + 3 مخاطر + أولويات الأسبوع القادم.

### 2. الـ Sprints المدفوعة هذا الأسبوع — Paid Sprints
- العدد المدفوع هذا الأسبوع: ___
- السعر لكل واحد (من سلّم السعر، لا ارتجال): ___
- هل أي منها خارج الـ 7-Day SLA؟ نعم/لا → السبب.

### 3. Proof Packs المُسلَّمة — Proof Packs Delivered
- العدد المُسلَّم: ___ من ___ المستحقّة.
- نسبة التسليم: ___% (الهدف 100%).
- جودة الـ Proof: كل ادعاء مربوط بمصدر؟ نعم/لا.

### 4. MRR / التحويلات المُدارة — Managed Conversions
- اشتراكات/retainers مفعّلة: ___
- MRR التقريبي: ___ SAR (تقديري، ليس مُتحقَّقًا).
- تحويلات من Sprint إلى علاقة مُدارة هذا الأسبوع: ___

### 5. حوادث الثقة — Trust Incidents (يجب أن تكون 0)
| نوع الحادث | عدد | يجب |
|---|---|---|
| إرسال تلقائي / واتساب جماعي | | 0 |
| ادعاء بلا دليل | | 0 |
| استخدام بيانات عميل بلا إذن | | 0 |
| نشر اسم عميل بلا موافقة | | 0 |
| إجراء خارجي بلا موافقة مؤسس | | 0 |

> أي رقم غير صفر = توقّف فوري + بند إلزامي في سجلّ القرار.

### 6. تحسين القوالب والهامش — Template / Margin Improvements

- العمل اليدوي لكل عميل هبوطًا؟ نعم/لا.
