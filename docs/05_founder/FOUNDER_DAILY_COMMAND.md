# Founder Daily Command — Daily Operating Checklist — قيادة المؤسس اليومية

The founder's single daily checklist. It turns the daily rhythm into a fixed sequence of blocks so no day drifts. Read the Launch Control Tower first, then execute top to bottom. Nothing here invents numbers; every count comes from a real, sourced event.

قائمة المؤسس اليومية الواحدة. تحوّل الإيقاع اليومي إلى تسلسل ثابت من الكتل بحيث لا ينحرف أي يوم. اقرأ برج التحكم في الإطلاق أولاً ثم نفّذ من الأعلى للأسفل. لا شيء هنا يختلق أرقاماً؛ كل عدّاد يأتي من حدث حقيقي موثّق.

---

## 0. Read first — اقرأ أولاً (08:55, 5 min)

Open the Launch Control Tower before anything else: [`../00_platform_truth/LAUNCH_CONTROL_TOWER.md`](../00_platform_truth/LAUNCH_CONTROL_TOWER.md). It answers where we are, what gate we are at, and today's three moves. Confirm the live counts (Paid Sprints, Proof Packs, MRR) still match reality. If a count is stale, fix the Tower before you start the day — a lying dashboard is worse than none.

افتح برج التحكم في الإطلاق قبل أي شيء. يجيب: أين نحن، ما البوابة، ما حركات اليوم الثلاث. تأكّد أن الأعداد الحيّة (السبرنتات المدفوعة، حِزم الإثبات، الإيراد الشهري) ما زالت تطابق الواقع. إن كان أي عدّاد قديماً، أصلِح البرج قبل بدء اليوم — لوحة تكذب أسوأ من لا لوحة.

## 1. Morning intelligence + approval block — كتلة الاستخبارات والاعتماد الصباحية (08:00–11:00)

| Time | Action | الإجراء |
|---|---|---|
| 08:00 | Run the Market Intelligence OS pass | تشغيل تمريرة استخبارات السوق |
| 09:00 | Read the Founder Command Brief | قراءة موجز قيادة المؤسس |
| 09:30 | Review the top 20 targets | مراجعة أعلى 20 هدفاً |
| 10:00 | Approve exactly 5 outreach drafts | اعتماد 5 مسودات تواصل بالضبط |
| 11:00 | Send the 5 drafts manually | إرسال الـ5 مسودات يدوياً |

Approval rules, enforced every morning:
- Every draft must carry **evidence** before approval — 100% of drafts, no exceptions (KPI: drafts with evidence 100%).
- No draft is sent by any agent. The founder sends manually. **No external action without approval** (non-negotiable #8).
- No scraping, no cold WhatsApp automation, no LinkedIn automation feeds this block (non-negotiables #1, #2, #3).
- If a draft has a claim, the claim must be sourced or it is cut (non-negotiable #7).

قواعد الاعتماد، تُطبَّق كل صباح:
- كل مسودة تحمل **دليلاً** قبل الاعتماد — 100% من المسودات بلا استثناء.
- لا تُرسَل أي مسودة عبر وكيل. المؤسس يرسل يدوياً. **لا فعل خارجي بلا اعتماد**.
- لا كشط، ولا أتمتة واتساب بارد، ولا أتمتة لينكدإن تغذّي هذه الكتلة.
- أي ادّعاء بلا مصدر يُحذف.

## 2. Midday sales block — كتلة المبيعات منتصف اليوم (13:00–15:00)

13:00 is for diagnostics and calls. This is where replies become conversations and conversations become Free Diagnostics or paid Sprints. Pricing is never improvised — quote only from the Price Authority: [`../01_go_to_market/PRICE_AUTHORITY.md`](../01_go_to_market/PRICE_AUTHORITY.md). The Sprint launch price is 499–1,500 SAR; the 499 is a proof gate, not the company's price. Never frame any price with guaranteed revenue (non-negotiable #5). Never offer a monthly subscription before a Sprint is paid and a Command Pack delivered.

الساعة 13:00 للتشخيص والمكالمات. هنا تتحوّل الردود إلى محادثات، والمحادثات إلى تشخيص مجاني أو سبرنت مدفوع. التسعير لا يُرتجل — اقتبس فقط من مرجع التسعير. سعر إطلاق السبرنت 499–1,500 ريال؛ الـ499 بوابة إثبات وليست سعر الشركة. لا تربط أي سعر بضمان إيراد. لا تعرض اشتراكاً شهرياً قبل دفع سبرنت وتسليم Command Pack.

Daily sales targets feed the 30-day KPIs: toward 30 manual messages, 5–10 replies, 3–6 diagnostic calls across the month. Log every opportunity and objection in the Revenue OS.

أهداف المبيعات اليومية تغذّي مؤشرات الـ30 يوماً: نحو 30 رسالة يدوية، 5–10 ردود، 3–6 مكالمات تشخيص خلال الشهر. سجّل كل فرصة واعتراض في نظام الإيراد.

## 3. Afternoon delivery block — كتلة التسليم بعد الظهر (15:00–17:00)

15:00 is delivery work. Build inside the customer-folder template; a project with no Proof Pack is not a project (non-negotiable #10), and no project ships without a Capital Asset (non-negotiable #11). At 17:00, assemble the Proof Pack and update the customer's progress. Module status governs what you may promise — only the Command Sprint wedge is sellable now; everything else is BETA, INTERNAL, or FUTURE: [`../00_platform_truth/MODULE_STATUS_MAP.md`](../00_platform_truth/MODULE_STATUS_MAP.md).

15:00 لعمل التسليم. ابنِ داخل قالب مجلد العميل؛ المشروع بلا Proof Pack ليس مشروعاً، ولا يُسلَّم مشروع بلا أصل رأسمالي. عند 17:00 جمّع حزمة الإثبات وحدّث تقدّم العميل. حالة الأنظمة تحكم ما يجوز وعده — فقط إسفين سبرنت القيادة قابل للبيع الآن؛ ما عداه BETA أو INTERNAL أو FUTURE.

Delivery quality gates checked daily:
- Client folders complete — target 100%.
- Proof packs delivered for every paid job — target 100%.
- No PII written into logs (non-negotiable #6).

بوابات جودة التسليم تُفحص يومياً:
- اكتمال مجلدات العملاء — الهدف 100%.
- تسليم حِزم الإثبات لكل عمل مدفوع — الهدف 100%.
- لا بيانات شخصية في السجلّات.

## 4. Evening learning loop — حلقة التعلّم المسائية (18:00)

18:00 closes the day. Capture what was learned, then write tomorrow's plan: set the three moves the Launch Control Tower will show in the morning. Confirm the day's auto-send incidents are 0 and public claims without approval are 0. If either is non-zero, that is a red-line — stop and fix before planning anything new (see [`CEO_SCORECARD.md`](./CEO_SCORECARD.md)).

18:00 تُغلق اليوم. دوّن ما تعلّمته، ثم اكتب خطة الغد: حدّد الحركات الثلاث التي سيعرضها البرج صباحاً. تأكّد أن حوادث الإرسال التلقائي 0 والادّعاءات العامة بلا اعتماد 0. إن كان أيٌّ منهما غير صفر فهذا خط أحمر — أوقِف وأصلِح قبل التخطيط لأي جديد.

## Honest baseline — الأساس الصادق

Today the baseline is 0 paid Sprints, 0 Proof Packs, 0 MRR. That is correct for a pre-revenue private launch. The daily command exists to move those from 0 to real, sourced numbers — not to make them look bigger than they are.

اليوم الأساس 0 سبرنت مدفوع، 0 حزمة إثبات، 0 إيراد شهري. هذا صحيح لإطلاق خاص قبل الإيراد. القيادة اليومية موجودة لتحريك هذه الأرقام من الصفر إلى أرقام حقيقية موثّقة — لا لتضخيمها.
