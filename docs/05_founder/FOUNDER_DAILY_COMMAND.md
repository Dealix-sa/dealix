# Founder Daily Command — إيقاع المؤسس اليومي — The Founder's Daily Operating Rhythm

> Dealix شركة نظام تشغيل أعمال بالذكاء الاصطناعي. المؤسس واحد، يلعب كل الأدوار بمساعدة الوكلاء والقوالب. هذه الوثيقة هي **الساعة التشغيلية** — متى تفعل ماذا، وما القرار، وما الأثر (artifact).
>
> القاعدة العليا — **The Supreme Rule:** أي عمل لا يجلب فرصة، أو يحوّلها، أو يسلّم قيمة، أو يثبت نتيجة، أو يوسّع داخل عميل — يُؤجَّل. لا استثناء.

## سلّم الموافقة — The Approval Ladder (مفتاح القراءة)

المؤسس هو **بوابة الموافقة البشرية**. لا إجراء خارجي يغادر بدون موافقة مُسجَّلة.

| المستوى — Tier | المعنى | مَن يوافق |
|---|---|---|
| **A0** | داخلي بحت — مسودة، بحث، تجهيز | الوكيل تلقائيًا، لا حاجة لموافقة |
| **A1** | داخلي حسّاس — تسعير مسودة، نص عرض | مراجعة سريعة من المؤسس |
| **A2** | خارجي — أي رسالة/عرض/ملف يغادر إلى عميل | **موافقة صريحة من المؤسس، مُسجَّلة في Approval Register** |
| **A3** | حسّاس استراتيجيًا — نشر اسم عميل، حالة دراسية، تغيير سعر دائم | موافقة المؤسس + موافقة العميل المكتوبة |

**كل ما هو A2+ يمرّ عبر طابور الموافقة أدناه. لا واتساب جماعي، لا إرسال تلقائي، لا ادعاء بلا دليل.**

## الساعة التشغيلية — The Daily Timetable

لكل بلوك: **المُدخَل (Input)** → **القرار (Decision)** → **الأثر (Artifact)**.

### 08:00 — Market Intelligence
- **Input:** إشارات السوق منذ الأمس (قطاعات الأسبوع، حركة حسابات مستهدفة، أخبار قطاعية عامة من مصادر مسموح بها).
- **Decision:** ما القطاع/الحساب الذي يستحق انتباه اليوم؟
- **Artifact:** قائمة Market Signals مُحدَّثة (10–20 سطرًا، كل سطر بمصدر).

### 09:00 — Founder Brief
- **Input:** Market Signals + حالة الـ pipeline من أمس + طابور الموافقة.
- **Decision:** ما أهم 3 تحرّكات اليوم؟
- **Artifact:** Founder Brief — صفحة واحدة: الوضع، الأولويات الثلاث، المخاطر.

### 09:30 — Review Top 20
- **Input:** أعلى 20 مرشّحًا في الـ pipeline حسب التأهيل (Targeting Scorecard).
- **Decision:** مَن يتقدّم، مَن يُسقَط، مَن ينتظر دليلًا إضافيًا.
- **Artifact:** Top-20 محدّثة بالحالة والخطوة التالية لكل سطر.

### 10:00 — Approve 5 Drafts
- **Input:** 5 مسودات تواصل/عرض جهّزها الوكيل (A0) ليلًا.
- **Decision:** موافقة / تعديل / رفض — هذه نقطة A2.
- **Artifact:** 5 سطور في **Approval Register** (مَن، متى، نسخة الرسالة، الحساب) — لا شيء يغادر قبل هذا.

### 11:00 — Manual Outreach
- **Input:** المسودات المعتمدة فقط.
- **Decision:** إرسال **يدوي** قناة بقناة، بموافقة، بسقف يومي (هدف ~30 رسالة يدوية موزّعة، لا دفعة واحدة).
- **Artifact:** Outreach Log — كل رسالة: الحساب، القناة، الوقت، حالة الموافقة.

### 13:00 — Diagnostics
- **Input:** الردود الواردة (هدف 5–10 ردود) + الحسابات التي حجزت مكالمة.
- **Decision:** أي رد يتحوّل إلى مكالمة تشخيص (هدف 3–6 مكالمات)؟
- **Artifact:** Diagnostic Notes لكل مكالمة + ملف العميل (Customer File) مفتوح.

### 15:00 — Offers / Proposals
- **Input:** Diagnostic Notes.
- **Decision:** أي تشخيص يستحق عرض Command Sprint؟ السعر من سلّم السعر، لا ارتجال.
- **Artifact:** مسودة عرض (A1) → تذهب لطابور الموافقة قبل الإرسال (A2).

### 17:00 — Delivery / Proof
- **Input:** الـ Sprints المدفوعة الجارية + موعد التسليم في الـ 7-Day SLA.
- **Decision:** أين نحن من الـ Command Pack؟ ما الذي يحتاج مراجعة المؤسس قبل التسليم؟
- **Artifact:** تقدّم Proof Pack + Approval Register للتسليم الخارجي (A2).

### 18:00 — Learning Loop
- **Input:** اعتراضات اليوم، ردود فعل العملاء، ما تعطّل.
- **Decision:** أي قالب يتحسّن؟ أي اعتراض يُضاف لبنك الاعتراضات؟
- **Artifact:** Learning Log — اعتراض مُسجَّل + قالب واحد محسّن (انظر أسفل).

### حواجز يومية ثابتة — Daily Guardrails
- لا إرسال تلقائي. لا واتساب جماعي. لا scraping خلف تسجيل دخول.
- لا ادعاء بلا دليل في أي مخرج.
- لا بيانات عميل لتدريب أي نموذج.
- لا نشر اسم عميل بلا موافقة مكتوبة (A3).
- لا إجراء خارجي يلمس عميلًا بدون موافقة مؤسس مُسجَّلة.
