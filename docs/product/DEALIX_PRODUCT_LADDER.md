# Dealix Product Ladder

> Seven rungs. Each rung has positioning, promise, deliverables, proof required, pricing guardrails, objections, sales script, upsell path.

## Rung 1 — Free Sample / Diagnostic

| Field | Value |
|---|---|
| Positioning | A real diagnostic of a real motion, in days, free |
| Audience | First-touch ICP-fit accounts |
| Promise | A signal-rich, evidence-bound diagnostic delivered within the sample time-box |
| Deliverables | One-page diagnostic; 3 sample drafts; founder review |
| Proof required | Sector signal; brand-pass |
| Price | Free |
| Objections | "What's the catch?", "How are you free?" |
| Sales script | Diagnostic as proof of method, not a hook for upsell |
| Upsell path | Rung 2 |

## Rung 2 — Revenue Sprint

| Field | Value |
|---|---|
| Positioning | Two weeks of evidence-led work on one revenue lever |
| Audience | Founders who want proof of method |
| Promise | A productised sprint that ships a real artifact and a real learning |
| Deliverables | One sprint plan; one playable lever; weekly operating reviews; closing brief |
| Proof required | Customer consent for proof; QA-pass artifacts |
| Price | Within Sprint guardrail (see `PRICING_GUARDRAILS.md`) |
| Objections | "Two weeks is short", "We've done sprints before" |
| Sales script | Be specific about the lever; map to the customer's bottleneck |
| Upsell path | Rung 3 or Rung 4 |

## Rung 3 — Managed Pilot

| Field | Value |
|---|---|
| Positioning | Six-week operated pilot with full QA |
| Audience | Teams ready to operate the cadence |
| Promise | Operating cadence + drafts + reviews against a single defined outcome |
| Deliverables | Operating cadence; drafts; reviews; closing report |
| Proof required | Outcome + consent |
| Price | Within Pilot guardrail |
| Objections | "Six weeks is long", "We can do this ourselves" |
| Sales script | Map the cadence to their bottleneck; show audit posture |
| Upsell path | Rung 4 |

## Rung 4 — Revenue Desk Retainer

| Field | Value |
|---|---|
| Positioning | Monthly retainer operating Distribution + Delivery |
| Audience | Operating customers |
| Promise | Trust-gated, founder-approved revenue motion on a monthly cadence |
| Deliverables | Monthly operating plan; weekly drafts; monthly review; quarterly business review |
| Proof required | NRR signal; customer green health |
| Price | Within Retainer guardrail |
| Objections | "Retainer is recurring", "We need predictability" |
| Sales script | Predictability comes from cadence, not promises |
| Upsell path | Rung 5 |

## Rung 5 — Founder Console / Command Center

| Field | Value |
|---|---|
| Positioning | Console + retainer hybrid for founder-led teams |
| Audience | Hands-on founders |
| Promise | Operate the company from one screen, with the founder at the centre |
| Deliverables | Console access; retainer; founder operating cadence |
| Proof required | Sustained green health + outcomes |
| Price | Within Command Center guardrail |
| Objections | "How is this different from a CRM?" |
| Sales script | Reframe to operating system; map surfaces |
| Upsell path | Rung 6 |

## Rung 6 — Enterprise Revenue Intelligence OS

| Field | Value |
|---|---|
| Positioning | Multi-tenant deployment with sovereign data |
| Audience | Larger orgs |
| Promise | Operating system at scale with KSA data sovereignty |
| Deliverables | Tenant deployment; SSO; sovereign storage; audit export; SLA |
| Proof required | Two reference customers in same sector / size |
| Price | Custom within Enterprise guardrail |
| Objections | "Procurement, security, sovereignty" |
| Sales script | Bring Trust Center and Sovereign first |
| Upsell path | Rung 7 |

## Rung 7 — Partner / White-label Revenue OS

| Field | Value |
|---|---|
| Positioning | Operate Dealix under a partner brand |
| Audience | Agencies, system integrators |
| Promise | Partner can deliver Revenue OS work under their own brand |
| Deliverables | Tenant; partner brand pack; partner enablement; co-sell motion |
| Proof required | Partner credential check |
| Price | Within Partner guardrail |
| Objections | "Margin", "Conflict with our team" |
| Sales script | Show the partner economics; show co-sell rules |
| Upsell path | Multi-tenant expansion within partner |

## Cross-rung rules

- **No guaranteed-result claims on any rung.**
- **No price commitments without founder approval** (non-negotiable #4).
- Each rung is independently sellable; ladders are guidance not gatekeeping.
