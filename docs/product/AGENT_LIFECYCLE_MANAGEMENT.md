# Agent Lifecycle Management

Every agent must go through:

1. Proposed  
2. Designed  
3. Simulated  
4. Evaluated  
5. Approved  
6. Monitored  
7. Improved  
8. Retired  

## No agent can be used in client delivery unless:

- agent card exists  
- autonomy level defined  
- allowed tools defined  
- evals pass  
- governance checks exist  
- owner assigned  

## Agent Retirement

Retire or disable an agent if:

- it is unused  
- errors repeat  
- scope is exceeded  
- risk outweighs value  
- a better workflow replaces it
