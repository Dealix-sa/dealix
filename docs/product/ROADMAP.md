# Roadmap

> A roadmap is a short list of things we will actually do this quarter.
> Not a wish-list.

## Q1 — Foundation (months 1–3)

### Build
- Revenue Sprint delivery kit (templates, QA checklist, handoff)
- Private revenue ledgers (`cash_collected.csv`, `pipeline_value.csv`)
- Approval queue + per-send token mechanism for A3 agents
- Doctrine verifier in CI
- Daily / weekly / audit CLI commands (`make daily`, `make weekly-close`)

### Defer
- Self-serve customer dashboard
- Multi-language report PDFs beyond AR + EN
- CRM integrations beyond manual export
- Advanced moat-asset registry tooling

### Kill
- Marketplace
- SaaS product
- Enterprise RFP motion
- Multi-sector expansion beyond 3 sectors

## Q2 — Repeatability (months 4–6)

### Build
- 1–2 templates promoted to automation (after Level 3 evidence)
- Weekly retainer report generator
- Capital asset registry (counts of approved case studies, etc.)
- Cost cap monitoring per AI pipeline

### Defer
- Anything Q1 deferred that did not earn evidence

### Kill
- Any tool not used weekly by us or a customer

## Q3 — Compounding (months 7–9)

### Build
- 2 sector-specific outreach packs (when sector has ≥ 3 sprints)
- Friction-log → policy automation
- Public proof site (with consent-bound case studies)

## Q4 — Optionality (months 10–12)

### Build
- SaaS readiness memo (only if Level 3 + 3 unprompted asks)
- Partner program (only if proof base supports it)

### Reassessed
- Whole roadmap re-cut based on year-1 evidence

## Roadmap Discipline

- Items move from Defer to Build only with named evidence.
- Items move from Build to Kill if effort or scope drifts.
- Roadmap is reviewed in the Monthly Strategy Review.
