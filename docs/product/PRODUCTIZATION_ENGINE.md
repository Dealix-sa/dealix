# Productization Engine

Anything done 3 times turns into a system.

This rule prevents two failure modes at once:

1. Building SaaS before the market validates it.
2. Staying manual forever.

## Rule
- If a manual workflow succeeds **3 times**, document it.
- If it succeeds **5 times**, template it.
- If it succeeds **10 times**, automate it.
- If customers ask for it repeatedly, consider productizing it.

## Stages

1. Manual
2. Documented
3. Template
4. Semi-automated
5. Productized
6. SaaS / OS feature

## Examples

| Workflow             | Current Stage | Next Step       |
|----------------------|---------------|-----------------|
| Lead scoring         | Template      | Semi-automate   |
| Outreach pack        | Manual        | Document        |
| Delivery report      | Template      | Generate        |
| Weekly client report | Manual        | Template        |

## Anti-pattern

Productizing a workflow before stage 4 is how teams burn engineering
time on features no customer paid for. Productizing later than stage 6
is how founders stay bottlenecks forever.
