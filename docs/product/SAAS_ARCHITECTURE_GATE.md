# SaaS Architecture Gate

## Purpose
Prevent Dealix from building SaaS architecture before repeated paid workflow evidence.

## Required Before SaaS Architecture
- 3 paid Revenue Sprints.
- 2 completed deliveries.
- 1 retainer attempt.
- 10 repeated workflow executions.
- stable data model.
- stable QA criteria.
- clear dashboard demand.
- trust boundary tested.
- support burden understood.

## If Gate Passes
Allowed to design:
- apps/web
- api
- db
- auth
- billing
- workers
- customer portal
- admin dashboard

## If Gate Fails
Continue:
- CLI
- scripts
- private ops
- local dashboard
- internal automation
- templates

## Rule
SaaS architecture starts only after workflow evidence, not before.
