# Feature Intake

Every feature request runs through this gate.

## Template

```
## Feature
[Name]

## Customer Evidence
Who asked for it? Link to the conversation or the recurring pattern.

## Revenue Impact
How does it support cash collected, MRR, retention, or founder leverage?

## Risk
What could go wrong? Any Claim Guard, suppression, or approval implications?

## Decision
Build / Fix / Kill / Defer.

## Owner
Who owns this decision and the work if approved.

## Review Date
YYYY-MM-DD
```

## Routing
- A feature without customer evidence or revenue impact defaults to **Defer**.
- A feature with one prospect but no payment defaults to **Defer**.
- A feature with regulatory implications goes to **Escalate**.

## Rule
The intake is the bouncer. Many great ideas wait in line — only the right ones get in tonight.
