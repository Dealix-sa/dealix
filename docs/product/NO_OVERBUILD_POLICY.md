# No Overbuild Policy

## Purpose
Prevent Dealix from building complex product layers before market evidence.

## Owner
Sami / Founder CEO.

## Review Cadence
Weekly.

## Overbuild Signals
- building SaaS before paid sprint.
- adding agents before manual workflow works.
- improving dashboard before daily execution is consistent.
- building investor material before revenue proof.
- adding features without customer request.
- automating workflow not repeated manually.

## Rules
- Revenue proof before product expansion.
- Manual before automation.
- Template before software.
- Customer evidence before feature.
- Trust control before external automation.
- Delivery proof before scale.

## Allowed Before First Payment
- founder-led outbound.
- samples.
- proposals.
- payment path.
- delivery templates.
- trust checks.
- daily/weekly CLI.

## Not Allowed Before First Payment
- multi-tenant SaaS.
- complex autonomous agents.
- investor data room.
- white-label.
- mobile app.
- advanced admin panel.

## Metrics
- deferred builds.
- killed features.
- revenue-linked builds.
- customer-requested features.

## Evidence
- Go/No-Go log.
- feature intake.
- weekly review.
- customer evidence.
