# Service → Product Path

How a **manual service** graduates into **repeatable product** without chaos. Example: Lead Intelligence.

## Lead Intelligence

### Stage 1 — Manual service

- Spreadsheets + manual checks
- Manual scoring rules
- Static report template

### Stage 2 — Assisted service

- **Import preview** + validation checklist
- Scripted or notebook scoring; human sign-off
- Templated proof pack sections

### Stage 3 — Internal tool

- Operator dashboard (internal-only)
- Reusable scoring module + tests
- Proof pack assembler / reporting hooks integrated with ledgers

### Stage 4 — Client feature

- Client upload / preview (governed)
- Scores + rationale surfaced; drafts **approval-first**

### Stage 5 — SaaS module

- Recurring workspace, monthly scoring cadence
- Integration adapters (CRM, etc.) behind contracts
- RBAC + audit per [`API_GOVERNANCE.md`](API_GOVERNANCE.md)

### Stage 6 — Enterprise workflow

- Multi-team permissions, SLA, data residency decisions — only after [`../enterprise/ENTERPRISE_DECISION.md`](../enterprise/ENTERPRISE_DECISION.md)

## Rule

Advance **one stage** only when prior stage has **ledger proof** (Delivery + Proof + Learning ledgers).

Decisions: [`PRODUCTIZATION_DECISION.md`](PRODUCTIZATION_DECISION.md), [`BUILD_DECISION.md`](BUILD_DECISION.md).
