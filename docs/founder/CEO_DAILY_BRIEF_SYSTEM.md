# CEO Daily Brief System

How the founder's morning brief is built, who owns each input, and what
"good" looks like.

## Purpose

Compress the company's state into a single, scrollable page the founder
reads before 07:30 — so the first decisions of the day are made on
evidence, not vibe.

## Owner

Founder. The brief is generated by `scripts/dealix_founder_daily_brief.py`
on a cron defined in the machine registry.

## Cadence

Daily at 07:00 Asia/Riyadh. The brief is *generated*; sending it externally
is forbidden — it lives only in the founder's local data folder and the
ops surface at `/ops/founder`.

## Source of Truth

| Section | Source |
| --- | --- |
| Pipeline | `data/commercial/pipeline_today.json` |
| Approvals queued | `approval_requests` table → `/ops/approvals` |
| Active machines | `registries/machine_registry.yaml` |
| Risk register top 3 | `docs/governance/RISK_REGISTER.md` |
| Yesterday's wins/losses | `data/founder_briefs/<yesterday>.md` |

## Inputs

- Pipeline snapshot (read-only)
- Approval queue depth (read-only)
- Drift report from `scripts/verify_everything.py`

## Outputs

- `data/founder_briefs/<date>.md` — the brief itself
- A short "top 3" block surfaced at `/ops/founder`
- Audit event recorded in the audit log

## KPI

- Brief produced ≤ 07:30 local
- All sections populated (none "no data" if sources are healthy)
- < 1 minute generation time

## Trust Boundary

Read-only. The brief generator may not call any external API. If it
needs an LLM, it uses the local mock provider unless explicitly run
with `DEALIX_USE_LIVE_LLM=1` by the founder.

## Failure Mode

- Empty sections → governance auditor flags within 24h.
- Brief skipped two days in a row → CEO OS failure mode.
- LLM returning banned claims → caught by
  `scripts/verify_prompt_output_quality.py`.

## Recovery Path

1. Re-run `make v5-digest` locally.
2. If a source is empty, fix the source — never paper over with hand
   typed numbers.
3. Log the recovery as a decision in `CEO_DECISION_LOG.md`.

## Verification

```bash
make business-os
python scripts/verify_everything.py --layer ceo_operating_system
```

## Next Action

Read today's brief. Pick the top three. Move them into the day's
calendar. Anything not in those three is, by definition, not today's job.
