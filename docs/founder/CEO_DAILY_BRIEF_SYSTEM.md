# CEO Daily Brief — System Spec

A single morning markdown file that answers seven questions in under 30
minutes of reading. Generated by
[`scripts/founder_ceo_daily_brief.py`](../../scripts/founder_ceo_daily_brief.py)
into `data/founder_briefs/ceo_daily_brief_<date>.md`.

## The seven questions

1. **What is the most important decision today?** — top item from PRIVATE_OPS `ceo/decisions.jsonl` flagged `pending`
2. **Where is the cash?** — sum of paid invoices in the last 7 days from existing payment events
3. **Where are we stuck?** — top 3 from [`scripts/dealix_bottleneck_radar.py`](../../scripts/dealix_bottleneck_radar.py) output (`docs/metrics/bottlenecks_recent.csv`)
4. **What am I personally on the hook for?** — open items from PRIVATE_OPS `ceo/decisions.jsonl` where `owner == "ceo"`
5. **What can I delegate today?** — items aged > 3 days in (4) (see [DELEGATION_DECISION_TREE](DELEGATION_DECISION_TREE.md))
6. **What are we doubling down on?** — sector / channel / offer that gained signal yesterday (from [../strategy/BEACHHEAD_SECTOR_SCORECARD](../strategy/BEACHHEAD_SECTOR_SCORECARD.md))
7. **What are we stopping?** — items that have not moved in 14 days

## Inputs

| Question | Source | PRIVATE_OPS? |
|---|---|---|
| 1, 4, 5 | `ceo/decisions.jsonl` | yes |
| 2 | proof ledger via existing `auto_client_acquisition.payment_ops` | no (in repo modules) |
| 3 | `scripts/dealix_bottleneck_radar.py` output | no |
| 6 | `docs/ops/pipeline_tracker.csv` + `docs/commercial/operations/evidence_events_tracker.csv` | no |
| 7 | Any task ledger row aged > 14 days | mixed |

When PRIVATE_OPS is not configured the brief still generates with the
non-sensitive sections populated and a clear banner: "Sensitive sections
skipped — PRIVATE_OPS not configured." See
[`dealix/private_ops.py`](../../dealix/private_ops.py) `missing_private_ops_note()`.

## Cadence

- Generated automatically at 06:00 KSA via [`founder_ceo_hypergrowth.yml`](../../.github/workflows/founder_ceo_hypergrowth.yml)
- Re-runnable any time with `make hyper-daily-brief`

## Read by

- The CEO (you)
- The frontend `CeoOsHub` component via `/api/v1/founder/ceo-os/daily-brief`

## Non-negotiables

The brief is read-only. It does not send messages. It does not call external
APIs beyond the documented integrations already running. See
[DO_NOT_SAY](DO_NOT_SAY.md).
