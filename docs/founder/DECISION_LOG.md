# Decision Log

Any decision that changes price, product, contracts, public messaging, data handling, or strategic priorities is logged here.

## Template

```
## Decision ID
DLX-DEC-0001

## Date
YYYY-MM-DD

## Decision
[Build / Fix / Kill / Defer / Approve / Reject / Escalate]

## Context
Why this decision exists.

## Options
1. Option A
2. Option B
3. Option C

## Chosen Option
[Selected option]

## Reason
Why this is the best decision.

## Expected Impact
Revenue / Delivery / Trust / Product / Founder leverage.

## Risk
Low / Medium / High

## Review Date
YYYY-MM-DD
```

## Rules
- One decision per entry.
- Decisions are append-only. If a decision is changed, log a new one that references the old one.
- The founder is the only person who can mark a decision as `APPROVE` at level A2 or A3 (see `docs/trust/APPROVAL_MATRIX.md`).
