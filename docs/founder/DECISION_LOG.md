# CEO Decision Log

> Every decision the founder makes that changes strategy, money, headcount,
> trust posture, or client-facing claims is recorded here. No exceptions.

## Format

Each entry uses the columns below. Append-only. Never edit a past entry —
add a new row with a reference if a decision is reversed.

| Date | Decision | Type | Reason | Owner | Outcome / Status |
|---|---|---|---|---|---|

## Decision Types

- **Build** — new system, feature, or process.
- **Fix** — broken system that must be repaired before new work.
- **Kill** — stop doing something that is not earning its keep.
- **Defer** — postpone to a defined later trigger.
- **Approve** — clear an A2 / A3 action through the Trust gate.
- **Reject** — block an action that violates doctrine or approval policy.

## Rules

1. Every A2 / A3 approval listed in `docs/trust/APPROVAL_MATRIX.md` requires a row here.
2. Reversals reference the original entry by date.
3. Outcomes are filled in within 14 days of the decision.
