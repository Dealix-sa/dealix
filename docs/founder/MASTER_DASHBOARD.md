# Master CEO Dashboard

> One screen. The state of the company.
> Generated by `make dashboard`.

## Panels

### 1. Revenue
- Cash collected (MTD)
- Cash expected (next 30 days)
- MRR
- Pipeline value (weighted)
- Active retainers
- Active Sprints

### 2. Pipeline
- DMs sent (week / target)
- Replies (week)
- Samples sent (week)
- Calls booked (week)
- Proposals sent (week)
- Pipeline by stage (counts)
- Days-in-stage outliers (deals stuck)

### 3. Trust
- Overclaim incidents (week, month)
- Approval queue depth
- Open risks
- Last audit date
- Last incident drill date

### 4. Delivery
- Sprints in flight
- Sprint cycle time (last 5)
- QA fail rate
- Approved Proof Packs (month)
- Retainer ask rate

### 5. Founder Focus
- Hours on revenue (today / week)
- Hours on admin (today / week)
- Bottlenecks open
- Decisions pending
- Approvals pending

### 6. Decision Queue
- Open decisions and their evidence-needed-by dates

### 7. Bottleneck Heatmap
- One row per system, 🟢/🟡/🔴

### 8. Weekly Learning
- One sentence from the last weekly review
- One sentence: what changes today because of it

### 9. Cash / Runway
- Cash on hand
- Net burn (last month)
- Runway (months)

### 10. Stage Readiness
- Workflows at Level 1, 2, 3 counts
- Workflows promoted this month
- Capital assets added this month

## The Single Question

```
> What must Sami do today to move Dealix closer to cash, proof, or
> retention?
```

The dashboard answers it explicitly: one sentence at the top of the
rendered output, derived from the highest-impact open action in the
pipeline + the highest-leverage moat investment available today.

## Refresh

`make dashboard` runs read-only against:

- `dealix-ops-private/revenue/*.csv`
- `dealix-ops-private/client_success/*.csv`
- `dealix-ops-private/founder/decision_queue.md`
- `dealix-ops-private/founder/approvals_waiting.md`
- `dealix-ops-private/trust/risk_log.md`
- `dealix-ops-private/finance/*.csv`

Output: stdout summary + optional file at
`dealix-ops-private/founder/master_dashboard.md`.

## Frequency

- Daily, first thing.
- Recomputed at end of day after `close-day`.

## Discipline

- A dashboard panel that nobody reads weekly is removed.
- A dashboard panel that triggers no decision is removed.
- Beauty is not a feature; clarity is.
