# CEO Operating System

## Purpose
Dealix runs on a system, not on the founder's mood. The CEO Operating System
is the single source of truth for how the founder spends time, makes
decisions, approves risky moves, and reviews company health.

## The Five CEO Loops

1. **Brief Loop** — Read the Daily Command Brief before any other work.
2. **Decide Loop** — Make the one decision that moves cash, proof, or trust.
3. **Approve Loop** — Clear the approval queue (A2/A3 items).
4. **Focus Loop** — Kill or defer anything that does not feed the loop above.
5. **Review Loop** — Run the Weekly CEO Review every Friday.

## Daily Cadence

| Time | Action | Output |
|---|---|---|
| 08:00 | Read Daily Command Brief | One CEO decision identified |
| 09:00 | Clear approval queue | Approved / rejected with reason in DECISION_LOG |
| 10:00 | Highest-leverage action | Cash, proof, or retainer outcome |
| 17:00 | End-of-day note | Result recorded in Daily Command Brief |

## Weekly Cadence

| Day | Action |
|---|---|
| Monday | Set operating theme (one sentence) |
| Wednesday | Mid-week pipeline check |
| Friday | Weekly CEO Review + scorecard update |

## CEO Anti-Patterns

The CEO does NOT:
- Start the day in the codebase.
- Build features without a revenue, delivery, or trust reason.
- Approve outbound or proposals informally — every A2/A3 item goes through DECISION_LOG.
- Add new offers before the previous offer has evidence.
- Carry context in their head — everything lives in `docs/founder/` or `docs/evidence/`.

## Hard Rules

1. No decision without a recorded reason.
2. No approval without an entry in DECISION_LOG.md.
3. No new system until the previous system has a verification script.
4. No claim made publicly without evidence on file.
5. The Daily Command Brief is created every working day, even if short.

## Companion Files

- `DAILY_COMMAND_BRIEF.md` — today's brief template + most recent entry.
- `WEEKLY_CEO_REVIEW.md` — weekly review template + history.
- `DECISION_LOG.md` — every CEO decision with reason and outcome.
- `RISK_REGISTER.md` — every active risk with owner and mitigation.
- `FOCUS_POLICY.md` — what we do, what we kill, what we defer.
- `COMPANY_HEALTH_SCORE.md` — rolling score across the 12 company systems.
- `CEO_DASHBOARD_SPEC.md` — what the CEO dashboard must show.
