# CEO Operating System

The single page the founder opens first every morning. If this document is
out of date, the operating system is out of date.

## Purpose

Make sure the founder decides on three things every day — revenue, risk,
roadmap — with the same inputs and the same cadence, so the company
compounds instead of drifting.

## Owner

Founder. No delegation. If the founder is unavailable for > 48h, the
acting-CEO clause in `docs/governance/HUMAN_IN_THE_LOOP_MATRIX.md`
takes over.

## Cadence

| Loop | Frequency | Output |
| --- | --- | --- |
| Daily brief | 07:00 Asia/Riyadh | `data/founder_briefs/<date>.md` |
| Weekly review | Sunday 09:00 | `docs/founder/CEO_WEEKLY_REVIEW.md` (rolling) |
| Decision log entry | per decision | `docs/founder/CEO_DECISION_LOG.md` |
| Capital allocation review | monthly | `docs/founder/CAPITAL_ALLOCATION_SYSTEM.md` |
| Quarterly OKR check | quarterly | linked OKR doc |

## Source of Truth

- Revenue: `data/commercial/pipeline_today.json` (built by `scripts/dealix_founder_daily_brief.py`)
- Risk: `docs/governance/RISK_REGISTER.md`
- Roadmap: `docs/ops/FOUNDER_OPERATING_SYSTEM_AR.md`

The CEO Operating System never invents numbers. If a source is empty, the
brief says "no data" rather than guessing.

## Inputs

- KPI dashboard snapshot
- Approval queue depth (`/ops/approvals`)
- War room signals from `docs/ops/DEALIX_REVENUE_WAR_ROOM_AR.md`
- Friction log (yesterday's blockers)

## Outputs

- Today's top three actions (max three — anything more is a planning failure)
- Decisions made (recorded in `CEO_DECISION_LOG.md`)
- One escalation, if any, into the war room

## KPI

- Daily brief generated by 07:30 (5/5 weekdays)
- ≥ 1 decision logged per day on operating days
- 0 days without a brief during a working week

## Trust Boundary

The CEO OS reads from internal systems only. It never publishes external
content and never sends customer messages. External actions require the
gate in `docs/trust/LIVE_SEND_SAFETY_GATE.md`.

## Failure Mode

- No brief generated → cron / verify_everything will flag
  `governance_auditor` machine failure within 24h.
- Brief generated but empty → `verify_business_os.py` fails CI.
- Decision log not updated for > 7 days → weekly review surfaces the gap.

## Recovery Path

1. Run `python scripts/dealix_founder_daily_brief.py` locally.
2. Inspect KPI sources; backfill missing inputs.
3. If KPI source is structurally broken, file a runbook entry under
   `docs/ops/DEALIX_MISSING_SYSTEMS.md` and create a fix branch.
4. Resume cadence the next morning — never skip two days in a row.

## Verification

Run:

```bash
make audit
make business-os
```

A missing section or empty file fails `verify_business_os.py`.

## Next Action

Open `data/founder_briefs/<today>.md`. Pick today's top three. Log every
decision in `CEO_DECISION_LOG.md` before the day closes.
