# Dealix CEO Operating System

> One operating system, twelve subsystems, one daily question.

## Purpose

Move Dealix from "execution engine" to a real company:
clear offer, daily sales, controlled delivery, high trust,
financial discipline, weekly learning, governance that
prevents random expansion.

## Universal System Contract

Every system **must** define:

1. **Owner** — single accountable person (today: Founder)
2. **Inputs** — what the system consumes
3. **Outputs** — what the system produces
4. **Rules** — non-negotiables that govern the system
5. **Metrics** — the numbers that prove the system is alive
6. **Evidence** — where proof lives (file, ledger, dashboard)
7. **Verifier** — script or human check that the system is honest
8. **Review cadence** — daily / weekly / monthly

> A system without Metrics and Evidence is not a system. It is decoration.

Strategy ───► Revenue ───► Sales ───► Delivery ───► Client Success ───► Productization
   │            │           │            │                │                     │

Trust & AI Governance sits **above** every system, not beside.

## Daily Operating Loop

1. `make daily` — open `DAILY_COMMAND_BRIEF.md`
2. Move the highest-leverage revenue action **first**
3. Update `revenue_action_log.csv`
4. Record one decision in `decision_queue.md`
5. `make close-day` — write evening evidence

## Weekly Operating Loop

1. `make weekly-close` — score the week
2. Decide: **Build / Fix / Kill / Defer / Continue**
3. Update `KILL_LIST.md` with anything killed or deferred
4. Update `90_DAY_PLAN.md` only if evidence demands it
5. Publish one external proof artifact (LinkedIn / case study / sample)

## Monthly Operating Loop

1. `MONTHLY_STRATEGY_REVIEW.md` — what changed in the thesis?
2. `MONTHLY_MOAT_REVIEW.md` — did the moat get deeper?
3. `BOARD_PACK_TEMPLATE.md` — even if board = self, write it
4. Update Capital Allocation priorities

## The Founder's Five Daily Questions

1. What is the highest-leverage **revenue** action today?
2. What evidence will I produce by end of day?
3. What am I refusing to do today (Kill List discipline)?
4. What decision needs to be made today, and on what evidence?
5. What did I learn yesterday that changes today?

## Non-Negotiables

- Start the day from `daily_brief`, not GitHub
- No bad revenue for cash
- No feature without revenue/delivery/trust/learning reason
- No important decision without evidence
- One weekly decision: Build / Fix / Kill / Defer / Continue
