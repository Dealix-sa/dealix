# Go / No-Go Decision System

## Purpose
Make stage advancement and major build decisions evidence-based.

## Owner
Sami / Founder CEO.

## Review Cadence
Weekly for sprint decisions, monthly for strategic decisions.

## Inputs
- stage roadmap.
- operating readiness level.
- revenue evidence.
- delivery evidence.
- trust risk.
- founder capacity.
- customer demand.

## Outputs
- Go.
- No-Go.
- Defer.
- Kill.
- Escalate.

## Go Criteria
A decision is Go only if:
- it supports revenue, delivery, trust, learning, or founder leverage.
- evidence exists.
- risk is understood.
- owner is clear.
- next action is clear.

## No-Go Criteria
A decision is No-Go if:
- no customer evidence.
- no revenue or delivery link.
- trust risk is unclear.
- creates complexity before proof.
- founder becomes bottleneck.
- public/private boundary is weak.

## Decision Table

| Decision | Go If | No-Go If |
|---|---|---|
| Build feature | repeated workflow + customer evidence | no revenue/delivery link |
| Send proposal | scope clear + A2 approval | unclear payment path |
| Publish case study | client approved | no public approval |
| Add agent | clear input/output/eval | no boundaries |
| Hire contractor | workflow documented | work still unclear |
| Start SaaS layer | 10 repeats + demand | no paid proof |

## Rules
- No major build without Go/No-Go entry.
- No stage advancement without exit criteria.
- No automation without approval level.
- No public claim without evidence.

## Metrics
- Go decisions.
- No-Go decisions.
- reversed decisions.
- decisions with evidence.
- blocked risky decisions.

## Evidence
- founder decision log.
- weekly review.
- approval log.
- GitHub checks.
- customer feedback.
