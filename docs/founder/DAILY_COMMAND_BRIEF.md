# Daily Command Brief

The Daily Command Brief is the founder's first artifact of every
working day. It is prepared by AI overnight from the previous day's
logs and approved (or amended) by the founder in the first 20 minutes
of the day.

## Purpose
Give the founder a single-page operating picture every morning:
Money, Sales, Delivery, Trust state, and the explicit Decisions
Required for the day. The brief replaces "checking inboxes" as the
first action of the day.

## Owner
Sami (Founder).

## Review Cadence
Daily. Generated by 07:00 local time, reviewed by 09:00 local time.

## Inputs
- Yesterday's pipeline movement (private repo `pipeline/pipeline_tracker.csv`).
- Yesterday's cash collected and invoices sent (private `revenue/cash_collected.csv`).
- Yesterday's delivery actions (private `delivery/` artifacts).
- Yesterday's approvals logged (private `trust/approval_log.csv`).
- Pending decisions queued (private `founder/decision_queue.md`).
- Calendar for today.

## Outputs
- The brief itself (overwritten daily, archived weekly).
- A short list of Decisions Required, each with a recommended option
  and the evidence backing it.
- Updates to the decision queue once the founder responds.

## Rules
- The brief must fit on one screen at default font size.
- Every line must be backed by a number or a named artifact, not a feeling.
- Decisions Required must be answerable in 60 seconds each.
- The brief never sends external messages by itself — it queues
  drafts; the founder approves separately.
- If a section has no data, it must say "No movement" — not be blank.

## Metrics
- Time to read brief (target: under 5 minutes).
- Number of Decisions Required answered before noon (target: 100%).
- Number of days the brief was actually opened (target: 5/5 working days).
- Number of days the brief was correct vs. needed correction (target: 4/5+).

## Evidence
- Latest brief artifact in private repo `founder/daily_brief.md`.
- Decision log entries from yesterday in `founder/decision_log.md`.
- Weekly archive in `founder/daily_brief_archive/YYYY-MM-DD.md`.

## Brief Template

The brief is generated into this exact structure every morning:

```
# Daily Command Brief — YYYY-MM-DD

## Money
- Cash collected yesterday: SAR X
- Cash collected month-to-date: SAR X
- Invoices outstanding: SAR X (N invoices)
- Burn yesterday: SAR X
- Runway: N months

## Sales
- New leads added yesterday: N
- DMs / emails sent yesterday: N
- Replies received yesterday: N
- Proposals sent yesterday: N
- Pipeline value: SAR X across N opportunities

## Delivery
- Active engagements: N
- Engagements in QA today: N
- Deliveries scheduled today: N
- Blockers: list or "None"

## Trust
- Approvals waiting: N
- Approvals granted yesterday: N
- Risk-flagged items: list or "None"
- Suppression list size: N

## Decisions Required
1. <decision> — recommended: <option> — evidence: <link>
2. <decision> — recommended: <option> — evidence: <link>
3. <decision> — recommended: <option> — evidence: <link>

## Today's One Priority
<the single thing that, if done, makes today a win>
```

## Failure Modes To Watch
- Brief becomes generic ("No movement" five days in a row → the system
  is broken, not the data).
- Decisions Required pile up unanswered (>3 days old) → escalate to
  the weekly CEO review as a meta-issue.
- Numbers stop matching the private trackers → the data pipeline
  feeding the brief is broken; treat as P0.
