# الإحاطة اليومية — Daily Command Brief

> The pre-filled daily page the founder reads at the Command Center.

## Purpose
Give the founder a single, structured page each morning summarizing yesterday's state, today's open items, and the variables that should set today's focus. Eliminate inbox-driven mornings.

## Owner
Founder/CEO. Brief is generated by `make ceo-daily` but reviewed by the founder.

## Inputs
- Yesterday's `dealix-ops-private/daily/<yesterday>.md`
- Pipeline state from `docs/revenue/REVENUE_COMMAND_CENTER.md`
- Cash from `docs/finance/CASH_CONTROL.md`
- Sprint board (delivery)
- Open decisions in `docs/founder/decisions/`

## Outputs
- `dealix-ops-private/daily/YYYY-MM-DD.md` — one file per day.
- Optionally a 3-line summary copied to the founder's notes app.

## Rules
1. Brief is generated before 8:30 AM Riyadh.
2. Brief is read in full at the Command Center, not skimmed.
3. Brief does NOT auto-send anything externally. It is internal-only.
4. If a section is empty, write "empty — no change since <date>". Never omit a section.
5. Brief never contains real customer names externally — anonymized labels only (Customer-A1, etc.) for non-private fields.

## Metrics
- Days a brief is generated: 6 / 7 minimum.
- Days the founder logs "read": 6 / 7.
- Mean brief length: 200–400 words.

## Cadence
Daily, before 8:30 AM.

## Evidence
`dealix-ops-private/daily/`.

## Verifier
`make brief-verify` — confirms today's file exists, has all sections, and is non-empty.

## Runtime Command
`make ceo-daily`

---

## Template

```
# Daily Brief — YYYY-MM-DD

## Cash
Cash today: SAR X,XXX,XXX
Runway: NN days
Change since yesterday: +/- SAR X

## Pipeline
- New qualified leads (24h): N
- Open proposals: N (total SAR X)
- Replies awaited: N (oldest waiting D days)
- Calls scheduled today: N
- Payments expected today: N (SAR X)

## Delivery
- Sprints in flight: N
- On-time: N | At-risk: N | Blocked: N
- One blocker to remove today: <text>

## Trust
- Open flags: N
- Refund/complaint: <text or none>

## Learning
- Yesterday's lesson: <one line>
- Doc to update today: <path or none>

## Open Decisions
- <decision id> — owner — deadline

## Yesterday's focus result
<status: Done / Moved / Killed>

## Today's recommended focus (founder confirms or overrides)
<text>
```

## Reading discipline
- Read top to bottom.
- Stop on the first Red signal. Resolve before moving on.
- Confirm or override the recommended focus in writing.

## What the brief is NOT
- Not a status report for anyone else.
- Not a place for analysis (analysis goes in the Weekly Review).
- Not a substitute for the Command Center's six questions.

## القواعد العربية
1. تولّد الإحاطة قبل الثامنة والنصف صباحًا.
2. تُقرأ كاملة، لا سريعًا.
3. لا قسم فارغ بدون تعليل. اكتب "لا تغيير منذ <تاريخ>".
