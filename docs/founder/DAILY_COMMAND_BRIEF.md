# Daily Command Brief

The one document the founder reads first thing every morning. If only one operating document is updated in a day, it is this one.

## Purpose
Give the founder, in under five minutes, the four pieces of information needed to start the day operating instead of reacting: Money, Sales, Delivery, Trust — plus the decisions queued for them today. Generated by the AI from the private ops repo, approved and acted on by the founder.

## Owner
Sami (Founder).

## Review Cadence
Daily, every working day, before 09:00 Riyadh.

## Inputs
- `revenue/cash_collected.csv` (private ops)
- `pipeline/pipeline_tracker.csv` (private ops)
- `delivery/active_engagements.md` (private ops)
- `trust/approvals_waiting.md` (private ops)
- `founder/decision_queue.md` (private ops)

## Outputs
- The four blocks below, filled with the live numbers.
- A decision list with one-line answers.
- A single "north star action for the day" that the founder commits to before 10:00.

## Rules
- Every number has a source line in the private ops repo.
- The brief never contains marketing language.
- No more than five decisions queued per day. Overflow waits.
- The founder confirms the brief is read by marking the file with the date at the top.

## Metrics
- Days completed (out of working days in the month).
- Number of decisions resolved same day.
- Average latency from decision queued to decision made.
- Number of days the north star action was completed.

## Evidence
- Date stamp at top of file each working day.
- Decision Log entry per decision.
- Approvals Log entry per approval.
