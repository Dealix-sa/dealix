# Daily Command Brief

The morning briefing the founder reads before opening any other tool.

## Format
A short, structured note generated (or written) each morning.

```
Date: YYYY-MM-DD

Cash: <SAR collected this week> / <pipeline value>
MRR:  <current> (Δ vs last week)
Pipeline: <new / qualified / replied / call booked / paid>
Delivery: <active clients / overdue / at risk>
Trust:    <approvals waiting / incidents>
Risk:     <top risk on the board>

Top decision today: <one decision>
Top opportunity today: <one opportunity>
Stop today: <one item>
Double down today: <one item>

Founder block (focus hours): <time>
```

## Source feeds
- Pipeline state (CRM or pipeline CSV)
- `dealix-ops-private/revenue/cash_collected.csv`
- `dealix-ops-private/revenue/mrr_tracker.csv`
- `docs/founder/RISK_REGISTER.md`
- `docs/trust/approval_log.csv` (private)
- `docs/learning/EXPERIMENT_LOG.md`

## Rules
- One page. No more.
- One decision, one opportunity, one stop, one double-down.
- If the brief cannot fit in one page, the picture is not clear enough.
- The brief is read first. Everything else is downstream.
