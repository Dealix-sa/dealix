# Decision Log System

Append-only JSONL ledger of every decision the CEO makes. One record per
decision. No edits — corrections are new records with `supersedes` set.

Stored at `$DEALIX_OPS_PRIVATE/ceo/decisions.jsonl`. Lives outside the repo
by design. Schema-validated by
[`scripts/founder_decision_log_append.py`](../../scripts/founder_decision_log_append.py).

The pattern mirrors the existing AgentCard JSONL persistence in
[`auto_client_acquisition/agent_os/agent_registry.py`](../../auto_client_acquisition/agent_os/agent_registry.py)
— append-only, one JSON object per line, no edits, no deletes.

## Record schema

| Field | Type | Required | Notes |
|---|---|---|---|
| `id` | str (uuid4) | yes | Generated by the script if not provided |
| `recorded_at` | ISO 8601 datetime | yes | Auto-stamped by [`dealix/private_ops.py`](../../dealix/private_ops.py) `write_jsonl_append()` |
| `decision` | str | yes | One sentence, present tense, active voice |
| `type` | str | yes | One of: `bet`, `cut`, `delegate`, `hire`, `automate`, `pricing`, `partnership`, `policy`, `other` |
| `owner` | str | yes | `ceo`, contractor handle, or team |
| `reversible` | bool | yes | True if reversible within a week without major cost |
| `assumption_ids` | list[str] | no | IDs from [STRATEGIC_ASSUMPTIONS_REGISTER](STRATEGIC_ASSUMPTIONS_REGISTER.md) |
| `expected_outcome` | str | no | Falsifiable expectation with a date |
| `kill_trigger` | str | no | "If X happens by date D, reverse this" |
| `links` | list[str] | no | URLs or repo paths supporting the decision |
| `supersedes` | str | no | UUID of an earlier decision this replaces |
| `status` | str | yes | `pending` / `executing` / `done` / `reversed` |

## Example

```json
{
  "id": "5f9a...",
  "recorded_at": "2026-05-24T08:30:00+00:00",
  "decision": "Pause LinkedIn outbound for two weeks; re-route the hours to sample factory",
  "type": "cut",
  "owner": "ceo",
  "reversible": true,
  "assumption_ids": ["a-li-conv-001"],
  "expected_outcome": "Sample throughput +50% by 2026-06-07",
  "kill_trigger": "If sample throughput does not move by 2026-06-14, revert",
  "links": ["docs/strategy/BEACHHEAD_SECTOR_SCORECARD.md"],
  "status": "executing"
}
```

## How to append

```bash
DEALIX_OPS_PRIVATE=/opt/dealix-ops-private python scripts/founder_decision_log_append.py \
  --decision "Pause LinkedIn outbound for two weeks; re-route the hours to sample factory" \
  --type cut --owner ceo --reversible \
  --expected-outcome "Sample throughput +50% by 2026-06-07" \
  --kill-trigger "If sample throughput does not move by 2026-06-14, revert"
```

PRIVATE_OPS unavailable → script exits 0 with the bilingual note from
[`dealix/private_ops.py`](../../dealix/private_ops.py) `missing_private_ops_note()`.
No record is written.

## API surface

- `GET /api/v1/founder/ceo-os/decisions?limit=20` — last N decisions; redacts on failure
- `POST /api/v1/founder/ceo-os/decisions` (admin-key gated) — append a decision; 503 if PRIVATE_OPS disabled

## Read by

- [CEO_DAILY_BRIEF_SYSTEM](CEO_DAILY_BRIEF_SYSTEM.md) — pulls the open items
- [CEO_WEEKLY_REVIEW](CEO_WEEKLY_REVIEW.md) — walks the week's records
- Frontend `CeoOsHub` component — surfaces the latest 5

## Non-negotiables

The ledger never auto-acts on a decision. It records intent. Execution
flows through the normal operational systems with their own approval gates.
See [DO_NOT_SAY](DO_NOT_SAY.md).
