# Weekly CEO Review

The Weekly CEO Review is the founder's most important hour of the
week. It closes the Learning Loop and opens the next week with a
single decision that compounds.

## Purpose
Force a structured pause every week so the founder reviews what
happened, what worked, what failed, what bottlenecks emerged, and
commits to exactly one change for the upcoming week. The review is
the only place the Company OS Scorecard moves.

## Owner
Sami (Founder).

## Review Cadence
Weekly. Recommended slot: Friday afternoon or Sunday morning. Not
skipped, ever. If skipped, the next review must explicitly note why.

## Inputs
- The week's five Daily Command Briefs.
- `docs/learning/WEEKLY_INTELLIGENCE_REVIEW.md` (AI-prepared draft).
- Pipeline movement for the week.
- Cash collected for the week.
- Deliveries shipped for the week.
- Approvals logged for the week.
- Customer feedback received (signed or unsigned).
- Last week's "One Change" — did it land?

## Outputs
- Updated `DEALIX_COMPANY_OS_SCORECARD.md`.
- A single committed decision for the next week ("One Change").
- Updates to `docs/learning/WEEKLY_INTELLIGENCE_REVIEW.md` archive.
- Adjustments to the offer ladder, pipeline stages, or approval
  matrix if the review surfaces structural issues.

## Rules
- The review must produce exactly **one** change for the next week —
  not three, not zero. Concentration beats breadth.
- The "One Change" must be measurable in seven days.
- A system on the scorecard can only move up if external evidence
  exists this month.
- A system on the scorecard moving *down* requires a written reason
  in the review.
- The review never proposes new offers or new motions on the spot —
  those go into the decision queue and get worked over the next week.

## Metrics
- Weeks reviewed in a row (target: streak).
- Number of "One Changes" that actually landed (target: ≥3 of 4 per
  month).
- Number of scorecard systems at PASS or COMPOUNDING (target: grows
  monotonically).
- Pipeline conversion rate week over week.
- Cash collected week over week.

## Evidence
- Archived reviews in `weekly_reviews/YYYY-MM-DD.md` (private repo).
- Scorecard delta committed in the same week.
- Decision log entries tagged with `weekly-review`.

## Review Template

```
# Weekly CEO Review — Week ending YYYY-MM-DD

## What happened this week
- Revenue: SAR X collected, N invoices sent.
- Sales: N leads, N DMs, N replies, N proposals.
- Delivery: N engagements active, N deliveries shipped.
- Trust: N approvals, N risk flags.

## What worked
- <specific action / system / message that produced evidence>

## What failed
- <specific action / system / message that did not produce evidence>

## What was the bottleneck
- <single biggest constraint this week>

## What will change next week
**One Change:** <one concrete, measurable action>

## Scorecard delta
- Founder OS: X → Y (reason)
- Strategy OS: X → Y (reason)
- Revenue OS: X → Y (reason)
- Delivery OS: X → Y (reason)
- Trust OS: X → Y (reason)
- Learning OS: X → Y (reason)

## Open questions for next review
- <question 1>
- <question 2>
```

## Failure Modes To Watch
- "One Change" becomes vague ("focus more on sales") → reject; rewrite
  until measurable.
- Scorecard never moves → review is theatrical, not operational.
- Same bottleneck named 3 weeks in a row with no progress → escalate
  to a structural decision (drop a system, hire help, change the
  motion).
