# Company Health Score

A single table the founder updates weekly. One line per area.

| Area | Score | Status | Signal | Action |
|---|---:|---|---|---|
| Revenue | 0–100 | PASS / FIX / BLOCKED | Cash, MRR, pipeline | Next revenue action |
| Sales | 0–100 | PASS / FIX / BLOCKED | DMs, replies, calls | Next outbound action |
| Delivery | 0–100 | PASS / FIX / BLOCKED | Active clients, overdue | Next delivery action |
| Product | 0–100 | PASS / FIX / BLOCKED | CI, bugs, feature demand | Next product action |
| Trust | 0–100 | PASS / FIX / BLOCKED | Approvals, incidents | Next risk action |
| Founder | 0–100 | PASS / FIX / BLOCKED | Focus, decisions, overload | Next CEO action |

## Status definitions
- **PASS** — area is on track. No urgent action.
- **FIX** — area is drifting. One action this week.
- **BLOCKED** — area is broken. Top priority of the week.

## Scoring guide
- 90–100: Excellent. Compounding.
- 70–89: Healthy. Maintain.
- 50–69: Drifting. Needs attention.
- 30–49: At risk. Active fix this week.
- 0–29: Broken. Stop other work.

## Rule
If any area is BLOCKED, the founder cannot start a new strategic bet that week.
