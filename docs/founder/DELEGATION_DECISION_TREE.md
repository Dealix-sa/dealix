# Delegation Decision Tree

For every recurring task currently on the CEO's plate: keep, coach, delegate,
automate, or hire. The tree below resolves in five questions.

## The five questions

1. **Is this differentiated work only I can do right now?**
   - Yes → **Keep** (e.g., first 10 customer demos, founding doctrine)
   - No → go to 2

2. **Is the bottleneck judgment, or capacity?**
   - Judgment → **Coach** an existing operator until their judgment is trusted, then delegate
   - Capacity → go to 3

3. **Is it repeatable with documented steps?**
   - Yes → go to 4
   - No → **Coach** first (write the SOP, then delegate)

4. **Does the volume justify a tool / agent over a person?**
   - Yes → **Automate** (with the trust gates from [`docs/00_constitution/NON_NEGOTIABLES.md`](../00_constitution/NON_NEGOTIABLES.md))
   - No → go to 5

5. **Are the hiring triggers crossed?** ([../people/HIRING_TRIGGER_SYSTEM](../people/HIRING_TRIGGER_SYSTEM.md))
   - Yes → **Hire** (full-time or contractor — see decision matrix in [../finance/HIRE_VS_AUTOMATE_VS_PARTNER](../finance/HIRE_VS_AUTOMATE_VS_PARTNER.md))
   - No → **Delegate** to an existing operator on a 30-day trial scorecard

## Quick reference table

| Situation | Action |
|---|---|
| Differentiated, judgment-heavy, < 5 hrs/week | Keep |
| Repeatable, < 5 hrs/week, no tool fit | Coach + delegate to existing operator |
| Repeatable, ≥ 5 hrs/week, tool fit available | Automate (with approval gates) |
| Repeatable, ≥ 10 hrs/week, no tool fit | Hire contractor |
| Repeatable, ≥ 20 hrs/week, sustained 4+ weeks | Hire full-time |

## Recording the decision

Every delegate / automate / hire decision goes through
[DECISION_LOG_SYSTEM](DECISION_LOG_SYSTEM.md) with `type` set accordingly.

## Anti-patterns

- "I'll just do it once more" — third occurrence triggers re-walk of the tree
- Coaching forever — coaching beyond 4 weeks → re-walk to delegate / hire
- Automating without an approval gate — see [`docs/00_constitution/NON_NEGOTIABLES.md`](../00_constitution/NON_NEGOTIABLES.md)

reviewed weekly. See [DO_NOT_SAY](DO_NOT_SAY.md).
