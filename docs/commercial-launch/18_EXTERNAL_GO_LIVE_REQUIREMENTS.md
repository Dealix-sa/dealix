# External Go-Live Requirements — متطلبات الإطلاق الخارجي

What must be true before any outreach or client-facing artifact goes live. A hard gate.

## Requirements — المتطلبات

**English.**
1. The governing rule is enforced: the system never sends externally.
2. All drafts pass the safety gates in [07_COMPLIANCE_AND_SAFETY_GATES.md](07_COMPLIANCE_AND_SAFETY_GATES.md).
3. CRM stages and suppression rules are live. See [22_LEAD_INTAKE_AND_CRM_OS.md](22_LEAD_INTAKE_AND_CRM_OS.md).
4. Final QA checklist passes. See [23_LEAD_OPS_FINAL_QA.md](23_LEAD_OPS_FINAL_QA.md).
5. Bilingual parity confirmed on every buyer-facing artifact.

**عربي.**
1. تُطبَّق القاعدة الحاكمة: النظام لا يرسل خارجيًا.
2. كل المسودات تجتاز بوابات السلامة.
3. مراحل CRM وقواعد الكبت مفعّلة.
4. قائمة الفحص النهائية تُجتاز.
5. التكافؤ ثنائي اللغة مؤكد على كل مادة للمشتري.

## Go-live checklist — قائمة الإطلاق

## Sign-off — الاعتماد

**English.** The founder signs off in writing before the first message goes out. No exceptions.

**عربي.** يعتمد المؤسس كتابةً قبل خروج أول رسالة. دون استثناء.
