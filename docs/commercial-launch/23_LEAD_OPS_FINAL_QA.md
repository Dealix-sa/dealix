# Lead Ops Final QA — الفحص النهائي لعمليات العملاء

The last checklist before the founder sends any message manually. Every box must pass.

## Per-message gate — بوابة كل رسالة

- [ ] One recipient only.
- [ ] Founder is the sender (manual). The system never sends externally.
- [ ] Opt-out line present.
- [ ] No faked familiarity ("as discussed" only if there was real prior contact).
- [ ] No fabricated urgency.
- [ ] WhatsApp only after opt-in.

بوابة كل رسالة: مستلم واحد؛ المؤسس هو المُرسِل يدويًا؛ سطر إلغاء الاشتراك موجود؛ لا افتعال ألفة؛ لا استعجال مُفتعَل؛ واتساب بعد موافقة فقط.

## Forbidden phrase scan — فحص العبارات المحظورة

Reject if any appears: "guaranteed ROI", "100%", "replace your team", "replace lawyers", "automate everything", "no human needed", "from our database", "as discussed" (without prior contact).

ارفض عند ظهور أي من: "عائد مضمون"، "100%"، "نستبدل فريقك"، "نستبدل المحامين"، "أتمتة كل شيء"، "بدون تدخل بشري"، "من قاعدة بياناتنا"، "كما اتفقنا" (دون تواصل سابق).

## Privacy scan — فحص الخصوصية

- [ ] No PII in any attached case asset.
- [ ] Value figures labeled estimated until verified.

فحص الخصوصية: لا بيانات شخصية في أي أصل حالة؛ أرقام القيمة مُوسومة تقديرية حتى التحقّق.

## Suppression check — فحص الكبت

- [ ] Recipient not on the suppression list.

فحص الكبت: المستلم ليس على قائمة الكبت.

## Sign-off — الاعتماد

**English.** Only after every box passes does the founder send by hand and update the CRM stage to `manually_contacted`.

**عربي.** فقط بعد اجتياز كل بند يرسل المؤسس يدويًا ويحدّث مرحلة CRM إلى `manually_contacted`.
