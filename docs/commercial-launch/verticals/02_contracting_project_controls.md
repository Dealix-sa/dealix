# Vertical Playbook — Contracting & Project Controls
## المقاولات وضبط المشاريع

### Sector thesis
Contractors lose margin where project status, variations and claims live in
spreadsheets and chats. Dealix installs a project-controls OS that keeps status,
variations and subcontractor follow-up current — with founder/PMO approval on
every external communication.

### Why now
Mega-project pipeline, tighter cost control, and audit/claim disputes make
real-time controls valuable now.

### ICP
Contractors and project controls / PMO functions (100–2,000 staff) running
multiple concurrent projects in KSA/GCC.

### Excluded ICP
Single-project micro-contractors; firms unwilling to structure project data.

### Buyer personas & titles
- **Decision maker:** Project Director / PMO Lead / Operations Director
- **Influencer:** Commercial Manager, Planning Engineer
- **User:** Project controllers, document control
- **Budget owner:** Director / Owner

### Top workflows
Project status roll-up → variation/claim tracking → subcontractor follow-up →
cost-vs-budget visibility → executive reporting.

### Top 10 pains
1. Status spread across spreadsheets and chats
2. Variation orders & claims tracked manually
3. Slow, reactive subcontractor follow-up
4. Cost overruns discovered too late
5. Executive reporting takes days
6. Document control fragmented
7. No early-warning on schedule slip
8. Claims evidence assembled under pressure
9. Inconsistent project KPIs across teams
10. Lessons-learned never captured

### Trigger events / buying signals
New project award, claim dispute, PMO setup, audit finding, cost-overrun review.

### Disqualification signals
No structured project data; expecting automated cold outreach; refusal to define
acceptance criteria.

### Offer ladder mapping
Audit (status & claims map) → Pilot on one project → Department OS (PMO controls)
→ Retainer (monitoring) → Enterprise (multi-project governance + integrations).

### Messaging angles
- **AR:** "تكتشف تجاوزات التكلفة متأخرًا؟ نبني نظام ضبط مشاريع يظهر الانحراف مبكرًا — أنت تعتمد كل خطوة."
- **EN:** "If cost overruns surface too late, you're managing blind. We install a project-controls OS that flags variance early — you approve every step."

### Cold email / follow-up / LinkedIn / website examples
Same governed pattern as vertical 01: single offer, clear pain, opt-out on email,
manual-only LinkedIn, no auto-submit on forms.

### Discovery questions
How is project status rolled up? Where do variations live? How are claims
evidenced? Which project is at risk now?

### Proposal angle
Anchor on the riskiest live project; propose Audit → Pilot with measurable
controls.

### Proof asset, delivery, metrics
Status roll-up template, variation/claim register, early-warning dashboard.
Success: reporting time ↓, variance caught earlier, claim evidence ready (targets).

### Risks & compliance
Commercial data is sensitive; data minimization + NDA; public contact only.

### What not to say / objection handling / founder notes
No guaranteed ROI. "We have a planner" → planning ≠ live controls + comms OS.
Strong second wedge; high deal sizes via Department OS.
