## Pricing logic / منطق التسعير

**EN —** Price follows scope, risk, and the size of the workflow we operate on — not headcount or seat count. Every tier is an entry point to the next. We start small, prove value on a limited dataset, then expand. All payment is handled manually outside this repository.

**AR —** السعر يتبع النطاق والمخاطرة وحجم المسار الذي نعمل عليه — لا عدد الموظفين ولا عدد المقاعد. كل مستوى مدخل للذي يليه. نبدأ صغيرًا، نثبت القيمة على بيانات محدودة، ثم نتوسع. تتم المدفوعات يدويًا خارج هذا المستودع.

## Included / excluded per tier / المشمول والمستثنى لكل مستوى

### Entry Diagnostic / التشخيص الأولي
- **Included / المشمول:** Workflow interviews, current-state map, prioritized pain list, pilot recommendation, risk notes.
- **Excluded / المستثنى:** Any build, any external sending, any data extraction, integrations.

### Paid Pilot / التجربة المدفوعة
- **Included / المشمول:** One working workflow on a limited, client-provided dataset; approval gates; draft outputs for founder/client review; a measurable result.
- **Excluded / المستثنى:** Full department rollout, auto-send, scraping, unlimited data, production integrations beyond pilot scope.

### Department OS / نظام تشغيل قسم
- **Included / المشمول:** Operating system for one department, approval-first workflows, dashboards, documented runbooks, training.
- **Excluded / المستثنى:** Cross-department rollout, autonomous external actions, regulated automation in sensitive sectors.

### Monthly Retainer / الاشتراك الشهري
- **Included / المشمول:** Monitoring, tuning, monthly improvements, monthly leadership report.
- **Excluded / المستثنى:** New department builds (scoped separately), unlimited change requests.

### Enterprise Custom OS / النظام المؤسسي المخصص
- **Included / المشمول:** Custom multi-workflow operating system, governance design, phased delivery.
- **Excluded / المستثنى:** Anything that removes human approval from sensitive actions.

## Payment terms / شروط الدفع

**EN —** Payment terms (deposit, milestones, invoicing) are agreed and processed manually with the client, outside this repository. Nothing in Dealix charges, invoices, or transacts automatically.

**AR —** تُتفق شروط الدفع (الدفعة المقدمة، المراحل، الفوترة) وتُعالَج يدويًا مع العميل خارج هذا المستودع. لا شيء في Dealix يُحاسب أو يُفوتر أو يُجري معاملة تلقائيًا.

## Upsell paths / مسارات الترقية

**EN —** Diagnostic → Pilot (proven pain). Pilot → Department OS (proven result). Department OS → Retainer (sustain + tune). Retainer → Enterprise Custom OS (expand to more departments). Each step requires a real, evidenced outcome from the prior step — never a promise.

**AR —** تشخيص ← تجربة (ألم مثبت). تجربة ← نظام قسم (نتيجة مثبتة). نظام قسم ← اشتراك (استدامة وضبط). اشتراك ← نظام مؤسسي مخصص (توسعة لأقسام أخرى). كل خطوة تتطلب نتيجة حقيقية مدعومة بأدلة من الخطوة السابقة — لا وعدًا.
