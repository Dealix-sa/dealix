# Dealix — Delivery Operating System — نظام التسليم التشغيلي

> **Golden rule / القاعدة الذهبية:**
> AI recommends and drafts. Deterministic workflows verify. Founder approves. Nothing is sent automatically.
> الذكاء يقترح ويصيغ، مسارات حتمية تتحقق، المؤسس يعتمد، لا شيء يُرسل تلقائيًا.

Related: [13_PROPOSAL_TEMPLATE_AR_EN.md](13_PROPOSAL_TEMPLATE_AR_EN.md) · [16_CLIENT_ONBOARDING.md](16_CLIENT_ONBOARDING.md) · [18_HANDOVER_AND_SUCCESS_REPORT.md](18_HANDOVER_AND_SUCCESS_REPORT.md)

**EN —** For each tier: what we receive, what we build, what we never build, timeline, acceptance, security boundary, success metrics, handover artifacts, and the next upsell. Every tier keeps a human approval gate.

**AR —** لكل مستوى: ما نستلمه، ما نبنيه، ما لا نبنيه أبدًا، الجدول، القبول، حدود الأمان، مؤشرات النجاح، مخرجات التسليم، والترقية التالية. كل مستوى يحتفظ ببوابة اعتماد بشري.

## Entry Diagnostic / التشخيص الأولي

- **Receive / نستلم —** Workflow interviews, sample documents, current pain points.
- **Build / نبني —** Current-state workflow map, prioritized pain list, pilot recommendation, risk notes.
- **Never build / لا نبني —** Any system, any sending, any data extraction.
- **Timeline / الجدول —** 3–7 days / أيام.
- **Acceptance / القبول —** Map + diagnosis + recommendation delivered and reviewed.
- **Security boundary / حدود الأمان —** Read-only on client-provided samples; nothing extracted.
- **Success metrics / مؤشرات النجاح —** Hours/week identified; leak points named (estimated).
- **Handover / التسليم —** Diagnostic document + recommendation.
- **Upsell / الترقية —** Paid Pilot on the highest-value workflow.

---

## Paid Pilot / التجربة المدفوعة

- **Receive / نستلم —** One limited, client-provided dataset; an approving owner.
- **Build / نبني —** One working workflow with draft outputs and approval gates; a measurable result.
- **Never build / لا نبني —** Full rollout, auto-send, scraping, unlimited data, production integrations beyond scope.
- **Timeline / الجدول —** 14–30 days / يومًا.
- **Acceptance / القبول —** Drafts produced, gates working, result reported.
- **Security boundary / حدود الأمان —** Limited dataset only; drafts internal until approved; retention agreed.
- **Success metrics / مؤشرات النجاح —** Workflow result vs. baseline (estimated until case-verified).
- **Handover / التسليم —** Working pilot + success report ([18](18_HANDOVER_AND_SUCCESS_REPORT.md)).
- **Upsell / الترقية —** Department OS.

---

## Department OS / نظام تشغيل قسم

- **Receive / نستلم —** Department scope, owners, access to the relevant dataset.
- **Build / نبني —** Approval-first operating system for one department, dashboards, runbooks, training.
- **Never build / لا نبني —** Cross-department rollout, autonomous external actions, regulated automation in sensitive sectors.
- **Timeline / الجدول —** 30–90 days / يومًا.
- **Acceptance / القبول —** Department runs the workflow with approval gates; team trained.
- **Security boundary / حدود الأمان —** Scoped department data; human approval on every external-facing output.
- **Success metrics / مؤشرات النجاح —** Throughput and turnaround improvements (estimated until case-verified).
- **Handover / التسليم —** Runbooks, dashboards, training records.
- **Upsell / الترقية —** Monthly Retainer.

---

## Monthly Retainer / الاشتراك الشهري

- **Receive / نستلم —** Ongoing access to the live department workflow.
- **Build / نبني —** Monitoring, tuning, monthly improvements, monthly leadership report.
- **Never build / لا نبني —** New department builds (scoped separately), unlimited change requests.
- **Timeline / الجدول —** Ongoing / مستمر.
- **Acceptance / القبول —** Monthly report delivered; agreed improvements shipped.
- **Security boundary / حدود الأمان —** Same scoped data; approval gates remain.
- **Success metrics / مؤشرات النجاح —** Trend of the workflow metric month over month.
- **Handover / التسليم —** Monthly leadership report.
- **Upsell / الترقية —** Enterprise Custom OS.

---

## Enterprise Custom OS / النظام المؤسسي المخصص

- **Receive / نستلم —** Multi-department scope, governance requirements.
- **Build / نبني —** Custom multi-workflow operating system, governance design, phased delivery.
- **Never build / لا نبني —** Anything that removes human approval from sensitive actions.
- **Timeline / الجدول —** Custom / مخصص.
- **Acceptance / القبول —** Phased acceptance per department.
- **Security boundary / حدود الأمان —** Governance-designed access; approval gates everywhere.
- **Success metrics / مؤشرات النجاح —** Per-department workflow metrics (estimated until case-verified).
- **Handover / التسليم —** Governance docs, runbooks, per-department dashboards.
- **Upsell / الترقية —** Additional departments and workflows.
