# Delivery Operating System / نظام التنفيذ

For each offer: inputs required, what Dealix builds, what it does not build, timeline, acceptance, security boundary, human-approval boundary, success metrics, handover artifacts, expansion path, retainer trigger.

## AI Workflow Audit
- Inputs: redacted workflow description, sample (non-sensitive) artifacts.
- Builds: workflow map, bottleneck diagnosis, risk map, opportunity score, pilot plan.
- Does NOT build: production system, integrations, sending.
- Timeline: 3–7 days. Acceptance: founder + client sign-off on the map.
- Security: sample data only. Approval boundary: client approves all outputs.
- Expansion: → Paid Pilot.

## Paid Pilot
- Inputs: sample dataset only; named approver.
- Builds: limited working pilot with approval gates + dashboard + report.
- Timeline: 14–30 days. Retainer trigger: successful pilot acceptance.

## Department OS / Retainer / Enterprise
- Production workflow OS, documentation, training, admin controls, reporting, handover.
- Retainer: monitoring, improvement backlog, monthly executive report, tuning, support, expansion.
