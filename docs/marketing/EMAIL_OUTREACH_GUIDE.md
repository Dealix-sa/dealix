# Email Outreach Guide

How Dealix drafts email outreach. The system **drafts**; the founder
**sends**.

## 1. Three template patterns

### Pattern A — Signal-anchored
> Hi {first_name},
> A signal at [Company] this month — [signal in one sentence] — lines
> up with how we run revenue motions at [sector]. I put together a
> short diagnostic for you. No pitch attached.
>
> If useful, reply "send" and I'll share it.

### Pattern B — Proof-anchored
> Hi {first_name},
> A [sector] founder we worked with said: "[approved quote, ≤ 12 words]."
> If a similar motion would be useful at [Company], happy to share
> what worked.

### Pattern C — Warm intro request
> Hi {first_name},
> [Mutual] suggested you'd find a 10-minute conversation useful — a
> short walk-through of how we set up a Saudi B2B revenue motion
> with founder approval. Up for it?

## 2. Length

- Subject: 4–7 words.
- Body: 70–120 words.
- One CTA. Never two.

## 3. Personalisation tokens

- `{first_name}` — required.
- `{company}` — required.
- `{sector}` — required.
- `{signal}` — required for pattern A.
- `{mutual}` — required for pattern C.

A draft missing any required token is blocked from the queue.

## 4. Compliance

- PDPL opt-out is honoured globally.
- Unsubscribe instructions are present in the footer.
- Sender domain matches the founder's authenticated mailbox.

## 5. Banned

- ❌ Misleading subject lines.
- ❌ Hidden tracking pixels that aren't disclosed.
- ❌ Auto-follow-up before a founder approval.
