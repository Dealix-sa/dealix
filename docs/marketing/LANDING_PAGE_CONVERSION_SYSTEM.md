# Landing Page Conversion System

How Dealix's landing pages convert curious traffic into approved,
queued conversations.

## 1. Page archetypes

| Archetype | Purpose | CTA |
|---|---|---|
| Hero (home) | Position Dealix, route to rung. | "Get the diagnostic." |
| Diagnostic | Sample request, named contact. | "Send my diagnostic." |
| Sector pulse | Sector signal report, capture sector. | "Send me this month's pulse." |
| Case study | Proof artifact, capture sector. | "Talk to the founder." |
| Founder console | Product walk-through. | "See the console." |

## 2. Page anatomy

- **Eyebrow** — sector or pillar.
- **Hero** — one-line positioning + one-line proof.
- **Section 1** — How the wedge works (3 cards).
- **Section 2** — Proof anchor (one consented artifact).
- **Section 3** — One specific outcome (no guarantees).
- **CTA** — single, primary; one form input minimum.

## 3. Conversion guardrails

- Forms require explicit consent.
- The "what happens next" block is always present.
- No dark patterns (hidden defaults, fake countdown).
- WCAG AA passes on every page.

## 4. Measurement

- Page → form completion rate.
- Form → diagnostic shipped.
- Diagnostic → meeting booked.
- Meeting → proposal sent.

## 5. Banned patterns

- ❌ Pop-ups blocking content.
- ❌ Auto-play media.
- ❌ Hype copy.
- ❌ Sender-name spoofing on confirmation emails.
