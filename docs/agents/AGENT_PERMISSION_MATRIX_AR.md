# Agent Permission Matrix

---

## الإطار المرجعي — Reference Framework

هذه المصفوفة مبنية على إطار Gartner لحوكمة الوكلاء المستقلين، مُكيَّف مع متطلبات Dealix التشغيلية ومتطلبات نظام حماية البيانات الشخصية (PDPL) السعودي.

This matrix is built on Gartner's agentic governance framework, adapted for Dealix's operational requirements and Saudi PDPL compliance.

---

## تعريف المستويات — Level Definitions

### المستوى 1 — Observe (يراقب فقط)

**التعريف:** الوكيل يقرأ البيانات ويحللها. لا يكتب، لا يُعدِّل، لا يُرسل.

**المسموح:**
- قراءة ملفات النظام والتقارير
- تحليل البيانات وإنتاج إحصائيات داخلية
- تسجيل الأنماط والتنبيهات في ذاكرته الداخلية

**الممنوع:**
- كتابة أي ملف
- إنتاج مخرجات تصل للعميل
- اتخاذ أي إجراء على نظام خارجي

---

### المستوى 2 — Advise (ينصح فقط)

**التعريف:** الوكيل يُنتج مسودات واقتراحات. لا ينفذ أي منها بدون موافقة المؤسس الصريحة.

**المسموح:**
- كتابة مسودات (رسائل، مقترحات، تقارير) في ملفات مسودة معزولة
- اقتراح إجراءات بلغة واضحة للمؤسس
- تحديث لوائح التتبع الداخلية

**الممنوع:**
- إرسال أي رسالة لجهة خارجية
- تعديل بيانات عميل مباشرة
- تنفيذ أي Sprint بشكل مستقل

**الإجراء المطلوب:** المؤسس يراجع ويوافق كتابياً قبل أي تنفيذ.

---

### المستوى 3 — Act with Approval (يتصرف بموافقة)

**التعريف:** الوكيل يُنفذ إجراءات محددة ومحدودة بعد موافقة صريحة موثقة.

**المسموح:**
- تحديث ملفات التتبع الداخلية بعد موافقة
- تشغيل Scripts محددة مسبقاً وموثقة
- إنتاج Proof Pack بعد اكتمال التسليم

**الممنوع:**
- إرسال أي اتصال خارجي مباشر
- حذف أي بيانات
- تعديل نطاق العقد

**الإجراء المطلوب:** موافقة مكتوبة من المؤسس + تسجيل كل خطوة في `reports/agents/AGENT_DAILY_ACTIVITY_REVIEW.md`.

---

### المستوى 4 — Act Autonomously (مستقل محدود)

**التعريف:** محظور في بيئة الإنتاج بدون استثناء. مسموح فقط في بيئات الاختبار المعزولة بشروط صارمة.

**الشروط للاستخدام في الاختبار:**
- بيئة معزولة تماماً عن بيانات العملاء
- لا يوجد اتصال بأي نظام خارجي
- المؤسس يُراقب الجلسة بالكامل
- يُوثَّق كل إجراء

## القيود الدائمة — Permanent Restrictions (لكل وكيل بلا استثناء)

1. **لا إرسال خارجي مستقل:** أي رسالة لجهة خارجية تمر بموافقة المؤسس أولاً
2. **لا حذف بيانات:** الحذف يتطلب قرار بشري موثق
3. **لا تعديل عقود:** أي تعديل على نطاق عقد يتطلب Change Request موقّع
4. **لا استخدام بيانات العميل في prompts خارجية:** البيانات تبقى داخل بيئة Dealix
5. **لا تجاهل خطأ أو استثناء:** كل خطأ يُسجَّل ويُرفع للمؤسس فوراً

## بروتوكول الانتهاك — Violation Protocol

إذا تصرف وكيل خارج صلاحياته:

1. **التوثيق الفوري:** تسجيل الإجراء غير المصرح في `reports/security/AGENT_AUDIT_LOG_REVIEW.md`
2. **إيقاف الوكيل مؤقتاً:** حتى مراجعة المؤسس
3. **تحليل السبب:** هل كان prompt injection؟ خطأ في الـ workflow؟ ثغرة في السياسة؟
4. **تحديث السياسة:** إذا كانت ثغرة، تُحدَّث هذه المصفوفة
5. **موافقة المؤسس على إعادة التشغيل**
