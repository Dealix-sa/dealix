# Module Status Map — خريطة حالة الأنظمة

This file keeps the **vision honest against reality.** Every Dealix system carries exactly one status. Nothing is described to a customer as more ready than its status here allows.

هذا الملف يبقي **الرؤية صادقة أمام الواقع.** كل نظام في Dealix يحمل حالة واحدة فقط. لا يُوصف أي نظام للعميل بأنه أجهز مما تسمح به حالته هنا.

---

## Status vocabulary — قاموس الحالات

| Status | Meaning | المعنى |
|---|---|---|
| LIVE | Usable with a paying customer today | قابل للاستخدام مع عميل يدفع اليوم |
| BETA | Usable in a controlled trial | قابل للتجربة المحكومة |
| INTERNAL | Founder/team use only | داخلي للفريق فقط |
| DOCS_ONLY | Documented, not built | موثّق وغير مبني |
| BLOCKED | Needs a fix before use | يحتاج إصلاحاً قبل الاستخدام |
| FUTURE | Planned, later | مُخطّط لاحقاً |
| DEPRECATED | Do not use | لا يُستخدم |

## Rule — القاعدة

Every module **must** carry a status in this table before it appears in any customer-facing material. A module with no status is treated as `FUTURE`.

كل نظام **يجب** أن يحمل حالة في هذا الجدول قبل أن يظهر في أي مادة موجّهة للعميل. النظام بلا حالة يُعامَل كـ`FUTURE`.
