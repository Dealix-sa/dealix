## EN — Launch Control Tower

### Sellable now
- **Dealix Command Sprint** (BETA) — the paid entry point.
- **Diagnostic** (BETA) — guided read that feeds the Sprint.
- **Business OS Score** (BETA) — self-serve signal routing to Diagnostic / Sprint.

Nothing else is offered, demoed as available, or priced. See [MODULE_STATUS_MAP.md](MODULE_STATUS_MAP.md).

### Gated (do not sell or demo as available)
- Enterprise OS, Partner OS, Academy OS — FUTURE.
- Adoption OS, Capital / Finance OS — INTERNAL.
- Lead inbox, ICP scoring, WhatsApp safe send, outreach windows — INTERNAL, approval-first, never auto-send or cold automation.

### Go / no-go criteria summary
Full criteria live in `docs/05_founder/LAUNCH_GO_NO_GO.md`. A layer ships only when all five hold:
1. **Truth** — claims match [MODULE_STATUS_MAP.md](MODULE_STATUS_MAP.md); no overclaim.
2. **Proof** — evidence or safe wording behind every customer-facing statement.
3. **Governance** — approval-first, PDPL-native, ZATCA-aware; no external action without founder approval.
4. **Delivery** — the Command Sprint can be delivered end to end in one cycle.
5. **Routing** — every growth asset has exactly one CTA to Score, Diagnostic, or Sprint.

If any one fails, the answer is no-go for that layer.

### Key metrics to watch (launch window)
- Score completions → Diagnostic conversion (estimated, watch the trend, not a target).
- Diagnostic → Command Sprint engagements booked.
- Sprint cycles delivered end to end within the committed window.
- Approval register completeness — every external action has a recorded approver.
- No-overclaim breaches — target zero; any breach is a stop.

All figures are estimated and operational, never published as guaranteed outcomes.

## AR — برج التحكّم في الإطلاق

### قابل للبيع الآن
- **Dealix Command Sprint** (BETA) — نقطة الدخول المدفوعة.
- **التشخيص** (BETA) — قراءة موجّهة تغذّي الـ Sprint.
- **Business OS Score** (BETA) — إشارة ذاتية توجّه إلى التشخيص / الـ Sprint.

لا شيء آخر يُعرض أو يُسعَّر أو يُقدَّم كمتاح. انظر [MODULE_STATUS_MAP.md](MODULE_STATUS_MAP.md).

### مُقيَّد (لا يُباع ولا يُعرض كمتاح)
- Enterprise OS، Partner OS، Academy OS — FUTURE.
- Adoption OS، Capital/Finance OS — INTERNAL.
- سجل الفرص، تقييم ICP، WhatsApp safe send، نوافذ التواصل — INTERNAL، بالاعتماد أولاً، لا إرسال تلقائي ولا أتمتة باردة.

### ملخّص معايير القرار (Go / No-Go)
التفصيل الكامل في `docs/05_founder/LAUNCH_GO_NO_GO.md`. تُطلَق الطبقة فقط عند تحقّق الخمسة:
1. **الصدق** — الادعاءات تطابق [MODULE_STATUS_MAP.md](MODULE_STATUS_MAP.md)؛ بلا مبالغة.
2. **الإثبات** — دليل أو صياغة آمنة خلف كل عبارة للعميل.
3. **الحوكمة** — الاعتماد أولاً، PDPL أصلاً، ZATCA-aware؛ لا إجراء خارجي دون موافقة المؤسس.
4. **التسليم** — يمكن تسليم الـ Command Sprint من البداية للنهاية في دورة واحدة.
5. **التوجيه** — لكل أصل نمو دعوة واحدة إلى Score أو التشخيص أو الـ Sprint.

إخفاق أي بند يعني No-Go لتلك الطبقة.

### المؤشرات الرئيسية (نافذة الإطلاق)
- إكمال الـ Score ← التحوّل إلى التشخيص (تقديري، نراقب الاتجاه لا الهدف).
- التشخيص ← ارتباطات Command Sprint المحجوزة.
- دورات الـ Sprint المُسلَّمة كاملة ضمن النافذة الملتزَم بها.
- اكتمال سجل الاعتماد — كل إجراء خارجي له معتمِد مُسجَّل.
- خروقات «بلا مبالغة» — الهدف صفر؛ أي خرق يوقف.

كل الأرقام تقديرية وتشغيلية، لا تُنشر كنتائج مضمونة.
