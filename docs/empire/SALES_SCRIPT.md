# Sales Script — سكربت البيع

**الغرض:** بيع **بداية صغيرة** لا اشتراك كبير. كل محادثة تنتهي بـ pilot أو commitment أو رفض واضح.

## رسالة الفتح — الوكالات

> أنتم تجيبون الاهتمام. **Dealix يثبت ماذا حدث بعد الاهتمام.**

العرض المرفق:

> نأخذ عميلاً واحداً فقط — نراجع ~10 leads، نجهّز المتابعة، نسلّم Proof Pack خلال ~7 أيام. بعدها نقرّر معاً: Pilot، Sprint، Retainer، أو شراكة إحالات.

تفاصيل الوتد: [AGENCY_WEDGE.md](AGENCY_WEDGE.md).

## جمل SOAEN في البيع

- **Lead بلا owner** = ليس pipeline.
- **Follow-up بلا evidence** = ليس تشغيلاً.
- **AI action بلا approval** = خطر.
- **Dashboard بلا next action** = تقرير فقط.

المعيار الكامل: [SOAEN_STANDARD.md](SOAEN_STANDARD.md).

## أسلوب الإغلاق

لا تقل: «تبغون تشتركون؟»

قل:

> لا نبدأ باشتراك كبير. نبدأ على **10 leads** أو **workflow واحد**، نسلّم **Proof Pack**، وإذا ظهر value واضح ننتقل إلى **Sprint** أو **Retainer**.

> القاعدة: **لا تخفض السعر أولاً — خفّض النطاق أولاً.** (انظر [OFFER_LADDER.md](OFFER_LADDER.md))

## بنية المكالمة (≈20 دقيقة)

| المرحلة | الزمن | الهدف |
|---------|-------|--------|
| سياق العميل | 4 د | أين تتوقف المتابعة بعد الـ lead؟ |
| إعادة التأطير | 3 د | CRM يخزّن، Dealix يحرّك |
| الديمو | 12 د | انظر [DEMO_FLOW.md](DEMO_FLOW.md) |
| النطاق + الإغلاق | 5 د | عرض pilot صغير + خطوة تالية محددة |

## الخطوة التالية بعد الموافقة المبدئية

- إرسال **Diagnostic Scope** مكتوب (نطاق، مخرجات، مدة، استثناءات).
- لا فاتورة قبل اتفاق نطاق واضح.
- لا وعد إيراد — نعد بـ Proof Pack موثّق فقط.
