# Daily Scorecard — لوحة القياس اليومية

**الغرض:** معيار اللوحة اليومية للتصريف التجاري. إذا أقل من **10 لمسات مباشرة/يوم** — اليوم **under-executed**.

> هذا المعيار هو القالب. النسخة التشغيلية الحيّة: `docs/ops/daily_scorecard.md`.

## نظام اللمسات اليومي

10 لمسات بشرية موافَق عليها يومياً:

| القناة | العدد | الشرط |
|--------|------|--------|
| Warm WhatsApp | 5 | فقط من تعرفهم أو لديك سياق معهم |
| Email | 3 | وكالات، عقار، خدمات B2B |
| LinkedIn | 1 | منشور founder-led أو DM يدوي |
| Partner conversation | 1 | محادثة شريك واحدة |

إضافة يومية: 1 X post · 1 scorecard update.

> ممنوع: scraping · mass DMs · cold WhatsApp blast — انظر [TRUST_LAYER.md](TRUST_LAYER.md).

## اللوحة اليومية

| الحقل | يُسجَّل |
|-------|---------|
| Messages sent | حسب السياسة |
| Follow-ups | عدد |
| Positive replies | عدد |
| Demos booked | عدد |
| Scopes requested | عدد |
| Invoices sent | عدد |
| Payments received | عدد |
| Proof Packs delivered | عدد |
| Partner conversations | عدد |
| Blocked risks | قنوات/أفعال مرفوضة |
| Best message | أفضل رسالة اليوم |
| Worst channel | أضعف قناة |
| Tomorrow priority | شريحة واحدة · حركة واحدة |

## بروتوكول التحديث

1. حدّث الصف بعد كل لمسة (لا تجميع آخر اليوم).
2. أي فعل خارجي يُسجَّل **بعد** تأكيد الإرسال — لا قبله.
3. جدول متابعات +2 / +5 / +10 يوم.
4. إذا اللمسات المباشرة < 10 — علّم اليوم **under-executed** واكتب السبب.

## المراجعة الأسبوعية

أفضل شريحة ICP · أفضل رسالة وعرض وقناة · أين يتوقف الـ funnel · أي اعتراض تكرر · أي شريك جلب quality — قرار صريح: **نضاعف / نوقف / لا نبنيه**.
