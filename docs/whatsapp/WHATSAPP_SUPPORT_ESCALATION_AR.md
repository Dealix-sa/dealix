# الدعم والتصعيد

كيف يطلب العميل الدعم، وكيف نصنّف الإلحاح، ومتى نحلّ ذاتيًا أو نفتح تذكرة أو نُحوّل لشخص. القائمة في `action_card_builder.py` والتدفق في `conversation_state.py`.

## قائمة الدعم

عند اختيار «طلب دعم» تظهر قائمة قصيرة:

```
وش نوع المشكلة؟
• النظام لا يعمل
• البيانات ناقصة
• الرسائل غير مناسبة
• التقرير غير واضح
• أحتاج شخص
```

رسالة الاستقبال (`support_escalation`):

```
وصلت طلبك للدعم.
صنّفت المشكلة، وإذا تبغى نصعّدها لشخص الآن أخبرني.
```

## تصنيف الإلحاح

| الإلحاح | أمثلة | المسار |
|---|---|---|
| high | النظام لا يعمل، شكوى، بيانات حسّاسة | تحويل بشري سريع |
| medium | البيانات ناقصة، الرسائل غير مناسبة | تذكرة + متابعة |
| low | التقرير غير واضح، استفسار عام | خدمة ذاتية أو إعادة عرض |

## خدمة ذاتية / تذكرة / شخص

| المسار | متى | ماذا يحدث |
|---|---|---|
| خدمة ذاتية | استفسار يُحلّ بإعادة عرض أو توضيح | «مراجعة تقرير أو عرض» تعيد إلى `recommendation`. |
| تذكرة | مشكلة تشغيلية تحتاج متابعة | تُسجّل وتُتابَع، تبقى المرحلة `support`. |
| شخص | «أحتاج شخص»، شكوى، حساس | تحويل بشري مع حزمة سياق. |

داخل مرحلة `support`: نية `review_report` تعيد إلى `recommendation`؛ نية `human_handoff` تنقل إلى `human_handoff`؛ و`unknown` تبقى في `support`.

## مسار «أحتاج شخص»

اختيار «أحتاج شخص» (أو كلمات «أحتاج شخص/كلّموني/اتصلوا») يُصنَّف فورًا كـ `human_handoff` بسبب `explicit_request`، وتُبنى حزمة تحويل بسياق كامل:

```
حوّلتك لشخص من فريق Dealix مع ملخّص محادثتك. بنرجع لك سريعًا.
```

الشكاوى تُكتشف كذلك من علامة الشكوى في الرسالة الواردة وتُصعَّد بسبب `complaint` (الإجراء المقترح: اتصال خلال ساعة + تلخيص المشكلة). التفاصيل في [WHATSAPP_HANDOFF_POLICY_AR.md](WHATSAPP_HANDOFF_POLICY_AR.md).

## لا حلقات مكررة

إذا تكرّر عدم فهم الطلب مرتين أو أكثر، نُصعّد لشخص (سبب `repeated_unknown`) بدل تكرار القوائم — احترامًا لوقت العميل.

## مقاييس الدعم

نقطة `GET /metrics` تشمل عدد التحويلات البشرية ومعدّلها وتوزيع أسبابها وتوزيع المراحل — محسوبة من السجلات، بلا أرقام مخترعة.
