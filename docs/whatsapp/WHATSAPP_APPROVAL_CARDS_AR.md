# الكروت — أنواعها وكروت الموافقة

كل رد على العميل عبارة عن كرت واحد أو أكثر. الكروت تُبنى في `action_card_builder.py` وتحمل حقول حوكمة: الخطورة، مستوى الدليل، علامة الموافقة، ومرجع الكتالوج. **لا كرت يرسل شيئًا**؛ حمولات واتساب توليد فقط.

## أنواع الكروت (CardKind)

| النوع | الغرض | مثال |
|---|---|---|
| menu | قائمة خيارات | القائمة الرئيسية، قائمة الدعم |
| action | إجراء واحد أو توضيح | بدء الفحص، كرت رفض آمن |
| approval | مسودة تحتاج موافقة قبل الإرسال | كرت اعتماد المسودة |
| permission | طلب صلاحية واحدة | كرت قراءة leads من CRM |
| recommendation | توصية مربوطة بالكتالوج | كرت أفضل خطوة |
| report | تقرير | تقرير الجاهزية |

كل كرت يحمل: `card_id`, `kind`, `title_ar`, `body_ar`, `options` (حتى ١٠)، `risk`, `evidence_level` (L0–L5)، `requires_approval`, `catalog_ref`, و`safety_summary = no_live_send_no_secrets_in_chat`.

## كرت الإجراء (Action Card)

مثال: «ما أعرف» يبدأ الفحص لخفض الاحتكاك.

```
تمام — بسأل أسئلة قصيرة وأقترح أفضل مسار
فحص الجاهزية يأخذ دقائق ويعطيك توصية واحدة واضحة.
• ابدأ الفحص
• أفضل أتحدث مع شخص
```
الخطورة: low · مستوى الدليل: L0 · يحتاج موافقة: لا.

## كرت الموافقة (Approval Card)

يظهر قبل أي إرسال خارجي. يحمل `requires_approval = true`.

```
جاهز Draft — يحتاج موافقتك قبل أي إرسال
«جهّزنا لك المسوّدة — تحتاج موافقتك قبل أي إرسال.»
• اعتماد
• تعديل
• رفض
• اختصرها
```
الخطورة: medium · مستوى الدليل: L2 · يحتاج موافقة: **نعم**.

> قاعدة صارمة: **كرت الموافقة يتطلب موافقة بشرية صريحة قبل أي إرسال خارجي.** النظام لا يرفع البوابة الحية أبدًا؛ أفضل نتيجة ممكنة هي «معتمد يدويًا» بعد موافقة. الاعتماد ينقل المرحلة من `draft_review` إلى `proposal`؛ الرفض يعيدها إلى `recommendation`.

## كرت الصلاحية (Permission Card)

```
نحتاج صلاحية واحدة لإكمال الربط
النظام: CRM
الغرض: قراءة leads فقط
الصلاحية: read contacts
الخطورة: medium · المدة: 30 يوم
🔒 لأمان البيانات: لا ترسل المفتاح هنا — أكمل عبر الرابط الآمن.
• أوافق
• أرفض
• اشرح لي
```
علامة الموافقة تُرفع تلقائيًا للمستويين L4 و L5. التفاصيل: [WHATSAPP_PERMISSION_ONBOARDING_AR.md](WHATSAPP_PERMISSION_ONBOARDING_AR.md).

## كرت التوصية (Recommendation Card)

يحمل `catalog_ref` (معرّف العرض) ومستوى الدليل من الفحص (L1) وخطورة الفحص.

```
بناءً على إجاباتك، هذي أفضل خطوة
النتيجة الكلية: 62/100
أنسب بداية: سبرنت إثبات الإيرادات (٤٩٩ ر.س)
السبب:
- عندكم leads لكن المتابعة تتسرّب.
- أفضل قيمة سريعة هي كشف التسرّب قبل بناء نظام كبير.
أول workflow: Lead intake → مسودة متابعة → موافقة → إرسال يدوي → تقرير
• ابدأ
• أرسل لي العرض
• احجز مكالمة
• اشرح أبسط
```

## كروت الرفض الآمن

| الكرت | متى | الخطورة |
|---|---|---|
| كرت رفض الأسرار | محاولة إرسال مفتاح/سر في الشات | high |
| كرت رفض غير الآمن | واتساب بارد/جماعي/scraping/أتمتة LinkedIn | high |

```
🔒 الأمان أولًا
لا ترسل المفاتيح أو كلمات السر هنا — لأمان بياناتك لا نحفظها في الشات.
• فتح الرابط الآمن
• أرسل خطوات يدوية
• صلاحية قراءة فقط
```

## حمولة واتساب التفاعلية

عند العرض، الكرت الذي يحمل ≤ ٣ خيارات يُصبح أزرارًا تفاعلية؛ وما زاد عن ٣ يُصبح رسالة قائمة (حتى ١٠ صفوف). التحويل توليد فقط — لا إرسال.
