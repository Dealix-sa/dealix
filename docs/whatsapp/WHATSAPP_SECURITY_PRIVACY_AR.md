# الأمان والخصوصية — قواعد لا تُكسر

أمان بيانات العميل أساس النظام. هذه القواعد مُنفَّذة في الكود (`whatsapp_policy_guard.py` و`permission_guard.py`)، وليست مجرد سياسة مكتوبة.

## القاعدة الأولى: لا أسرار في الشات أبدًا

لا تُطلب مفاتيح API أو رموز (tokens) أو كلمات سر داخل واتساب نهائيًا. النظام يكشفها، **يرفض حفظ القيمة**، ويوجّه العميل لمسار آمن.

```
🔒 الأمان أولًا
لا ترسل المفاتيح أو كلمات السر هنا — لأمان بياناتك لا نحفظها في الشات.
• فتح الرابط الآمن
• أرسل خطوات يدوية
• صلاحية قراءة فقط
```

ما يُكتشف ويُحجب (من `whatsapp_policy_guard.py`): مفاتيح OpenAI/Anthropic، مفاتيح AWS، رموز HubSpot، رموز Bearer، الرموز الطويلة العامة، وأي صيغة `key/secret/token/password/كلمة السر/المفتاح = ...`. القيمة المكتشفة **لا تُعاد ولا تُسجَّل أبدًا**.

## مسار البوابة الآمنة

البديل الوحيد لإدخال أي بيانات اعتماد:

```
واتساب
   ↓ (رابط البوابة الآمنة)
بوابة آمنة (خارج الشات)
   ↓ (إدخال مشفّر)
خزنة الأسرار (vault)
   ↓ (تأكيد فقط — لا قيمة في الشات)
سجل تدقيق (audit log)
```

بدائل العميل دائمًا ثلاثة: (١) الرابط الآمن، (٢) خطوات الربط اليدوية، (٣) صلاحية قراءة فقط. أي ربط يحمل بيانات اعتماد يُفعّل تلقائيًا مسار البوابة الآمنة بغضّ النظر عن مستوى الصلاحية.

## L5 والعمليات الحسّاسة

العمليات الحسّاسة (دفع، عقود، بيانات حسّاسة) **لا تتم عبر واتساب وحده**؛ تُحوَّل إلى بوابة آمنة و/أو مراجعة بشرية. التفاصيل: [WHATSAPP_PERMISSION_ONBOARDING_AR.md](WHATSAPP_PERMISSION_ONBOARDING_AR.md).

## وعي بنظام حماية البيانات (PDPL)

- لا تُخزَّن بيانات شخصية (PII) خام في السجلات؛ نستخدم معرّفات معتمة (`client_handle`).
- وجود بيانات شخصية أو حسّاسة في فحص الجاهزية يرفع المخاطرة إلى «high» ويفرض مراجعة بشرية ومسارًا آمنًا قبل أي ربط.
- العقود واتفاقيات معالجة البيانات (DPA) تُحوَّل لمسار بشري قبل أي تنفيذ.

```
⚠️ يوجد بيانات حسّاسة — أي ربط يحتاج مراجعة بشرية ومسار آمن.
```

## لا PII في السجلات

كل إجراء مُدوَّن في سجلات JSONL، لكن:
- المعرّفات معتمة فقط (handles)، لا أرقام جوال أو إيميلات أو هويات خام.
- حزمة التحويل البشري تحفظ السياق وآخر الرسائل **بلا تسريب PII**.
- قيم الأسرار لا تدخل السجلات إطلاقًا.

## القائمة الممنوعة

ممنوع تمامًا كخدمات أو طلبات (تُحجب مع بديل آمن):

| الممنوع | البديل |
|---|---|
| واتساب بارد (cold WhatsApp) | تواصل فقط مع جهات على علاقة قائمة + موافقة صريحة. |
| إرسال جماعي / رسائل blast | مسودة فردية + موافقتك قبل الإرسال. |
| قوائم مشتراة | لا تُستخدم إطلاقًا. |
| Scraping / كشط البيانات | بيانات يرسلها العميل عبر مسار آمن فقط. |
| أتمتة LinkedIn / بوتات | لا تُقدَّم كخدمة. |
| إرسال خارجي بلا موافقة | موافقة بشرية صريحة شرط لأي إرسال. |

```
هذا الطلب خارج سياسة Dealix.
نتواصل فقط مع جهات على علاقة قائمة، بمسودة + موافقة صريحة منك.
```

## ملخص الأمان على كل كرت

كل كرت يحمل `safety_summary = no_live_send_no_secrets_in_chat`: لا إرسال حي، ولا أسرار في الشات. النظام لا يرفع «البوابة الحية» (live gate) أبدًا؛ أفضل نتيجة ممكنة هي «معتمد يدويًا» بعد موافقة بشرية.
