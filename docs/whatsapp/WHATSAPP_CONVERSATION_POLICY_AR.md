# سياسة المحادثة — قواعد تجربة واتساب

كيف يتحدث Client OS مع العميل. الهدف: وضوح، قرار واحد في كل خطوة، واحترام وقت العميل التنفيذي. لا حشو، لا وعود.

> النموذج اللغوي يصيغ النص فقط؛ التوجيه والقرارات حتمية في `conversation_state.py` و`intent_router.py`.

## القواعد السبع

| # | القاعدة | السبب |
|---|---|---|
| 1 | رسائل قصيرة (≤ ٥ أسطر) | القراءة على الجوال أثناء العمل. |
| 2 | خيار واحد واضح لكل رسالة | قرار سريع بلا تشتت. |
| 3 | وجود «ما أعرف — اقترح علي» دائمًا | لا نُلزم العميل بمعرفة المسار. |
| 4 | تقدّم ظاهر «الخطوة X من Y» | يعرف كم بقي. |
| 5 | لا نسأل ١٠ أسئلة دفعة واحدة | سؤال واحد لكل كرت. |
| 6 | توجيه حتمي؛ النموذج يصيغ فقط | لا قرارات عشوائية. |
| 7 | كل كرت يحمل ملخص أمان | لا إرسال حي ولا أسرار في الشات. |

## رسائل قصيرة وخيار واحد

كل كرت يحمل سؤالًا واحدًا أو إجراءً واحدًا، وأزراره محدودة (واتساب يدعم حتى ٣ أزرار، وما زاد يتحوّل إلى قائمة حتى ١٠ صفوف). مثال على رسالة ترحيب وقائمة:

```
أهلًا، أنا مساعد Dealix.
أقدر أساعدك في تشخيص المبيعات والمتابعة وتشغيل أول نظام داخل شركتك.
اختر الخطوة:
• تشخيص سريع للشركة
• تجهيز حملة أو متابعة
• ربط CRM / تقويم
• مراجعة تقرير أو عرض
• طلب دعم
• ما أعرف — اقترح علي
```

## «ما أعرف — اقترح علي»

هذا الخيار حاضر في القائمة الرئيسية ويظهر أيضًا عند عدم الفهم. عند اختياره، يبدأ النظام فحص الجاهزية مباشرة (يخفض الاحتكاك بدل ترك العميل حائرًا):

```
تمام — بسأل أسئلة قصيرة وأقترح أفضل مسار.
الخطوة ١ من ١٠.
```

## التقدّم الظاهر «الخطوة X من Y»

أثناء فحص الجاهزية، كل سؤال يحمل عنوانه «الخطوة X من Y» حيث Y = ١٠ (عدد المحاور). العميل يعرف موقعه ومتى ينتهي.

```
الخطوة ٣ من ١٠
هل في متابعة موحّدة للـ leads؟
• نعم، عملية واضحة
• جزئيًا / يدوي
• لا، تضيع كثير
```

## التوجيه الحتمي

تصنيف النية يتم بالترتيب: (١) معرّف الزر المهيكل، (٢) حارس السياسة (الأسرار والطلبات غير الآمنة تُقصِّر المسار فورًا)، (٣) خريطة كلمات مفتاحية عربية/إنجليزية، (٤) `unknown`. لا يوجد إرسال حر للنموذج اللغوي ليقرر.

عند `unknown` نُعيد عرض القائمة مع «ما أعرف». وإذا تكرّر عدم الفهم مرتين أو أكثر، نُصعّد لشخص بدل تكرار القوائم (انظر [WHATSAPP_HANDOFF_POLICY_AR.md](WHATSAPP_HANDOFF_POLICY_AR.md)).

```
ما فهمت تمامًا — اختر من القائمة أو «ما أعرف — اقترح علي».
```

## ما لا نفعله في المحادثة

- لا نطلب مفتاح API أو كلمة سر داخل الشات أبدًا (انظر [WHATSAPP_SECURITY_PRIVACY_AR.md](WHATSAPP_SECURITY_PRIVACY_AR.md)).
- لا نَعِد بأرقام مبيعات أو نسب تحويل؛ كل رقم معه مستوى دليل.
- لا نرسل أي رسالة خارجية نيابة عن العميل بلا موافقة صريحة.
- لا نترك العميل في حلقة قوائم مكررة.
