# تجربة العميل — قبل وأثناء وبعد الشراء

كيف يرى العميل التنفيذي السعودي رحلته مع Dealix عبر واتساب، وكيف ننافس اللاعبين الكبار بوضوح بلا مبالغة.

## قبل الشراء

| اللحظة | ما يحدث |
|---|---|
| أول رسالة | قائمة قصيرة بخيار واحد واضح + «ما أعرف — اقترح علي». |
| الفحص | أسئلة قصيرة («الخطوة X من ١٠») بلا التزام. |
| التوصية | توصية واحدة مربوطة بمنتج حقيقي ومستوى دليل، مع سبب مختصر. |
| القرار | «أرسل لي العرض» أو «احجز مكالمة» — بلا ضغط. |

لا وعود بأرقام. كل توصية تحمل سببًا ومستوى دليل (L0–L5) ومرجعًا في الكتالوج.

## أثناء الشراء

```
جهّزت لك عرضًا مبدئيًا.
النطاق والسعر واضحان في الكرت — راجع واعتمد أو احجز مكالمة.
```

- العرض يحمل نطاقًا وسعرًا واضحين في كرت واحد.
- الدفع يتم عبر **رابط آمن خارج الشات**، بعد موافقتك.
- أي تفاوض على السعر أو شرط تعاقدي يُحوَّل لشخص — خارج صلاحية المساعد.

## بعد الشراء

| المرحلة | ما يحصل عليه العميل |
|---|---|
| الإعداد | صلاحيات واضحة، ملفات مطلوبة، ثم أول workflow. |
| التشغيل | مسودات متابعة جاهزة — تحتاج موافقتك قبل أي إرسال. |
| التقرير | موجز أسبوعي: أهم تحسّن + الخطوة القادمة. |
| التجديد | اقتراح استمرار/توسعة بناءً على النتائج، **بلا التزام تلقائي**. |

أول workflow ثابت ومفهوم: `Lead intake → مسودة متابعة → موافقة → إرسال يدوي → تقرير`.

## كيف ننافس اللاعبين الكبار

| المحور | موقف Dealix |
|---|---|
| سعودي أولًا | عربي أولًا، نبرة أعمال سعودية، وعي بنظام حماية البيانات (PDPL). |
| الموافقة أولًا | لا إرسال خارجي بلا موافقة بشرية صريحة. |
| سرعة الوصول للقيمة | أول مخرج سريع (تشخيص خلال ٢٤ ساعة، أو سبرنت ٧ أيام). |
| واتساب أصيل | يعمل داخل القناة التي يستخدمها العميل أصلًا، بأزرار وقوائم. |
| مدفوع بالأدلة | كل رقم معه مستوى دليل؛ لا مبالغة ولا «إثبات» مزيّف. |
| تصعيد بشري | عند الغموض أو الحساسية أو السعر أو العقد أو الشكوى نُحوّل لشخص. |

> Dealix ليس شات بوت عام. هو مساعد سير عمل محكوم، لكل خطوة فيه حدّ أمان وسجل.

## وعود نتجنّبها

- لا «نضمن مبيعات»؛ بدلًا منها: فرص مُثبتة بأدلة ومستوى دليل واضح.
- لا نِسب تحويل أو ROI كحقيقة؛ كل رقم تقديري أو نمط آمن الحالة.
- لا إرسال نيابة عنك بلا موافقتك.
