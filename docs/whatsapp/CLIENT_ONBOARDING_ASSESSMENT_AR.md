# فحص جاهزية الشركة — Dealix Company Readiness Scan

«فحص جاهزية الإيرادات والمتابعة» — فحص قصير ومهني (وليس «استبيان») يُنتج: ثلاث ركائز فرعية + نتيجة كلية ٠–١٠٠، نطاق مخاطرة، **توصية واحدة** مربوطة بكتالوج الخدمات الحقيقي، رسم لأول workflow، الصلاحيات المطلوبة، والخطوة القادمة. التقييم حتمي ومشتق من `assessment.py` (مرآته في `data/whatsapp/assessment_questions.yaml`).

## الركائز الثلاث والأوزان

| الركيزة | المحاور | الوزن |
|---|---|---|
| جاهزية الإيراد (revenue_readiness) | lead_flow · decision_maker · offer_clarity · urgency | 0.40 |
| نضج المتابعة (follow_up_maturity) | channel_chaos · follow_up · crm · reporting | 0.35 |
| جاهزية الأتمتة (automation_readiness) | automation_readiness · compliance_privacy | 0.25 |

> النتيجة الكلية = `round(0.40 × revenue + 0.35 × follow_up + 0.25 × automation)`.
> درجة كل ركيزة = متوسط درجات محاورها (٠–١٠) × ١٠ ← مقياس ٠–١٠٠.

## المحاور العشرة (نص الأسئلة والخيارات)

كل خيار يحمل درجة جاهزية ٠–١٠ (الأعلى = أكثر جاهزية).

**١. lead_flow — جاهزية الإيراد**
«هل عندكم استفسارات أو leads شهريًا؟»
- نعم، كثير (50+ شهريًا) = ٩ · نعم، متوسط = ٦ · قليل = ٣ · لا / غير متأكد = ١

**٢. channel_chaos — نضج المتابعة**
«وين تجيكم الاستفسارات غالبًا؟»
- قناة واحدة منظمة = ٩ · قناتين (واتساب + إيميل) = ٦ · قنوات متعددة ومبعثرة = ٣ · غير متأكد = ٢

**٣. follow_up — نضج المتابعة**
«هل في متابعة موحّدة للـ leads؟»
- نعم، عملية واضحة = ٩ · جزئيًا / يدوي = ٥ · لا، تضيع كثير = ٢

**٤. crm — نضج المتابعة**
«هل تستخدمون CRM؟»
- نعم، فعّال = ٩ · موجود لكن غير منظم = ٥ · لا / Excel فقط = ٢

**٥. decision_maker — جاهزية الإيراد**
«مين يقرر الشراء عندكم؟»
- واضح (شخص/لجنة محددة) = ٩ · أكثر من شخص = ٦ · غير واضح = ٣

**٦. offer_clarity — جاهزية الإيراد**
«هل عروضكم وأسعاركم واضحة؟»
- نعم، واضحة وموثّقة = ٩ · واضحة جزئيًا = ٥ · غير واضحة = ٢

**٧. reporting — نضج المتابعة**
«هل الإدارة تشوف تقرير أسبوعي للمبيعات؟»
- نعم، أسبوعي = ٩ · شهري فقط = ٥ · لا يوجد تقرير = ٢

**٨. automation_readiness — جاهزية الأتمتة**
«هل عندكم أدوات مربوطة (CRM/تقويم/واتساب أعمال)؟»
- نعم، مربوطة = ٩ · بعضها = ٥ · لا شيء مربوط = ٢

**٩. compliance_privacy — جاهزية الأتمتة** (محور حساسية البيانات)
«هل بياناتكم منظمة وواضحة الحساسية؟»
- لا بيانات حساسة = ٩ · بيانات شخصية (أرقام/إيميلات) = ٦ (حسّاس) · بيانات حساسة (مالية/صحية/هويات) = ٣ (حسّاس) · غير متأكد = ٤ (حسّاس)

**١٠. urgency — جاهزية الإيراد**
«وش تبغون تحققون خلال 14 يوم؟»
- نتيجة سريعة الآن = ٩ · خلال هذا الشهر = ٦ · نستكشف فقط = ٣

## نطاقات المخاطرة

| النطاق | الشرط |
|---|---|
| high | وجود بيانات حسّاسة (خيار حسّاس بدرجة ≤ ٤) **أو** النتيجة الكلية < ٤٠ |
| low | النتيجة الكلية ≥ ٧٥ |
| medium | ما عدا ذلك |

وجود بيانات حسّاسة يفرض مراجعة بشرية ومسارًا آمنًا قبل أي ربط.

## خريطة التوصية (أول تطابق يفوز — أسرع قيمة)

كل توصية مربوطة بمعرّف حقيقي في `service_catalog` وتحمل مستوى دليل (L1) وسببًا.

| الشرط | المعرّف | الاسم |
|---|---|---|
| overall < ٤٥ | `free_mini_diagnostic` | التشخيص المجاني المختصر |
| follow_up_maturity < ٥٥ ووجود leads | `revenue_proof_sprint_499` | سبرنت إثبات الإيرادات (٤٩٩ ر.س) |
| automation_readiness < ٥٠ | `data_to_revenue_pack_1500` | حزمة من البيانات إلى الإيراد (١٥٠٠ ر.س) |
| overall ≥ ٧٥ و automation ≥ ٦٥ | `executive_command_center_7500` | غرفة قيادة الإدارة (٧٥٠٠ ر.س / شهر) |
| ما عدا ذلك | `growth_ops_monthly_2999` | عمليات النمو الشهرية (٢٩٩٩ ر.س / شهر) |

> «وجود leads» = درجة محور lead_flow ≥ ٤. أول workflow ثابت: `Lead intake → مسودة متابعة → موافقة → إرسال يدوي → تقرير`.

## نموذج التقرير

من نقطة `GET /assessment/{id}/report`:

```
# تقرير جاهزية — شركة مثال
- النتيجة الكلية: **62/100**
- جاهزية الإيراد: 68
- نضج المتابعة: 45
- جاهزية الأتمتة: 60
- المخاطرة: medium

## التوصية: سبرنت إثبات الإيرادات (٤٩٩ ر.س)
- عندكم leads لكن المتابعة تتسرّب.
- أفضل قيمة سريعة هي كشف التسرّب قبل بناء نظام كبير.
- أول مخرج خلال 7 أيام.

**أول workflow:** Lead intake → مسودة متابعة → موافقة → إرسال يدوي → تقرير
**الخطوة القادمة:** ابدأ Sprint إثبات الإيرادات (7 أيام).

## الصلاحيات المطلوبة
- L2: قراءة leads من CRM (بعد موافقة)
- L1: ملف/رابط يرسله العميل
- L1: رابط حجز التقويم
- L4: إرسال متابعة بعد موافقة صريحة
```

الخطوة القادمة تختلف حسب التوصية: التشخيص المجاني (٢٤ ساعة)، سبرنت الإيرادات (٧ أيام)، حزمة البيانات (١٤ يوم)، أو مكالمة ١٠ دقائق لتأكيد النطاق.

> القيمة التقديرية ليست قيمة مُتحقَّقة / Estimated value is not Verified value.
