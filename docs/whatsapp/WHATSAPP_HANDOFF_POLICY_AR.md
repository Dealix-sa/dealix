# التحويل البشري — متى وكيف نُصعّد لشخص

المساعد محكوم، وعند حدوده نُحوّل لشخص (مؤسس/مسؤول نجاح العميل) مع سياق كامل. المنطق في `handoff_router.py`، والتجاوزات في `conversation_state.py`.

## أسباب التحويل التسعة

تُقيَّم بالترتيب؛ أول تطابق يفوز.

| # | السبب | المعنى | الإجراء المقترح |
|---|---|---|---|
| 1 | explicit_request | العميل طلب التحدث مع شخص. | اتصال ١٠ دقائق. |
| 2 | secrets_attempt | محاولة إرسال مفتاح/سر في الشات — مراجعة أمنية. | تأكيد عدم تسجيل السر + إرشاد للبوابة الآمنة. |
| 3 | permission_l5 | صلاحية حسّاسة (L5) لا تتم عبر واتساب وحده. | إكمال عبر بوابة آمنة + تأكيد بشري. |
| 4 | complaint | شكوى تحتاج تدخلًا بشريًا. | اتصال خلال ساعة + تلخيص المشكلة. |
| 5 | sensitive_data | وجود بيانات حسّاسة يحتاج مراجعة بشرية. | مراجعة DPA + مسار آمن قبل أي ربط. |
| 6 | pricing_negotiation | تفاوض على السعر — خارج صلاحية المساعد. | مكالمة مؤسس لمناقشة النطاق والسعر. |
| 7 | contract | طلب يتعلق بعقد — يحتاج مراجعة بشرية/قانونية. | تحويل لمراجعة العقد + توقيع DPA. |
| 8 | ambiguity | غموض في الطلب يحتاج توضيحًا بشريًا. | اتصال ١٠ دقائق لتوضيح الهدف. |
| 9 | repeated_unknown | تكرار عدم فهم الطلب — نصعّد بدل تكرار القوائم. | اتصال قصير لفهم الاحتياج. |

> ترتيب الاكتشاف في الكود: طلب صريح ← محاولة سرّ ← L5 ← شكوى ← بيانات حسّاسة ← تفاوض سعر (عند `proposal`/`payment_handoff` مع `reject`/`edit`) ← تكرار unknown (مرتين فأكثر).

## متى تُكتشف الأسباب

- **explicit_request:** نية `human_handoff` (زر «أحتاج شخص» أو كلمات «أحتاج شخص/كلّموني/اتصلوا»).
- **secrets_attempt:** حارس السياسة يكتشف مفتاحًا/سرًا في النص.
- **permission_l5:** أي صلاحية بمستوى L5.
- **complaint:** علامة الشكوى في الرسالة الواردة.
- **sensitive_data:** خيار حسّاس في فحص الجاهزية.
- **pricing_negotiation:** عند مرحلتي `proposal`/`payment_handoff` مع نية `reject`/`edit`.
- **repeated_unknown:** تكرار عدم الفهم (≥ ٢).

## حزمة التحويل (Handoff Packet)

عند التحويل تُبنى حزمة (`HandoffRequest`) تُسلَّم لإنسان:

| الحقل | المحتوى |
|---|---|
| `summary_ar` | ملخص: العميل + المرحلة + سبب التحويل. |
| `last_messages` | آخر الرسائل (حتى ١٠، كل رسالة مقتطعة عند ٢٨٠ حرفًا). |
| `suggested_action_ar` | الإجراء المقترح حسب السبب. |
| `urgency` | درجة الإلحاح (low/medium/high). |
| `reason` | أحد الأسباب التسعة. |

مثال على الملخص:

```
عميل (شركة مثال) عند مرحلة «proposal» — تفاوض على السعر — خارج صلاحية المساعد.
```

رسالة العميل عند التحويل:

```
حوّلتك لشخص من فريق Dealix مع ملخّص محادثتك. بنرجع لك سريعًا.
```

## حفظ السياق بلا PII

- الحزمة تحفظ السياق وآخر الرسائل، لكن السجلات تستخدم معرّفات معتمة فقط (`client_handle`).
- علامة الأمان على الحزمة: `context_preserved_no_pii_in_logs`.
- قيم الأسرار لا تدخل الحزمة ولا السجلات إطلاقًا.

## نقطة الـ API

`POST /handoff/human` تفرض تحويلًا بشريًا مع السياق؛ السبب الافتراضي `explicit_request`، وإن لم يكن السبب ضمن الأسباب التسعة يُستبدل بـ `explicit_request`. المرحلة `human_handoff` نهائية حتى يستأنف إنسان.
