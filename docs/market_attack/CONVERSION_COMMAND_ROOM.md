# Conversion Command Room

> The weekly room where the founder sees the entire funnel in one
> place: where signals come from, where they get stuck, where deals
> close, where they die.

## Inputs

| Source                                            | Used for                       |
| ------------------------------------------------- | ------------------------------ |
| `campaigns/campaign_results.csv`                  | top funnel by channel          |
| `market_attack/offer_market_fit_tests.csv`        | which offers survived          |
| `market_attack/strategic_accounts.csv`            | named accounts in motion       |
| `market_attack/objection_library.csv`             | what kills deals               |
| `sales_assets/sales_asset_registry.csv`           | which assets shipped this week |
| `partners/partner_pipeline.csv`                   | partner-sourced motion         |
| `authority/founder_posts.csv`                     | authority loop status          |

## Output report

`<PRIVATE_OPS>/market_attack/conversion_command_room.md`, generated by
`scripts/generate_campaign_command_report.py`. Sections:

1. **Top of funnel** — impressions / clicks by channel (read-only).
2. **Signal stage** — positive_replies by channel × sector.
3. **Proposal stage** — proposals sent, by offer and sector.
4. **Payment stage** — payments collected, by offer (always quoted
   with the customer's own confirmation, never auto-claimed).
5. **Frictions** — top 5 objections in the last 7 days.
6. **Next 5 actions** — discrete tasks for the founder for the coming
   week.

## Decision triggers

- Channel produced ≥ 3 positive replies in 14 days → mark `scale`.
- Channel had ≥ 30 sends and 0 positive replies → mark `kill`.
- Asset cited in ≥ 3 positive replies → promote in
  `sales_asset_registry.csv` (`status = champion`).
- Asset cited in ≥ 3 objections → flag in `objection_library.csv`
  (`asset_needed = revise`).
