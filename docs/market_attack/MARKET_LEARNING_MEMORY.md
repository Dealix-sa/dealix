# Market Learning Memory

> Dealix's compounding asset. Every test, win, loss, objection,
> partner conversation, and pricing experiment gets logged here. Memory
> is what turns 90-day cycles into a strategic moat.

## Structure

For each entry, append to
`<PRIVATE_OPS>/market_attack/market_learning_memory.md`:

```
## YYYY-MM-DD — <one-line headline>

- Context:       (sector / offer / channel / account)
- What we did:   (hypothesis, experiment)
- What happened: (observed signal, numbers)
- Why:           (hypothesis on cause)
- Decision:      (scale / fix / kill / hold)
- Asset created: (path, if any)
- Re-test at:    (date)
```

## Sources that feed memory

- Offer-market fit test postmortems.
- Campaign postmortems (`CAMPAIGN_POSTMORTEM_SYSTEM.md`).
- Lost-deal interviews.
- Won-deal interviews.
- Partner conversations.
- Authority post performance (replies and DMs, not vanity metrics).

## Read pattern

- Before launching a new campaign in a sector, founder must read the
  last 5 entries in that sector. The verifier does **not** enforce
  this, but the playbook does.
- Before changing a price or contract, founder must read all entries
  tagged `pricing` or `objection`.

## Doctrine

Memory entries are **facts and decisions**, not claims. The verifier
flags entries containing "guaranteed", "always", "every customer",
"100%" — these are not knowledge, they are marketing.
