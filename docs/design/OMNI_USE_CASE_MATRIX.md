# Dealix Design OS Omni Use Case Matrix

## Purpose

This matrix shows how to apply the Dealix Design Command Room OS across business, product, sales, delivery, trust, and engineering.

| Use case | Template | Generated artifact | Next owner action |
|---|---|---|---|
| Daily pipeline clarity | `revenue-command-room` | `reports/design/revenue-command-room-*.md` | Approve drafts, pick next 10 actions |
| Founder daily operating view | `founder-war-room` | `reports/design/founder-war-room-*.md` | Decide the highest-value action |
| Client diagnosis handoff | `client-proof-pack` | `reports/design/client-proof-pack-*.md` | Review assumptions and proof |
| Prospect pitch | `sales-deck` | `reports/design/sales-deck-*.md` | Convert into deck/PDF after claims review |
| Website copy/design | `landing-page` | `reports/design/landing-page-*.md` | Promote to isolated web preview after approval |
| Multi-channel growth ops | `client-growth` | `reports/design/client-growth-*.md` | Review channel actions before any send |
| Delivery workflow control | `delivery-os` | `reports/design/delivery-os-*.md` | Move client to scope/blueprint/proof pack |
| Executive intelligence | `company-brain` | `reports/design/company-brain-*.md` | Convert signals into decisions |

## Recommended cadence

| Cadence | Action |
|---|---|
| Daily | Generate `revenue-command-room` or `founder-war-room` |
| Weekly | Generate `sales-deck`, `landing-page`, and `client-proof-pack` |
| Per client | Generate `delivery-os` and `client-growth` |
| Before release | Review generated artifact against claims/safety/production handoff docs |

## Practical rule

If the next action is unclear, generate a draft artifact. If the artifact is unsafe or unsupported by proof, keep it internal.
