# Dealix Sales Deck Artifact Spec

## Goal

Create a clear, client-ready Dealix sales deck that explains the outcome, method, pilot scope, proof plan, and next step.

## Required sections

1. Client pain
2. Why now
3. What Dealix does
4. The specific operating system proposed
5. Before / after workflow
6. Pilot scope
7. Implementation timeline
8. Proof pack and review cadence
9. Commercial model
10. Next step

## Claims rule

Use source-backed or assumption-labeled claims only. Do not use guaranteed ROI, fake logos, fake testimonials, or unverifiable market dominance claims.

## Output recommendation

Start as markdown in `reports/design/sales-deck-v0.md`, then export manually or through an approved document pipeline.
