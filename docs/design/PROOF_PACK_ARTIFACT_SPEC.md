# Client Proof Pack Artifact Spec

## Goal

Create a client-facing proof artifact that shows what Dealix diagnosed, prepared, built, validated, and recommends next.

## Primary user

Client decision maker, founder, head of sales, operations lead, or transformation owner.

## Required sections

1. Client context
2. Current workflow pain
3. What Dealix prepared
4. What was reviewed
5. What was validated
6. Before/after map
7. Risks and assumptions
8. Next 30 days
9. Approval and handoff

## Required safety checks

- no fake result claims
- assumptions labeled
- sensitive data removed or masked
- approval state visible
- generated vs verified distinction clear

## Output recommendation

Start as markdown in `reports/proof/<client>/PROOF_PACK_ARTIFACT.md` or `reports/design/client-proof-pack-v0.md`.
