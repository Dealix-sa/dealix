# Dealix Design Claims Review

Use this before approving any design artifact for sales, client delivery, public website, or production UI.

## Blocked claims

- guaranteed revenue
- guaranteed ROI
- fake customer outcomes
- fake testimonials
- fake client logos
- official partnership claims without proof
- hidden automation claims
- claims that imply live outbound is enabled when it is not

## Safer alternatives

Use:

- designed to help
- can reduce manual follow-up work
- draft-first workflow
- reviewed before external send
- pilot before scale
- proof pack generated after validation
- source-backed recommendation

## Approval statement

```text
Claims reviewed:
Proof available:
Assumptions labeled:
External-send implication checked:
Approved by:
Date:
```
