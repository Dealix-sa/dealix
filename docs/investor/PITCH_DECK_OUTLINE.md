# Pitch Deck Outline

The canonical structure for any Dealix deck shared externally.

## Slides
1. Cover.
2. The problem in Saudi B2B (plain language).
3. The shift making the solution possible now.
4. Dealix — what it is and how it works.
5. The offer ladder.
6. Proof — case studies, sector reports, named results.
7. Traction — cash collected, MRR trend, retainer count.
8. Trust posture — Claim Guard, approval matrix, audit.
9. Market — sectors, size, where Dealix focuses.
10. Roadmap — 12 / 24 months.
11. Team — founder + named delegation triggers.
12. The ask — what we want from this conversation.

## Production rules
- One concept per slide.
- One chart per slide where data is shown.
- Sources cited for every external claim.
- No banned phrases from `CLAIM_GUARD.md`.

## Storage
- Latest deck in `dealix-ops-private/investor/decks/`.
- Public extract (if any) in `docs/public/`.

## Rule
A deck is a conversation tool, not a brochure. If a slide cannot be defended in one sentence, cut it.
