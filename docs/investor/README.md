# Investor

## Purpose
Define how Dealix talks to current and prospective investors so updates are honest, evidence-based, and trust-safe.

## Owner
Sami / Founder CEO.

## Review Cadence
Monthly, plus before any investor conversation.

## Inputs
- Cash collected and MRR.
- Pipeline value and best next close.
- Delivery proof (samples, reports, retainers).
- Learning loop summaries.
- Risk and incident log.

## Outputs
- Investor update template.
- Confidential investor data room (private, not stored here).
- Pre-call briefs.
- Post-call action log.

## Rules
- Investor updates use the same evidence the founder uses internally - no embellishment.
- No guaranteed revenue or compliance claims appear in any investor material.
- Confidential customer or financial data goes to the data room, never to this public folder.
- Material changes to the company story require A2 founder approval before being shared externally.

## Metrics
- Updates sent on cadence.
- Time from investor question to evidence-backed answer.
- Investor commitments tracked against actuals.

## Evidence
- update template in this folder.
- private data room (external).
- meeting notes (private).
- approval log entries for external commitments.
