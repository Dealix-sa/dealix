# معالجة بيانات العملاء
# Client Data Handling

---

## المبدأ — Principle

بيانات العميل تُستخدم فقط للخدمة المتفق عليها. لا تُشارَك، لا تُحلَّل خارج السياق المصرَّح به، لا تبقى بعد الحاجة.

Client data is used only for the agreed service. It is not shared, not analyzed outside the authorized context, and not retained beyond necessity.

**الامتثال:** نظام حماية البيانات الشخصية السعودي (PDPL) — المرسوم الملكي م/19، 1443هـ.

---

## القسم الأول — البيانات المسموح بجمعها

**بيانات تشغيلية مسموح بها:**

| نوع البيانات | الغرض | الأساس القانوني |
|------------|-------|----------------|
| اسم الشركة + القطاع | تخصيص Sprint | تنفيذ العقد |
| بيانات العملاء التجاريين (B2B) مجمَّعة | تحليل Account Intelligence | تنفيذ العقد |
| بيانات الأداء التشغيلي (مبيعات، فواتير) | إنتاج تقارير القيمة | تنفيذ العقد |
| اسم المسؤول المُفوَّض + بريده | التواصل التعاقدي | مصلحة مشروعة |
| تقييمات العميل ورأيه | تحسين الخدمة | موافقة صريحة |

---

## القسم الثاني — البيانات الممنوعة

**لا تُجمَع هذه البيانات في أي حال:**

- الرقم الوطني أو الإقامة لأي فرد
- البيانات الصحية أو البيومترية
- بيانات الأطفال دون 18
- معلومات حسابات بنكية أو بطاقات ائتمان مباشرة
- بيانات الموقع الجغرافي الدقيق
- أي بيانات S3 (حسب تصنيف `docs/04_data_os/PII_CLASSIFICATION.md`)

إذا أرسل العميل ملفاً يحتوي على هذه البيانات عن طريق الخطأ:
1. إبلاغ العميل فوراً
2. حذف الملف بدون معالجة
3. تسجيل الحادثة في سجل الأمان

---

## القسم الثالث — مدة الاحتفاظ

| نوع البيانات | مدة الاحتفاظ | إجراء ما بعد المدة |
|------------|------------|-------------------|
| بيانات تشغيلية نشطة (Sprint جارٍ) | حتى Sign-Off | أرشفة في ملف العميل |
| بيانات عميل نشط (Managed Ops) | مدة العقد + 12 شهراً | حذف أو أرشفة |
| بيانات عميل منتهي | 12 شهراً | حذف |
| سجلات Proof Pack | 24 شهراً | مراجعة قبل الحذف |
| بيانات DNC | إلى أجل غير مسمى | لا حذف (حق الاعتراض دائم) |

---

## القسم الرابع — حق العميل في الحذف

**الإجراء عند طلب الحذف:**

1. استلام الطلب وتأكيده للعميل خلال 3 أيام عمل
2. تحديد كل مكان تخزين البيانات (ملفات, Notion, أدوات AI)
3. تنفيذ الحذف الكامل خلال 30 يوماً
4. إرسال تأكيد مكتوب للعميل

**استثناءات الحذف:**
- بيانات مطلوبة لامتثال قانوني (وتُعلَّم كذلك)
- بيانات مُجمَّعة تماماً (بدون تحديد هوية)

---

## القسم الخامس — حماية البيانات في بيئة LLM

**القواعد التشغيلية:**

1. **لا بيانات PII في prompts:** أي بيانات شخصية تُزال أو تُرمَّز قبل إدخالها في أي LLM
2. **لا بيانات عميل في نماذج عامة:** البيانات تبقى في بيئة Dealix المحكومة فقط
3. **تسجيل الاستخدام:** كل استخدام لبيانات عميل في LLM يُسجَّل في AI_RUN_LEDGER
4. **حذف السياق:** بعد كل جلسة LLM تستخدم بيانات حساسة، السياق يُحذف
5. **بيانات S3 لا تعبر الحدود:** البيانات ذات الحساسية العالية لا تُرسَل لخوادم خارج المملكة بدون أساس قانوني موثق

**المرجع:** `dealix/masters/constitution.md` — المادة الحادية عشرة (الوضع السعودي)

## القسم السادس — التدريب والوعي

- المؤسس يراجع هذه السياسة كل 6 أشهر
- أي موظف أو متعاون يُضاف يوقّع على الاطلاع على هذه السياسة قبل بدء العمل
- أي وكيل يُنشَّط يُحمَّل هذه السياسة ضمن System Prompt
