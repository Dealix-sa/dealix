> هذا المستند هو نقطة الحقيقة الواحدة لـ "أين نحن، وما الذي يمنع الانطلاق، وماذا أفعل الآن".

## 1) الملخص التنفيذي (٦٠ ثانية)

- **الموقع يبني نظيفًا** (Next.js — كل المسارات) و**الـ API يعمل** (944 مسار).
- **أُزيلت مخالفة جوهرية**: الصفحة الرئيسية كانت فيها **شعارات عملاء وهميين + إحصائيات مختلقة (+500 عميل، 3.2x، 99.9%) + شهادات بأسماء وهمية**. هذا ينتهك مبدأ "لا ادّعاءات ملفّقة" وهو خطر قانوني/سمعة قاتل لشركة شعارها الثقة والامتثال. استُبدِل بمحتوى صادق يبيع بقوة دون كذب.
- **أُضيفت صفحة `/contact`** = "تحدّث معنا / اطلب AI مخصّص" — نموذج يُدخل **ليدز حقيقية موافَق عليها** مباشرةً إلى صندوق المؤسِّس ويعيد رابط الحجز. هذا يحوّل الموقع إلى **قمع توليد ليدز فعلي**.
- **العائق الوحيد المتبقّي للإيراد**: قائمة العملاء الدافئة (`data/warm_list.csv`) **فارغة**. لا يمكن لأي نظام تعبئتها (نمنع الـ scraping عقائديًا) — تأتي من شبكتك أنت. هذه هي خطوتك رقم ١.

## 4) العوائق الجذرية وحلولها

1. **التلفيق في الموقع** (مخالفة عقيدة + خطر قانوني) → ✅ **حُلّ**: استُبدِل بكل المحتوى الصادق.
2. **روابط ميتة في الأسعار/الفوتر** → ✅ **حُلّ**: أُنشئت `/contact` وأُعيد توجيه الروابط.
3. **لا مسار لطلب "AI مخصّص"** (طلبك الصريح) → ✅ **حُلّ**: نموذج `/contact` يغطّي AI مخصّص + تشغيل مُدار + شراكة.
4. **"ليدز حقيقية" بدون scraping** → ✅ **حُلّ هيكليًا**: الموقع الآن يلتقط ليدز واردة موافَقًا عليها؛ والليدز الصادرة تأتي من قائمتك الدافئة (الخطوة ١ أدناه). النظام يسجّل ويرتّب ويجهّز المسودّات — أنت توافق وترسل.
5. **النشر/الدومين/المدفوعات** → 🟡 **مفاتيح بيدك**: الأوامر الدقيقة في القسم ٥.

## 5) ماذا تفعل الآن — إجراءات المؤسِّس (مرتّبة، بأوامرها)

> هذه أشياء **لا يستطيع أي وكيل تنفيذها نيابةً عنك** (تتطلّب مفاتيح/حساباتك/شبكتك). كل أمر جاهز للنسخ.

### الخطوة ١ — عبّئ قائمتك الدافئة (تفتح الإيراد) ⏱️ ١٥ دقيقة
```bash
cp data/warm_list.csv.template data/warm_list.csv
# عبّئ الأعمدة: name, role, company, sector, relationship (warm/active/cold), city, linkedin_url, notes
# ضع ٢٠ اسمًا حقيقيًا من شبكتك (لا scraping — المصدر شبكتك أو مقدمة شخصية فقط)
python3 scripts/warm_list_outreach.py     # → data/outreach/warm_list_drafts.md (مسودّات مؤهَّلة — لا ترسل، تراجَع وتوافق)
```

### الخطوة ٢ — موجزك اليومي (روتين الصباح) ⏱️ ٥ دقائق
```bash
python3 scripts/dealix_founder_daily_brief.py --out data/founder_briefs/today.md
# يطبع: الاختناقات + إجراء اليوم الواحد + حالة العملاء المدفوعين
```

### الخطوة ٣ — انشر الموقع على Railway (مرة واحدة) ⏱️ ٣٠ دقيقة
```bash
# املأ مفاتيح الإنتاج أولًا (APP_SECRET_KEY, DATABASE_URL, ...)
python3 scripts/railway_launch_env_check.py        # تحقّق من اكتمال المفاتيح
bash scripts/railway_prod_bootstrap.sh             # Alembic + بذرة War Room (مرة واحدة)
bash scripts/official_launch_verify.sh             # يجب أن يطبع OFFICIAL_LAUNCH_VERDICT=PASS
```

### الخطوة ٤ — اربط الدومين dealix.me
- DNS: سجّل CNAME/A إلى Railway · فعّل HTTPS · راجع `docs/ops/DOMAIN_OPERATIONS_RUNBOOK.md`.

### الخطوة ٥ — فعّل محتوى الإطلاق (مسودّات جاهزة)
- راجع `docs/commercial/launch_kit/` → انسخ منشورات LinkedIn / الإيميل / الواتساب → **انشر/أرسل بنفسك بعد الموافقة**.

### الخطوة ٦ — المدفوعات الحيّة (فقط عند أول صفقة)
```bash
python3 scripts/moyasar_live_cutover.py            # حوّل من sandbox للوضع الحي — بيدك أنت
```
