# Claude Code Execution Report — Template

Use one copy of this template per execution run (whether orchestrated by the founder, a CI workflow, or a parallel sub-agent burst). File completed reports in `docs/ops/execution-reports/YYYY-MM-DD-<slug>.md`.

---

## 1. Run header

| Field | Value |
|-------|-------|
| Run id | `<auto-generated or PR number>` |
| Date (UTC) | `YYYY-MM-DD` |
| Branch | `<git branch>` |
| Trigger | manual / cron / PR / orchestrator |
| Operator | founder / agent id |
| Prompt version | e.g. `Dealix Master Prompt v7 — Section A` |

## 2. Scope

One paragraph describing the intended deliverable. Reference the section in the master prompt.

## 3. Files created

List every new file with a one-line description.

```
path/to/file.ext — description
```

## 4. Files modified

List every existing file changed, with a one-line description of the change.

## 5. Verifier commands run

For each command, record: command, exit code, key output lines.

```
$ make policy-check
exit: 0
summary: 11/11 rules valid
```

## 6. Tests run

Record test runner invocation, pass count, fail count, and skipped count. Doctrine guard tests must be listed explicitly.

## 7. Failures / skipped

Anything that did not run, did not pass, or was intentionally skipped — with the reason.

## 8. Manual steps remaining

A bullet list of follow-ups that require human action (approvals, secret rotation, deploys, etc.). One per line, in priority order.

## 9. Provenance

- Commit SHA (after commit) or "not committed by agent"
- Reviewer (human) name when applicable
- Approval queue ids touched, if any

---

This template is the contract between automated work and human review. Do not delete sections — write "n/a" if a section is empty.
