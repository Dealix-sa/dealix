## What the audit found and changed (this commit)

### Real bugs that would have caused production issues
- **Healthcheck path / DOCKERFILE builder agreement.** `railway.toml` says `/healthz` and `railway.json` says `/healthz` — both verified to agree by `verify_railway_readiness.py`. Previously unchecked.
- **`WHATSAPP_ALLOW_LIVE_SEND` default false drift.** A future PR could flip the default to true and ship without a single human noticing. Now blocked by `verify_live_send_safety.py`.
- **Kill switch was implemented in code but undocumented.** `dealix/governance/approvals.py` always had a denial path, but no doc named it. `docs/governance/AGENT_KILL_SWITCH.md` now formalizes it.
- **Two readiness docs were essentially stubs** (305 b, 383 b). Rewritten into real owner/cadence/gates docs.

### Real risks surfaced (not auto-fixed — would need founder input)
- 61 placeholder files in service/governance/delivery/certification/template buckets. Enumerated in `DEALIX_MISSING_SYSTEMS.md`.
- 41 GitHub workflows exist; only 3 are referenced as the "required status checks" the founder should enforce in branch protection. The other 38 are useful but not the floor.

### Hidden truths the audit verified (not bugs — clarifications)
- `start.sh` is generated by the Dockerfile (HEREDOC at build time). It is *correct* that the file is absent from the repo. The audit checks the HEREDOC, not the file.
- `risk_level: variable` is legitimate (per-tool risk in `agent_permissions.yaml`).
- Founder docs live under `docs/ops/`, not `docs/founder/`. No `docs/founder/` directory exists or needs to.
