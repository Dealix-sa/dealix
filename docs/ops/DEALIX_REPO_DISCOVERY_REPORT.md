## Quirks future auditors should NOT re-flag as bugs

1. **`start.sh` is not on disk by design.** The Dockerfile generates `/app/start.sh` at build time via `COPY <<'EOF' /app/start.sh` HEREDOC. `verify_wiring.py` and `verify_railway_readiness.py` check for the HEREDOC, not the file.
2. **`risk_level: variable` is legitimate.** The `agent_tool_action` class delegates per-tool risk to `agent_permissions.yaml`. `verify_policy_as_code.py` accepts it.
3. **Two frontend surfaces are intentional.** `frontend/` is the customer-facing app, `apps/web/` is internal control-plane (agents, approvals, safety pages). Both have `package.json` + a Next config.
4. **Founder docs live under `docs/ops/` not `docs/founder/`.** There is no `docs/founder/` directory in this repo.
5. **Banned-claim scanner is context-aware.** It skips lines containing negation cues (❌, "we do not", "Article 8", "claims", etc.) and files whose entire purpose is to enumerate forbidden patterns (e.g. `STOP_DOING.md`, `FORBIDDEN_ACTIONS.md`, `what_we_do_not_do.md`).
