# Capacity Planning System

## Purpose
Prevent Dealix from selling more work than it can deliver.

## Owner
Sami / Founder CEO.

## Capacity Units
- founder sales hours
- research hours
- delivery hours
- QA hours
- trust approval time

## Weekly Capacity Review
| Work Type | Capacity | Used | Remaining |
|---|---:|---:|---:|
| Outbound | | | |
| Samples | | | |
| Proposals | | | |
| Delivery | | | |
| QA | | | |

## Rules
- No more proposals than delivery capacity can support.
- No rushed delivery without QA.
- If delivery capacity is full, raise price or defer start.
- Repeated bottlenecks become delegation candidates.

## Metrics
- delivery hours per sprint
- samples per week
- QA time
- founder bottleneck count
