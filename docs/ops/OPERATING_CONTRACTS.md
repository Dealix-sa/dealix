# Operating Contracts

## Purpose
Define the input/output contracts between Dealix company systems.

## Owner
Sami / Founder CEO.

## Review Cadence
Monthly.

## Contract: Revenue → Delivery
### Revenue Provides
- signed/approved scope
- offer type
- client objective
- start condition
- payment / PO / written approval
- delivery date

### Delivery Produces
- intake
- report
- QA checklist
- handoff
- feedback request

### Rule
Delivery does not start without payment, PO, or written approval.

## Contract: Delivery → Client Success
### Delivery Provides
- final report
- client feedback
- usage signals
- next opportunity

### Client Success Produces
- health score
- retainer ask
- renewal plan
- case study request if approved

## Contract: Trust → All Systems
### Trust Provides
- approval matrix
- safe language
- public/private boundary
- claim guard

### All Systems Must
- avoid unsupported claims
- protect client data
- log A2/A3 decisions
- escalate risky outputs

## Contract: Learning → Productization
### Learning Provides
- repeated issues
- repeated manual workflows
- customer requests
- playbook updates

### Productization Produces
- template candidate
- automation candidate
- SaaS candidate
- build/fix/kill/defer decision

## Contract: Finance → CEO
### Finance Provides
- cash collected
- pipeline value
- expenses
- MRR
- runway
- pricing learnings

### CEO Produces
- capital allocation decision
- pricing decision
- kill/defer decision
