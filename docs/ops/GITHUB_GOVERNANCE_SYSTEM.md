# GitHub Governance System

## Purpose
Use GitHub Actions and branch protection to make Dealix quality non-negotiable.

## Required Protections
- Require pull request before merging.
- Require status checks to pass.
- Require branches to be up to date.
- Require conversation resolution.
- Do not allow bypassing.
- Enable secret scanning.
- Enable push protection.
- Enable Dependabot alerts.

## Required Checks
- Dealix Implementation Audit
- Dealix Data Architecture Checks
- Dealix Execution Assurance Checks
- Dealix Revenue Ops Checks
- Dealix Delivery Client Success Checks
- Dealix Finance Pricing Checks
- Dealix Trust Compliance AI Risk Checks
- Dealix Brand Proof Content Checks
- Dealix People Partner Checks
- Dealix Productization Engineering Checks
- Dealix Board-Level OS Checks
- Dealix Strategic CEO Checks
- Security Checks

## Rule
No merge to main if required checks fail.
