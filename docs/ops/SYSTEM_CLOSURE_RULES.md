# System Closure Rules

## Purpose
Define when a Dealix system is considered complete enough for the current stage.

## Closure Levels

### C0 — Not Started
No file, no owner, no evidence.

### C1 — Documented
System has operating document.

### C2 — Templated
System has template/checklist.

### C3 — Verified
System has verifier and passes checks.

### C4 — Used
System has been used in private ops.

### C5 — Produces Result
System produced market, revenue, delivery, trust, or learning result.

## Closure Rule
A system can be called "done for now" only at C4 or C5.

## Current Stage Closure Requirements
| System | Minimum Closure |
|---|---:|
| Revenue | C5 |
| Trust | C4 |
| Delivery | C4 |
| CEO Command | C4 |
| Learning | C4 |
| Finance | C3 |
| Productization | C2 |

## Rules
- Do not polish C2 systems while Revenue is below C5.
- Do not automate systems below C4.
- Do not scale systems below C5.
