# Weekly Learning Governance

## Purpose
Force Dealix to improve from real operating evidence every week.

## Owner
Sami / Founder CEO.

## Review Cadence
Weekly.

## Inputs
- weekly metrics
- replies
- calls
- proposals
- payments
- delivery QA
- trust escalations
- founder bottlenecks

## Outputs
- one learning decision
- one updated playbook/checklist/template/policy
- one next-week experiment

## Rules
- The week is not closed without a learning decision.
- The week is not closed without one system update or an explicit reason.
- Repeated issues must become checklist/template/playbook changes.
- Metrics are used to improve the system, not to decorate reports.

## Metrics
- learning decisions per week
- playbook updates
- repeated issues reduced
- conversion improvement
- delivery rework reduction

## Evidence
- weekly_reviews/YYYY-MM-DD.md
- git commit
- updated file
- decision log
