# إيقاع التشغيل الأسبوعي — Weekly Operating Rhythm — Dealix

---

## Arabic / عربي

### المبدأ
Dealix تعمل بإيقاع مدروس لا بردود أفعال عشوائية. كل يوم له غرض. كل أسبوع له ناتج. كل شهر له مراجعة.

---

### الإيقاع اليومي (كل يوم عمل)

| الوقت | المهمة | المخرج |
|---|---|---|
| 08:30 | تشغيل الحزمة التجارية اليومية | تقرير pipeline + أولويات اليوم |
| 09:00 | مراجعة قائمة الانتظار (approval queue) | قرارات: موافقة / تعديل / رفض |
| 09:30 | تحديث CRM بالحالة الحالية | سجل دقيق لكل عميل محتمل |
| طوال اليوم | ردود على الإيميل والواتساب | خلال 24 ساعة في أيام العمل |
| 17:00 | توثيق أي قرار اتُّخذ اليوم | سجل القرارات |

**قاعدة:** لا إجراء تجاري يتم خارج هذا الإيقاع (لا إرسال عشوائي، لا عروض بدون قرار).

---

### الأسبوع — يوماً بيوم

#### الاثنين — موجز الرئيس التنفيذي + مراجعة المسار

**الهدف:** وضوح الأولويات قبل أي تنفيذ.

**المهام:**
- [ ] مراجعة تقرير الأسبوع الماضي (pipeline، إيرادات، تسليمات)
- [ ] تحديد ثلاثة أولويات للأسبوع الحالي
- [ ] مراجعة حالة كل عميل نشط (تسليم في الوقت؟ عقبات؟)
- [ ] مراجعة حالة كل فرصة في المسار (اتصال؟ عرض؟ انتظار قرار؟)
- [ ] تحديد القرار الواحد الأكثر أهمية هذا الأسبوع

**المخرج:** موجز الاثنين (وثيقة داخلية، 1 صفحة).

---

#### الثلاثاء — مراجعة مسودات التواصل + إعداد العروض

**الهدف:** ضمان جودة كل تواصل خارجي قبل إرساله.

**المهام:**
- [ ] مراجعة مسودات التواصل المعلّقة (إيميل، رسائل)
- [ ] الموافقة أو التعديل أو الرفض لكل مسودة
- [ ] إعداد أي عرض تقديمي مطلوب هذا الأسبوع
- [ ] التحقق من اكتمال وثائق النطاق للعروض الجديدة

**قاعدة الموافقة:** لا يُرسل أي تواصل خارجي بدون مراجعة المؤسس.

---

#### الأربعاء — مكالمات الاكتشاف + التحضير للاجتماعات

**الهدف:** تنفيذ الاجتماعات بجودة عالية.

**قبل كل اجتماع:**
- [ ] مراجعة ملف العميل وآخر تواصل
- [ ] تحضير الأسئلة الرئيسية (بناءً على سكريبت الاكتشاف)
- [ ] تحديد الهدف الواحد من الاجتماع

**بعد كل اجتماع:**
- [ ] توثيق الملاحظات خلال 30 دقيقة
- [ ] تحديد الخطوة التالية وتاريخها
- [ ] إرسال ملخص الاجتماع للعميل خلال 24 ساعة

---

#### الخميس — تسليم العروض + تنفيذ المتابعة

**الهدف:** تحريك الفرص المفتوحة نحو قرار.

**المهام:**
- [ ] إرسال العروض الجاهزة (بعد مراجعة المؤسس)
- [ ] تنفيذ متابعات اليوم 3 واليوم 7 المجدولة
- [ ] تحديث حالة كل فرصة في CRM
- [ ] تسليم أي مخرجات عملاء مستحقة هذا اليوم

---

#### الجمعة — التقرير الأسبوعي + مراجعة المؤشرات + تخطيط الأسبوع القادم

**الهدف:** إغلاق الأسبوع بوضوح واستعداد.

**المهام:**
- [ ] توليد التقرير الأسبوعي (pipeline، إيرادات، تسليمات، عقبات)
- [ ] مراجعة مؤشرات الأداء الأسبوعية:
  - عدد اجتماعات الاكتشاف
  - عدد العروض المرسلة
  - معدل تحويل العروض إلى عقود
  - عدد العملاء النشطين
- [ ] توثيق درس واحد من هذا الأسبوع
- [ ] تحديد أولويات الأسبوع القادم

**ملاحظة:** التقرير الأسبوعي للاستخدام الداخلي — لا يُرسل تلقائياً.

---

### الإيقاع الشهري

| الأسبوع | المهمة الرئيسية |
|---|---|
| الأسبوع الأول | إنتاج حزم الدليل للعملاء الذين أكملوا الشهر الأول |
| الأسبوع الثاني | محادثات التجديد مع العملاء في نهاية العقد |
| الأسبوع الثالث | مراجعة الأداء التجاري الشهري (إيرادات، عقود، خسائر) |
| الأسبوع الرابع | مذكرة الإدارة الداخلية + تخطيط الشهر القادم |

**مراجعة شهرية إضافية:**
- مراجعة أسعار العروض (هل التسعير الحالي مناسب للسوق؟)
- مراجعة جودة البيانات المُستخدمة
- مراجعة امتثال PDPL (هل بيانات المنتهية صلاحيتها حُذفت؟)

## English

### Principle
Dealix operates on a deliberate rhythm, not reactive improvisation. Every day has a purpose. Every week has an output. Every month has a review.

---

### Daily Rhythm (every working day)

| Time | Task | Output |
|---|---|---|
| 08:30 | Run daily commercial pack | Pipeline report + today's priorities |
| 09:00 | Review approval queue | Decisions: approve / revise / reject |
| 09:30 | Update CRM with current status | Accurate record for every prospect |
| Throughout | Email and WhatsApp replies | Within 24 hours on working days |
| 17:00 | Document any decision made today | Decision log |

**Rule:** No commercial action happens outside this rhythm (no random outreach, no proposals without a decision).

---

### The Week — Day by Day

#### Monday — CEO Brief + Pipeline Review + Priority Setting

**Objective:** Clarity of priorities before any execution.

**Tasks:**
- [ ] Review last week's report (pipeline, revenue, deliveries)
- [ ] Identify three priorities for the current week
- [ ] Review status of each active client (on time? blockers?)
- [ ] Review status of each pipeline opportunity (contact? proposal? awaiting decision?)
- [ ] Identify the single most important decision this week

**Output:** Monday brief (internal document, 1 page).

---

#### Tuesday — Outreach Draft Review + Proposal Preparation

**Objective:** Ensure quality of all external communications before sending.

**Tasks:**
- [ ] Review pending outreach drafts (email, messages)
- [ ] Approve, revise, or reject each draft
- [ ] Prepare any proposal needed this week
- [ ] Verify scope documents are complete for new proposals

**Approval rule:** No external communication is sent without founder review.

---

#### Wednesday — Discovery Calls + Meeting Preparation

**Objective:** Execute meetings at high quality.

**Before each meeting:**
- [ ] Review client file and last communication
- [ ] Prepare primary questions (based on discovery script)
- [ ] Identify the single objective for the meeting

**After each meeting:**
- [ ] Document notes within 30 minutes
- [ ] Identify next step and its date
- [ ] Send meeting summary to client within 24 hours

---

#### Thursday — Proposal Delivery + Follow-up Execution

**Objective:** Move open opportunities toward a decision.

**Tasks:**
- [ ] Send ready proposals (after founder review)
- [ ] Execute scheduled Day 3 and Day 7 follow-ups
- [ ] Update status of each opportunity in CRM
- [ ] Deliver any client outputs due today

---

#### Friday — Weekly Report + KPI Review + Next Week Planning

**Objective:** Close the week with clarity and readiness.

**Tasks:**
- [ ] Generate weekly report (pipeline, revenue, deliveries, blockers)
- [ ] Review weekly performance indicators:
  - Number of discovery meetings held
  - Number of proposals sent
  - Proposal-to-contract conversion rate
  - Number of active clients
- [ ] Document one lesson from this week
- [ ] Set priorities for next week

**Note:** Weekly report is for internal use — not sent automatically.

---

### Monthly Rhythm

| Week | Primary Task |
|---|---|
| Week 1 | Produce proof packs for clients completing their first month |
| Week 2 | Renewal conversations with clients approaching contract end |
| Week 3 | Monthly commercial performance review (revenue, contracts, losses) |
| Week 4 | Internal management memo + next month planning |

**Additional monthly reviews:**
- Pricing review (is current pricing appropriate for market?)
- Data quality review for active systems
- PDPL compliance review (have expired-retention records been deleted?)
