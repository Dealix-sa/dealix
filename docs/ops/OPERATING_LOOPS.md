# Dealix Operating Loops

The company does not run on tasks. It runs on loops. Anything that does
not live inside one of these loops is, by definition, drift.

## Revenue Loop
Lead → Reply → Call → Payment → Delivery → Proof → Retainer

## Delivery Loop
Intake → Research → Score → Draft → QA → Handoff → Feedback

## Trust Loop
Classify → Guard → Approve → Log → Review → Improve

## Learning Loop
Experiment → Measure → Learn → Decide → Update

## Product Loop
Customer Signal → Feature Intake → Build/Fix/Kill/Defer → Ship → Measure

## CEO Loop
Brief → Decide → Approve → Focus → Review

## Content Loop
Insight → Draft → Approve → Publish → Measure → Learn

## Client Success Loop
Deliver → Feedback → Health Score → Upsell/Renew → Retain

## Loop Contract

Each loop must declare, in `docs/ops/OPERATING_METRICS_CONTRACT.md`:

- input metric
- output metric
- quality metric
- risk metric
- review cadence
