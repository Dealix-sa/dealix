# Operating Loops

## Purpose
Define the five compounding loops that run Dealix as a Revenue Company Operating System.

## Owner
Sami / Founder CEO.

## Review Cadence
Weekly until stable, then monthly.

## Inputs
- Lead activity.
- Delivery feedback.
- Trust escalations.
- Learning reviews.
- Founder decisions.

## Outputs
- Clear weekly cadence.
- Loop owners and steps.
- Bottleneck visibility per loop.

## Loops

### Revenue Loop
Lead → Reply → Call → Proposal → Payment → Delivery → Proof → Retainer.
Owner: Sami.
Cadence: daily activity, weekly review.

### Delivery Loop
Intake → Research → Score → Draft → QA → Handoff → Feedback.
Owner: Sami / Delivery owner.
Cadence: per engagement, weekly QA review.

### Trust Loop
Classify → Guard → Approve → Log → Review → Improve.
Owner: Sami / Trust owner.
Cadence: continuous, weekly review.

### Learning Loop
Experiment → Measure → Learn → Decide → Update.
Owner: Sami / Learning owner.
Cadence: weekly intelligence review.

### CEO Loop
Brief → Decide → Approve → Focus → Review.
Owner: Sami.
Cadence: daily brief, weekly review.

## Rules
- Every loop must have an owner.
- Every loop must produce evidence per cycle.
- Bottlenecks in any loop must surface in the weekly review.
- Loops must not be skipped to chase short-term wins.
- AI prepares, workflows execute, trust governs, founder approves.

## Metrics
- Revenue Loop conversion rates by stage.
- Delivery Loop QA pass rate and on-time rate.
- Trust Loop A3 blocks and incidents.
- Learning Loop decisions per week.
- CEO Loop brief completion rate.

## Evidence
- Pipeline tracker.
- Delivery QA checklists.
- Approval log.
- Weekly intelligence review.
- Daily command brief.
