# Operating Loops

## Purpose
Define the five repeating loops that make Dealix operate as a company instead of a project.

## Owner
Sami / Founder CEO.

## Review Cadence
Weekly during the first 90 days, then monthly.

## Inputs
- Revenue activity (leads, replies, calls, proposals, payments).
- Delivery activity (intake, research, scoring, drafts, QA, handoff, feedback).
- Trust activity (classify, approve, log, escalate).
- Learning activity (experiments, win/loss, sector data, message data).
- Founder activity (briefs, decisions, approvals, focus).

## Outputs
- A repeating motion across revenue, delivery, trust, learning, and CEO time.
- Clear handoffs between loops.
- Visible bottlenecks per loop.
- Evidence that the company is running, not improvising.

## The Five Loops

### Revenue Loop
Lead → Reply → Call → Payment → Delivery → Proof → Retainer

Goal: turn cold attention into recurring cash.

### Delivery Loop
Intake → Research → Score → Draft → QA → Handoff → Feedback

Goal: make every delivery useful, accurate, and trust-safe.

### Trust Loop
Classify → Guard → Approve → Log → Review → Improve

Goal: prevent claims, actions, and automations from outrunning evidence.

### Learning Loop
Experiment → Measure → Learn → Decide → Update

Goal: convert weekly results into system changes (ICP, message, pricing, playbook, productization).

### CEO Loop
Brief → Decide → Approve → Focus → Review

Goal: keep the founder spending time on the highest-leverage move every day.

## Rules
- Every operating action must belong to a loop. No off-loop work.
- No loop runs without producing evidence (file, log, decision).
- Loops feed each other: Delivery feedback enters the Learning Loop; Learning decisions update the Revenue and Delivery Loops; Trust gates every external commitment.
- The CEO Loop is the only loop authorized to approve A2/A3 actions.
- Loops must be observable in the Company State Schema; if a loop's state cannot be read, the loop is broken.

## Metrics
- Revenue Loop: replies per 25 DMs, call rate, proposal rate, payment rate, retainer conversion.
- Delivery Loop: on-time rate, QA pass rate, rework count, client feedback score.
- Trust Loop: A3 blocked actions, approval turnaround time, incidents.
- Learning Loop: learning decisions per week, playbook updates, experiments completed.
- CEO Loop: daily brief streak, decisions logged per week, focus block discipline.
