# Client Onboarding — تأهيل العميل بعد البيع

## Purpose
Define the first 7 days after a signed proposal. The first-week experience determines whether the client treats Dealix as a vendor or as a partner. Operational, not ceremonial.

## Day 0 (signature day)
- Send welcome email: scope, named contact, weekly cadence, escalation channel.
- Create client folder `docs/client_success/clients/<anonymized-label>/`.
- Log signed proposal and advance invoice in registers.
- Create a private channel (e.g., shared workspace or chat group) for working communication.

## Day 1-2 (kickoff)
- 45-minute kickoff call. Agenda fixed:
  1. Confirm objectives and definitions of success.
  2. Confirm data access, contacts, and decision-makers.
  3. Walk through delivery lifecycle per [`docs/delivery/DELIVERY_LIFECYCLE.md`](../delivery/DELIVERY_LIFECYCLE.md).
  4. Confirm weekly report cadence and standing meeting time.
  5. Confirm escalation path.
- Issue kickoff summary email within 24 hours, including any open items and owners.

## Day 3-5 (data + readiness)
- Receive any client-provided data; classify per `docs/governance/PDPL_DATA_RULES.md`.
- Redact and store per `docs/governance/PII_REDACTION_POLICY.md`.
- Run readiness check per `docs/delivery/RETAINER_READINESS.md` or `docs/delivery/DEMO_READINESS.md` depending on engagement.
- Identify any blockers; escalate in writing.

## Day 6-7 (first cycle starts)
- First working deliverable started.
- First weekly report drafted per [`WEEKLY_REPORT_TEMPLATE.md`](WEEKLY_REPORT_TEMPLATE.md).
- Confirm health score baseline per [`CLIENT_HEALTH_SCORE.md`](CLIENT_HEALTH_SCORE.md).

## First-week checklist
- [ ] Signed proposal stored
- [ ] Advance invoice issued and paid
- [ ] Kickoff call held; summary sent
- [ ] Client folder created
- [ ] Data access confirmed and classified
- [ ] Working channel created
- [ ] Weekly cadence agreed
- [ ] Escalation path agreed
- [ ] Readiness check completed
- [ ] Baseline health score recorded
- [ ] First deliverable started

## Rules
- No work starts before advance payment clears unless explicitly approved by founder in writing.
- No PII handled outside the redaction policy.
- No undisclosed third-party subcontracting.

## Evidence
- Onboarding checklist stored in client folder; each line links to artefact.

## Owner & cadence
- Owner: Founder until Client Success hired.
- Cadence: per new client signature.

## القسم العربي

**الغرض:** أول 7 أيام بعد التوقيع تحدد ما إذا كان العميل يعاملنا كمورد أو شريك.

**يوم 0:** بريد ترحيب يحدد النطاق وقناة التواصل ومسار التصعيد. إنشاء مجلد العميل. تسجيل العقد والفاتورة.

**يوم 1-2:** مكالمة انطلاق 45 دقيقة بأجندة ثابتة. ملخص مكتوب خلال 24 ساعة.

**يوم 3-5:** استلام بيانات العميل وتصنيفها وتنقيتها وفق سياسات الحوكمة. فحص الجاهزية. تصعيد المعوقات كتابيًا.

**يوم 6-7:** بدء أول مُخرج، صياغة أول تقرير أسبوعي، تسجيل نقاط الصحة المرجعية.

**القواعد:** لا عمل قبل دفع المقدّم إلا بإذن مكتوب. لا PII خارج سياسة التنقية. لا تعاقد فرعي غير معلن.

**المالك:** المؤسس. **الإيقاع:** عند كل عميل جديد.
