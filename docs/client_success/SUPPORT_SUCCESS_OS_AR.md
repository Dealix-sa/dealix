# نظام دعم ونجاح العميل

> النسخة الإنجليزية القانونية: [`SUPPORT_SUCCESS_OS.md`](./SUPPORT_SUCCESS_OS.md).

## مرجع الدوكترين
- الالتزامات: #1، #2، #5.
- القرارات المثبّتة: الموافقة-أولًا.

## الغرض

دعم العملاء بعد التسليم وحماية الاحتفاظ. الدعم هنا مصدر بيانات الاحتفاظ والملاحظات وطلبات الـ retainer ومرشّحات الإثبات.

## الطوابير

| الطابور | المصدر | SLA |
|--------|--------|-----|
| أسئلة التهيئة | عميل جديد نشط | نفس يوم العمل |
| مشاكل تسليم | تفاعل نشط | 24 ساعة |
| طلبات مراجعة (داخل النطاق) | تفاعل نشط | 48 ساعة |
| طلبات خارج النطاق | فرصة نطاق جديد | 48 ساعة؛ عرض سعر خلال 3 أيام عمل |
| ملاحظات | بعد التسليم | تُسجَّل نفس الأسبوع |
| طلب احتفاظ | ملاحظات قوية + وضوح نطاق | المؤسس يجدول خلال أسبوع |
| مخاطر تجديد | انخفاض health-score | مراجعة نفس اليوم |
| استفسارات استرداد | أي مصدر | مراجعة المؤسس نفس اليوم |

## مدخلات Health Score (لمخاطر التجديد)

- حداثة آخر تفاعل إيجابي.
- عدد مشاكل التسليم المفتوحة.
- عدد طلبات المراجعة المفتوحة.
- زمن منذ آخر ملاحظات.
- زمن منذ آخر محادثة احتفاظ/توسع.
- علم شكوى أو تصعيد.

انخفاض الـ health-score = إشارة، مو حكم. المراجعة تبحث الـ "لماذا" قبل أي فعل.

## القواعد الجوهرية

- لكل عميل نشط مالك علاقة مسمى.
- كل تفاعل يُسجَّل بزمن ونتيجة.
- الدعم ما يأخذ قرارات تسعير أو نطاق أو عقد بدون المؤسس.
- الدعم ينتج بيانات احتفاظ ترجع لغرفة العمليات ودورة الحياة.
- ذكر عميل علنًا يحتاج موافقة كتابية مسجّلة.

## الإيقاع

- يومي: فرز المفتوحات، تصعيد تجاوزات SLA.
- أسبوعي: مراجعة health-score لكل عميل نشط.
- شهري: بطاقة احتفاظ: مخاطر churn، مرشّحات توسع، مرشّحات إثبات.
