# Feedback & Retention System

## Purpose
Turn delivery into feedback, proof, retention, and product learning.

## Owner
Sami / Client Success owner.

## Flow
Delivery → Feedback Request → Health Score → Retainer Ask → Proof Approval → Learning Update

## Feedback Questions
1. Were the opportunities relevant?
2. Was the outreach pack useful?
3. What was the most useful part?
4. What was missing?
5. Would weekly opportunity support help?
6. Can we use anonymized proof?

## Retention Triggers
- client asks for more leads
- client likes lead quality
- client wants help with outreach
- client lacks sales ops
- client wants weekly/monthly support

## Metrics
- feedback received
- positive feedback
- retainer ask sent
- retainer accepted
- proof approved
- repeat work requested

## Retention Reference Metrics
For subscription/retainer motion (used once retainers exist):
- Logo retention = clients retained / clients at start of period.
- GRR (Gross Revenue Retention) = recurring revenue retained,
  excluding expansion.
- NRR (Net Revenue Retention) = recurring revenue retained,
  including expansion and contraction/churn inside the existing base.

NRR above 100% means existing clients are growing faster than they churn.
NRR is the strongest signal that Dealix is delivering compounding value.

## Rules
- Ask for feedback after every delivery.
- Do not ask for public proof before trust is earned.
- Do not push retainer if value is unclear.
- Record every retention signal.
