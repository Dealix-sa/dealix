# Upsell Playbook

> How (and when) to move a client up the rung ladder.

## Rung Ladder Recap

```
Free Diagnostic → Sprint (499) → Data Pack (1,500) → Managed Ops (2,999 → 4,999) → Custom AI (5K–25K)
```

## Upsell Timing Rules

Don't upsell:
- Mid-Sprint (focus on delivery)
- Same week as a delivery miss
- When client's health score < 60
- When founder hasn't earned the right (< 1 successful delivery)

Do upsell:
- At handoff (soft mention of next rung)
- Month 3 of Managed Ops (renewal conversation)
- Month 6 of Managed Ops (sector benchmark window)
- When client explicitly asks for "more"
- When client describes a problem the next rung solves

## Per-Transition Plays

### Sprint → Data Pack
- Trigger: Sprint shipped well, client asks for "more leads" or "more sector context"
- Pitch: "Data Pack is 1,500 SAR, 5 days, gives you 200 enriched leads + sector benchmark"
- Soft mention at Day-7 handoff; hard pitch only if asked

### Sprint → Managed Ops (skip Data Pack)
- Trigger: Sprint shipped well, client says "I want this monthly"
- Pitch: Managed Ops at 2,999/mo, 30-day cancel anytime, productized
- Confirm: they're truly ready for recurring commitment (not just enthusiasm)

### Data Pack → Managed Ops
- Trigger: Data Pack delivered, client asks "what next" or sends more outreach signal
- Pitch: Managed Ops with explicit "this is the same work, but recurring + with reporting"
- Show: the value of the monthly trust audit + sector report

### Managed Ops 2,999 → 4,999
- Trigger: ≥ 3 months at 2,999 with growth in prospect volume need / additional sector
- Pitch: 1-page "what changes" doc
- Earn: must show clear value-add (more volume + sector benchmark + advisor sync)

### Managed Ops → Custom AI (Revenue Desk)
- Trigger: ≥ 3 months at 4,999, sector or integration need clearly beyond productized scope
- Pitch: SOW conversation (separate sales process)
- Gate: founder + advisor approval; client must qualify per `revenue_desk/OFFER.md`

## Anti-Patterns (forbidden upsells)

- Upselling to fix a delivery miss (apology should not have a price tag)
- Bundling extras at discount to close (cheapens productization)
- "Limited-time" upgrade pressure
- Upselling someone whose health score is < 60
- Skipping rungs (Data Pack → Custom AI without Managed Ops)

## Soft Pitch Template (use this language)

```
You've used the Sprint output well. If you're getting value, the next
rung is {Data Pack | Managed Ops} — {price} — {what they get}.

No pressure: you can also keep running on the Sprint output for another
14 days and decide then. Want me to send a one-pager?
```

## Hard Pitch (only when asked)

```
Here's the one-pager for {next rung}. Pricing: {SAR}. Scope: {bullets}.
Cancellation: {terms}. Trust posture stays the same.

Want to kick off {date}?
```

## Win-Back

If they declined upsell and 90 days pass:
- Single check-in: "Quick update — are you still using the Sprint outputs?"
- If yes: light suggestion of next rung
- If no: ask why; learn

## Metrics

- Sprint → next rung conversion: target ≥ 40%
- Managed Ops → expansion (2,999 → 4,999) at month 3: target ≥ 30%
- Managed Ops → Revenue Desk by month 12: track count (every Custom AI is a slow earned move)

## What This Refuses

- "Upsell or churn" framing
- Cross-selling things we don't have
- Reselling the client's own data back to them as "insights"
- Treating every customer call as a sales pitch
