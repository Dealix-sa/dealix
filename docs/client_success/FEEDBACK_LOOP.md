# Feedback Loop

> Customer feedback is the cheapest research we will ever get.

## Sources

1. **Structured request after every Sprint.**
   3 questions:
   - What worked?
   - What did not?
   - What would you change?

2. **Monthly review of every retainer.**
   - One open question: "What is the one thing we should change?"

3. **Loss debriefs** (every lost deal, every churned retainer).
   - "What would have made this a yes / a continuation?"

4. **Unsolicited replies** (LinkedIn, email).
   - Logged with the same weight as solicited.

## Triage

All feedback enters `dealix-ops-private/client_success/feedback/yyyy-mm/`
as a single Markdown file per piece, with:

- Source
- Customer
- Date
- Verbatim quote
- Pattern tag (controlled vocabulary):
  - `scope_confusion`
  - `delivery_pace`
  - `arabic_quality`
  - `english_quality`
  - `evidence_strength`
  - `tone`
  - `pricing`
  - `agent_overclaim`
  - `process_friction`
  - `tooling_request`

## Weekly review

In the weekly CEO review, the founder:

- Counts feedback items by pattern tag.
- Identifies the dominant tag (if any).
- Names one change to make next week if a tag repeats ≥ 3 times.

## Quarterly review

- All feedback summarised by pattern.
- Patterns that repeated > 5 times become formal policy or process
  changes.
- Patterns that never repeated are noted as one-offs.

## What we do **not** do

- Treat one customer's request as a roadmap line.
- Discount feedback that arrived "informally".
- Reply with marketing language ("Thank you for your feedback. We'll
  take that into consideration.").

## What we do

- Acknowledge promptly, in plain language.
- Tell the customer if their feedback changed something — that is the
  highest-trust signal we can send.
