# Weekly Client Report — Template (Managed Ops)

> Sent every Sunday EOD to active Managed Ops clients.
> Single page. Skimmable.

```markdown
# Dealix Weekly Update — {client_name} — Week ending YYYY-MM-DD

## TL;DR
{One sentence: what you should know if you read nothing else.}

## What we did this week
- {Deliverable 1} — link/file
- {Deliverable 2} — link/file
- {Deliverable 3} — link/file

## Numbers
- New scored prospects this week:        _____
- Outreach drafts sent for your approval: _____
- Outreach drafts you approved + sent:    _____
- Replies you received (from prior weeks):_____
- Calls booked:                           _____

## Trust note
- Approvals logged:                       _____ (link to summary)
- claim_guard flags this week:            _____
- Suppression-list updates:               _____

## What's next week
- {Plan 1}
- {Plan 2}
- {Plan 3}

## Questions for you
- {Question 1 if any — bold}
- {Question 2 if any — bold}

## Anything else
- {one-line observation or recommendation}

— Bassam
```

## Discipline

- Sent before Sunday midnight, every week
- Same format every time (predictability beats variety)
- Never bigger than one screen
- Never missed without a heads-up
- Always asks for input if there's a question

## What Goes Into "Numbers" Section

- Use only verifiable numbers
- If client provides numbers we can't verify (e.g., their reply count), label them "client-reported"
- Never extrapolate ("if you sent more, you'd get more") — show only what happened

## What Doesn't Go In

- Vanity metrics
- Speculation
- Anything that hasn't been done
- Anything that wasn't approved

## When To Send More Than Weekly

- SEV-1/SEV-2 incident (immediate, separate channel)
- Major win (one-off shout)
- Major loss (proactive heads-up)
- Approval queue overflow (need their attention)

## Storage

- Each report archived in `clients/{client}/reports/YYYY-MM-DD.md` (private)
- Aggregated to monthly summary

## What This Template Refuses

- Filler weeks ("nothing much happened")
- Hiding bad news in a footnote
- Sending without numbers when numbers are due
- Skipping the trust note section
