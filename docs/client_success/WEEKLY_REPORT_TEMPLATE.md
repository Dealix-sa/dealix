# Weekly Report Template — قالب التقرير الأسبوعي للعميل

## Purpose
Defines the structure of the weekly report Dealix sends to active clients during a sprint or retainer. Short, evidence-led, no fluff. Reports are not a creative exercise — same shape every week.

## Cadence
- Sent every working week of an active engagement, by Sunday 4pm KSA.
- One report per client. No combined reports across clients.

## Structure (fixed sections)

### 1. Headline
One sentence: what happened this week that moves the engagement forward.

### 2. Done this week
Bullet list of deliverables completed, with link to artefact. Each item has a one-line evidence note.

### 3. In progress
Bullet list of items currently being worked. Expected completion date for each.

### 4. Blockers
Bullet list of blockers, with the named owner. If the owner is on the client side, the ask is explicit.

### 5. Numbers (when applicable)
Table of relevant metrics for the engagement: e.g., reports built, leads scored, contacts reviewed. Labels follow [`docs/content/PROOF_LIBRARY.md`](../content/PROOF_LIBRARY.md).

### 6. Decisions needed
Items requiring a client decision. Default is escalated next call.

### 7. Next week
Bullet list of what is planned. Conservative; reflects actual capacity.

### 8. Disclosure (always present)
"Estimated value is not Verified value / القيمة التقديرية ليست قيمة مُتحقَّقة"

## Rules
- No new claim in a weekly report. Only outputs the proof library and the signed proposal support.
- Numbers are sourced from the delivery log, not memory.
- No external party copied without client consent.
- AR + EN sections balanced when the client is Saudi business; otherwise primary language follows the contract.

## Operations
- Drafted by delivery analyst on Sunday morning.
- Founder reviews and sends by 4pm KSA.
- Stored in client folder `docs/client_success/clients/<anonymized-label>/weekly_NNN.md`.

## Evidence
- Each weekly report links to the source artefacts.
- Receipt of report logged by founder.

## Owner & cadence
- Owner: Delivery analyst drafts; founder approves and sends.
- Cadence: weekly during active engagements.

## القسم العربي

**الغرض:** هيكل التقرير الأسبوعي الذي يُرسل لعملاء سبرنت/Retainer.

**الإيقاع:** كل أسبوع عمل بحلول الأحد 4م بتوقيت السعودية. تقرير واحد لكل عميل.

**الأقسام الثابتة:**
1. عنوان واحد — أهم تقدم هذا الأسبوع.
2. ما أُنجز — قائمة مع رابط دليل لكل بند.
3. قيد التنفيذ — مع تاريخ متوقع.
4. المعوقات — مع المالك المُسمّى.
5. الأرقام — جدول مقاييس متعلقة بالعقد.
6. قرارات مطلوبة من العميل.
7. خطة الأسبوع القادم — متحفظة.
8. الإفصاح الدائم: "القيمة التقديرية ليست قيمة مُتحقَّقة / Estimated value is not Verified value".

**القواعد:** لا ادعاء جديد، الأرقام من سجل التسليم لا من الذاكرة، لا نسخ طرف ثالث بدون موافقة، AR + EN متوازيان للعملاء السعوديين.

**المالك:** المحلل يصوغ، المؤسس يعتمد ويرسل. **الإيقاع:** أسبوعي.
