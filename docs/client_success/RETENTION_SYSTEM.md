# Retention System

## Purpose
Convert one-time Revenue Sprint delivery into recurring Revenue Desk support.

## Owner
Sami / Client Success owner.

## Retention Flow
Delivery → Feedback → Health Score → Next Opportunity → Retainer Ask → Monthly Support

## Retainer Triggers
- client liked the lead quality
- client asks for more targets
- client wants follow-up support
- client lacks internal sales ops
- client needs weekly opportunity flow

## Retainer Offer
Revenue Desk:
- weekly qualified opportunities
- message improvement
- pipeline review
- monthly learning report
- opportunity intelligence

## Rules
- Ask for feedback after every delivery.
- Ask for retainer only when value is clear.
- Do not overpromise meetings or revenue.
- Public proof requires approval.

## Metrics
- feedback received
- retainer asks
- retainer conversion
- client health score
