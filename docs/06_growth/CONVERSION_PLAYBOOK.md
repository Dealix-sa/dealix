## The one-CTA rule — قاعدة المسار الواحد

Every page and every tool routes to exactly one of three CTAs: **Business OS Score**, **Diagnostic**, or **Command Sprint**. Pages do not offer competing buttons. Secondary links educate but never compete for the primary click.

كل صفحة وأداة توجّه إلى مسار واحد فقط من ثلاثة: Business OS Score أو Diagnostic أو Command Sprint. لا أزرار متنافسة. الروابط الثانوية تُثقّف ولا تنافس الزر الأساسي.

## Objection handling — معالجة الاعتراضات

Address objections on the page where they arise, in the reader's words.

| Objection | Safe response | صياغة عربية |
|---|---|---|
| "Is this just a chatbot?" | Governed AI operations, not a chatbot. Revenue OS is the first wedge. | تشغيل مُحوكَم وليس روبوت محادثة |
| "Is my data safe / PDPL?" | PDPL-aware handling, no scraping behind login, data provenance recorded. | معالجة متوافقة مع PDPL |
| "Will you spam my customers?" | No auto-send, no cold WhatsApp automation. Every external message is a founder-approved draft. | لا إرسال تلقائي |
| "Can you prove results?" | We deliver a Proof Pack, not promises. Estimated value is labeled as estimated. | حزمة إثبات لا وعود |
| "Too expensive / unclear value" | Start with the Diagnostic; the Command Sprint is scoped before commitment. | ابدأ بالتشخيص |

No objection is answered with a guaranteed-revenue claim. Mirrors [../05_governance_os/CLAIM_SAFETY.md](../05_governance_os/CLAIM_SAFETY.md).

## Safe proof framing — تأطير الإثبات الآمن

- Use **estimated / observed / verified** value tiers; never blur them. See [../08_value_os/ESTIMATED_OBSERVED_VERIFIED_VALUE.md](../08_value_os/ESTIMATED_OBSERVED_VERIFIED_VALUE.md).
- Case references are **case-safe summaries** only — no PII, no real names unless approved. See [../07_proof_os/CASE_SAFE_SUMMARY.md](../07_proof_os/CASE_SAFE_SUMMARY.md).
- Never publish a result without a delivered Proof Pack behind it.
- Numbers on marketing pages are illustrative until verified, and labeled as such.

استخدم طبقات القيمة التقديرية والملاحَظة والمُتحقَّقة دون خلط. المراجع ملخصات آمنة بلا بيانات شخصية. لا نتيجة منشورة بدون حزمة إثبات.

## Form design — تصميم النماذج

- Ask for the minimum: name, work email, company, sector. Add fields only when they change routing.
- Value first, email second: free tools show partial value before requesting contact.
- No pre-checked consent boxes. Explicit PDPL-aware opt-in only.
- One primary button per form, labeled with the outcome ("Get my Business OS Score"), not "Submit".
- Inline validation, Arabic and English error states.

اطلب الحد الأدنى. القيمة قبل البريد. لا مربعات موافقة مُحددة مسبقاً. زر واحد بعنوان النتيجة.

---

## Trust signals — إشارات الثقة

- PDPL-aware data handling statement on every capture surface.
- Governance and approval-gate language ("nothing is sent without your approval").
- Link to the Trust Pack ([../14_trust_os/TRUST_PACK.md](../14_trust_os/TRUST_PACK.md)).
- No fake logos, no invented testimonials, no inflated counters.

---

## Arabic-first / RTL — العربية أولاً واتجاه RTL

- Arabic is the default; English is the parallel, not an afterthought.
- Full RTL layout: navigation, forms, tables, and number alignment mirror correctly.
- Numerals and dates follow Saudi conventions; currency in SAR.
- Tone is executive Saudi business: clear, decisive, evidence-forward. No marketing fluff, no emojis.
- Parallel AR/EN blocks match in length and structure on every page.

العربية هي الأصل والإنجليزية الموازية. تخطيط RTL كامل. أرقام وتواريخ بصيغة سعودية والعملة بالريال. نبرة تنفيذية واضحة بلا حشو.
