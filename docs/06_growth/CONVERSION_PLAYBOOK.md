# Conversion Playbook — Dealix Self-Growth OS

تحويل أخلاقي وقوي. كل تكتيك هنا مسموح عالميًا ولا يعتمد على خداع أو ضغط
كاذب. القاعدة العليا: **Proof-before-Price** و**CTA واحد في كل مكان**.

---

## 1. Scarcity حقيقية فقط

لا «مقاعد محدودة» كذبًا. الصياغة الصادقة:

> «نقبل 3 Command Sprints فقط هذا الشهر لأن التسليم يتم بمراجعة مؤسس
> وProof Pack يدوي.»

الندرة مسموحة فقط حين تكون **سعة التسليم محدودة فعلًا**.

## 2. Risk Reversal (قابل للتنفيذ فقط)

> «إذا لم نطلع لك 3 نقاط تعطل واضحة وNext Action Board خلال 7 أيام، نمدد
> العمل يومين بدون تكلفة.»

لا وعد برد أموال إلا إذا كان منفّذًا فعلًا. **لا ضمان إيراد.**

## 3. Proof-before-Price

قبل ذكر السعر، اعرض: Sample Command Pack · Proof Register · before/after
clarity (نوعي، لا أرقام غير مُتحقَّقة).

## 4. Micro-Commitment

```
Business OS Score (مجاني) → Diagnostic → Sprint → Managed OS
```

## 5. نسخة لكل قطاع (Segment-specific copy)

| القطاع | الزاوية |
|---|---|
| وكالة تسويق | Proof + client reporting |
| شركة IT | Delivery + support + client memory |
| شركة تدريب | academy + conversion + proof |

لا ترسل نفس الرسالة لكل قطاع.

## 6. Objection Library (كل اعتراض = أصل)

| الاعتراض | الأصل التسويقي |
|---|---|
| عندنا CRM | مقال: لماذا CRM لا يكفي بدون operating rhythm |
| AI خطر | صفحة: Approval-first AI |
| السعر مرتفع | Calculator: تكلفة فوضى المتابعة |
| ما عندنا بيانات | Diagnostic: نبدأ من الموجود |
| ما نحتاج مبيعات | Command OS: القرار والتسليم والإثبات |

## قمع التحويل

```
Visitor → Free Score → Result → Sample Command Pack → Book Diagnostic
→ Command Sprint Checkout → Intake → Delivery Workspace → Proof Pack
→ Managed OS Upsell
```

### Sales-Assisted Self-Serve

- **Score 70+** → Diagnostic booking → call → offer → payment link → intake.
- **Score 40–69** → nurture → template download → retargeting → diagnostic later.
