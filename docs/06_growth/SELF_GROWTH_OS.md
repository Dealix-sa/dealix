# Self-Growth OS / نظام النمو الذاتي

**Status:** PLANNED (governed) · **Owner:** Founder · **Updated:** 2026-06-06
**Hard boundary:** this is a **drafts-only** growth loop. It never sends, never
scrapes, never runs cold/bulk outreach. Every output is a draft for founder
approval. حلقة نمو محكومة — مسودات فقط، بلا إرسال تلقائي وبلا scraping.

> Scope companions: [`../SELF_GROWTH_OS_SCOPE.md`](../SELF_GROWTH_OS_SCOPE.md),
> [`../SELF_GROWTH_OS_PACKAGE.md`](../SELF_GROWTH_OS_PACKAGE.md).

---

## 1. What it is / ما هو

A governed loop that turns delivered Proof into the **next** safe, consented
opportunity — without violating any non-negotiable.

```
delivered Proof  ──►  pattern (what worked)  ──►  warm shortlist (consented)
        ▲                                                   │
        │                                          drafts for approval
   case-safe story  ◄── founder approval ◄── founder reviews ◄┘
```

---

## 2. Inputs (allowed sources only) / المدخلات

- Delivered Proof Packs (internal learnings).
- **Warm** lists: existing relationships, inbound, referrals, opt-in.
- Public, non-scraped sector signals (read, not harvested).

Forbidden inputs: scraped contacts, purchased cold lists, bulk-harvested data.

---

## 3. Outputs (drafts only) / المخرجات

- A ranked **shortlist** with a declared `consent_basis` per row.
- Personalized **outreach drafts** (never sent automatically).
- A case-safe story draft (only publishable with consent + approval).

All outputs land in the outreach approval queue
(`reports/revenue/outreach_approval_queue.md`) as `approval_required`.

---

## 4. Guardrails / الحواجز

| Rule | Enforced by |
|---|---|
| No auto-send | Human Approval Policy |
| No scraping | Source / consent_basis required per row |
| No guaranteed-revenue language | Claims Register |
| No publishing without consent + approval | Proof Pack v6 standard |

---

## 5. Current state

PLANNED — the loop is documented and the manual founder version runs today via
the Diagnostic → Command Sprint → Proof Pack motion. Automation of the
*drafting* steps stays gated behind the Human Approval Policy and is never
auto-send.
