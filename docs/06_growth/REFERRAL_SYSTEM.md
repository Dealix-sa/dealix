# Referral System — Dealix Self-Growth OS

كل عميل راضٍ = إحالة محتملة. الإحالة أرخص قناة وأعلاها ثقة، لكن تبدأ
**بعد التسليم وProof** — لا قبله.

---

## متى نطلب الإحالة

بعد تسليم Proof Pack ناجح (وليس قبل الإثبات). السؤال:

> «هل تعرف شركة خدمات ثانية عندها نفس فوضى المتابعة أو ضعف الإثبات؟»

---

## المكافأة (اختر واحدة، صريحة ومكتوبة)

| المكافأة | متى تناسب |
|---|---|
| 10% referral credit | العميل يكمل مع Dealix |
| شهر report مجاني | عميل managed OS |
| workshop مجاني | عميل يقدّر التعليم |

> لا مكافآت غامضة. الشروط مكتوبة في `referral_terms` ضمن Partner Kit.
> أكواد الإحالة موجودة في البنية الحالية
> ([`scripts/seed_referral_codes.py`](../../scripts/seed_referral_codes.py)،
> [`data/seeded_referrals.json`](../../data/seeded_referrals.json)) — استخدمها.

---

## التدفق

```
Proof Pack delivered → اطلب الإحالة → كود/شروط الإحالة
→ المُحال يدخل عبر Diagnostic → Command Sprint → مكافأة عند التأهل
```

كل مُحال يدخل من **نفس القمع** (Diagnostic → Sprint)، لا مسار جانبي.

---

## الحوكمة

- لا حوافز على إحالات وهمية أو شهادات مزيّفة.
- لا إرسال نيابة عن العميل بدون إذنه.
- المكافأة تُصرف فقط عند تأهل حقيقي (Sprint مدفوع/اشتراك).
- القيمة التقديرية ليست قيمة مُتحقَّقة.

---

## الربط بالشركاء

نظام الإحالة هو المستوى الأول في برنامج الشركاء
(انظر [`PARTNER_DISTRIBUTION.md`](PARTNER_DISTRIBUTION.md)): **Referral →
Co-sell → Co-delivery**.
