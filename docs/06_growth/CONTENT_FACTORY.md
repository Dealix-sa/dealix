# Content Factory — مصنع المحتوى

محتوى منضبط لا عشوائي: مصادر ثابتة، أنواع محدّدة، تقويم أسبوعي، ونظام إعادة تدوير يحوّل فكرة واحدة إلى 12 أصلًا. الهدف من كل قطعة: أن تقود إلى Score أو Diagnostic أو Sprint.

> القاعدة: بِع الألم لا "الذكاء الاصطناعي". المحتوى يصف وجعًا يعرفه العميل، ثم يقدّم إطارًا، ثم يدعو لخطوة واحدة. لا مسودة AI تُنشر بدون مراجعة بشرية.

---

## 1) المصادر الخمسة الأسبوعية

| المصدر | ماذا يولّد |
| --- | --- |
| Market Intelligence | زوايا قطاعية، اتجاهات، أسئلة شائعة |
| اعتراضات العملاء | محتوى يعالج الاحتكاك الحقيقي |
| أسئلة الـ Diagnostic | محتوى تعليمي حول كل ألم نقيسه |
| دروس التسليم | Build Log صادق من ميدان العمل |
| Proof Packs المُعقّمة | رؤى مبنية على نتائج (بموافقة فقط) |

كل أسبوع: اسحب فكرة واحدة من كل مصدر = 5 أفكار جذعية.

---

## 2) أنواع المحتوى العشرة

| النوع | الوظيفة |
| --- | --- |
| Pain | يصف وجعًا تشغيليًا محددًا |
| Framework | إطار عمل قابل للتطبيق |
| Build Log | شفافية بناء المؤسس |
| Proof Insight | رؤية من Proof Pack مُعقّم |
| Objection | يعالج اعتراضًا متكررًا |
| Sector | زاوية قطاع سعودي |
| Founder Lesson | درس صريح من الميدان |
| Tool | يبرز أداة مجانية |
| Partner | يخاطب الشركاء المحتملين |
| Case-safe | قصة بموافقة، مُعقّمة |

---

## 3) التقويم الأسبوعي

| اليوم | النوع | مثال نسخة |
| --- | --- | --- |
| الأحد | Pain | "كل صفقة تتأخر متابعتها 48 ساعة هي تسريب إيراد صامت. كم تخسر هذا الربع؟" |
| الإثنين | Framework | "ثلاث طبقات تجعل تشغيلك نظامًا لا صندوق وارد: Revenue Map، Proof Register، Decision Rhythm." |
| الثلاثاء | Build Log | "هذا الأسبوع بنينا Next Action Board لعميل خدمات. الدرس: الوضوح يسبق الأتمتة." |
| الأربعاء | Proof Insight | "في Proof Pack مُعقّم: 23% فقط من ادعاءات الشركة كانت مدعومة بدليل. الباقي رأي." |
| الخميس | Objection | "‘عندنا CRM’ — جيد. لكن هل يعطيك Executive Command Brief كل صباح؟ هذا فرق Business OS." |
| الجمعة | Sector / Founder Lesson | "وكالات التسويق تخسر المعرفة عند خروج account manager. Client Brain يحلّها." |
| السبت | Review | مراجعة الأداء: ما الذي قاد لأكبر عدد Scores؟ كرّره. |

كل منشور ينتهي بـ CTA واحد فقط.

---

## 4) من فكرة واحدة إلى 12 أصلًا

```
1 فكرة جذعية (مثال: تسريب المتابعات)
   ├─ 1 صفحة سؤال/جواب (SEO/GEO)
   ├─ 1 منشور Pain
   ├─ 1 Framework
   ├─ 1 Build Log مرتبط
   ├─ 1 Proof Insight
   ├─ 1 Objection
   ├─ 1 زاوية قطاعية
   ├─ 1 newsletter
   ├─ 1 سكربت أداة (Revenue Leakage Calculator)
   ├─ 1 درس Academy
   ├─ 1 قالب مرتبط (Next Action Board)
   └─ 1 رسالة شريك
```

فكرة واحدة قوية تكفي أسبوعًا كاملًا عبر القنوات، مع الحفاظ على CTA موحّد.

---

## الحواجز في هذه الصفحة

- لا نشر خارجي بدون موافقة المؤسس.
- لا proof مزيّف — Proof Insight من Proof Pack حقيقي مُعقّم فقط.
- كل قطعة تقود إلى Score / Diagnostic / Sprint.
