# Revenue Sprint Kit

The Revenue Sprint is Rung 2 of the Offer Ladder. It is the small, paid first commitment that turns a buyer into a customer. This kit is the full operating contract for the sprint.

## Files in this kit

- [SCOPE.md](./SCOPE.md) — what is in scope and out of scope.
- [PRICING.md](./PRICING.md) — 499 SAR, pre-paid, no discounts without A2.
- [DELIVERABLES.md](./DELIVERABLES.md) — what the buyer receives.
- [TIMELINE.md](./TIMELINE.md) — fourteen-day calendar.
- [PROPOSAL_TEMPLATE.md](./PROPOSAL_TEMPLATE.md) — proposal generator template.
- [REFUND_POLICY.md](./REFUND_POLICY.md) — refund-on-request rules.
- [CASE_STUDY_TEMPLATE.md](./CASE_STUDY_TEMPLATE.md) — public write-up rules.
- [README.md](./README.md) — this index.

## How the kit is used

- Sales OS reads [SCOPE.md](./SCOPE.md) when qualifying a lead.
- Sales OS uses [PROPOSAL_TEMPLATE.md](./PROPOSAL_TEMPLATE.md) when generating a proposal.
- Delivery uses [TIMELINE.md](./TIMELINE.md) and [DELIVERABLES.md](./DELIVERABLES.md) during the sprint.
- Revenue Ops uses [PRICING.md](./PRICING.md) and [REFUND_POLICY.md](./REFUND_POLICY.md) at payment time.
