# Feedback Request — `<client>`

> Sent with the handoff. **3 sentences.** Anything longer goes unanswered.

## Message

```
Hi <first-name>,

Quick favour for the next sprint to be better than this one. Three sentences,
that's it:

1. What was the most useful part?
2. What was the least useful?
3. If you were going to recommend this sprint to one peer, what would you say?

Reply by <date + 7 days>. If I don't hear back I'll assume "good enough" and
stop bothering you.

— <founder>
```

## What to do with the answers

- File the written reply in `clients/<client>/feedback.md`.
- If the answer to #3 is something concrete, log it as a candidate playbook
  update in `learning/<ISO_week>.md` and ship the update before Friday.
- If the answer to #1 contradicts what we thought the sprint was about,
  flag it in `founder/decision_queue.md` for the next weekly review.

## What not to do

- Do not ask "would you do another one" inline. That is the
  `retainer_ask.md`'s job.
- Do not chase more than once.
- Do not promise to "implement all feedback".
