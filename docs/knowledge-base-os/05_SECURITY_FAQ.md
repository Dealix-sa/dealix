# أسئلة الأمان والبيانات | Security & Data FAQ

## الغرض | Purpose

**عربي:** مرجع للأسئلة المتكررة حول الأمان وحماية البيانات، بصياغة واعية بنظام حماية البيانات الشخصية (PDPL) ودون كشف أي أسرار أو تفاصيل حساسة. تُجهَّز الإجابات بالذكاء الاصطناعي وتُعتمد من المؤسّس.

**English:** A reference for recurring security and data-protection questions, phrased with PDPL awareness and without exposing any secrets or sensitive details. Answers are AI-prepared and founder-approved.

---

## الأسئلة والأجوبة | Q&A

**س: كيف تحمون بيانات العملاء؟**
ج: نطبّق ضوابط وصول ومبدأ أقل صلاحية، ونلتزم بـ PDPL. لا نشارك بيانات دون إذن موثّق.

**Q: How do you protect client data?**
A: We apply access controls and least-privilege, and comply with PDPL. We never share data without documented consent.

---

**س: هل تُخزَّن المفاتيح أو الأسرار في الوثائق؟**
ج: لا إطلاقًا. لا تُكتب أسرار أو مفاتيح API في أي مستند أو أصل معرفي.

**Q: Are keys or secrets stored in documentation?**
A: Never. No secrets or API keys are written in any document or knowledge asset.

---

**س: كيف تتعاملون مع طلبات أصحاب البيانات؟**
ج: عبر إجراء موثّق متوافق مع PDPL، مع التحقق من الهوية قبل أي إجراء.

**Q: How do you handle data subject requests?**
A: Through a documented, PDPL-aligned procedure, with identity verification before any action.

---

**س: هل تُرسَل البيانات لأطراف خارجية تلقائيًا؟**
ج: لا. لا إرسال خارجي تلقائي؛ كل مشاركة بيانات يدوية ومعتمَدة.

**Q: Is data sent to external parties automatically?**
A: No. There is no automated external sending; every data share is manual and approved.

---

## مبادئ الأمان | Security Principles

- أقل صلاحية وأقل بيانات | Least privilege, data minimization.
- لا أسرار في الوثائق | No secrets in docs.
- التزام PDPL | PDPL compliance.

---

## قواعد السلامة | Safety Guardrails

- لا كشف أسرار أو مفاتيح API.
- لا إرسال خارجي تلقائي ولا كشط.
- لا ادعاءات أمنية غير مثبتة.
- الذكاء الاصطناعي يجهّز، المؤسّس يعتمد، الإجراء يدوي.
