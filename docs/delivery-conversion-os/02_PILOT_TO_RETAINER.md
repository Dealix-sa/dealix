# Pilot to Retainer — من التجربة إلى العقد الشهري

## الغرض (AR)
يحدد هذا المستند كيف تتحوّل التجربة المُسلَّمة إلى عقد شهري متكرر داخل نظام تحويل التسليم. التحويل يُبنى على نتائج التجربة الموثّقة مقابل معايير النجاح المتفق عليها. كل العروض شفافة وبلا ضمان عائد، والتواصل يدوي.

## Purpose (EN)
This document defines how a delivered pilot converts into a recurring retainer within the Delivery Conversion OS. The conversion is built on documented pilot results against agreed success criteria. All offers are transparent with no guaranteed ROI, and contact is manual.

## شرط البداية / Entry Condition
- الحالة `pilot_delivered`.
- نتائج التجربة موثّقة مقابل معايير النجاح المتفق عليها مسبقًا.

Stage `pilot_delivered`, with pilot results documented against pre-agreed success criteria.

## خطوات التحويل / Conversion Steps
1. **مراجعة النتائج** / Review results: قارن المخرجات بمعايير النجاح المكتوبة.
2. **صياغة القيمة المستمرة** / Frame ongoing value: ما الذي يستمر العميل في كسبه شهريًا.
3. **هيكلة العقد** / Structure: نطاق شهري، تسعير ثابت، مدة، وإيقاع مراجعة.
4. **العرض والإغلاق** / Propose and close: اتفاق مكتوب، ثم تسجيل `retainer_started` يدويًا.

## معايير الجاهزية للعقد / Retainer Readiness Checklist
- [ ] تحققت معايير نجاح التجربة (أو فُسِّر الانحراف).
- [ ] توجد قيمة مستمرة واضحة للعميل.
- [ ] النطاق الشهري قابل للتسليم ضمن قدرتنا.
- [ ] أصول إثبات جاهزة دون كشف بيانات العميل.

## معالجة الاعتراضات / Objection Handling
- "لماذا الاستمرار؟" / "Why continue?": التحسين تراكمي والقيمة مستمرة.
- "السعر" / "Price": اربطه بالنطاق الشهري، لا بوعد عائد.

## السلامة / Safety
- لا ضمان عائد، لا أرقام مفبركة، لا ادعاءات غير مثبتة.
- العقد يدوي ومعتمد من المؤسس بالكامل؛ لا إرسال تلقائي.

No guaranteed ROI, no fabricated numbers, no unproven claims. The contract is fully manual and founder-approved; no auto-sending.
