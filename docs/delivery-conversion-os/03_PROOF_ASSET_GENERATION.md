# Proof Asset Generation — توليد أصول الإثبات

## الغرض (AR)
يحدد هذا المستند كيف نوثّق إثبات القيمة من التسليم **دون كشف بيانات العميل**. أصول الإثبات تدعم التحويل والتوسّع، لكنها تخضع لضوابط خصوصية صارمة وموافقة العميل. لا ادعاءات غير مثبتة ولا أرقام مخترعة.

## Purpose (EN)
This document defines how we document value proof from delivery **without exposing client data**. Proof assets support conversion and expansion, but are subject to strict privacy controls and client consent. No unproven claims and no invented numbers.

## أنواع أصول الإثبات / Proof Asset Types

| النوع / Type | المثال / Example |
|---|---|
| دراسة حالة مجهّلة / Anonymized case study | "شركة لوجستية متوسطة" بدل الاسم / "A mid-size logistics firm" |
| نمط نتيجة / Outcome pattern | تحسّن في زمن العملية موثّق داخليًا / Documented process-time improvement |
| شهادة بموافقة / Consented testimonial | اقتباس معتمد كتابيًا من العميل / Written-approved client quote |
| ملخص منهجية / Methodology summary | كيف عملنا، دون بيانات حساسة / How we worked, no sensitive data |

## قواعد الخصوصية / Privacy Rules
- إخفاء الهوية افتراضيًا (لا اسم، لا شعار، لا بيانات تعريفية) ما لم يوافق العميل كتابيًا.
- لا مشاركة بيانات خام أو معلومات سرية للعميل.
- لا أرقام لا يمكن إثباتها داخليًا.
- موافقة كتابية إلزامية قبل أي نشر منسوب.

Anonymize by default (no name, logo, or identifiers) unless the client consents in writing. Never share raw data or client confidential information. No numbers that cannot be internally substantiated. Written consent is mandatory before any attributed publication.

## سير العمل / Workflow
1. اجمع النتائج الموثّقة من التسليم.
2. جرّد البيانات المعرّفة (anonymize).
3. صُغ الأصل بصيغة مجهّلة.
4. اطلب موافقة العميل إن كان النشر منسوبًا.
5. خزّن الأصل للاستخدام في التحويل اليدوي.

## السلامة / Safety
- لا أسرار أو بيانات عملاء في المستودع.
- لا جذب وهمي ولا ادعاءات غير مثبتة.
- النشر والمشاركة يدويان وبموافقة المؤسس والعميل.

No secrets or client data in the repo. No fake traction or unproven claims. Publishing and sharing are manual and require founder and client consent.
