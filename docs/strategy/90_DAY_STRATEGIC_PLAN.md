# 90-Day Strategic Plan

> Complementary to `90_DAY_PLAN.md` (tactical 4-week execution).
> This file holds the **strategic** 90-day plan: north-star milestones, kill criteria, capital posture.

## North-Star Milestone (T+90)

By 2026-08-21 (90 days from 2026-05-23):
- **3 paid sprints delivered** with proof packs
- **1 retainer signed** (Managed Ops, ≥ SAR 2,999/mo)
- **1 public case study published** (Trust-approved)
- **Cash collected ≥ SAR 15K over the quarter**
- **Trust incidents = 0**

If we miss any of these, we don't pivot — we run a 7-day diagnosis sprint, fix the weakest leg, and continue.

## 30-60-90 Strategic Posture

### Days 1–30 — Foundation + Proof
- Lock 12 Super Systems (this PR)
- Ship Trust modules (approval matrix, claim guard, policy engine)
- Run 1 paid Revenue Sprint end-to-end, produce evidence pack
- Publish 1 weekly LinkedIn post starting week 1
- Build outreach queue of 100 Tier 1 prospects

**Strategic kill criterion:** If no paid sprint by day 30 → review ICP (likely wrong sector or wrong message), not the offer.

### Days 31–60 — Repeatability
- Run 2 more paid sprints (different sectors if possible)
- Convert first sprint customer to retainer conversation
- Produce first sector playbook from first sprint
- Hire first contractor only if delivery throughput < required for 2 active sprints (per HIRING_TRIGGERS)

**Strategic kill criterion:** If sprint #2 delivery time > sprint #1 → process bug, fix before sprint #3.

### Days 61–90 — Compounding
- Sign first retainer (Managed Ops)
- Publish first public case study (Trust-approved)
- Lock second sector playbook
- Run Monthly Strategy Update; decide quarter 2 focus

**Strategic kill criterion:** If retainer conversion fails twice → revisit pricing (likely too high vs. demonstrated value) or fit (likely wrong buyer).

## Strategic Bets (and what we're betting against)

| Bet | Bet against | Evidence we're winning | Evidence we're losing |
|---|---|---|---|
| Productized > project-priced | Local agency norm | Buyers ask "what exactly do I get" and pay quickly | Buyers want custom scoping calls before any commitment |
| Trust > speed | "Just automate everything" pitch | Buyers comment positively on the approval gate | Buyers leave because of "too much manual approval" |
| Saudi mid-market > enterprise | "Go upmarket for big checks" | Mid-market deals close in < 21 days | Mid-market never gets above 499 → 1500 |
| Founder-first delivery > team-first | "Hire fast" advice | Sprint margin > 60% with founder | Sprint margin < 30%, burning founder hours |
| Public learning > hidden moves | "Don't telegraph strategy" | Inbound leads cite the posts | No inbound; posts feel like shouting |

Each bet has an evidence column. Update monthly.

## Capital Posture

- **No external capital this quarter.**
- Reinvest 100% of cash collected into Trust + Delivery infrastructure
- Founder draws nothing until cash collected ≥ SAR 25K for 2 consecutive months
- If runway drops below 60 days → trigger "lean mode" (defer all build, focus on cash collection only)

## Headcount Posture

- **No hires this quarter** unless `HIRING_TRIGGERS.md` thresholds explicitly fire
- First likely hire: **part-time SDR/VA** (only when outreach volume > 100/week sustained)
- Second likely hire: **delivery analyst** (only when > 3 active sprints sustained)
- No engineering hires this quarter

## Strategic Risks (this 90 days)

- **R-A:** Founder burnout → mitigated by Focus Policy + forced rest Saturdays
- **R-B:** First sprint takes > 14 days → reveal process gaps; fix before sprint #2
- **R-C:** Trust incident in public → response plan in `docs/trust/INCIDENT_RESPONSE.md`
- **R-D:** Wrong sector chosen → defer to Tier 2 sector by day 45 if no traction
- **R-E:** Public learning posts don't land → reformat by day 30 if engagement < target

## End-Of-Quarter Review

On 2026-08-22:
- Score every milestone (hit / partial / miss)
- For each miss: root cause + corrective rule added to `DEALIX_DECISION_RULES.md`
- Rewrite this file for the next 90 days

## What This Plan Refuses

- Shifting the milestone definitions mid-quarter
- Adding a new strategic bet without removing one
- Hiring to "look bigger"
- Building features not in PRODUCT_STRATEGY's top 5
- Extending scope of any sprint past 7 days for any client this quarter
