# Sector One-Pager System

> Each beachhead sector gets one canonical one-pager that explains:
> who Dealix is for in this sector, the pain it solves, the proofs, and
> the next step. Used in warm intros, partner conversations, and event
> follow-up.

## Filename convention

`assets/sales/one_pagers/onepager_<sector>_v<NN>.md`

## Structure

```
# Dealix for <Sector>

## The buyer
(title, company size, region, decision style)

## The pain
(3 bullets max — taken from objection_library + lost-deal interviews)

## What Dealix does
(3-5 bullets — only verifiable capabilities)

## Proof
(2-3 links — each linked to a proof-pack id; no unverified claims)

## How we engage
(offer ladder reference)

## Trust
(PDPL Art. 5/13/14/18/21, ZATCA Phase 2, audit trail, approval-first)
```

## Banned in one-pagers

- Bench-marketing claims with no source.
- "AI" used as a synonym for "magic".
- Promises about ROI without a footnote linking to the assumption set.
