# Objection Response Library

> The customer-facing companion to `OBJECTION_INTELLIGENCE_SYSTEM.md`.
> Each high-frequency objection gets a one-page response sheet stored
> in `assets/sales/objections/`.

## Structure of a response sheet

```
# Objection: <verbatim customer wording>

- Sector(s):
- Stage(s) it appears in:
- Why it appears:
- Our short response (one paragraph):
- Proof / link to artifact:
- What to do next (concrete next step for the buyer):
- What we DO NOT promise here:
```

## Production rules

1. Sheets are produced for objections with `frequency >= 3`.
2. Each sheet is approved before being used in customer-facing
   conversation.
3. Each sheet must include a "what we DO NOT promise" section — this
   is the proof-safe guardrail.

## Examples we expect to see

- "We're already using <competitor>."
- "We can't share data because of PDPL."
- "We need to see ROI before signing."
- "Can we just buy the lead list?"
- "Will you guarantee X leads per month?" — the response sheet must
  explicitly state we do not guarantee.
