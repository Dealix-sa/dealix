# Win/Loss Review

Every closed-won and every closed-lost gets a 10-minute debrief recorded in
the structured format below. The point is to learn faster than competitors,
not to feel good or bad about the result.

## Cadence

- Per-deal: within 48 hours of close (won or lost)
- Aggregate: monthly during the [PRICING_COUNCIL](PRICING_COUNCIL.md)

## Per-deal interview SOP

Five questions, in order. Keep answers to one sentence.

1. **Why did this deal close (or not)?** — the buyer's stated reason
2. **What was the trigger?** — the event that turned interest into action
3. **Who was the real decision-maker?** — title and decision style
4. **What was our weakest moment?** — where did we almost lose / lose?
5. **What would we do differently?** — one change, named

## Ledger

Recorded in [`docs/commercial/operations/founder_meeting_debrief_template.yaml`](../commercial/operations/founder_meeting_debrief_template.yaml)

## Reason codes (canonical)

| Code | Meaning |
|---|---|
| `price_too_high` | Buyer cited price |
| `price_too_low` | We left money on the table |
| `competitor_chosen` | Lost to a named competitor |
| `built_internal` | Buyer chose to build it themselves |
| `no_budget` | Real budget block |
| `wrong_timing` | Real timing block, may re-open |
| `trust_concern` | Security / proof / brand concern |
| `scope_mismatch` | Our offer did not fit the need |
| `champion_left` | Internal champion exited or lost authority |
| `other` | Document the reason in the notes |
