# Bad Revenue Filter

## Purpose
Refuse revenue that damages trust, repeatability, or focus, even when cash is offered.

## Owner
Sami / Founder CEO.

## Review Cadence
Per opportunity; full review weekly.

## Disqualifiers (auto-reject)
- Buyer asks Dealix to violate PDPL or any applicable law.
- Buyer asks for scraping, fake reviews, fake leads, or misleading claims.
- Buyer requires guaranteed revenue language.
- Buyer requires hidden subcontracting that Dealix cannot disclose.
- Buyer has no decision authority and no path to authority.
- Engagement requires data export Dealix cannot legally hold.

## Yellow Flags (founder review required)
- Scope drifts beyond ladder rung without paid expansion.
- Buyer demands extreme discount (> 30% off ladder).
- Buyer expects 24/7 SLA that Dealix cannot honor.
- Buyer is a known bad-payer in market signal.

## Acceptance Criteria
Revenue is "good" only if it improves at least two of:
- cash position
- proof asset library
- repeatable playbook
- trust position
- learning loop

## Process
1. Run the filter before sending any proposal above Free Diagnostic.
2. Log the result in pipeline notes.
3. If yellow flag triggered, founder must confirm before proceeding.
4. If disqualifier triggered, decline politely using SAFE_LANGUAGE_LIBRARY.
