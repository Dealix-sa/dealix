# Revenue Command Center — مركز قيادة الإيرادات

## Purpose

The single page the founder opens at the start of the day to know what to do next. Daily and weekly rhythm. KPIs visible at a glance. If a number is missing, the day starts with finding it.

## Daily KPIs (read before noon)

| KPI | Definition | Today's target |
|---|---|---|
| New qualified leads | Leads scored ≥ 70 on [SCORING_RULES.md](../delivery/revenue_sprint/SCORING_RULES.md) | 3 |
| DMs sent (approved) | A1-approved outreach messages | 5 |
| Replies received | Inbound from outreach | track only |
| Samples shared | Diagnostic sample delivered to a buyer | 1 |
| Proposals out | A1-approved proposals sent | 1 |
| Payments collected | SAR in account | track only |

## Weekly KPIs (Sundays)

| KPI | Definition | Weekly target |
|---|---|---|
| Pipeline value (SAR) | Sum of open opportunities × stage probability | track |
| Conversion stage-to-stage | Reply → Sample → Proposal → Payment | track |
| Sprints sold | 499 SAR diagnostics paid | 3 |
| Cash collected | Net SAR received | track |
| Retainers active | Managed-ops clients on monthly | track |
| Win rate | Paid / proposals out (4-week rolling) | track |

## Daily rhythm

1. Open this page. Read yesterday's row in the revenue log.
2. Clear A1 approvals queue (proposals, outbound).
3. Run lead surfacing block (90 min). Output goes to staging.
4. Founder review block (45 min): approve, edit, or reject what agents prepared.
5. Send approved outbound. Log every send.
6. Close-of-day: update the revenue log row.

## Weekly rhythm (Sundays)

1. Reconcile pipeline value vs. cash collected.
2. Review losses; write one paragraph per lost deal into the loss log.
3. Update [OFFER_EVOLUTION_SYSTEM.md](./OFFER_EVOLUTION_SYSTEM.md) inputs.
4. Confirm bad-revenue filter caught what it should have ([BAD_REVENUE_FILTER.md](./BAD_REVENUE_FILTER.md)).
