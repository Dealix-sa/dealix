# Revenue Operations Playbook
## Purpose
Turn Dealix revenue work into a repeatable daily operating machine.
## Owner
Sami / Founder CEO.
## Review Cadence
Daily during execution sprint, weekly after.
## Revenue Flow
ICP → Lead Source → Qualification → DM → Follow-up → Sample → Proposal → Payment/PO → Delivery → Feedback → Retainer
## Core Rule
Every revenue activity must create one of:
- cash
- proof
- retention
- learning
- trust-safe pipeline
## Daily Revenue Inputs
- pipeline_tracker.csv
- revenue_action_log.csv
- founder_dm_pack.md
- sample_pack_template.md
- proposal_fast_template.md
- payment_followup_templates.md
## Daily Revenue Outputs
- leads added
- DMs sent
- replies logged
- samples prepared
- proposals sent
- payment follow-ups
- next actions updated
## Weekly Revenue Outputs
- message learning
- sector learning
- win/loss learning
- pricing learning
- one playbook update
## Non-Negotiable Rules
- Every lead has next_action.
- Every proposal has follow-up date.
- No guaranteed revenue claim.
- No full delivery without payment, PO, or written approval.
- No public proof without approval.
- No SaaS expansion before repeated workflow evidence.
