# Pricing Council

Monthly forum to consider pricing changes. Currently solo CEO; rituals
documented so the same discipline holds when the team grows.

## Cadence

First Monday of each month, 30 minutes. Inputs ready before the meeting.

## Inputs

- [WIN_LOSS_REVIEW](WIN_LOSS_REVIEW.md) — last month's reasons
- [`docs/revenue/PRICING_AND_PACKAGING.md`](PRICING_AND_PACKAGING.md) — current list
- Discount log from [DEAL_DESK_SYSTEM](DEAL_DESK_SYSTEM.md) decisions
- Competitor signals (recorded ad-hoc, dated)

## The five questions

1. Are we winning the deals we should be winning? (price too high?)
2. Are we winning deals we should be losing? (price too low, attracting wrong customer?)
3. Is any package being heavily discounted to close? (re-price or kill it)
4. Has the customer's value shifted? (re-anchor the value story)
5. Should a new package emerge from observed scope patterns?

## Output

Any pricing change is logged in [`docs/founder/DECISION_LOG_SYSTEM.md`](../founder/DECISION_LOG_SYSTEM.md)
with `type: pricing`, and the linked assumption is added or updated in
[`docs/founder/STRATEGIC_ASSUMPTIONS_REGISTER.md`](../founder/STRATEGIC_ASSUMPTIONS_REGISTER.md).

[PRICING_AND_PACKAGING](PRICING_AND_PACKAGING.md) is the canonical source of
the live list and is updated only after the council decision is logged.

## Non-negotiables

Pricing is published; commitments to refunds, payment terms beyond list, or
discounts not in the package list require a deal-desk decision. See
[`docs/founder/DO_NOT_SAY.md`](../founder/DO_NOT_SAY.md).
