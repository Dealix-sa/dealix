# ERP / CRM Acquisition Playbook — Dealix

## Target buyer

- COO / CFO / Head of Operations في شركات ERP/CRM إقليمية بحجم 50–500 موظف.
- المنطقة: السعودية + GCC.
- اللغة: عربي رئيسي، English للـ technical.

## Pain hypotheses

- بيانات العمليات موزعة على modules غير متصلة.
- تقارير end-of-quarter تستغرق أسابيع.
- زبائنهم يطلبون insights AI لكن لا يعرفون كيف يبنوها.

## Signals

- post-implementation phase (≥6 شهور بعد go-live).
- ذكر "data quality" / "reporting" في LinkedIn posts.
- partner publicly مع SAP/Oracle/MS Dynamics.

## Offer (default)

- **Data Pack** بسعر 1,500 SAR — تشخيص قطاع + KPIs قابلة للقياس.
- Upgrade path: Diagnostic 4,999 SAR → Custom AI ≥10K.

## Proof needed

- Sample diagnostic من قطاع مشابه (مخفي البيانات الحقيقية).
- ZATCA-ready invoice.
- شهادة Trust gate قبل النشر.

## Draft angles

- "كيف يمكن لـ AI تحويل بيانات ERP إلى قرار خلال 14 يوم؟"
- "ROI Diagnostic لمنفذي SAP/Oracle".
- "نموذج Data Pack مفصَّل لقطاع التجزئة".

## Trust risks

- AIR-001 (hallucination) — Diagnostic لا يدّعي أرقام بدون evidence.
- REV-002 (delayed payment) — sample مدفوع.
- AIR-007 (bias) — scoring يخضع لمراجعة بشرية.

## Approval class

- `medium` — proposal يحتاج founder sign-off.

## Sample path

1. Discovery call 30 دقيقة.
2. توقيع NDA إن طُلب.
3. Diagnostic preview عام (لا بيانات عميل).
4. عرض Data Pack مع scope محدد.

## Proposal path

1. ZATCA-compliant quote.
2. دفع 50% مقدم.
3. Delivery خلال 14 يوم.
4. Done definition مكتوب قبل البيع.

## Follow-up path

- D+3: تأكيد استلام proposal.
- D+7: عرض sample إضافي إن لزم.
- D+14: قرار Kill أو Push.

## Objections (with responses)

| اعتراض | الرد |
| --- | --- |
| "عندنا BI tool" | "Data Pack ليس BI — هو تشخيص قطاع موجه لقرار محدد" |
| "غالي" | "1,500 SAR < ساعتين من وقت COO" |
| "ما نثق بـ AI" | "كل output يُراجع بشريًا قبل التسليم — Audit log كامل" |

## Disqualification rules

- شركة <20 موظف.
- بدون ERP/CRM running.
- لا قرار شراء داخلي (الـ contact مجرد technical).

## الملكية

- Owner: Sales lead.
- Reviewer: Founder.
