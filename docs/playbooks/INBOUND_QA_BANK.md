## Usage rules (per send)

1. Personalize EVERY merge bracket before send (no template smell)
2. Match the language to what the inquirer used (AR or EN)
3. Founder approves every send (Doctrine #1)
4. Log the send in `proof_ledger` with the Q&A code (e.g.
   `inbound_qa_C1`)
5. If the inquiry doesn't match any Q here, draft a new one and
   add to this bank after sending

## Quarterly review

Every quarter, review which Qs were actually used. Retire dead
ones. Add Qs from real conversations that surprised the founder.
Keep the bank tight (~30 Qs max).
