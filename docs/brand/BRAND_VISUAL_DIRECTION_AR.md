## 1. المبدأ البصري — Visual Principle

Dealix علامة **نظام تشغيل جادّ، لا شات بوت مرح**. الخلفيات داكنة تنفيذية، الهرمية واضحة، والادعاءات الكبيرة لا تظهر إلا مع دليل بصري بجوارها. الصورة تقول «غرفة قرار»، لا «روبوت لطيف».

A serious operating-system brand, not a playful chatbot. The image says "decision room," never "cute robot."

## 2. استخدام نظام الألوان — Using the Color System

الألوان الأساسية (من [الهوية البصرية](./DEALIX_VISUAL_IDENTITY_AR.md))، وكيف تُستخدَم:

| اللون | الكود | الدور البصري |
|-------|-------|--------------|
| Midnight Navy | `#06111F` | خلفية تنفيذية أساسية لكل سطح |
| Electric Cyan | `#22D3EE` | لمسة أساسية: روابط، حواف نشطة، رؤوس البيانات |
| Emerald Proof | `#34D399` | **الدليل فقط**: نتيجة مُثبتة، حالة «تمّ»، Proof block |
| Amber Warning | `#FBBF24` | تنبيه، انتظار موافقة، حدّ حذر |
| Slate Text | `#CBD5E1` | نصّ الجسم على الخلفية الداكنة |

**قواعد:** الأخضر لا يُستخدم للزينة — هو لون الدليل، فمتى ظهر توقّع رقماً مُثبتاً. الكهرماني يعني «قرار بشري مطلوب». لا تدرّجات صاخبة، لا أكثر من لمستين في الشاشة الواحدة.

## 3. أنماط التخطيط — Layout Patterns

أربعة أنماط معتمدة تتكرّر عبر الديك والموقع والتقارير:

- **بطاقات البيانات / Data cards** — رقم كبير + تسمية + مصدر صغير. تُستخدم للمقاييس. الرقم بـ Slate، المصدر بخطّ أصغر؛ لا رقم بلا مصدر.
- **خطوط الـ Pipeline / Pipeline flow** — مراحل أفقية من اليمين لليسار (عربي): فرصة → تسجيل → مسودة → موافقة → إرسال → ردّ. الكهرماني عند بوابة الموافقة.
- **كتل الإثبات / Proof blocks** — إطار بحافة Emerald يحوي: الادعاء + الدليل + مستوى الدليل ([سياسة الادعاءات](./BRAND_CLAIMS_POLICY_AR.md)). لا كتلة إثبات بلا مصدر.
- **خرائط القرار / Decision maps** — مدخلات → معايير → قرار → Decision Passport. تُظهر أن الوكيل يقترح والإنسان يقرّر.

ترتيب كتل الإثبات داخل الديك يتبع [PROOF_STACK_ORDER_AR.md](../commercial/operations/PROOF_STACK_ORDER_AR.md).

## 4. الهرمية والتخطيط — Hierarchy & Composition

- رسالة واحدة لكل شاشة/شريحة ([دليل العرض](./DEALIX_PRESENTATION_STYLE_GUIDE_AR.md)).
- الأرقام كبيرة، النصّ المحيط صغير.
- مساحة بيضاء (داكنة) سخيّة؛ الازدحام يقتل الثقة.
- عربي أولاً في التخطيط؛ الإنجليزية موازية لا مزدحمة.
- الشعار يُعامَل كعلامة نظام جادّ، لا كأيقونة مرحة.

## 5. ما يُتجنَّب — What to Avoid

- **روبوتات كرتونية / Cartoon robots** — تتناقض مع «نظام تشغيل جادّ».
- **صور AI عامة** — أدمغة متوهّجة، شبكات عصبية زخرفية، أيدي آلية.
- **الازدحام / Clutter** — أكثر من فكرة في الشاشة، فقرات كثيفة، ألوان متزاحمة.
- **ادعاءات كبيرة بلا دليل بصري** — أي رقم أداء بلا بطاقة مصدر بجواره.
- **الإيموجي** و**أسماء النماذج** — ممنوعة في كل سطح.
- **الأخضر الزخرفي** — يكسر معنى «الأخضر = دليل».

## 6. قائمة فحص بصرية سريعة — Quick Visual Check

قبل نشر أي أصل بصري:
- [ ] خلفية Midnight Navy، لمسة واحدة أو لمستان فقط؟
- [ ] كل رقم له بطاقة مصدر؟
- [ ] الأخضر مقصور على الدليل، الكهرماني على الموافقة؟
- [ ] رسالة واحدة لكل شاشة؟
- [ ] لا روبوت، لا إيموجي، لا اسم نموذج؟
- [ ] سطر الإفصاح حاضر في الأصول النصية؟
