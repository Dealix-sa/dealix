# Brand Content Rules — قواعد المحتوى

**Status:** القواعد الملزمة لكل أصل عام — كل محتوى يربط ستّ ركائز، لا نسخ عام، تذييل إفصاح إلزامي.

## 1. المبدأ — The Principle

المحتوى العام ليس «وعياً بالعلامة»؛ هو **أداة بيع محكومة**. كل أصل يجب أن يخدم قراراً تجارياً واضحاً ويُسنَد بدليل. النسخ العام الذي يصلح لأي شركة لا يصلح لـ Dealix.

Public content is a governed sales tool, not brand awareness. Generic copy that fits any company fits none.

## 2. القاعدة الست-رؤوس — The Six-Anchor Rule

كل أصل عام يجب أن يربط الرؤوس الستّة. إن غاب أحدها، الأصل غير جاهز:

| الركيزة | السؤال | مثال |
|---------|--------|------|
| **قطاع / Sector** | لمن تحديداً؟ | وكالات، عيادات، مقاولات، تجزئة B2B |
| **ألم / Pain** | أيّ وجع ملموس؟ | فرص مبعثرة، متابعات ضائعة، قرارات بلا دليل |
| **Workflow** | أيّ تدفّق نُصلحه؟ | ترتيب → تسجيل → مسودة → موافقة → تقرير |
| **Proof** | أيّ دليل؟ | Proof Pack، رقم مُتحقَّق، نمط case-safe ([PROOF_STACK_ORDER_AR.md](../commercial/operations/PROOF_STACK_ORDER_AR.md)) |
| **Offer** | أيّ عرض؟ | حزمة من [DEALIX_REVOPS_PACKAGES_AR.md](../commercial/DEALIX_REVOPS_PACKAGES_AR.md) |
| **CTA** | أيّ خطوة واحدة؟ | «احجز تشخيصاً» — CTA واحد لا أكثر |

قاعدة عملية: اقرأ مسوّدتك واسأل عن كل رأس «أين هو؟». لو لم تجده، أضِفه أو لا تنشر.

## 3. لا نسخ عام — No Generic Copy

- ممنوع: «نساعد الشركات على النمو» — بلا قطاع ولا ألم ولا دليل.
- مطلوب: «وكالات التسويق السعودية تفقد فرصاً في جداول متفرّقة؛ نرتّبها ونُسجّل أعلى 50 ونُخرج Proof Pack — بانتظار موافقتك.»
- كل أصل عربي له مقابل إنجليزي موازٍ (بنية وطول متماثلان).
- لا كلمات hype ([BRAND_VOICE_AR.md](./BRAND_VOICE_AR.md))، لا أرقام أداء كحقيقة ([BRAND_CLAIMS_POLICY_AR.md](./BRAND_CLAIMS_POLICY_AR.md)).

## 4. تذييل الإفصاح الإلزامي — Mandatory Disclosure Footer

كل أصل عام يُختَم بسطر الإفصاح أدناه، على سطر مستقلّ، عربي وإنجليزي. لا استثناء. هذا يربط L1/L2 بحقيقتها: التقدير ليس تحقُّقاً ([مستويات الدليل](./BRAND_CLAIMS_POLICY_AR.md)).

دراسات الحالة تُوسَم صراحةً «Hypothetical / case-safe template» إن لم يُسمَّ عميل حقيقي، وبلا أيّ PII.

## 5. قواعد حسب نوع الأصل — Rules by Asset Type

| النوع | قاعدة خاصة |
|-------|-----------|
| منشور LinkedIn | صوت المؤسس، نشر يدوي، الستّ ركائز ([نظام السوشال](./BRAND_SOCIAL_SYSTEM_AR.md)) |
| بريد بارد | CTA واحد، إفصاح وآلية إلغاء، يمرّ على البوابات الستّ ([نظام الـ Outbound](./BRAND_OUTBOUND_SYSTEM_AR.md)) |
| دراسة حالة | مُجهّلة، موسومة case-safe، نمط لا ضمان |
| تقرير قطاع | منهجية وأنماط مُجمَّعة فقط، لا أرقام سرّية |
| ديك/عرض | رسالة لكل شريحة، الستّ ركائز موزّعة ([دليل العرض](./DEALIX_PRESENTATION_STYLE_GUIDE_AR.md)) |

## 6. بوابة قبل النشر — Pre-Publish Gate

قبل نشر أي أصل عام:
- [ ] الرؤوس الستّة حاضرة (قطاع · ألم · workflow · proof · offer · CTA)؟
- [ ] CTA واحد فقط؟
- [ ] لا نسخ عام، لا كلمة ممنوعة؟
- [ ] كل رقم موسوم بمستوى دليل ومصدر؟
- [ ] مقابل عربي/إنجليزي متوازٍ؟
- [ ] لا PII، دراسات الحالة موسومة case-safe؟
- [ ] تذييل الإفصاح حاضر؟
- [ ] موافقة المؤسس الصريحة؟

إن سقط بند: لا تنشر. أصلِح أو اعرض على المؤسس. خريطة الأصول وحالتها في [BRAND_ASSET_CHECKLIST_AR.md](./BRAND_ASSET_CHECKLIST_AR.md).
