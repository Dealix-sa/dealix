## 1. Voice Characteristics | خصائص الصوت

The Dealix voice is the personality that stays constant across every context. Tone shifts by situation; voice does not.

### 1.1 The four voice pillars

| Pillar | Arabic | What it means | What it rejects |
|---|---|---|---|
| **Confident** | واثق | We state what we do and do what we say. No hedging on capabilities we have built. No exaggeration of capabilities we have not. | Hype, bravado, "revolutionary", "game-changing". |
| **Precise** | دقيق | We name the specific OS, the specific gate, the specific proof artifact. We do not generalize. | Vague "AI-powered" sloganeering. |
| **Operational** | تشغيلي | We speak like operators running a system, not marketers selling a dream. Language of pipelines, gates, drafts, approvals, deltas. | Aspirational "transform your business" marketing-speak. |
| **Arabic-first** | عربي أولاً | Arabic is the primary voice. Saudi MSA (فصحى مبسطة), not overly colloquial. English mirrors are parallel and equal-weight. | Translated-afterthought Arabic. Generic global English-first voice. |

### 1.2 Voice personality (reference)

From [`BRAND_GUIDELINES_AR.md`](BRAND_GUIDELINES_AR.md) and [`VISUAL_IDENTITY.md`](VISUAL_IDENTITY.md):

- **رسمي ولكن ودود** — Formal but approachable. We address founders in language they understand.
- **واثق لكن متواضع** — Confident but humble. No exaggerated promises. We say what we do and do what we say.
- **شفاف** — Transparent. We state the real product state (pilot, draft-only, when we will be ready).
- **رصين، موثوق، قائم على الدليل — لا صخب ولا مبالغة** — Composed, trustworthy, evidence-first — no noise, no hype.

## 2. Tone by Context | النبرة حسب السياق

Tone is the adjustment of voice for a specific situation. The voice pillars stay constant; the emotional register shifts.

### 2.1 Website (marketing)

| Attribute | Arabic | English |
|---|---|---|
| **Register** | طموح، واضح، يركز على القيمة | Ambitious, clear, value-focused |
| **Sentence length** | متوسط (15-25 كلمة) | Medium (15-25 words) |
| **CTA style** | "احجز تشخيص" / "ابدأ التشخيص" | "Book a diagnosis" / "Start diagnosis" |
| **Emotion** | ثقة هادئة، لا حماس مبالغ | Quiet confidence, not hype |
| **Disclosure** | إلزامي على كل صفحة عامة | Mandatory on every public page |

Example — Arabic:
> "ديليكس يبني نظام تشغيل لكل وظيفة تجارية — اكتشاف، قرار، تسليم، إثبات — تحت موافقات صريحة. لا روبوت محادثة، لا CRM، لا لوحة معلومات."

Example — English:
> "Dealix builds an operating system for every commercial function — discovery, decision, delivery, proof — under explicit approvals. Not a chatbot, not a CRM, not a dashboard."

### 2.2 Sales (deck, proposal, discovery call)

| Attribute | Arabic | English |
|---|---|---|
| **Register** | دقيق، احترافي، يركز على الألم والنتيجة والتسليم | Precise, professional, focused on pain, outcome, deliverables |
| **Rule** | كل عرض يوضح: الألم، النتيجة، المُسلَّمات، السعر، الخطوة التالية | Every offer shows: pain, outcome, deliverables, price, next step |
| **Proof** | حزم إثبات، خط أساس → بعد → فرق موثّق | Proof packs, baseline → after → documented delta |
| **Emotion** | جِدّة تنفيذية، ثقة بالمنتج | Executive seriousness, product confidence |

Example — Arabic:
> "الألم: متابعات ضائعة في الواتساب والإيميل. النتيجة: مسودات محوكمة مع بوابات موافقة. المُسلَّمات: نظام متابعة الواتساب والبريد، مسودات يومية، سجل تدقيق. السعر: تشخيص 4,999 ريال. الخطوة التالية: أسبوع تشخيص مدفوع."

Example — English:
> "Pain: follow-ups lost across WhatsApp and email. Outcome: governed drafts with approval gates. Deliverables: WhatsApp/Inbox Follow-up OS, daily drafts, audit trail. Price: SAR 4,999 diagnosis. Next step: one-week paid diagnostic."

### 2.3 Delivery (client comms, build updates, success reports)

| Attribute | Arabic | English |
|---|---|---|
| **Register** | متعاطف، سريع، يركز على الحل | Empathetic, fast, solution-focused |
| **Transparency** | نذكر الحالة الحقيقية: قيد البناء، قيد الاختبار، جاهز | State the real state: building, testing, ready |
| **Proof** | تقرير نجاح لكل عميل: اكتشاف → بناء → اختبار → نتيجة | Success report per client: discovery → build → test → result |
| **Emotion** | هادئ، مسؤول، لا وعود بدون دليل | Calm, accountable, no promises without proof |

Example — Arabic:
> "مرحلة البناء: اكتمل اكتشاف البيانات، نعمل الآن على مسودات المتابعة. جميع الإرسالات افتراضياً مسودة فقط. سنريك بوابات الموافقة قبل أي إرسال."

Example — English:
> "Build phase: data discovery complete, we are now working on follow-up drafts. All sends are draft-only by default. We will show you approval gates before any send."

### 2.4 Support / WhatsApp client comms

| Attribute | Arabic | English |
|---|---|---|
| **Register** | متعاطف، سريع، يركز على الحل | Empathetic, fast, solution-focused |
| **Rule** | كل مسودة تمر ببوابة موافقة قبل الإرسال | Every draft passes an approval gate before send |
| **Emotion** | ودود، عملي، لا لغة مؤسسية ثقيلة | Friendly, practical, no heavy corporate language |

Example — Arabic:
> "استلمنا رد العميل. جهّزنا مسودة متابعة، بانتظار موافقتك قبل الإرسال. تقدر تعدّلها أو ترفضها."

Example — English:
> "We received the client's reply. We have prepared a follow-up draft, awaiting your approval before send. You can edit or reject it."

### 2.5 Crisis / incident / outage

| Attribute | Arabic | English |
|---|---|---|
| **Register** | شفاف، معتذر، واضح، يركز على الإصلاح | Transparent, apologetic, clear, fix-focused |
| **Rule** | قل ما حدث، ما التأثير، ما الإصلاح، متى التحديث القادم | State what happened, the impact, the fix, when the next update is |
| **Emotion** | هادئ، مسؤول، لا دفاعية | Calm, accountable, no defensiveness |
| **Forbidden** | إخفاء النطاق، لوم العميل، وعود بدون أساس | Hiding scope, blaming the client, baseless promises |

Example — Arabic:
> "حدث تأخير في توليد المسودات اليوم بين الساعة 9 و11 صباحاً. السبب: بطء في مزود الذكاء الاصطناعي. التأثير: مسودات المتابعة تأخرت ساعتين. الإصلاح: تم التحويل لمزود بديل. التحديث القادم: خلال ساعة."

Example — English:
> "Draft generation was delayed today between 9 and 11 AM. Cause: slow LLM provider response. Impact: follow-up drafts delayed by two hours. Fix: failed over to alternate provider. Next update: within one hour."

### 2.6 Investor / international partner

| Attribute | Arabic | English (primary for this audience) |
|---|---|---|
| **Register** | دقيق، قائم على الأدلة، يركز على الفئة والمنتج | Precise, evidence-based, category and product focused |
| **Rule** | كل توقع سيناريو بمستوى ثقة وافتراضات صريحة | Every forecast is a scenario with confidence level and explicit assumptions |
| **Emotion** | جِدّة تنفيذية، ثقة بالفئة الجديدة | Executive seriousness, confidence in the new category |

Example — English:
> "Dealix creates the category of AI Operating Systems for B2B in Saudi. 14 productized OSes share one trust kernel. Every forecast is a scenario with a confidence level. Proof Packs are the renewal artifact."

## 3. Do's and Don'ts | افعل ولا تفعل

### 3.1 Do

| ✓ Do (AR) | ✓ Do (EN) |
|---|---|
| استخدم العربية السعودية السليمة (فصحى مبسطة) | Use clean, precise English |
| سمِّ النظام المحدد (غرفة قيادة الإيرادات، نظام متابعة الواتساب) | Name the specific OS (Revenue Command Room OS, WhatsApp Follow-up OS) |
| اذكر بوابات الموافقة والإثبات | Mention approval gates and proof |
| استخدم الأمثلة الواقعية من السوق السعودي | Use real Saudi market examples |
| اكشف دائماً: "القيمة التقديرية ليست قيمة مُتحقَّقة" | Always disclose: "Estimated value is not Verified value" |
| تأكد من الدقة الشرعية والقانونية لـ PDPL | Ensure PDPL legal and regulatory accuracy |
| استخدم لغة العمليات: مسودات، بوابات، تدقيق، فرق | Use operational language: drafts, gates, audit, delta |

### 3.2 Don't

| ✗ Don't (AR) | ✗ Don't (EN) | Why |
|---|---|---|
| "حوّل عملك" | "Transform your business" | Hype, no proof |
| "خارق" / "ثوري" | "Supercharge" / "Revolutionary" | Hype |
| "مدعوم بالذكاء الاصطناعي" كشعار | "AI-powered" as a slogan | Not a differentiator; every B2B product is |
| "نتائج مذهلة فورًا" | "Instant amazing results" | Fake, unverifiable |
| "نضمن لك مبيعات" | "Guaranteed sales" | Prohibited by no-overclaim register |
| "معتمد من PDPL/ZATCA" | "PDPL/ZATCA-certified" | We are "aware", not "certified" |
| "روبوت ذكي" | "Smart robot" | Chatbot positioning, not OS |
| "تلقائي تماماً" | "Fully automatic" | Implies autonomous sending — prohibited |
| استخدم الخط المائل للعربية | Use italic for Arabic | Most Arabic fonts do not support it; prohibited by typography rules |
| اترك الإفصاح خارج صفحة عامة | Drop disclosure from a public page | Mandatory on every public page |

## 4. Arabic vs English Voice Differences | الفروق بين العربية والإنجليزية

### 4.1 Parallel and equal-weight

Arabic and English are never half-translations of each other. Each is written natively in its language and kept in sync. The voice pillars (confident, precise, operational, Arabic-first) apply to both, but the linguistic expression differs.

### 4.2 Linguistic differences

| Dimension | Arabic | English |
|---|---|---|
| **Register** | فصحى مبسطة (simplified MSA), not overly colloquial | Professional, direct, no slang |
| **Direction** | RTL | LTR |
| **Letter spacing** | 0 (Arabic requires no letter-spacing; avoid it) | Negative letter-spacing for large headings |
| **Line height** | 1.7–1.8 (larger than English) | 1.5–1.6 for body, 1.05–1.3 for headings |
| **Italic** | Never — most Arabic fonts do not support it | Allowed for emphasis (sparingly) |
| **Numbers** | Western Arabic numerals (1, 2, 3) acceptable; tabular figures for data | Standard |
| **Tone warmth** | Slightly warmer, more relational (Saudi business culture) | Slightly more direct, less relational |
| **Formality** | Formal but not stiff | Formal but not stiff |
| **Technical terms** | Translate where possible; keep English in parentheses on first use if needed | English native |
| **Punctuation** | Arabic punctuation (، ؟ ؛) in Arabic text | English punctuation in English text |

### 4.3 Term mapping

| Concept | Arabic (primary) | English (mirror) |
|---|---|---|
| Operating system | نظام تشغيل | Operating system |
| Approval gate | بوابة موافقة | Approval gate |
| Draft-only by default | مسودة فقط افتراضياً | Draft-only by default |
| Proof Pack | حزمة إثبات | Proof Pack |
| Documented delta | فرق موثّق | Documented delta |
| Baseline | خط أساس | Baseline |
| No-overclaim register | سجل عدم المبالغة | No-overclaim register |
| Trust kernel | نواة الثقة | Trust kernel |
| Audit trail | مسار تدقيق | Audit trail |
| Governed | محوكَم | Governed |
| Estimated value is not Verified value | القيمة التقديرية ليست قيمة مُتحقَّقة | Estimated value is not Verified value |
