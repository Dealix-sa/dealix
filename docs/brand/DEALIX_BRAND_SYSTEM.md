# Dealix Brand System

> Single source of truth for how Dealix shows up — visually, verbally, and
> behaviourally — across the product, founder console, landing pages,
> proposals, proof packs, social, and internal artifacts.

## 1. Brand essence

- **Name:** Dealix
- **Wordmark:** `DEALIX`
- **Tagline (EN):** Intelligent Deals. Real Growth.
- **Tagline (AR):** صفقات ذكية. نمو حقيقي.
- **Positioning:** Saudi B2B Revenue Operating System for intelligent
  deal flow, founder-approved growth, and trust-gated AI execution.

## 2. Pillars

1. **Built on Trust** — policy-as-code, approval-first, audit-by-default.
2. **Driven by Growth** — every machine outputs scored, ranked, prioritized work.
3. **Closing Deals** — proposal factory, payment capture, delivery trigger.
4. **Focused on Results** — performance loop wired to revenue KPI tree.
5. **Global Mindset, Local Impact** — Saudi B2B native; PDPL / ZATCA aware.

## 3. What Dealix is — and is not

| ✅ Is | ❌ Is not |
|---|---|
| A founder-controlled Revenue OS | A CRM replacement |
| AI-assisted execution with trust gates | An open agent that sends external messages on its own |
| Draft / queue / score / recommend by default | A scraping or spam machine |
| Saudi B2B native (PDPL/ZATCA aware) | A US tool with translated screens |
| Approval-first for any commitment | A guaranteed-revenue service |

## 4. Operating doctrine for the brand

- Every external action is **trust-gated**: drafts go to an approval queue,
  not to the recipient.
- No agent makes pricing, contract, refund, or payment commitments.
- No guaranteed revenue or sales claims appear in any artifact.
- Fallback data is labelled `source=fallback` everywhere it surfaces.
- Brand colours are accessible per WCAG 2.x (see Accessibility Guide).

## 5. Reference files

- `docs/brand/brand-tokens.json` — machine-readable design tokens.
- `apps/web/lib/brand-tokens.ts` — TypeScript bindings for the web.
- `apps/web/styles/brand.css` — CSS variables and primitive classes.
- `apps/web/components/brand/` — Logo, cards, badges, CTA, nav.
- `assets/brand/source/` — SVG sources (logo / mark / wordmark / social).
- `apps/web/public/brand/` — Public assets served by the web app.
