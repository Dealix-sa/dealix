## 1. What Dealix Is NOT | ما ليست عليه ديليكس

Dealix is defined as much by what it refuses to be as by what it is. These are category boundaries, not feature gaps.

| Dealix is NOT | Why | What Dealix IS instead |
|---|---|---|
| **A chatbot** | Chatbots converse. They do not run a function end-to-end. No approval gates. No proof. | An operating system per commercial function: discovery → decision → delivery → proof. |
| **A CRM** | CRMs store data. They require humans to do the operational work. No draft generation, no approval routing, no proof. | An OS that generates drafts, routes approvals, and produces proof. Can work alongside a CRM. |
| **An agency** | Agencies are black-box human labor. No audit trail. No transferable system. No proof discipline. | A transparent, auditable system the client can see, audit, and eventually operate. |
| **A dashboard / BI tool** | Dashboards observe and visualize. They cannot execute, decide, or prove. | A system that closes the loop: observe → recommend → approve → execute → prove. |
| **Generic automation (Zapier, n8n)** | Automation tools connect APIs. No AI reasoning, no approval gates, no proof. | An OS for a full function with AI reasoning, governance, and proof. |
| **RPA** | RPA automates deterministic clicks. No AI exploration, no proof discipline. | Governed automation with AI reasoning, approval gates, and Proof Packs. |
| **A "set it and forget it" tool** | Implies autonomous operation. Violates the governed doctrine. | A system where AI recommends and humans approve critical external commitments. |
| **A foreign product translated to Arabic** | Arabic as a localization layer is not Arabic-first. | A Saudi-native company where Arabic is the primary language and English is a mirror. |

## 2. Prohibited Claims | ادعاءات ممنوعة

### 2.1 The no-overclaim register (binding)

These are strictly forbidden in any brand expression, by any person, on any surface, at any stage.

| # | Prohibited claim (AR) | Prohibited claim (EN) | Why | Allowed alternative |
|---|---|---|---|---|
| 1 | "نضمن عائد الاستثمار" / "نضمن نمو الإيرادات" | "We guarantee ROI" / "We guarantee revenue growth" | No guarantee of revenue, sales, or return is permitted. | "كل توقع سيناريو بمستوى ثقة وافتراضات صريحة" / "Every forecast is a scenario with confidence level and explicit assumptions" |
| 2 | "زيادة الإيراد بنسبة 300%" (كضمان) | "Increase revenue by 300%" (as a guarantee) | Specific percentages as guarantees are prohibited. | "سيناريو محافظ: 7,500–12,000 ريال شهرياً، ثقة متوسطة-عالية" / "Conservative scenario: SAR 7,500–12,000/month, medium-high confidence" |
| 3 | عملاء وهميون، شهادات ملفقة، دراسات حالة مختلقة | Fake clients, fabricated testimonials, invented case studies | No invented names, logos, quotes, or case studies. | No public client reference without a signed Proof Publication Consent. |
| 4 | "معتمد من PDPL" / "معتمد من ZATCA" | "PDPL-certified" / "ZATCA-certified" | We are "aware", not "certified" by regulators. | "مدرك لـ PDPL" / "مدرك لـ ZATCA" — "PDPL-aware" / "ZATCA-aware" |
| 5 | "إرسال تلقائي للعملاء" | "Automatic sending to clients" | Implies autonomous external sending — prohibited. Default is draft-only. | "مسودات جاهزة للمراجعة، إرسال يدوي بعد الموافقة" / "Drafts ready for review, manual send after approval" |
| 6 | "تلقائي تماماً" / "بدون تدخل بشري" | "Fully automatic" / "No human intervention" | Violates governed doctrine. Human approval is a feature. | "ذكاء اصطناعي يكتشف ويوصي، إنسان يوافق" / "AI explores and recommends, humans approve" |
| 7 | "نتائج فورية" / "نتائج مذهلة" | "Instant results" / "Amazing results" | Fake, unverifiable. | "قيمة تقديرية موثّقة المصدر" / "Source-backed estimated value" + disclosure |

### 2.2 Scenario-based forecasting rules

Every external forecast must carry ALL of:
1. **Confidence level** (e.g., Medium, Medium-high, Low-medium).
2. **Explicit assumptions** (e.g., "single OS, one function, light ops").
3. **Description as a scenario**, not a guarantee (e.g., "Conservative scenario: ...").

A forecast missing any of these three elements is a guardrail violation.

### 2.3 Client references

- No client is named publicly without a signed **Proof Publication Consent** on file (see [`../wave8/PROOF_PUBLICATION_CONSENT_TEMPLATE.md`](../wave8/PROOF_PUBLICATION_CONSENT_TEMPLATE.md)).
- No testimonial is published without a real, documented Proof Pack behind it.
- No logo is displayed without written permission.
- At the founder stage, if no real Proof Pack exists, no results claim is made. Period.

## 3. Messaging Guardrails | ضوابط الرسائل

### 3.1 Banned words and phrases

| Banned (AR) | Banned (EN) | Category | Use instead |
|---|---|---|---|
| "حوّل عملك" | "Transform your business" | Hype | "نظام تشغيل محوكَم" / "A governed operating system" |
| "خارق" | "Supercharge" | Hype | "عمليات محوكمة مع إثبات" / "Governed operations with proof" |
| "ثوري" | "Revolutionary" | Hype | Name the specific OS and mechanism |
| "مدعوم بالذكاء الاصطناعي" (كشعار) | "AI-powered" (as a slogan) | Non-differentiator | Name the specific OS and approval-gated workflow |
| "نتائج مذهلة فورًا" | "Instant amazing results" | Fake claim | "قيمة تقديرية موثّقة المصدر" / "Source-backed estimated value" |
| "نضمن لك مبيعات" | "Guaranteed sales" | No-overclaim violation | "فرص مُثبتة بأدلة" / "Evidenced opportunities" |
| "روبوت ذكي" | "Smart robot" | Chatbot positioning | "نظام تشغيل" / "Operating system" |
| "معتمد" (للجهات المنظمة) | "Certified" (for regulators) | Unearned certification | "مدرك لـ" / "Aware of" |
| "تلقائي تماماً" | "Fully automatic" | Autonomous claim | "تحت موافقات صريحة" / "Under explicit approvals" |
| "حلّ سحرى" | "Magic solution" | Hype | Name the mechanism |

### 3.2 Required disclosures

Every public page and external document must carry:

**Arabic:** "القيمة التقديرية ليست قيمة مُتحقَّقة."
**English:** "Estimated value is not Verified value."

Placement: footer or bottom of page, visible, not hidden in a collapsed section.

### 3.3 Arabic-first messaging guardrails

- Arabic is the primary language on all Saudi-market-facing surfaces.
- English mirrors are parallel and equal-weight — never a half-translation.
- No italic for Arabic (most Arabic fonts do not support it).
- RTL for Arabic, LTR for English. Never mix directions in one paragraph.
- Arabic punctuation (، ؟ ؛) in Arabic text. English punctuation in English text.

## 4. Visual Guardrails | ضوابط المرئيات

### 4.1 Color guardrails

| Rule | Detail |
|---|---|
| Navy is the primary brand color. | Used for backgrounds, headings, primary buttons, navbar, footer. |
| Gold is for accent only. | Used sparingly: secondary buttons, feature icons, accent borders, badges, `pulse-gold` on the single primary CTA. |
| Gold is never used for large background fills. | No Gold hero backgrounds, no Gold section backgrounds. |
| Emerald is for proof and success only. | Used for: approved status, success indicators, checkmarks, proof pack indicators, positive numbers. |
| Emerald is never used as a brand color. | It is a semantic/success color, not a primary brand color. |
| Off-palette colors are prohibited. | No colors outside the defined Navy/Gold/Emerald + neutral slate scale. (See [`COLOR_SYSTEM.md`](COLOR_SYSTEM.md) and [`VISUAL_IDENTITY.md`](VISUAL_IDENTITY.md).) |
| WCAG AA minimum. | All color pairings must meet AA (4.5:1 for small text, 3:1 for large text). See [`COLOR_SYSTEM.md`](COLOR_SYSTEM.md) §5. |

### 4.2 Logo guardrails

From [`LOGO_USAGE.md`](LOGO_USAGE.md):

| Prohibited | Why |
|---|---|
| Recoloring the logo | Only Navy + Gold (or white on dark). |
| Rotating the logo | Never rotate. |
| Stretching / squashing | Preserve aspect ratio always. |
| Adding drop shadows or glows | No effects on the logo. |
| Placing on low-contrast backgrounds | Must have clear contrast. |
| Placing on busy photo backgrounds | Keep clear space. |
| Changing fonts | Only the defined font stack. |
| Removing elements | Icon + "Dealix" + "ديليكس" stay together. |
| Reordering elements | English and Arabic text order is fixed. |
| Adding external borders/frames | No frames around the logo. |

### 4.3 Imagery guardrails

| Prohibited | Why | Use instead |
|---|---|---|
| Generic AI robot imagery | Chatbot positioning, not OS. | Dashboards, decision rooms, revenue maps, proof packs. |
| Stock photos | Inauthentic. | Real product screenshots, clean data visualizations, brand-color graphics. |
| Overly colorful icons | Off-brand. | Gold icons on Navy, or Navy icons on light. |
| Low-quality images | Unprofessional. | Crisp, high-resolution, brand-consistent visuals. |
| Stereotypical "AI" visuals (brains, neural networks as decoration) | Cliché. | Operational visuals: pipelines, approval gates, audit trails, proof cards. |

### 4.4 Typography guardrails

From [`TYPOGRAPHY.md`](TYPOGRAPHY.md):

| Prohibited | Why |
|---|---|
| Italic for Arabic | Most Arabic fonts do not support it. |
| Excessive letter-spacing for Arabic | Arabic requires no letter-spacing. |
| Off-stack fonts | Only: Noto Kufi Arabic, Noto Naskh Arabic, Cairo, Noto Sans Arabic, Tajawal, IBM Plex Mono (AR); Poppins, Inter, JetBrains Mono (EN). |
| Mixing RTL and LTR in one paragraph | Confusing, unprofessional. |
| Arbitrary pixel values for font sizes | Use the defined modular scale (see [`TYPOGRAPHY.md`](TYPOGRAPHY.md) §2). |

### 4.5 Layout guardrails

| Rule | Detail |
|---|---|
| Spacing scale: 4 / 8 / 12 / 16 / 24 / 32 / 48 / 64px. | Stick to multiples. No arbitrary values. |
| Section rhythm: 64–96px vertical between major sections. | Consistent vertical rhythm. |
| Card padding: 24px inner. | Consistent card internals. |
| Radius: use `--radius` tokens (`lg`, `md`, `sm`). | No arbitrary border-radius. |
| Motion: 0.2–0.4s, quiet. `fade-in`, `slide-in-right`. | No distracting or flashy motion. |

## 5. Partner Co-Branding Rules | قواعد العلامة المشتركة للشركاء

### 5.1 When co-branding is allowed

Co-branding is allowed only when:
1. A signed partnership agreement is on file.
2. The partner's brand is complementary, not competitive (not a chatbot, CRM, agency, or dashboard company).
3. The Dealix brand remains the lead brand in Saudi-market-facing materials.
4. The partner has reviewed and approved the co-branded material.

### 5.2 Co-branding layout rules

| Rule | Detail |
|---|---|
| Dealix logo first (left in LTR, right in RTL). | Dealix is the lead brand. |
| Partner logo secondary, equal or smaller size. | Never larger than the Dealix logo. |
| Clear space respected for both logos. | Per [`LOGO_USAGE.md`](LOGO_USAGE.md). |
| No logo color changes. | Both logos use their official color palettes. |
| No combining logos into a single mark. | Keep them separate with clear space. |
| Shared materials use Dealix brand colors. | Navy primary, Gold accent. Partner colors do not override. |

### 5.3 Co-branding messaging rules

| Rule | Detail |
|---|---|
| All Dealix messaging guardrails apply. | No hype, no fake claims, no guaranteed ROI. |
| The disclosure line is present. | "Estimated value is not Verified value" on every public co-branded page. |
| No implied certification by the partner. | A partnership is not a regulator certification. |
| No claims about the partner's customers. | No borrowed social proof. |
| Arabic-first applies to co-branded materials. | Arabic is primary; English is the mirror. |

### 5.4 Co-branding prohibited scenarios

| Prohibited | Why |
|---|---|
| Co-branding with a chatbot company | Conflicts with "not a chatbot" positioning. |
| Co-branding with a CRM company as an "integration partner" without clear distinction | Must clearly state Dealix is an OS, not a CRM extension. |
| Co-branding with an agency | Conflicts with "not an agency" positioning. |
| Letting the partner's brand override Dealix visual identity | Dealix brand colors and fonts are primary. |
| Publishing co-branded results claims without a Proof Pack | No proof, no claim — applies to co-branded material too. |
| Using the partner's client logos as Dealix social proof | No borrowed social proof. |

## 6. Enforcement | التطبيق

### 6.1 Pre-publication checklist

Before any external material ships, verify (from [`DEALIX_BRAND_OS.md`](DEALIX_BRAND_OS.md) §5.4):

- [ ] Arabic is primary; English mirror is parallel and equal-weight.
- [ ] Disclosure line present: "Estimated value is not Verified value."
- [ ] No hype words (see §3.1 above).
- [ ] No guaranteed ROI, no fake clients, no specific percentages as guarantees.
- [ ] No "certified" claims (PDPL-aware, not certified).
- [ ] Visuals use Navy primary, Gold sparingly, Emerald for proof only.
- [ ] No generic AI robot imagery. No stock photos.
- [ ] Logo used per [`LOGO_USAGE.md`](LOGO_USAGE.md).
- [ ] Fonts per [`TYPOGRAPHY.md`](TYPOGRAPHY.md). No italic for Arabic.
- [ ] Every forecast is a scenario with confidence level + assumptions.
- [ ] Spacing scale respected. No arbitrary pixel values.
- [ ] For co-branded material: partnership agreement on file, Dealix is lead brand.

### 6.2 Violation handling

| Violation type | Action |
|---|---|
| No-overclaim register violation (guaranteed ROI, fake clients, fake percentages) | Immediate removal of the material. Founder review. Log in trust KPIs (target: 0 violations). |
| Missing disclosure line | Add before publication. No page ships without it. |
| Visual guardrail violation (Gold fills, Emerald as brand color, off-palette) | Redesign before publication. |
| Logo misuse | Replace with correct logo asset. |
| Arabic-first violation (English primary on Saudi-facing surface, half-translation) | Rewrite with Arabic as primary. |
| Co-branding violation (partner overrides Dealix, borrowed social proof) | Redesign or remove. Review partnership terms. |

### 6.3 Trust KPIs (from company OS)

These are tracked continuously and reflect brand governance health:
- Approval gates bypassed (target: 0)
- Send flags in correct state per environment
- PDPL controls active
- No-overclaim register violations (target: 0)
- Audit trail completeness (target: 100%)
