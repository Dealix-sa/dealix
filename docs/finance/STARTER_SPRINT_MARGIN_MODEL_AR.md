# نموذج الهامش — Sprint المبتدئ
# Starter Sprint Margin Model

---

## المبدأ — Principle

كل Sprint يُحسب هامشه قبل قبول العقد. نموذج سالب = لا نبيع. نطاق غير واضح = لا نبيع.

Every Sprint margin is calculated before accepting the contract. Negative margin = we do not sell. Unclear scope = we do not sell.

## افتراضات النموذج — Model Assumptions

- **تكلفة AI:** LLM tokens + API calls. يُراجَع شهرياً في `docs/06_llm_gateway/COST_GUARD.md`
- **معدل الوقت:** يُقدَّر بـ 50 ريال/ساعة (وقت مؤسس في مرحلة الإطلاق). يُراجَع كل 90 يوم.
- **لا تكاليف ثابتة مُدرَجة** في هذا النموذج (infrastructure, domain, etc.) — تُدار في نموذج P&L منفصل

## متى لا نبيع — When to Decline

### شرط الهامش السالب
إذا كان الهامش الصافي المحسوب < 100 ريال لأي Sprint بسعر 499 → رفض أو إعادة التفاوض.

### شرط عدم وضوح النطاق
إذا لم يستطع العميل تحديد المشكلة بجملة واحدة بعد جلستين → Sprint ليس مناسباً بعد.

### شرط غياب المدخلات
إذا لم تتوفر المدخلات المطلوبة (بيانات، ملفات) → لا نبدأ. وقت التسليم يبدأ من GATE_PASSED.

### شرط الطلب غير المتوافق
إذا طلب العميل ضمان ROI أو مبيعات محددة → رفض فوري.

### شرط الطاقة
إذا كانت الطاقة الحالية ممتلئة (راجع `docs/scale/DELIVERY_CAPACITY_PLANNING_AR.md`) → رفض أو تأجيل.
