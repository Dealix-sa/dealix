# Default Alive System

## Purpose
Determine whether Dealix can survive and grow based on current revenue, burn, and pipeline.

## Owner
Sami / Founder CEO.

## Review Cadence
Monthly.

## Definitions

### Default Alive
Dealix can reach sustainable profitability or enough recurring revenue without needing external funding.

### Default Dead
Dealix will run out of cash or founder capacity before reaching sustainable revenue unless something changes.

## Inputs
- cash on hand
- monthly expenses
- MRR
- cash collected
- weighted pipeline
- proposal conversion
- delivery capacity
- founder capacity

## Rules
- If net burn is high and no pipeline exists, fix revenue immediately.
- If pipeline exists but no payment conversion, fix sales/payment path.
- If paid work exists but delivery is weak, fix delivery before scaling.
- If MRR exists, prioritize retention.

## Monthly Decision
Default Alive / Default Dead / Unknown

## CEO Actions
- If Default Dead: cut scope, push revenue, kill non-essential build.
- If Unknown: improve finance tracking.
- If Default Alive: invest carefully in repeatability and retention.
