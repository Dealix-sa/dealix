# Payment Path System
## Purpose
Make it easy and clear for prospects to start a Revenue Sprint.
## Owner
Sami / Finance owner.
## Payment Paths
- bank transfer.
- invoice.
- purchase order.
- written approval.
- payment link later.
## Rules
- Every proposal must include start condition.
- Every payment follow-up must be logged.
- PO or written approval can start work only if risk acceptable.
- Discounts above 20% require CEO approval.
- Refunds are A3 and never automatic.
## Metrics
- payment follow-ups.
- payment conversion.
- overdue decisions.
- days from proposal to start condition.
