# Dealix Master Operating Blueprint

## Purpose
Integrate all Dealix systems into one operating blueprint for execution, governance, revenue, delivery, finance, productization, and founder leverage.

## CEO Principle
Revenue drives.
Trust governs.
Delivery proves.
Finance disciplines.
Learning improves.
Productization follows evidence.
Founder leverage compounds.

## Company Operating Layers
| Layer | Purpose | Main Evidence |
|---|---|---|
| CEO Control | decide what Sami does next | mission_control.md |
| Revenue Ops | create pipeline and cash path | pipeline_tracker.csv |
| Delivery | create client value and proof | delivery_report.md |
| Client Success | convert value into retention | health_score.md |
| Finance | protect cash, margin, runway | finance_command_report.md |
| Trust & AI Risk | protect reputation and data | approval_log.csv |
| Content & Proof | turn learning into trust and demand | proof_library.md |
| Productization | turn repetition into software | candidates.csv |

## Daily CEO Loop
1. Run mission-control.
2. Run company-check.
3. Run stage.
4. Execute top CEO action.
5. Update private evidence.
6. Run advance.
7. Run close-day.

## Weekly CEO Loop
1. Run ceo-weekly.
2. Run weekly-close.
3. Run trust-full.
4. Run finance-full.
5. Run strategic-decision.
6. Update one system.
7. Commit changes.

## Monthly CEO Loop
1. Run board.
2. Run monthly.
3. Run productization.
4. Run pricing.
5. Review capital allocation.
6. Review SaaS readiness gate.
7. Review hiring triggers.

## Stage Advancement Rule
No stage advancement without:
- evidence
- trust review
- finance awareness
- delivery readiness
- learning update

## No-Overbuild Rule
Do not build SaaS, agents, white-label, mobile app, or enterprise platform before:
- 3 paid Revenue Sprints
- 2 completed deliveries
- 1 retainer attempt
- 10 repeated workflow executions
- trust controls tested
- data model stable

## Current Priority
Stage 1: Revenue Sprint Execution.

## Current Minimum Evidence
- 25 leads.
- 25 founder-led DMs.
- 3 sample packs.
- 1 proposal.
- 1 payment / PO / written approval follow-up.
- 1 weekly learning review.
- 1 system update.

## Definition of Operating
Dealix is operating when:
- company-check passes.
- business-score is generated.
- assurance score is generated.
- mission-control gives a top action.
- revenue evidence exists.
- trust checks pass.
- finance report exists.
- weekly learning updates a system.
