# Dealix Delivery Engine

## Delivery philosophy
Every client delivery must become a reusable asset.

## Stages

### 1. Diagnostic Sprint
Outputs:
- Workflow map
- Data map
- Leakage map
- Success metrics
- Recommended first system

### 2. Prototype
Outputs:
- Screens
- Data schema
- Workflow states
- Report sample
- Automation map

### 3. Build
Outputs:
- Dashboard
- Forms/API
- Automations
- Reports
- SOPs
- Training docs

### 4. Pilot
Outputs:
- Live usage
- Bugs
- Adoption feedback
- KPI baseline
- Weekly report

### 5. Scale
Outputs:
- Add workflows
- Add users
- Add integrations
- Add governance
- Add SLA

## Never start with
- Full enterprise build
- Undefined custom scope
- Unlimited integrations
- AI agents without workflow ownership
