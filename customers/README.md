# Customers Directory — Dealix Delivery Folders — مجلّدات تسليم العملاء

This directory holds one per-customer delivery folder for each Command Sprint, at `customers/{company}/`. A folder is created **only after payment is confirmed** — before payment there is no folder and no delivery work.

No PII belongs in any committed file here: no email, phone, national ID, or real personal name (non-negotiable #6). Folders use an anonymized company slug, and operational contact details, when needed, stay outside the committed repository.

The structure of each folder — every file `00`–`10`, its purpose, and what "complete" means — is documented in [`../docs/04_delivery/CUSTOMER_FOLDER_TEMPLATE.md`](../docs/04_delivery/CUSTOMER_FOLDER_TEMPLATE.md). The delivery runbook is [`../docs/04_delivery/COMMAND_SPRINT_DELIVERY_OS.md`](../docs/04_delivery/COMMAND_SPRINT_DELIVERY_OS.md).

---

هذا الدليل يحفظ مجلّد تسليم واحداً لكل عميل عن كل سبرنت قيادة، في `customers/{company}/`. ويُنشأ المجلّد **بعد تأكيد الدفع فقط** — قبل الدفع لا مجلّد ولا عمل تسليم.

لا مكان لأي بيانات تعريف شخصية في أي ملف محفوظ هنا: بلا بريد أو هاتف أو هوية وطنية أو اسم حقيقي (المبدأ غير القابل للتفاوض رقم 6). وتستخدم المجلّدات اسماً مختصراً مجهول الهوية، وتبقى تفاصيل التواصل التشغيلية، عند الحاجة، خارج المستودع المحفوظ.

هيكل كل مجلّد — كل ملف `00`–`10` وغرضه ومعنى "مكتمل" — موثّق في [`../docs/04_delivery/CUSTOMER_FOLDER_TEMPLATE.md`](../docs/04_delivery/CUSTOMER_FOLDER_TEMPLATE.md). ودليل التسليم هو [`../docs/04_delivery/COMMAND_SPRINT_DELIVERY_OS.md`](../docs/04_delivery/COMMAND_SPRINT_DELIVERY_OS.md).
